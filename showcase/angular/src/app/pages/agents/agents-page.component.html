Meet the FSB Skill

The canonical OpenClaw onboarding path for FSB. Loads cleanly into a fresh OpenClaw, runs the full diagnostic flow, and installs the bridge for any other MCP host on your machine.

Doctor first protocol

Runs npx -y fsb-mcp-server doctor, parses the failing layer (one of package, bridge, extension, active-tab, content-script, config), and prints the next step recommendation. Exit code surfaces failure to the host.

Stdio config printer

Emits the canonical OpenClaw stdio block to paste into your MCP config. The block stays parity locked with mcp/src/install.ts so the printer never drifts from the rest of the installer.

Consent gated host install

Detects other MCP hosts (Claude Desktop, Cursor, Codex, etc.) and asks y/n per host before installing. Never --all without consent. Never auto writes the OpenClaw config.

Why FSB MCP is powerful

Browser automation usually means a headless instance you cannot see. FSB drives the user's actual Chrome session, and the MCP surface is built around four hard requirements: precision, observability, safety, and concurrency.

50+

MCP tools, one surface

Read only first (read_page, get_dom_snapshot, get_page_snapshot, get_site_guide), interaction (click, type_text, press_enter), navigation, vault, and observability tools. All typed. All scoped to the calling agent.

N agents

Multi agent identity and tab ownership

Each MCP session gets a server minted agent_id. Tabs are owned per agent. Cross agent access rejects with typed TAB_NOT_OWNED. The concurrency cap is configurable from 1 to 64 (default 8) with a fail loud AGENT_CAP_REACHED error. Background tabs never steal user focus.

0 leaks

Vault boundary, by design

Passwords and payment details resolve inside the FSB Chrome extension's encrypted storage via fill_credential and use_payment_method. Never in chat. Never in tool args. requires.env is mandatory empty.

Live

Real Chrome, real session

The user's logged in cookies, extensions, and saved state are the runtime. Visual feedback shows what is being targeted. change_report on every action keeps the agent aware of mutations without an extra read_page round trip.

Install in 3 Steps

Get from a clean machine to a green doctor in under five minutes.

  1. 1

    Install the FSB Chrome extension

    Primary path. Paste this Chrome Web Store URL into Chrome's address bar:

    https://chromewebstore.google.com/detail/badgafnfchcihdfnjneklogedcdkmjfk

    Fallback path. Download the latest .zip from GitHub Releases, then load unpacked at chrome://extensions with Developer mode on.

  2. 2

    Install the FSB MCP server config

    Print the canonical OpenClaw stdio block and paste it into your MCP host config:

    node scripts/print-stdio.mjs

    Or use the FSB CLI directly. Discover supported hosts on your machine:

    npx -y fsb-mcp-server install --list

    Then run the installer for one host, for example:

    npx -y fsb-mcp-server install --claude-desktop
  3. 3

    Verify with the doctor

    Run the six layer diagnostic. Expect [OK] on every layer.

    node scripts/doctor.mjs

    If any layer fails, the doctor prints a one line next step. The full recovery table lives in USAGE.md inside the skill.

Try It

Once the doctor is green, paste any of these into your OpenClaw chat to confirm the loop works.

Manual mode (default)

Each prompt calls a single FSB tool so you can see the round trip plainly.

  • read_page "Open https://example.com and read the page. Summarize the visible text in two sentences."
  • click "On https://example.com, click the 'More information...' link and tell me what page loads."
  • type_text "Open https://duckduckgo.com, type fsb-mcp-server into the search box, and press Enter. Then list the first three result titles."

Autopilot (explicit delegation only)

NEVER call run_task unless the user has explicitly said something like "use FSB autopilot", "delegate this to FSB", "run the whole task autonomously", or named run_task directly. Manual mode tools are the default for everything else, including tasks with many steps.

  • run_task "Use FSB autopilot to find the first GitHub repo for fsb-mcp-server and report a one paragraph summary of its README."
