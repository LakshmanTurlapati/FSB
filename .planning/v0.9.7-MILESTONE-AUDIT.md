## Tech Debt

### 1. Missing Formal Verification (Low Severity)
43 phases (54-96) completed without running the gsd-verifier agent. All phases went through:
- Smart discuss (context gathering)
- Planning (2 plans each)
- Execution (site guide + diagnostic report)
- Human checkpoint approval

The missing VERIFICATION.md files are a documentation gap, not a functional one.

**Remediation:** Run `/gsd:validate-phase N` retroactively for any phase needing formal verification.

### 2. PARTIAL Diagnostic Outcomes (Medium Severity)
46 of 50 phases produced PARTIAL outcomes because the MCP server WebSocket bridge was disconnected during executor agent sessions. The orchestrator ran live MCP tests for phases 49-54 (using navigate, click_at, drag, press_key directly), confirming:
- CDP drag works on Google Maps canvas (Phase 49)
- CDP tools work through iframes (Phase 50 Solitaire, Phase 57 Vimeo)
- Photopea renders entire UI on canvas -- no DOM (Phase 51)
- CDP drag rotates Sketchfab 3D viewers (Phase 52)
- press_key plays piano notes via debuggerAPI (Phase 54 -- PASS)

**Remediation:** Connect WebSocket bridge and re-run diagnostic tests for individual phases as needed.

### 3. Site Guide Selector Inaccuracies (Low Severity)
Several site guides have research-based selectors that don't match live DOM:
- Photopea: All selectors wrong (canvas-rendered, no DOM)
- Google Solitaire: Game inside iframe, selectors can't reach game DOM
- Virtual Piano: Keyboard mapping corrected (was home row, actually middle row)
- Various CSS class mismatches documented in diagnostic reports

**Remediation:** Inaccuracies documented in diagnostic reports. Site guides updated where live testing was possible.

## Integration Assessment
