# Requirements: FSB v0.9.46 Site Discoverability (SEO + GEO)

**Defined:** 2026-04-30
**Core Value:** Reliable single-attempt execution -- the AI decides correctly, the mechanics execute precisely

## Milestone Goal

Make `full-selfbrowsing.com` discoverable to traditional search engines (Google, Bing) and generative AI search (ChatGPT, Claude, Perplexity, Gemini, Google AI Overviews) by prerendering the Angular SPA marketing routes and shipping LLM/crawler-aware root files. Verified live: the site currently returns only the literal string "FSB" to non-JS crawlers -- empty `<app-root>`. AI crawlers do not execute JavaScript; static prerender is the load-bearing fix that unblocks every downstream discoverability play.

## Already Validated (counted toward this milestone)

None. v0.9.46 is a greenfield discoverability milestone -- no prior phase landed any of these requirements.

## v1 Requirements (in scope for v0.9.46)

### Static Prerender Foundation (PRE)

- [ ] **PRE-01**: `@angular/ssr@^19` is installed as a devDependency in `showcase/angular/package.json` and the project's prerender scaffolding (`main.server.ts`, `app.config.server.ts`, `server.ts`) is generated by `ng add @angular/ssr`
- [ ] **PRE-02**: `angular.json` for the `showcase-angular` project has `outputMode: "static"` and `prerender: { discoverRoutes: false, routesFile: "prerender-routes.txt" }` configured under the build target
- [ ] **PRE-03**: A `prerender-routes.txt` file at `showcase/angular/prerender-routes.txt` lists exactly the four marketing routes (`/`, `/about`, `/privacy`, `/support`) -- explicitly excluding `/dashboard` so its interactive runtime state is never baked into static HTML
- [ ] **PRE-04**: `localStorage.getItem('fsb-showcase-theme')` access in `showcase/angular/src/index.html` (lines 8-15) is wrapped in a `typeof localStorage !== 'undefined'` guard so the inline theme bootstrap survives the prerender environment without throwing
- [ ] **PRE-05**: A production build (`npm --prefix showcase/angular run build`) emits per-route `index.html` files at `showcase/dist/showcase-angular/browser/about/index.html`, `/privacy/index.html`, `/support/index.html`, plus the root `/index.html`; no `dashboard/index.html` is produced

### Per-Route Metadata (META)

- [ ] **META-01**: Each prerendered route component (Home, About, Privacy, Support) calls Angular's `Title.setTitle(...)` and `Meta.updateTag(...)` in `ngOnInit` so the prerendered HTML emits a unique route-specific `<title>` and `<meta name="description">` (no two prerendered pages share the same title)
- [ ] **META-02**: Each prerendered route emits a `<link rel="canonical" href="https://full-selfbrowsing.com/<route>">` tag in the prerendered HTML, set via Angular's `DOCUMENT` token + `Renderer2` (Angular's `Meta` service does not natively support `<link>` tags)
- [ ] **META-03**: Each prerendered route emits OpenGraph tags (`og:title`, `og:description`, `og:url`, `og:type`, `og:image`, `og:site_name`) and Twitter Card tags (`twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`) appropriate for the route, with a single shared OG image asset for v0.9.46 (per-route OG images deferred)
- [ ] **META-04**: The `/dashboard` route emits `<meta name="robots" content="noindex, nofollow">` at runtime so search engines do not index the interactive runtime surface (it is not prerendered, but Googlebot can still execute JS and reach it)

### Structured Data / JSON-LD (LD)

- [ ] **LD-01**: An `Organization` JSON-LD block is embedded in `showcase/angular/src/index.html` (so it is inherited by every prerendered route) with `@id` cross-reference, official URL, logo, `sameAs` (GitHub repo URL), and contact info; the JSON payload is escaped with `JSON.stringify(obj).replace(/</g, '\\u003c')` to defeat `</script>` injection
- [ ] **LD-02**: A `SoftwareApplication` JSON-LD block is embedded only on the home route (`/`) via the home component, with `applicationCategory: "BrowserApplication"`, `operatingSystem: "Chrome"`, `offers: { price: "0" }`, `softwareVersion`, and `sameAs` linking to the GitHub repo; payload uses the same `</script>` escaping pattern as LD-01
- [ ] **LD-03**: All emitted JSON-LD passes Google's Rich Results Test (https://search.google.com/test/rich-results) on the deployed URL with zero errors and zero warnings

### Crawler Root Files (CRAWL)

- [x] **CRAWL-01**: A `robots.txt` file lives at `showcase/angular/public/robots.txt` with explicit `User-agent: <name>` + `Allow: /` blocks for the verified 2026 AI crawler universe -- GPTBot, ChatGPT-User, OAI-SearchBot, ClaudeBot, Claude-User, Claude-SearchBot, PerplexityBot, Perplexity-User, Google-Extended, Applebot-Extended, Amazonbot, Bytespider, CCBot, Meta-ExternalAgent, DuckAssistBot, plus a standard `User-agent: *` allow-all -- and a final `Sitemap: https://full-selfbrowsing.com/sitemap.xml` line; the file is served at `https://full-selfbrowsing.com/robots.txt` with `Content-Type: text/plain`
- [ ] **CRAWL-02**: A `sitemap.xml` file at `showcase/angular/public/sitemap.xml` lists the four prerendered marketing routes (no `/dashboard`) with `<loc>` (full HTTPS URL) and `<lastmod>` (build-date ISO 8601) only -- no `<changefreq>` or `<priority>` (Google ignores both in 2026); served at `https://full-selfbrowsing.com/sitemap.xml` with `Content-Type: application/xml`
- [x] **CRAWL-03**: An `llms.txt` file at `showcase/angular/public/llms.txt` follows the llmstxt.org markdown convention -- project name as H1, one-paragraph description (definition-first), then a Docs section linking to `/about`, `/support`, `/privacy`, the GitHub repo, and `llms-full.txt`; served at `https://full-selfbrowsing.com/llms.txt` with `Content-Type: text/plain`
- [ ] **CRAWL-04**: An `llms-full.txt` file at `showcase/angular/public/llms-full.txt` contains a curated long-form markdown dump for LLM ingestion (project description, capabilities list, install instructions, key concepts, comparison framing vs Browser Use / Project Mariner / Operator), kept under 256KB; served at `https://full-selfbrowsing.com/llms-full.txt` with `Content-Type: text/plain`
- [ ] **CRAWL-05**: A Node prebuild script at `showcase/angular/scripts/build-crawler-files.mjs` regenerates `sitemap.xml` (lastmod) and `llms-full.txt` (assembled from source markdown) at build time; wired via npm `prebuild` script in `showcase/angular/package.json`; zero new npm dependencies (pure Node fs + Date)

### Express Server Wiring (SRV)

- [ ] **SRV-01**: The Express SPA-fallback at `server/server.js:111-117` is replaced with `express.static({ extensions: ['html'] })` semantics (or an equivalent route-aware lookup that prefers `path.join(staticPath, req.path, 'index.html')` before falling back to root `index.html`) so that requests for `/about`, `/privacy`, `/support` return their prerendered route-specific HTML, not the root index
- [ ] **SRV-02**: The `/dashboard` route preserves its existing SPA behavior -- the server falls back to the root `index.html` for `/dashboard` only (whitelisted in the patched fallback), so the interactive runtime continues to bootstrap from the SPA shell as it does today
- [ ] **SRV-03**: `*.txt` files (`robots.txt`, `llms.txt`, `llms-full.txt`) and `*.xml` files (`sitemap.xml`) are served by `express.static` with `Content-Type` set correctly (`text/plain; charset=utf-8` and `application/xml; charset=utf-8` respectively) and `Cache-Control: public, max-age=3600`; no SPA-fallback shadowing

### Production Validation Smoke (SMOKE)

- [ ] **SMOKE-01**: `curl -A "GPTBot" https://full-selfbrowsing.com/` returns a complete HTML document with a populated `<title>`, `<meta name="description">`, `<link rel="canonical">`, OpenGraph tags, and at least one `<script type="application/ld+json">` block; the `<app-root>` element is non-empty (contains prerendered DOM)
- [ ] **SMOKE-02**: `curl -A "GPTBot" https://full-selfbrowsing.com/about` (and `/privacy`, `/support`) each return the route-specific prerendered HTML with route-specific `<title>` and `<meta name="description">` -- not the home page; canonical href matches the requested route
- [ ] **SMOKE-03**: `curl https://full-selfbrowsing.com/robots.txt`, `/sitemap.xml`, `/llms.txt`, `/llms-full.txt` each return HTTP 200 with the correct `Content-Type` header and non-empty body; the URLs listed in `sitemap.xml` resolve to HTTP 200 prerendered HTML
- [ ] **SMOKE-04**: Google Search Console "Test live URL" passes for `/`, `/about`, `/privacy`, `/support` (rendered HTML matches prerendered HTML; no `noindex` leaks; no JavaScript-only content); Rich Results Test passes for the home page's `Organization` and `SoftwareApplication` JSON-LD with zero errors

## Out of Scope (explicit exclusions)

These are explicitly NOT being done in v0.9.46:

- **Angular Universal full SSR** -- overkill for static marketing pages; static prerender via `@angular/ssr` outputMode is the right tool. Angular Universal adds runtime infra and complexity for no gain on routes whose content is build-time-known.
- **Webpack-era prerender plugins (`@nguniversal/express-engine`, `prerender-spa-plugin`)** -- the project uses `@angular/build:application` (modern esbuild-based builder); legacy plugins do not apply.
- **`changefreq` and `priority` in sitemap.xml** -- Google explicitly ignores both in 2026; including them is dead weight.
- **`noai` / `noimageai` meta tags** -- FSB *wants* AI ingestion (the entire point of GEO is to be cited). These tags would defeat the milestone goal.
- **`HowTo` JSON-LD** -- Google deprecated rich snippet support for `HowTo`. Not worth the schema surface.
- **`WebSite` JSON-LD with `SearchAction`** -- the showcase has no on-site search; misleading schema would invite Google's "site search" entity to be wrong.
- **Multiple `Organization` JSON-LD blocks** -- exactly one Organization at site root, inherited by every prerendered route via `index.html`. Duplicate blocks confuse Google's entity resolution.
- **Angular 19 -> 20 migration** -- Angular 19 EOL is 2026-05-19; tracked separately. v0.9.46 ships under Angular 19; the upgrade is a future milestone.
- **New build system / bundler / transpiler** -- project explicitly forbids it (CLAUDE.md). The Node prebuild script is plain ESM, no transpiler.
- **WebP / AVIF image conversion** -- out of scope for the discoverability milestone; revisit during a future performance pass if Core Web Vitals INP becomes a problem.
- **Service worker / PWA manifest** -- no offline capability is needed for a marketing site; would also conflict with the prerender lifecycle if added carelessly.

## Known Tech Debt

- **Angular 19 EOL: 2026-05-19** -- the showcase Angular shell must migrate to Angular 20 before this date. Out of scope for v0.9.46; explicit milestone-after-next deadline tracked in PROJECT.md.
- **`/dashboard` runtime-state HTML is not prerendered** -- accepted by design (interactive surface). Search engines will JS-render it at crawl time; META-04's `noindex` ensures they do not index a transient state snapshot.
- **Single-region Fly.io (`sjc`) + `auto_stop_machines = 'stop'`** -- non-US AI crawlers may hit cold-start latency. Mitigation: `min_machines_running = 1` already set. Monitor via Search Console crawl stats post-launch; revisit if crawl budget becomes a concern.
