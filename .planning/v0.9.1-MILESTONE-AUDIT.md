## End-to-End Flows

### Flow A: "Send a LinkedIn message" -- COMPLETE
10 integration points verified:
1. User types task in sidepanel -> background.js receives startAutomation
2. content.js extracts DOM with correct viewport detection (Phase 1)
3. ai-integration.js builds prompt with 15K budget, form-task prioritization (Phase 2)
4. AI responds with actions, background.js executes
5. After each action, changeSignals detect what changed (Phase 3)
6. Memory records "clicked 'Send' (#send-btn)" with element text (Phase 4)
7. Critical action registry records the send action with cooldown (Phase 5)
8. detectCompletionSignals finds success confirmation (Phase 5)
9. validateCompletion scores multi-signal and ends session (Phase 5)
10. Session idled (Phase 6), visible in history (Phase 7), replayable (Phase 8)

### Flow B: "Follow-up command in same conversation" -- COMPLETE
5 integration points verified:
1. First command completes -> session goes idle (Phase 6)
2. User sends follow-up in same popup/sidepanel
3. conversationId matches -> reactivateSession preserves AI history (Phase 6)
4. injectFollowUpContext adds separator to AI conversation (Phase 6)
5. Logger appends to same session entry (Phase 6)

### Flow C: "Replay a previous session" -- COMPLETE
7 integration points verified:
1. User opens history view (Phase 7)
2. Sees replay button on completed session with actions (Phase 8)
3. Clicks replay -> background.js loads actionHistory from storage
4. executeReplaySequence re-executes actions step-by-step via sendMessageWithRetry
5. Progress shown in sidepanel with percentage
6. Stop button can abort mid-replay
7. No AI involved, no token cost (isReplay flag, no AI instances)

**Score: 3/3 E2E flows verified (100%)**
