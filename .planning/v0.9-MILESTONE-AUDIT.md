## Gaps (Critical)

### 1. MCP-05: Autopilot Progress Streaming Bug

**Location:** `mcp-server/src/tools/autopilot.ts` lines 27-31

`run_task` sends `mcp:start-automation` via `bridge.sendAndWait()` but passes no `onProgress` callback. The `WebSocketBridge.progressListeners` map is never populated for that request. When `background.js` sends `mcp:progress` packets back (via `broadcastMCPProgress`), the bridge tries `progressListeners.get(resp.id)` but finds nothing.

Additionally, there is an id-space mismatch: `background.js` keys progress by `sessionId` (`mcpProgressCallbacks.set(result.sessionId, id)`), while `WebSocketBridge` expects messages keyed by its own WS message id from `generateId()`. The two never match.

**Impact:** MCP autopilot tasks complete successfully, but AI agents using `run_task` receive no intermediate progress updates.

### 2. MCP-10: Orphaned Native Host Manifest

**Location:** `mcp-server/native-host-manifest/com.fsb.mcp.json`

The install script (`install-host.cjs`) was correctly deleted by Phase 46-01, but the manifest template it installed was left behind. No code references this file. It is misleading dead weight that implies native messaging is still used.

**Impact:** No functional impact. Developer confusion only.
