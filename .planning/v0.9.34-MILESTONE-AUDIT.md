## Critical Gaps (Blockers)

### 1. MCP-04: Payment Confirmation Flow Broken

**Location:** `ws/mcp-bridge-client.js` `_handleUsePaymentMethod()`
**Issue:** Sends `{action: 'paymentFillConfirmation'}` via `_dispatchToBackground()` but background.js has no case handler for this action. Returns "User declined" instantly without user seeing the dialog.
**Fix:** Adopt the two-phase confirmation pattern from `ai/tool-executor.js`:
1. Broadcast `paymentFillConfirmation` to sidepanel
2. Register temporary `chrome.runtime.onMessage` listener for `paymentFillApproved`/`paymentFillDenied`
3. Timeout fallback (30s)

### 2. SEC-02: fill_credential Domain Not Derived from Active Tab

**Location:** `ws/mcp-bridge-client.js` `_handleFillCredential()` line 691
**Issue:** Uses `payload.domain` from MCP request for credential vault lookup. A malicious MCP client could request bank.com credentials while active tab is evil.com.
**Fix:** Derive lookup domain from active tab URL (same pattern as `_handleUsePaymentMethod` uses for merchant domain).
