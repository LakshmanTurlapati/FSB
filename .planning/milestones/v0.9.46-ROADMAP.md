## Overview

Make `full-selfbrowsing.com` discoverable to traditional search engines (Google, Bing) and generative AI search (ChatGPT, Claude, Perplexity, Gemini, Google AI Overviews) by prerendering the Angular SPA marketing routes (`/`, `/about`, `/privacy`, `/support`) and shipping LLM/crawler-aware root files. Pre-milestone, the site returned only the literal string "FSB" to non-JS crawlers — empty `<app-root>`. Static prerender + crawler files unblocks every downstream discoverability play.

## Phases

### Phase 215: Prerender Foundation, Per-Route Metadata & Structured Data

**Goal**: Production build emits per-route prerendered `index.html` files for the four marketing routes with route-specific metadata and structured data baked into the served HTML, so AI crawlers and traditional search engines can read what FSB is without executing JavaScript.
**Depends on**: Nothing (keystone phase)
**Plans**: 3 plans

Plans:

- [x] 215-01: Prerender pipeline foundation — install `@angular/ssr@^19`, configure `outputMode: "static"` + `provideServerRouting`, prerender-safe `localStorage`/`document` guards, emit per-route HTML to `dist/showcase-angular/browser/{,about,privacy,support}/index.html`
- [x] 215-02: Per-route metadata — `Title`/`Meta` services drive route-specific title, description, canonical, OG, and Twitter Card tags; runtime `noindex,nofollow` on `/dashboard`
- [x] 215-03: Structured data — Organization JSON-LD baked into `src/index.html` (inherited by every prerendered route); SoftwareApplication JSON-LD injected on home via `Renderer2` with `publisher.@id` cross-reference; T-LD-01 mitigated via `\u003c` escape

**Requirements satisfied:** PRE-01..05, META-01..04, LD-01, LD-02 (11 of 24)

### Phase 216: Crawler Root Files, Express Wiring & Production Validation

**Goal**: Ship LLM/crawler-aware root files at site root, wire the Express SPA fallback to prefer per-route prerendered HTML over the SPA shell while preserving `/dashboard` as a runtime route, and validate end-to-end against the live production deploy.
**Depends on**: Phase 215 prerender output
**Plans**: 5 plans

Plans:

- [x] 216-01: Static crawler root files — hand-author `robots.txt` (15 named LLM bot allowlists + default + Sitemap), `llms.txt` (llmstxt.org-format), and `llms-full.source.md` (long-form GEO source)
- [x] 216-02: Prebuild pipeline — single-file ESM Node script (`build-crawler-files.mjs`) generates `sitemap.xml`, `llms-full.txt`, and overwrites `version.ts` from `manifest.json.version` via npm `prebuild` lifecycle hook; zero new dependencies
- [x] 216-03: Express wiring — `server/server.js` patched: `setHeaders` callback pins `.txt`/`.xml` to `Cache-Control: public, max-age=3600`; custom middleware serves per-route prerendered HTML for `/`, `/about`, `/privacy`, `/support`; `/dashboard` exact-match returns SPA shell; 404 otherwise
- [x] 216-04: Production smoke — `smoke-crawler.mjs` (46 assertions: META + CRAWL + sitemap traversal) targets `BASE_URL` (defaults to production); zero new deps, uses Node 18+ `fetch`
- [x] 216-05: Manual UAT scaffold — `216-HUMAN-UAT.md` with 5 entries (Rich Results Test on home + 4× Search Console Live URL); `verify-uat.sh` asserts scaffold structure; sign-off footer marks the milestone gate
