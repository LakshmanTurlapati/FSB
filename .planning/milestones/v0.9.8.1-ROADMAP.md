# Milestone v0.9.8.1: npm Publishing

**Status:** ✅ SHIPPED 2026-04-02
**Phases:** 105-106
**Total Plans:** 2

## Overview

v0.9.8.1 turned the FSB MCP server into a real installable product instead of a repo-clone workflow. Phase 105 established the npm packaging and publish pipeline, and the milestone closed by finishing documentation, publishing the package publicly, and polishing the MCP install surface so users can launch FSB with `npx -y fsb-mcp-server` or the optional local HTTP mode without changing the browser extension.

## Phases

### Phase 105: Package & Distribution

**Goal**: Users can install and run the FSB MCP server via `npx -y fsb-mcp-server` without cloning the repo
**Depends on**: Nothing (first phase of milestone)
**Plans**: 2 plans

Plans:

- [x] 105-01: Package metadata, files whitelist, `.npmignore`, and `prepublishOnly` script
- [x] 105-02: GitHub Actions publish workflow and end-to-end local verification

**Details:**
Package metadata, npm filtering, and publish safeguards were added first. The server package now has a stable npm identity (`fsb-mcp-server`), constrained publish contents, and a build-before-publish guard. CI/CD then moved publishing to a tag-driven GitHub Actions flow with local verification for clean tarball output, CLI startup, and publish readiness.

### Phase 106: Documentation

**Goal**: Users can configure the FSB MCP server in their preferred MCP client by following the README
**Depends on**: Phase 105
**Plans**: Completed during release closeout (no standalone plan files)

Plans:

- [x] Release closeout: package README and root README updated for npm install and MCP host setup
- [x] Release closeout: local HTTP mode, diagnostics commands, and release notes finalized for the shipped package

**Details:**
Documentation closed the gap between a technically publishable package and a usable install experience. The MCP docs now explain stdio usage, optional local HTTP transport, setup/status/doctor/wait-for-extension commands, and the unchanged extension pairing model on `ws://localhost:7225`. The release was published publicly as `fsb-mcp-server@0.4.0` with matching GitHub release notes and registry metadata.

## Milestone Summary

**Decimal Phases:**

- None

**Key Decisions:**

- Keep the extension pairing contract unchanged on `ws://localhost:7225` so MCP install improvements happen entirely on the server side.
- Add local Streamable HTTP as an optional second transport on `127.0.0.1:7226` instead of replacing stdio.
- Defer `.mcpb` packaging to a follow-up release instead of blocking the npm release.

**Issues Resolved:**

- Users no longer need to clone the repo to run the MCP server.
- The MCP package now has a public npm release, release workflow, and packaged CLI entrypoint.
- MCP installation docs now cover actual host setup and troubleshooting instead of a source-only workflow.

**Issues Deferred:**

- `.mcpb` packaging for one-click desktop distribution
- Broader remote/public MCP deployment beyond the local extension companion model

**Technical Debt Incurred:**

- v0.9.8.1 never had a canonical milestone-scoped `REQUIREMENTS.md`; the requirements archive for this milestone is retrospective and reconstructed from roadmap, summaries, verification artifacts, and shipped package state.
