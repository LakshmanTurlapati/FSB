      1. Looks up the tool's `_emitChangeReport` flag from `tool-definitions.js`.

      3. Calls `buildChangeReport(beforeState, afterState, mutations, { crossOrigin, partial })` where `crossOrigin` is true if pre/post `chrome.tabs.get(tabId).url` origins differ AND content script communication failed for afterState (D-08), and `partial` is true if the 500ms safety net was hit (D-09).

    - On any error during harvest/build (try/catch around the entire wrap), the dispatcher logs a diagnostic and returns the original response WITHOUT change_report -- the action result itself is never blocked by report failures.
    - The pre-existing response shape (`success`, `message`, tool-specific fields) is byte-identical when `change_report` is absent. Adding `change_report` is purely additive.

        const stableP = waitForDOMStable(/* existing args */);
        const safetyP = new Promise(r => setTimeout(() => r('safety'), 500));
        const settleResult = await Promise.race([stableP, safetyP]);
        const partial = settleResult === 'safety';

        const afterUrl = (await chrome.tabs.get(tabId)).url;
        const beforeOrigin = new URL(beforeUrl).origin;
