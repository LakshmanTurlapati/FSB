  <files>extension/utils/action-verification.js</files>
  <behavior>
    - startMutationHarvest(targetSelector) returns a handle with { observer, mutations: [], startedAt, scopeRoot }; when targetSelector resolves to an element, scopeRoot is the nearest stable ancestor (closest <form>, <dialog>, <main>, or 3 levels up); when targetSelector is null/unresolvable, scopeRoot is document.documentElement.
    - stopMutationHarvest(handle) disconnects the observer, returns { mutations: MutationRecord-like[], settle_ms: Date.now() - handle.startedAt }.
    - buildChangeReport(beforeState, afterState, mutations, options) returns the D-04 shape; applies filter rules from D-03 (drop style-only attribute mutations, drop animation-class-only mutations matching /-(enter|leave|active|show|hide)|^animate-|^motion-/, drop scrollTop/scrollLeft attribute mutations, drop mutations on aria-hidden=true elements that stayed hidden, drop text mutations <3 chars).
    - buildChangeReport with options.crossOrigin === true short-circuits to URL-only report with cross_origin: true, mutation_count: 0, settle_ms: 0, all arrays empty, truncated: false (D-08).
    - dialogs_opened detected by added nodes matching <dialog>, [role="dialog"], [role="alertdialog"], or .modal/.popup with offsetWidth > 0.
    - inputs_changed derived from beforeState.inputValues vs afterState.inputValues delta (matches existing comparePageStates logic).
    - focus_shift derived from before/after document.activeElement selectors (caller passes activeElement before/after via beforeState.activeElementSelector / afterState.activeElementSelector).
    - applyChangeReportSizeCap(report) measures JSON.stringify(report).length; if > 2400 bytes, slice arrays to top N (3 dialogs_opened, 5 nodes_added, 5 nodes_removed, 8 attrs_changed), set truncated: true; returns { report, truncated }. The 600-token (~2400 byte) cap is per D-04.
    - All new functions exported via the existing module.exports (or window.* / globalThis.* mirror if file currently uses script-tag globals -- match existing pattern of capturePageState/comparePageStates).
  </behavior>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| page DOM -> harvest pipeline | Untrusted page content (selectors, text, attribute values) flows into change_report |
| change_report -> MCP response | Serialized report flows out to the MCP host (Claude Code, etc.) over WS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-245-01 | I (Information disclosure) | buildChangeReport | mitigate | Cap text snippets at 80 chars per added/removed node; never include full innerHTML; existing capturePageState already truncates bodyText to 1000 chars |
| T-245-02 | D (Denial of service) | startMutationHarvest scope | mitigate | Scope per D-02 (ancestor-of-target, not document) keeps observer cheap; size cap per D-04 prevents pathological serialization; <25ms p95 budget per D-09 |
| T-245-03 | T (Tampering -- malicious page injects misleading mutations) | buildChangeReport selectors | accept | Selectors are advisory; the agent treats change_report as hint, not as authoritative state; downstream `read_page` still available |
| T-245-04 | I | cross-origin path D-08 | mitigate | When `options.crossOrigin: true`, builder skips DOM inspection entirely and emits URL-only report -- no leak of cross-origin DOM into report |
</threat_model>
