# Changelog

## 0.8.0 (yyyy-mm-dd)

### Multi-Agent Tab Concurrency
- New per-session/task `agent_id` minted by FSB via `crypto.randomUUID()`. Callers cannot supply IDs.
- Configurable concurrency cap (default 8, range 1-64) persisted in `chrome.storage.local`. The (N+1)th agent claim rejects with `AGENT_CAP_REACHED { cap, active }`.
- Tab-ownership enforcement on every MCP tool dispatch via the new gate in `extension/ws/mcp-tool-dispatcher.js`. Cross-agent calls reject with `TAB_NOT_OWNED`. Incognito tabs reject with `TAB_INCOGNITO_NOT_SUPPORTED`. Cross-window tabs reject with `TAB_OUT_OF_SCOPE`.
- Per-bindTab `ownership_token` (fresh `crypto.randomUUID()` per binding) prevents tab-ID-reuse exploitation.
- Forced-new-tab pooling via `chrome.tabs.onCreated + openerTabId`: opening a forced-new tab does NOT count as a new agent against the cap; the pool releases only when its last tab closes.
- Lock release on: task/session ends, MCP client disconnects (after `RECONNECT_GRACE_MS` ~10s grace keyed by `connection_id`), user closes the tab. No idle timeout.
- Service-worker eviction recovery: agent registry mirrors to `chrome.storage.session` (write-through); on SW wake, hydrate reconciles persisted records against `chrome.tabs.query()` and reaps ghosts before servicing any request.

### `back` Tool (BACK-01..05)
- New ownership-gated `back` MCP tool: single-step browser-history back on the agent's owned tab.
- Structured result: `{ status: 'ok' | 'no_history' | 'cross_origin' | 'bf_cache' | 'fragment_only', resultingUrl, historyDepth }`.
- `pageshow`-based settle verification with 2s timeout; cross-origin transitions reuse the v0.9.11 BF-cache resilience path to re-inject the content script.
- Background-tab compatible: does not steal focus.

### `run_task` Return-on-Completion (Phase 236 reborn -- MCP-03..06)
- `run_task` now returns when the underlying automation actually completes via `fsbAutomationLifecycleBus.dispatch('automationComplete')`, not at an arbitrary 300s ceiling.
- 300s ceiling raised to 600s safety net (kept until cleanup-path audit confirms zero dropped lifecycle events).
- 30s heartbeat ticks emitted via `_sendProgress` / `notifications/progress` with rich fields under `params._meta` (alive, step, elapsed_ms, current_url, ai_cycles, last_action) so MCP host clients (Claude Code, Cursor, Codex, OpenClaw) do not hit per-tool timeouts.
- Task lifecycle persisted in `chrome.storage.session` keyed by `task_id`; SW eviction during a long task can resume and still settle the originating MCP call. The server emits a `sw_evicted` partial outcome when the bridge cannot recover.

### Tool Description Updates (MCP-07)
- Every manual tool description now documents: `agent_id` is FSB-issued and required, `tab_id` is agent-scoped, the cap is configurable, ownership is enforced, and typed error codes are enumerated.
- `run_task`, `start_visual_session`, `end_visual_session`, and `back` carry the full multi-agent contract block.

### UI / Observability
- v0.9.36 trusted-client badge extended with short `agent_id` (e.g. "Claude / agent_a3f1") on the page overlay and dashboard mirror.
- Sidepanel and popup show a read-only "owned by Agent X" badge on owned tabs (visibility only, no enforcement).
- `options.html` Advanced Settings exposes a Concurrency Cap control (numeric input 1-64, default 8, helper text, current-active counter, reset-to-default).

### Background-Tab Execution
- Audited 25+ MCP / autopilot tools for `chrome.tabs.update({active: true})` side effects. Tools that genuinely require focus opt in via a per-tool `force_foreground` flag in `tool-definitions.cjs` (default false). All others execute on background tabs without focus-stealing.
- `webNavigation.onCommitted` detects user-initiated navigation on agent-owned tabs and emits a pause signal.

### Dependencies
- `@modelcontextprotocol/sdk` bumped from `^1.27.1` to `^1.29.x`. Zod stays on `^3.x`.

### Migration Notes
- Existing single-agent surfaces (popup, sidepanel, autopilot) continue to work unchanged via synthesized `agent_id = 'legacy:<surface>'` -- no v0.9.36 / v0.9.50 regression.
- MCP clients that currently send `agent_id` will have it ignored (server captures via `agent:register`).
- The default cap of 8 is sufficient for almost all multi-agent workflows; raise via `options.html` Advanced Settings if needed.
