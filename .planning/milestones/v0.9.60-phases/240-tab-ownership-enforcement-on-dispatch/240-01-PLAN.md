## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP server -> extension SW | Tool dispatch crosses bridge with `{agentId, ownershipToken}` payload — token never returns to MCP server in plain form except via the agent:register response (Open Question 1 resolution: yes, ownershipTokens map carried in response per Plan 02) |
| Extension SW -> registry storage | Registry writes through to `chrome.storage.session` — token persists for browser-session lifetime; never crosses to disk |
| registry consumer (gate) -> registry data | Gate reads via sync APIs (`isOwnedBy`, `getTabMetadata`); MUST NOT mutate registry state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-240-01 | Spoofing | Stolen agentId attack on bindTab/isOwnedBy | mitigate | D-04 ownershipToken: bindTab mints fresh per-bind token; isOwnedBy 3-arg requires token match. Even with stolen agentId, attacker without token fails the gate. **Phase 240 owns the enforcement; Plan 01 owns the registry-side check.** |
| T-240-02 | Tampering | Malicious bindTab spoofing — attacker tries to bind a tab they don't own | mitigate | bindTab inside `withRegistryLock` with cross-owner displacement-returns-false at line 277-280 (Phase 237 contract); Plan 02's chokepoint requires ownership BEFORE bindTab fires for non-creating handlers (D-08 sites bind only on success path, after the gate has already validated agentId/ownershipToken on caller's existing tabs) |
| T-240-03 | Information Disclosure | Token leaks across the bridge | accept (limited) | Token crosses bridge via agent:register response (Open Q1) and back via every subsequent dispatch payload. Same channel as agentId; no new disclosure surface. **Defense-in-depth assumption: a fully-malicious MCP host has both. Token defends against agentId-only attacks (most common: log scraping)** |
| T-240-04 | Elevation of Privilege | legacy:* prefix used by an attacker to register `legacy:adversary` | mitigate | getOrRegisterLegacyAgent's ALLOWED map hard-codes the 3 surfaces; arbitrary surface returns `{error: 'unknown_legacy_surface'}` — no enumeration. registerAgent (the public entry point) ignores any caller-supplied id including `legacy:*` per D-12 |
| T-240-05 | Denial of Service | Same-agent re-entry race causing concurrent bindTab on same tab | mitigate | bindTab is wrapped in `withRegistryLock` (Phase 237 promise-chain mutex); concurrent calls serialize. Last-write-wins on token (a re-bind mints a fresh token; old token is invalidated as Pitfall 2 documents). **Visual-session same-agent resume (Plan 03 D-03) is the user-facing path; this plan's bindTab is the underlying invariant.** |
| T-240-06 | Backward-Compat Regression | Phase 237 callers that ignored bindTab's boolean return now break on shape change | mitigate | bindTab return is `{agentId, tabId, ownershipToken}` for SUCCESS and `false` for failure. Callers that use truthy-check (`if (await bindTab(...)) ...`) still work. Callers that explicitly compare to `=== true` would break — Plans 02/03 audit each call site. **All Phase 237 callers in the codebase use truthy-check (verified at Phase 237 plan-03 wiring)**. |
