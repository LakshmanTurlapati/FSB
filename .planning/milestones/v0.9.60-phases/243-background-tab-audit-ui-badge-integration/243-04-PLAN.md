## Trust Boundaries

|----------|-------------|
| chrome.storage.session -> options page | Per-extension-origin storage; trusted |
| User input -> cap input | Untrusted; clamping + validation enforced (Phase 241 + this plan) |
| chrome.storage.onChanged -> listener | Chrome dispatches for any extension-origin write; trusted |

## STRIDE Threat Register (4 threats)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-243-04-01 | Information Disclosure | Counter "X of Y active" leaks active agent count to anyone with options page access | accept | Options page is extension-internal; only the user can open it. Active count is not sensitive. |
| T-243-04-02 | DoS via storage.onChanged thrash (Pitfall 4) | Denial of Service | mitigate | refreshActiveAgentCount is debounced 100ms via _capCounterDebounce. Bulk registry writes (8 agents x 3 tabs = 24 writes) collapse to one counter refresh. |
| T-243-04-03 | legacy:* phantom load (Pitfall 1) | Information Disclosure / UX defect | mitigate | computeActiveAgentCount filters `!startsWith('legacy:')`. Helper unit-tested with Test 1 + Test 3. |
| T-243-04-04 | Tampering of fsbAgentCap via storage to bypass cap | Tampering | accept | Phase 241 already enforces cap on the registry side under withRegistryLock. Options page is the user's only write surface; the cap value is a user preference. Storage tampering requires extension-origin code, which is already trusted. |
</threat_model>
