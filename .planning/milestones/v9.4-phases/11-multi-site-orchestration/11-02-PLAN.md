```
// Detect multi-company career task
if (typeof extractCompaniesFromTask === 'function') {
  const companies = extractCompaniesFromTask(session.task);
  if (companies && companies.length >= 2) {
    // Initialize multi-site orchestration state
    session.multiSite = {
      originalTask: session.task,
      companyList: companies,
```

**2. Helper functions (add near the orchestrator code):**

`extractSearchQuery(taskString)`: Extract role keywords from the task. Take words before "at" in the string, filter out verbs (find, search, look, get, show, list, check, browse). Return as lowercase space-joined string. Example: "find DevOps engineer jobs at Microsoft, Amazon" -> "devops engineer jobs".

`buildSingleCompanyTask(originalTask, companyName)`: Rewrite the multi-company task to single-company. Replace the company list portion with just the target company name. Example: "find DevOps jobs at Microsoft, Amazon, and Google" with companyName="Amazon" -> "find DevOps jobs at Amazon". Use regex to replace the "at [company list]$" portion with "at [companyName]".

`initMultiSiteAccumulator(session)`: Read existing accumulator from chrome.storage.local ('fsbJobAccumulator'). Call checkAccumulatorRelevance with existing accumulator and session.multiSite.searchQuery. If result is 'clear', write a fresh accumulator: `{sessionId: session.sessionId, searchQuery: session.multiSite.searchQuery, startedAt: Date.now(), companies: {}, totalJobs: 0, completedAt: null}`. If 'keep', update the sessionId and startedAt on the existing accumulator.

**4. handleMultiSiteCompletion(sessionId, session, aiResponse) async function:**

This is the core orchestrator logic. Steps:

a. Get current company: `session.multiSite.companyList[session.multiSite.currentIndex]`
b. Check AI result for auth wall: if `aiResponse.result` contains "AUTH REQUIRED", defer the company:
   - Push to `session.multiSite.deferredCompanies` with name and reason
   - Update accumulator in chrome.storage.local: set company status to 'auth_required' with error message
   - Log the deferral
c. Check AI result for failure (PAGE ERROR, timeout): push to `session.multiSite.failures` with company, reason, error text
d. Advance index: `session.multiSite.currentIndex++`
e. Check if more companies remain (index < companyList.length):
   - YES: Reset session for next company:
     - `session.task = buildSingleCompanyTask(session.multiSite.originalTask, nextCompany)`
     - `session.iterationCount = 0`
     - `session.stuckCounter = 0`
     - `session.actionHistory = []`
     - `session.domHashes = []` (reset stuck detection)
     - `session.taskSummary = "Job search: ${currentIndex+1}/${total} companies"`
     - Send ProgressOverlay update via sendSessionStatus with taskSummary
     - `setTimeout(() => startAutomationLoop(sessionId), 500)` (small gap for page transition)
     - Return true (handled)
   - NO, but deferred companies exist:
     - Start deferred company processing (see step f)
     - Return true (handled)
   - NO, no deferred companies:
     - Call `finalizeMultiSiteSearch(sessionId, session)`
     - Return false (let normal completion flow run with finalized data)

f. Deferred company handling: For each deferred company (auth-walled):
   - Check if user may have logged in: look at the session's action history or check chrome.tabs for URL on that company's domain that is NOT a login page (heuristic: URL does not contain /login, /signin, /auth, /sso)
   - If login detected: add company back to the queue as next to search, reset and re-launch loop, return true
   - If login NOT detected: mark in accumulator as 'auth_required', add to failures with "User did not authenticate"
   - After all deferred companies processed: call finalizeMultiSiteSearch, return false

**5. finalizeMultiSiteSearch(sessionId, session) async function:**

a. Read accumulator from chrome.storage.local
b. Flatten all jobs across all companies
c. Call deduplicateJobs on the flat array
d. Update accumulator: set dedupedJobs count, set completedAt timestamp
e. Write updated accumulator back to chrome.storage.local
f. Build final result string:
   - If zero jobs total: check if all failures vs no results. Per locked decision: AI gives specific summary distinguishing "no results found" from "site failure"
   - If has jobs: set session result to a concise summary. Per locked decision: final chat summary is context-dependent -- concise natural language (not full data table) unless user explicitly asked for listing
   - Append failure summary: "Failures: [company]: [reason]" for each failure. Per locked decision: all failures listed in final summary
g. Set aiResponse.result on the session or store for the completion handler to use
h. Persist orchestration state to chrome.storage.session via persistSession
