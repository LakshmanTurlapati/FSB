**1. Existing sheet append mode detection:**
In the Sheets data entry directive (the block added in Plan 12-01), add an append-mode instruction section.

After the "WRITING PROCEDURE:" block, add:
```
EXISTING SHEET DETECTION (if switching to an existing Sheets tab):
If the sheet already has data:
1. Click Name Box, type "A1", press Enter
2. Read formula bar -- if A1 has content, this is an existing sheet
3. Tab through columns to read existing headers via formula bar
4. Map existing headers to your column order (fuzzy match: "Role" = Title, "Firm" = Company, "Link" = Apply Link, etc.)
5. Press Ctrl+End to find the last row with data
6. Read the Name Box to get the last cell reference
7. Navigate to the first empty row below existing data
8. Write data matching the EXISTING column order (not the default order)
9. Do NOT rewrite headers -- append data only
```

**2. Ensure 'multitab' task type for Sheets data entry:**
In the `detectTaskType` method, add an early check: if the task string contains "job listings to Google Sheets" (the rewritten task from the orchestrator), return 'multitab' immediately. This ensures the correct tool set is available. Place this check early in the method, before the general keyword matching.

This early check is necessary because the rewritten task from the orchestrator is: "Write N job listings to Google Sheets...". The word "write" is NOT present in the `gatherActions` arrays (which contain: find, search, research, get, look up, check, go to, visit). Without this early check, the existing output-destination detection (`hasOutputDest && hasGatherAction`) will match "google sheet" but fail on `hasGatherAction` because "write" is not a gather action. The task would fall through to a generic type and lack the multi-tab tools (switchToTab, openNewTab, listTabs, waitForTabLoad) required for Sheets tab management.
