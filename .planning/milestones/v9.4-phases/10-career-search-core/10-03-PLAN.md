Modify the `_buildTaskGuidance` method (lines ~3921-3983) to support company-name-based guide lookup for ALL career tasks, regardless of whether a URL-based siteGuide already matched.

The current flow: `_buildTaskGuidance(taskType, siteGuide, currentUrl)` -- if siteGuide is null, falls back to `TASK_PROMPTS[taskType]`.

IMPORTANT: `getGuideForTask` keyword fallback often returns a non-null siteGuide for career tasks (the first matching career guide in the registry), even when that guide is NOT the specific company the user asked about. Therefore, company name extraction must ALWAYS run for career tasks, not only when siteGuide is null.

New flow for career tasks:
1. Modify the method signature to accept an optional 4th parameter: `_buildTaskGuidance(taskType, siteGuide, currentUrl, task = null)`
2. If `taskType === 'career'` (ALWAYS, regardless of siteGuide being null or not):
   a. If task is available, call `extractCompanyFromTask(task)` to get company name
   b. If company name found, call `getGuideByCompanyName(companyName)` to get the company-specific guide
   c. If company-specific guide found AND it differs from siteGuide (different `.site` value):
      - Use the company-specific guide as siteGuide (override the keyword-fallback match)
   d. If the active guide (company-specific or original siteGuide) has `careerUrl`:
      - Prepend to the career prompt: `\n\nDIRECT CAREER URL: Navigate directly to ${guide.careerUrl} -- do NOT Google search for this company's career page.`
   e. If no company name extracted or no guide found: keep original siteGuide as-is
   f. If siteGuide is still null after all lookups: fall back to `TASK_PROMPTS.career` (generic career prompt)
3. For non-career tasks, existing logic unchanged

Also update the call site in `buildPrompt` (line ~2212) to pass the task string:
```javascript
// Change from:
${this._buildTaskGuidance(taskType, siteGuide, currentUrl)}
// To:
${this._buildTaskGuidance(taskType, siteGuide, currentUrl, task)}
```

The `task` variable is already available in buildPrompt scope (it's the first parameter).

IMPORTANT: The functions `extractCompanyFromTask` and `getGuideByCompanyName` are defined in site-guides/index.js which is loaded via importScripts in background.js. They are global functions accessible in the service worker scope. Use `typeof extractCompanyFromTask === 'function'` guard before calling (same pattern used for other site-guide functions in this method).
