<objective>
Fix batch action behavior on Google Sheets so that multiple type actions for different spreadsheet cells are never batched together.

Purpose: When the AI batches type actions for cells A1, B1, C1 on Google Sheets, all values concatenate into the active cell because Sheets uses a canvas-based grid (cells are not independent DOM elements). This causes data corruption and wasted iterations. The fix adds both a prompt-level warning (teach the AI) and an execution-level safety check (catch mistakes).

Output: Updated BATCH_ACTION_INSTRUCTIONS constant with Sheets-specific "WHEN NOT TO BATCH" entry, plus a URL-based batch suppression guard in executeBatchActions with graceful fallback handling in startAutomationLoop.
</objective>

**Change A: URL-based safety check in executeBatchActions (line 5891)**

After `const sessionId = session?.sessionId;` (line 5897) and before the `automationLogger.info('Starting AI-declared batch execution'` log (line 5904), insert the Google Sheets batch suppression check:

```javascript
// SAFETY: Suppress type-heavy batches on Google Sheets (canvas-based grid)
// Sheets cells are canvas-rendered, not DOM elements. Batching multiple type actions
// concatenates all values into the active cell instead of typing into separate cells.
try {
  const tab = await chrome.tabs.get(tabId);
  const currentUrl = tab?.url || '';
  if (/docs\.google\.com\/spreadsheets\/d\//i.test(currentUrl)) {
    const typeActionCount = actions.filter(a => a.tool === 'type').length;
    if (typeActionCount >= 2) {
      automationLogger.warn('Batch suppressed: Google Sheets canvas grid -- type actions cannot target separate cells', {
        sessionId,
        typeActionCount,
        totalActions: actions.length,
        tools: actions.map(a => a.tool)
      });
      return {
        batched: false,
        suppressed: true,
        reason: 'google_sheets_canvas_grid',
        actions,
        results: [],
        totalCount: actions.length,
        successCount: 0,
        failedAt: -1,
        duration: 0,
        skippedActions: actions
      };
    }
  }
} catch (urlCheckError) {
  // URL check is a safety measure -- don't block batch execution if it fails
  automationLogger.debug('URL check for batch suppression failed', { error: urlCheckError.message });
}
```

Key details:
- URL regex: `/docs\.google\.com\/spreadsheets\/d\//i` -- requires document ID path segment (not just homepage)
- Threshold: 2+ type actions (single type action is valid on Sheets)
- Return shape: matches existing `executeBatchActions` return structure but with `batched: false` and `suppressed: true`
- Wrapped in try/catch because URL check is non-critical (don't break batch execution if chrome.tabs.get fails)
