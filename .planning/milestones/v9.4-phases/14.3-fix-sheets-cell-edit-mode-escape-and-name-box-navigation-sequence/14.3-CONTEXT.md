# Phase 14.3: Fix Sheets cell edit mode escape and Name Box navigation sequence - Context

**Gathered:** 2026-02-26
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix the AI's failure to exit cell edit mode before Name Box navigation on Google Sheets. When the AI types data into a cell and then tries to navigate to another cell via the Name Box, the cell reference gets appended to the current cell's content instead of going into the Name Box. The fix adds an explicit Escape step before every Name Box navigation in prompts, site guide, and continuation reminders.

</domain>

<decisions>
## Implementation Decisions

### Escape before Name Box navigation
- Every Name Box navigation MUST be preceded by pressing Escape to exit cell edit mode
- This mirrors the existing formatting STEP 0 pattern: "Press Escape to ensure you are not in cell edit mode"
- The 5-step procedure in TASK_PROMPTS becomes 6 steps: Escape -> Click Name Box -> Type ref -> Enter -> Type data -> Tab/Enter

### Prompt locations to fix (3 locations)
- **Site guide** (`site-guides/productivity/google-sheets.js`): Add Escape step to the NAME BOX NAVIGATION section and all workflows that begin with "Click the Name Box"
- **Task prompt** (`ai/ai-integration.js` TASK_PROMPTS multitab section): Insert Escape as Step 0 before the current Step 1 (Click Name Box), renumber steps to 0-5
- **Continuation reminder** (`ai/ai-integration.js` buildMinimalUpdate GOOGLE SHEETS REMINDER): Add Escape step to the cell navigation reminder line

### Workflow updates in site guide
- These workflows start with "Click the Name Box" and need Escape prepended: `setupHeaderRow`, `enterRowData`, `navigateToCell`, `dataEntrySequential`, `formulaBarVerification`, `appendToExistingSheet`, `applyAlternatingColors`, `applyLinkColumnBlueText`
- Workflows that already have Escape (formatting workflows): `renameSheet`, `formatHeaderRow` -- leave as-is
- The `enterHyperlinkFormula` workflow says "Navigate to the target cell via Name Box" -- add explicit note to Escape first

### Warning addition
- Add a new warning to the site guide warnings array: "CRITICAL: Press Escape before clicking the Name Box to exit cell edit mode. If you type a cell reference while still in edit mode, the reference will be appended to the current cell's content instead of navigating."

### Claude's Discretion
- Exact wording of the Escape step in each location (as long as intent is clear)
- Whether to use keyPress Escape or a more descriptive phrase
- Whether to add a verification note (e.g., "confirm Name Box is focused before typing cell reference")

</decisions>

```
STEP 0 -- EXIT EDIT MODE:
```

Apply the same pattern to data entry Name Box navigation.
