<objective>
Fix the AI confusing Name Box cell navigation with cell data entry on Google Sheets by adding explicit disambiguation prompts at three injection points: the site guide guidance text, the multitab task prompt, and the continuation iteration minimal update.

Purpose: The AI currently conflates "type a cell reference into the Name Box" with "type a data value into the active cell" because both use the `type` tool but target different elements. This causes the AI to type cell references (e.g., "b1") as data values, corrupting spreadsheet content. The fix is purely prompt-level -- the execution engine works correctly.

Output: Enhanced prompt text in google-sheets.js and ai-integration.js that clearly separates the two-step process (navigate via Name Box, then type value) and reinforces it on continuation iterations.
</objective>

**Implementation:**

After the stuck warning block (after line 780, before the taskType detection at line 783), add a new conditional block:

```javascript
// Sheets-specific reminder on continuation iterations
// The full site guide guidance is only in the first-iteration system prompt.
// Reinforce the critical Name Box vs cell value distinction on every continuation.
const currentUrl = context?.currentUrl || domState.url || '';
if (/docs\.google\.com\/spreadsheets\/d\/|sheets\.google\.com/i.test(currentUrl)) {
  update += `\n\nGOOGLE SHEETS REMINDER:`;
  update += `\n- To navigate to a cell: click Name Box (#t-name-box), type cell reference (e.g., "B1"), press Enter.`;
  update += `\n- To enter data: type the value (keystrokes go to the active cell). Do NOT type cell references as data values.`;
  update += `\n- Tab = move right, Enter = move down. Use Tab between columns in the same row.`;
}
```

This is 4 lines of reminder text -- compact enough to not bloat the continuation prompt but explicit enough to prevent the confusion. The URL regex uses `/d/` to match only open Sheets documents (same pattern as the Phase 14.1 batch suppression guard).

**Why this location:** The buildMinimalUpdate method constructs the user message content for continuation iterations (iteration 2+). The system prompt for continuations is MINIMAL_CONTINUATION_PROMPT which has zero Sheets guidance. By appending to the user message content in buildMinimalUpdate, we inject the reminder without modifying the system prompt constant. This is the same approach used for CAPTCHA, stuck, and modal warnings.

Do NOT modify the MINIMAL_CONTINUATION_PROMPT constant itself (line 361-387). The constant should remain generic for all site types. The Sheets-specific content belongs in the conditional injection within buildMinimalUpdate.
