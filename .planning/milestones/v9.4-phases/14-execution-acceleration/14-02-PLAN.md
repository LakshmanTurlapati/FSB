<objective>
Implement the batch action execution engine that lets the AI return multiple same-page actions in a single response, executed sequentially with DOM-based completion detection between each action.

Purpose: Currently every action requires a full AI round-trip (DOM analysis, API call, response parsing). Batching same-page actions (form fills, checkbox selections, data extraction) eliminates redundant round-trips, reducing task completion time by 50-80% for multi-action pages.

Output: executeBatchActions() function in background.js, batchActions response format support in ai-integration.js, batch instructions in AI system prompt.
</objective>

- Signature: `async function executeBatchActions(batchActions, session, tabId)`
- Cap batch at MAX_BATCH_SIZE = 8 actions (slice input array regardless of AI compliance)
- Track batchStartTime for duration measurement
- Loop through actions sequentially. For each action:
  a. Route to appropriate handler using same logic as existing action loop:
     - `multiTabActions.includes(action.tool)` -> `handleMultiTabAction(action, tabId)`
     - `backgroundDataTools.includes(action.tool)` -> `handleBackgroundAction(action, session)`
     - Otherwise -> `sendMessageWithRetry(tabId, { action: 'executeAction', tool: action.tool, params: action.params, visualContext: { taskName: session.task?.substring(0, 50), stepNumber: i + 1, totalSteps: actions.length, iterationCount: session.iterationCount, isBatchedAction: true, animatedHighlights: session.animatedActionHighlights ?? true } })`
  b. Record result in results array: `{ tool, params, success, error, duration }`
  c. Push to session.actionHistory with batched: true, batchIndex: i, batchTotal: actions.length
  d. **STOP ON FAILURE**: if `!actionResult?.success`, log warning and break
  e. **STOP ON NAVIGATION**: if `actionResult?.navigationTriggered === true` and this is not the last action, log info and break (treat remaining as skipped, not failed)
  f. **COMPLETION DETECTION** between actions (skip after last action):
     - Navigation tools (navigate, searchGoogle, goBack, goForward) or navigationTriggered -> use `pageLoadWatcher.waitForPageReady(tabId, { maxWait: 5000 })`
     - All other tools -> use `sendMessageWithRetry(tabId, { action: 'executeAction', tool: 'waitForPageStability', params: { maxWait: 5000, stableTime: 300, networkQuietTime: 200 } })`
     - Wrap completion detection in try/catch -- on communication failure, log warning and break batch (content script may have disconnected)

- Return object: `{ batched: true, results, totalCount, successCount, failedAt (index or -1), duration, skippedActions }`

- Return them in a "batchActions" array instead of "actions"

- When any action might trigger page navigation (put it last or don't batch)
