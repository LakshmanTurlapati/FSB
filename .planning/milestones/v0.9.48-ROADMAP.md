## Milestone Summary

**Decimal Phases:** None.

**Key Decisions:**

- **Schematic-driven migration**: `ng update` did 95% of the work. `provideServerRouting` removal was the only meaningful API change and was auto-rewritten correctly.
- **Zoneless explicitly deferred**: Touching change detection mid-migration multiplies risk. zone.js retained at v20-compatible version; zoneless adoption is a future milestone.
- **TypeScript bump**: 5.6 → 5.9.3 picked up automatically by ng update; satisfies Angular 20's >=5.8 peer constraint.
- **CI Node version**: `node-version: '20'` in setup-node@v4 resolves to latest 20.x (≥20.19), which meets Angular 20's Node minimum.
- **Lockfile handling**: Phase 221's executor-generated lockfile was incomplete (missing nested @types/node entry); a follow-up fix commit regenerated it cleanly. Future migrations should validate `npm ci` from a clean tree before pushing.

**Issues Resolved:**

- Pre-milestone: `showcase/angular/` was on Angular 19 with EOL approaching 2026-05-19. Resolved.
- Pre-milestone: `provideServerRouting` API was scheduled for removal. Resolved (now using `provideServerRendering(withRoutes(...))`).

**Issues Deferred:**

- **Zoneless mode** — `provideExperimentalZonelessChangeDetection` (or stable equivalent) — future milestone.
- **Dead `serve:ssr:showcase-angular` npm script**: Angular 20 + `outputMode: "static"` no longer writes the SSR runtime to disk. The script references `dist/showcase-angular/server/server.mjs` which doesn't exist post-build. Production unaffected (Express serves static prerendered output). Cleanup belongs in a future docs/scripts pass.

**Technical Debt Incurred:**

- Dead `serve:ssr:showcase-angular` npm script — see above.
- Lockfile-drift risk: dev machines on Node 25 vs CI on Node 20 produced subtly different lockfiles. The Phase 221 follow-up fix addressed it; future migrations should regenerate the lockfile from a clean state before committing.
