# Requirements: FSB v0.9.3 Memory Tab Overhaul

**Defined:** 2026-03-16
**Core Value:** One automation = one memory -- a consolidated reconnaissance report with visual graph

## v0.9.3 Requirements

### Memory Model

- [ ] **MEM-01**: Unified Task Memory schema -- single object per session combining what happened (episodic), what was learned (semantic), and reusable patterns (procedural) into one memory with `type: "task"`
- [ ] **MEM-02**: Session-to-memory pipeline -- `extractAndStoreMemories` produces exactly one Task Memory per session instead of 1-5 fragmented type-based memories
- [ ] **MEM-03**: AI extraction prompt rewrite -- single consolidated prompt that returns one recon-style report (task, outcome, steps timeline, selectors discovered, site structure, patterns learned) instead of separate typed memories

### Display

- [ ] **DISP-01**: Task Memory card in Memory list -- one card per task showing task description, outcome badge (success/partial/failure), domain, duration, step count, and age
- [ ] **DISP-02**: Expanded detail view -- click card to see full recon report: task narrative, step-by-step timeline with outcomes, selectors/patterns discovered, site structure encountered, AI analysis
- [ ] **DISP-03**: Graph visualization per task -- each Task Memory renders a force-directed graph showing pages visited, elements interacted with, navigation paths taken during the task

### Storage & Migration

- [ ] **STOR-01**: New storage schema for Task Memory objects in `fsb_memories` with backward-compatible reader (old type-based memories still render until migrated)
- ~~**STOR-02**: Migration utility~~ -- DESCOPED: old memories stay as-is, no migration needed

### Consolidation

- [ ] **CONS-01**: Domain + task similarity deduplication -- consolidator matches task memories by domain and task description similarity (>= 0.7 threshold), merging repeat runs of the same task into one memory with combined run history, selectors, patterns, and procedures

### Theme & Export

- [ ] **THEME-01**: All Memory tab elements respect active theme — no white-on-dark or dark-on-light mismatches in cards, detail panels, graph containers, badges, collapsible sections, stats bar
- [ ] **EXPORT-01**: Export all memories as downloadable JSON file and import previously exported JSON with duplicate detection

## Out of Scope

| Feature | Reason |
|---------|--------|
| 3D Knowledge Graph overhaul | Existing knowledge graph is a separate visualization; focus on per-task graphs |
| Memory search rewrite | Existing hybrid search works; just needs updated index fields for new schema |
| Cross-site pattern analysis | Keep existing cross-site consolidation as-is; it operates on the new schema's learned patterns |
| Site Explorer / Research Results | These sections of Memory tab are unrelated to task memory display |
