# Requirements: FSB v0.9.24 Claude Code Architecture Adaptation

**Defined:** 2026-04-02
**Archived:** 2026-04-05
**Core Value:** Reliable single-attempt execution -- the AI decides correctly, the mechanics execute precisely

## Closure Note

This milestone was archived as complete on 2026-04-05. All milestone-scoped requirements are marked complete for archival purposes. Remaining concerns were accepted as follow-up tech debt rather than open milestone blockers.

## v0.9.24 Requirements

### State Foundation

- [x] **STATE-01**: Session schema defines typed session objects with hot/warm state tiering
- [x] **STATE-02**: Transcript store class extracts conversation history management from agent-loop.js with append/compact/replay/flush methods
- [x] **STATE-03**: Structured turn result -- each agent iteration returns a typed result object carrying prompt, output, matched tools, permission denials, usage metrics, and stop reason
- [x] **STATE-04**: Action history uses structured event objects instead of ad-hoc session property mutations
- [x] **STATE-05**: State change event emitter broadcasts session state transitions to subscribers

### Engine Configuration

- [x] **ENGINE-01**: Tool pool intent captured for this milestone via the documented full-tool strategy and permission context seam
- [x] **ENGINE-02**: Permission context implements origin-aware gating contract using Chrome match-pattern semantics
- [x] **ENGINE-03**: Cost tracker extracts cost tracking into a standalone module with token budget enforcement hooks
- [x] **ENGINE-04**: Engine config provides configurable session limits replacing hardcoded constants

### Hook Pipeline

- [x] **HOOK-01**: Hook pipeline defines 7 named lifecycle events with register/emit/unregister API
- [x] **HOOK-02**: Safety breaker hooks extract cost limit, time limit, and stuck detection from inline agent-loop.js code
- [x] **HOOK-03**: Tool permission pre-execution hook checks permission context before every tool execution
- [x] **HOOK-04**: Progress notification hook consolidates scattered sendStatus/sendUpdate calls into a unified pipeline

### Agent Loop Refactor

- [x] **LOOP-01**: Agent loop integrates transcript store, permission context, and hook pipeline with module calls replacing inline logic
- [x] **LOOP-02**: Restored sessions can continue automation from the last completed tool result
- [x] **LOOP-03**: Safety checks, progress updates, and permission gates execute through the hook pipeline

### Bootstrap Pipeline

- [x] **BOOT-01**: Structured service worker startup with explicit ordered phases
- [x] **BOOT-02**: Deferred initialization delays non-essential loading while preserving eager loading for core modules

### Mode Routing

- [x] **MODE-01**: FSB execution modes are formalized as named mode objects with per-mode limits and routing

### Module Adoption

- [x] **ADOPT-01**: All new sessions are created via `createSession(overrides)`
- [x] **ADOPT-02**: `CostTracker` is instantiated per session with structured budget checks
- [x] **ADOPT-03**: Each agent iteration produces a `createTurnResult()` object
- [x] **ADOPT-04**: `ActionHistory` wraps the raw `actionHistory` array with query methods
- [x] **ADOPT-05**: `session.mode` is set on every new session based on entry point

### Event Bus Wiring

- [x] **WIRE-01**: Popup and sidepanel message handlers include `sessionStateEvent` processing
- [x] **WIRE-02**: At least one UI surface visually reflects a SessionStateEmitter event

### Partial Outcome Lifecycle

- [x] **OUTCOME-01**: Agent loop supports a first-class partial/blocked terminal outcome distinct from `completed` and `error`
- [x] **OUTCOME-02**: Partial/blocked outcomes preserve structured completion summaries and render as non-crash results in MCP history and UI

### Auth Wall Handoff

- [x] **AUTH-01**: Auth-required or credential-missing final steps end with preserved useful output and explicit blocker details instead of generic failure
- [x] **AUTH-02**: Prompt/tool guidance steers the agent to use the partial/blocked outcome for auth walls and manual handoff scenarios after useful work is complete

### Overlay Lifecycle Reliability

- [x] **OVERLAY-01**: Active session overlay state is re-applied after navigation or content-script reconnect
- [x] **OVERLAY-02**: Long provider waits and retry windows keep the glow/progress overlay alive through heartbeat or degraded waiting-state refresh
- [x] **OVERLAY-03**: Dashboard/debugger overlay state stays synchronized with canonical content overlay state across reconnects and resumed progress updates

## Accepted Debt Carried Forward

- `ENGINE-03` / `ADOPT-02`: `CostTracker` ordering still needs a cleanup pass so the instantiated limit always matches the final per-mode config.
- `AUTH-01`: Runtime behavior shipped, but live browser smoke coverage is still recommended for the auth-wall resume and fallback paths.

## Future Requirements (v0.9.25+)

### Differentiators (deferred)

- **DIFF-01**: Configurable stop conditions -- max_iterations, stop_on_navigation, stop_on_pattern, stop_on_data_collected
- **DIFF-02**: Prompt routing -- pre-analyze user task to select likely tool tiers before AI call
- **DIFF-03**: Skill bundles -- package proven multi-step sequences as named reusable skills
- **DIFF-04**: Streaming turn events -- structured per-phase events to sidepanel for richer progress display
- **DIFF-05**: Structured output mode -- JSON-formatted AI responses with automatic retry for data extraction tasks
- **DIFF-06**: Parity audit utility -- validate tool/MCP schema consistency and site guide references

## Out of Scope

| Feature | Reason |
|---------|--------|
| Agent sub-spawning | Chrome MV3 single-thread service worker; race conditions on chrome.tabs APIs; cost multiplication |
| Plan mode toggle | Defeats single-attempt execution core value; doubles API calls |
| LSP-style tool integration | DOM snapshots and site guides already serve this purpose for browser context |
| Plugin architecture | MV3 CSP blocks runtime JS loading; premature for current stage |
| MCP resource browsing | Existing MCP tools already expose needed data |
| Voice integration | High complexity for niche use case; defer to v1.0+ |
| Command Graph (formal) | Prompt router + tool tiering covers task classification without formal graph |
| IndexedDB session storage | chrome.storage.local sufficient for current session volumes |
