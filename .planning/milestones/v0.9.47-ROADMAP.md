## Overview

Reorganized the repo into three clearly-bounded top-level packages — `extension/`, `mcp/`, `showcase/` — each owning its own assets, README, and tests. Cross-package boundaries are now explicit. Mechanical reorg only; no functional changes, no bundler changes, no Angular 19 → 20 migration (deferred to v0.9.48).

Pre-milestone the repo had extension code, MCP server, showcase, and deploy backend interleaved at the root. Post-milestone, the layout is:

```
extension/        # Chrome extension (was scattered at root)
mcp/              # was mcp-server/ — npm package fsb-mcp-server (name unchanged)
showcase/
  angular/        # Angular SSR site (unchanged)
  server/         # was server/ at root — Express + SQLite deploy backend
README.md         # repo overview pointing at the three packages
LICENSE
.github/, .planning/, scripts/, tests/
```

## Milestone Summary

**Decimal Phases:** None.

**Key Decisions:**

- **Folder names**: `extension/` + `mcp/` + `showcase/` (not `mcp-server/` or monorepo-style `packages/*`). Short top-level names. The npm package `fsb-mcp-server` keeps its name; only the folder renames.
- **Deploy backend ownership**: `server/` moved into `showcase/server/`, since it's the deploy backend for the showcase site. Single fly.io artifact, clear ownership.
- **`tests/` stays at repo root**: Tests cover both extension and mcp surfaces; splitting them would force a separate decision on which tests own which package and is out of scope. Test files now use `require('../extension/...')` / `require('../mcp/...')` paths.
- **Phase 217 → Phase 218 hand-off**: `mcp-server/package.json` build script's `cp ../ai/tool-definitions.js` was updated in Phase 217 to `cp ../extension/ai/tool-definitions.js` so CI stayed green during the inter-phase window. Phase 218 then renamed the dir; the cp path was already correct.
- **CI gating preserved**: Three jobs (`extension`, `mcp`, `showcase`) + `all-green` aggregator. Branch protection on `all-green` held throughout — every PR (#7, #8, #9, #10, #11) gated cleanly.
- **No tooling changes**: Folder rename only. No npm workspaces, no Yarn workspaces, no Nx, no Turborepo. Future migration is a separate milestone if ever.

**Issues Resolved:**

- Pre-milestone: extension files (`background.js`, `content/`, `ui/`, etc.) interleaved at root with mcp-server, showcase, server, deploy infra. Resolved.
- Pre-milestone: cross-package boundaries were implicit; moving the extension required path-archeology across many consumers. Resolved (now explicit folder boundaries).
- Pre-milestone: showcase prebuild script hardcoded root path to `manifest.json` — broke when manifest moved. Resolved (path candidates with fallback).

**Issues Deferred:**

- Angular 19 → 20 migration (deadline 2026-05-19; first candidate for v0.9.48).
- Splitting `tests/` per package (no decision yet on ownership).
- Lean root README + extension README absorbing landing-page content (preserved as-is for now per landing-page importance).

**Technical Debt Incurred:**

- Cross-package import path: `mcp/package.json` build script does `cp ../extension/ai/tool-definitions.js`. Couples mcp build to extension layout. Tolerable; revisit if `mcp/` should own its own copy of tool-definitions instead.
- Dockerfile + fly.toml hardcoded paths: manual update on any future restructure. Mitigation: deploy.yml runs flyctl on every push to main; broken paths fail visibly.
