## Key Decisions

- CDP tools (cdpClickAt, cdpDrag, cdpDragVariableSpeed, cdpScrollAt) work for canvas interaction
- Keyboard-first interaction (press_key for tool selection, cdpDrag for drawing) is the primary pattern
- Text entry uses cdpInsertText on transient textarea, not type tool
- Canvas operations use keyboard shortcuts exclusively -- no toolbar DOM clicks
- Edge coordinate formula documented inline for AI to calculate arrow start/end points
- Coordinate templates use concrete pixel values for direct AI follow-through
- CDP expression generator pattern: getCanvasPixelFallback returns self-contained IIFE string
- Canvas scene reads via getCanvasScene() with three-stage cascade: interceptor, rerender trigger, pixel fallback
- session.url does not exist -- use domResponse.structuredDOM.url with session.lastUrl fallback
