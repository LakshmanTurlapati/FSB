# Milestone v0.9.21: UI Retouch & Cohesion

**Status:** ✅ SHIPPED 2026-04-02
**Phases:** 140-144
**Total Plans:** 9

## Overview

Polish the existing FSB UI across the primary operator-facing surfaces without redesigning the product. The milestone established a shared UI baseline, retouched the sidepanel, popup, and control panel, made overlay feedback target-aware, and closed with a final regression sweep.

## Phases

### Phase 140: Shared Surface Audit & Design Corrections

**Goal**: Shared UI building blocks look and feel consistent across audited FSB surfaces while preserving the existing brand aesthetic
**Depends on**: Nothing (first phase of milestone)
**Plans**: 2 plans

Plans:

- [x] 140-01-PLAN.md -- Audit repeated UI primitives and define the retouch baseline across shared surfaces
- [x] 140-02-PLAN.md -- Implement shared style corrections for buttons, cards, badges, spacing, and core surface chrome

**Details:**
- Introduced a shared `fsb-ui-core.css` baseline consumed by sidepanel, popup, control panel, and showcase surfaces
- Normalized shared tokens, icon buttons, badges, status indicators, dense rows, and form primitives
- Preserved the existing FSB aesthetic while removing obvious cross-surface drift

### Phase 141: Sidepanel & Popup Retouch

**Goal**: The sidepanel and popup feel like polished versions of the same product surface, with cleaner hierarchy, spacing, and state feedback
**Depends on**: Phase 140
**Plans**: 2 plans

Plans:

- [x] 141-01-PLAN.md -- Retouch sidepanel layout, header/actions, history surface, and chat composer states
- [x] 141-02-PLAN.md -- Align popup hierarchy and shared control styling with the sidepanel retouch baseline

**Details:**
- Retouched the sidepanel into a clearer persistent workspace with cleaner header hierarchy, history chrome, composer polish, and footer metadata
- Retouched the popup into a compact quick-launch sibling with improved pinned/running/error states and tighter compact message/composer treatment
- Preserved existing behavior while making high-frequency chat interactions feel more deliberate

### Phase 142: Control Panel & Dashboard Retouch

**Goal**: Settings and monitoring surfaces feel cohesive with the rest of FSB while fixing visual noise, card drift, and awkward responsive composition
**Depends on**: Phase 140
**Plans**: 2 plans

Plans:

- [x] 142-01-PLAN.md -- Retouch control panel navigation, content sections, cards, forms, and settings composition
- [x] 142-02-PLAN.md -- Polish dashboard utility surfaces, pairing flow, help/docs cards, and naming consistency

**Details:**
- Flattened the dashboard shell toward a black-neutral dark mode and reduced the oversized density of the control panel
- Retouched nav, sidebar, forms, cards, analytics, and settings surfaces so they align with the shared milestone baseline
- Hid the raw server endpoint from display, enlarged the pairing QR presentation, updated docs/help destinations, and renamed `Site Explorer` to `Reconnaissance`

### Phase 143: Context-Aware Overlay Feedback

**Goal**: Automation highlight feedback looks precise and appropriate for the target type rather than relying on one padded rectangle treatment
**Depends on**: Phase 140
**Plans**: 2 plans

Plans:

- [x] 143-01-PLAN.md -- Redesign highlight target selection and presentation rules for text-centric versus control-centric targets
- [x] 143-02-PLAN.md -- Implement and verify overlay animation, positioning, and DOM stream parity for the new highlight behavior

**Details:**
- Compact links and inline text now receive text-style highlight treatment built from text rect fragments
- Inputs, buttons, and larger controls keep a fitted box highlight with tighter geometry
- Computed highlight geometry now flows into the DOM stream path instead of mirroring only raw element bounds

### Phase 144: UI Sweep & Regression Cleanup

**Goal**: The milestone ends with a visible quality uplift across all touched surfaces and no obvious regressions from the retouch
**Depends on**: Phase 141, Phase 142, Phase 143
**Plans**: 1 plan

Plans:

- [x] 144-01-PLAN.md -- Run a full UI consistency sweep, close polish gaps, and fix regressions across touched surfaces

**Details:**
- Aligned popup chrome and dark-mode tokens with the cleaned-up sidepanel direction
- Normalized lingering `Background Agents` copy to `Agents`
- Flattened remaining dashboard sidebar/nav elevation so it matches the updated control-panel shell

## Milestone Summary

**Key Decisions:**

- Use the control panel/options surface as the strongest visual source of truth while preserving the existing FSB aesthetic
- Keep dark-mode operator shells neutral black instead of warm orange-brown tinted
- Treat inline text and compact links differently from larger controls in automation highlight feedback
- Prefer small high-signal polish passes over broad redesigns

**Issues Resolved:**

- Cross-surface styling drift between sidepanel, popup, and control panel
- Warm-tinted dark shells in sidepanel/control panel surfaces
- Oversized dashboard density and inconsistent utility chrome
- Raw server endpoint exposure in the dashboard pairing flow
- Oversized rectangular highlight behavior on text and link targets

**Issues Deferred:**

- Showcase high-fidelity replica work continued separately in v0.9.22
- Dashboard stream and remote-control reliability work continued separately in v0.9.23

**Technical Debt Incurred:**

- Phase 142-144 artifacts were backfilled after the code landed directly in the repo
- No automated visual regression suite exists for the retouched UI surfaces
