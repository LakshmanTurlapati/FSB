# Milestone v0.9: FSB Reliability Improvements

**Status:** SHIPPED 2026-02-14
**Phases:** 1-11
**Total Plans:** 24

## Overview

This milestone transformed FSB from an unreliable "hit or miss" automation tool into a precise, single-attempt execution engine. The journey started with fixing element targeting (the core broken functionality), added visual feedback so users can see what's happening, improved context quality for better AI decisions, added verification to confirm actions succeeded, provided debugging tools for when things go wrong, and finally optimized execution speed. Each phase built on the previous, creating a foundation of reliability before adding polish.

## Milestone Summary

**Key Decisions:**
- Focus on mechanics, not AI (user confirmed AI intent is correct, execution layer is the problem)
- Visual feedback with orange glow (user specifically requested seeing what's being targeted)
- Single-attempt reliability over retry sophistication (core value is precision)
- 50 element limit for focused AI context (down from 300)
- Shadow DOM for visual overlays (complete style isolation from page CSS)
- Outcome-based dynamic delays (replaced static category delays)

**Issues Resolved:**
- Fixed unreliable selector generation with uniqueness scoring
- Fixed elements being clicked when not visible, disabled, or obscured
- Fixed missing coordinate fallback (code was unreachable)
- Fixed AI receiving noisy DOM context (300+ elements reduced to ~50)
- Fixed actions proceeding without verification of success
- Fixed control panel with orphaned UI references and disconnected settings

**Issues Deferred:**
- Phase 10: waitForActionable() dead code removal (~80 lines)
- Phase 10: Adaptive ElementCache sizing (currently hardcoded to 100)

**Technical Debt Incurred:**
- Debug mode storage change listener deliberately omitted (loads on startup/restart instead)
- waitForActionable() still present but unused in content.js
