Add ElementCache class near the top of content.js (after DOMStateManager class).

The class should:
1. Use a Map for cache storage: selector -> { ref: WeakRef(element), version: stateVersion, timestamp }
2. Track stateVersion that increments on invalidation
3. Initialize MutationObserver on document.body with:
   - childList: true, subtree: true
   - attributes: true with attributeFilter: ['class', 'id', 'disabled', 'hidden', 'style']

8. invalidate() method: clear cache, increment stateVersion
9. destroy() method: disconnect observer, clear cache

Also add global instance: `const elementCache = new ElementCache();`
And call `elementCache.initialize()` in the DOMContentLoaded listener.

Key decision from research: Use WeakRef for auto-cleanup when elements are garbage collected. Check deref() result before using.

<task type="auto">
  <name>Task 2: Create performQuickReadinessCheck and smartEnsureReady fast-path</name>
  <files>content.js</files>
  <action>
Add performQuickReadinessCheck function near the ensureElementReady function.

The function should:
1. Accept element parameter
2. Get bounding rect and perform 4 quick checks:
   - hasSize: rect.width > 0 AND rect.height > 0
   - notDisabled: !element.disabled AND element.getAttribute('aria-disabled') !== 'true'
   - visible: getComputedStyle shows display !== 'none', visibility !== 'hidden', opacity > 0
   - receivesEvents: document.elementFromPoint at center returns element or element.contains(result)
3. Also check inViewport: rect.top >= 0 AND rect.bottom <= window.innerHeight (for informational)
4. Return object:
   - definitelyReady: all 4 main checks pass AND inViewport
   - definitelyNotReady: hasSize OR notDisabled OR visible is false
   - concern: 'scroll' if !inViewport, 'obscured' if !receivesEvents, null otherwise
   - checks: object with all individual check results

Also add smartEnsureReady function that:
1. Calls performQuickReadinessCheck(element)
2. If definitelyReady: return { ready: true, scrolled: false, fastPath: true, checks }
3. If definitelyNotReady OR has concern: fall through to ensureElementReady(element, actionType)
4. Return result of ensureElementReady

Key decision from research: Quick check should be conservative - when in doubt, do full check.
  </action>
