# Milestone v0.9.27: Usage Dashboard Fix

**Status:** ✅ SHIPPED 2026-04-14
**Phases:** 171-172
**Total Plans:** 3
**Requirements:** 7

## Overview

v0.9.27 fixed the control-panel usage dashboard so tracked analytics data renders reliably again. The milestone shipped an init-safe storage-backed refresh path, deferred off-screen refresh handling, null-safe dashboard label updates, a guarded cost-breakdown render path, regression coverage for the refresh contract, and local end-to-end smoke verification confirming that real task completion updates metrics and chart data.

## Key Accomplishments

- `refreshAnalyticsDashboard()` now waits for analytics initialization and reloads usage data from `chrome.storage.local` instead of relying on stale in-memory state.
- Off-screen analytics updates are tracked through `dashboardState.analyticsNeedsRefresh` and replayed when the operator returns to the dashboard.
- `updateTimeRangeLabels()` no longer throws when dashboard label nodes are missing, and the dashboard refresh lifecycle is protected by `tests/dashboard-analytics-refresh.test.js` in the main `npm test` suite.
- Local unpacked-extension smoke verification confirmed that a real task completion updates request count, token totals, cost, and chart data on the options page.
