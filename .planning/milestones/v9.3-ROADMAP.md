## Overview

## Milestone Summary

**Key Decisions:**
- window.FSB namespace pattern for module communication (not ES modules, not bare globals)
- Programmatic injection via chrome.scripting.executeScript (not manifest content_scripts)
- Shadow DOM + Popover API for visual overlays (top-layer rendering, z-index fallback)
- Store-first-enrich-second pattern for memory AI analysis (enrichment never blocks storage)
- Pure heuristic cross-site pattern analysis (no AI calls, keyword-based classification)
- Preset dropdown pattern (50/100/200/500/Custom) for bounded numeric settings
- AI-only extraction with badge error indicator (removed local fallback to surface config errors visibly)
- Dual-format registry (old category + new per-site) for backward compatibility during site guide migration
- IIFE pattern for viewer JS to avoid global namespace pollution

**Issues Resolved:**
- content.js 13K-line monolith split into 10 logical modules
- waitForActionable dead code eliminated (158 lines + orphaned files)
- ElementCache hardcoded size replaced with configurable value
- UniversalProvider constructor bug fixed (was receiving wrong arguments, causing AI extraction to always fail)
- Memory tab had no detail viewers -- now has expandable panels for all types
- No cost visibility for memory operations -- now tracked separately with 30-day window

**Issues Deferred:**
- Site Guides Viewer design mismatch (UAT blocker: should display like memories with mind maps, not custom accordion) -- needs redesign in future milestone

**Technical Debt Incurred:**
- Site Guides Viewer overengineered with accordion sub-sections instead of memory-style display (blocker from UAT)
- 53 script tags for per-site guide files in options.html (could be bundled)
