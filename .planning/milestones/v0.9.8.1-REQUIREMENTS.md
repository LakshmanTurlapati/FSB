# Requirements Archive: FSB v0.9.8.1 npm Publishing

**Status:** ✅ SHIPPED 2026-04-02
**Archive note:** v0.9.8.1 never had a canonical top-level `REQUIREMENTS.md`. This archive reconstructs the milestone requirements from `.planning/ROADMAP.md`, Phase 105 summaries and verification, the shipped MCP package, and the published `fsb-mcp-server@0.4.0` release state.

## Reconstructed Requirements

### Packaging

- [x] **PKG-01**: The MCP server is represented as a real npm package with package metadata, versioning, repository identity, and runtime constraints.
- [x] **PKG-02**: The published tarball is restricted to runtime artifacts and package documentation rather than source and development clutter.
- [x] **PKG-03**: The package exposes a working CLI entrypoint so users can launch it through `npx -y fsb-mcp-server`.
- [x] **PKG-04**: Publishing rebuilds the package before release so shipped artifacts stay in sync with source.

### Distribution

- [x] **DIST-01**: A repeatable tag-driven CI workflow can publish the package to npm.
- [x] **DIST-02**: Local verification proves the tarball contents and CLI startup are correct before release.
- [x] **DIST-03**: The package is publicly installable from npm without cloning the repository.

### Documentation

- [x] **DOCS-01**: The package README explains how to install and configure the MCP server in supported MCP hosts.
- [x] **DOCS-02**: The root project README surfaces the npm-distributed MCP server and its extension-pairing model.
- [x] **DOCS-03**: Operational docs cover the shipped server modes and diagnostics commands (`serve`, `setup`, `status`, `doctor`, `wait-for-extension`).
