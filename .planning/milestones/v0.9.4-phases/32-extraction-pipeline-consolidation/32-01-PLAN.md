  <name>Task 1: Rewrite extraction prompt for single Task Memory JSON</name>
  <files>lib/memory/memory-extractor.js</files>
  <action>
Rewrite `_buildExtractionPrompt()` to ask the AI for a single consolidated recon report instead of 1-5 typed memories.

**System prompt changes:**
- Frame as intelligence/reconnaissance report extraction: "Analyze this completed browser automation session and produce a consolidated intelligence report"
- AI must return ONE JSON object (not an array) with shape: `{ text: "Task description -- outcome", session: {...}, learned: {...}, procedures: [...] }`
- `text` field: natural language summary like "Searched Amazon for wireless mouse -- success" (AI generates this)
- `session` section: task (string), outcome ("success"|"partial"|"failure"), domain, duration (seconds), iterationCount, finalUrl, timeline (array of {action, target, result, timestamp}), failures (array of strings)
- `learned` section: selectors (array of {name, selector, context}), siteStructure (array of strings describing page layout/navigation), patterns (array of strings describing learned behaviors)
- `procedures` section: array of {name, steps[], successRate, targetUrl} -- reusable sequences discovered during the task
- Rules: no sensitive data, be specific to the domain, include full URLs in timeline entries, include element descriptions alongside selectors

**User prompt changes -- send EVERYTHING per CONTEXT.md decisions:**
- Send ALL actions from newActions (not just last 15) with full details: tool, params (including full URL), result.success, result description/element role
- Format each action as: `[{index}] {tool}({params_summary}) -> {success|fail}: {result_description}`
- Include full URLs per action so AI can map navigation paths
- Include element description/role from result data
- Include session._enrichedData sections:
  - hardFacts.workingSelectors (all of them, not truncated)
  - hardFacts.criticalActions (all)
  - sessionMemory.stepsCompleted (all, not last 10)
  - sessionMemory.failedApproaches (all)
  - compactedSummary (full text)
- Include session.error if present
- Token management: if total action log exceeds ~4000 chars, include first 5 and last 15 actions with "[...N actions omitted...]" in between (Claude's discretion on exact truncation strategy)

**Also rewrite `_parseExtractedMemories()` to parse single JSON into createTaskMemory():**
- Strip markdown code blocks (existing pattern)
- JSON.parse the response -- expect a single object, not an array
- If array received (AI returned old format), take first element and adapt
- Map parsed JSON to `createTaskMemory(parsed.text, metadata, { session: parsed.session, learned: parsed.learned, procedures: parsed.procedures })`
- metadata: domain from context, taskType inferred from session task, tags from parsed.tags or auto-generated from domain+task keywords, confidence from parsed.confidence or 0.9, sourceSessionId from session.sessionId
- Return array with exactly one memory (for backward compat with manager.add() which iterates)
- On parse failure: log error, return empty array (same as current behavior)

Keep all other methods unchanged: _getProvider, _isAuthError, _isTransientError, _setBadgeError, _extractDomain, extract() retry loop, enrich(), _buildEnrichmentPrompt (enrichment changes in Plan 02), _parseEnrichmentResponse.

Update `add()` in memory-manager.js to handle the new single-Task-Memory extraction result.

**Changes to add():**
- The extractor now returns an array with exactly one Task Memory (or empty on failure)
- The existing loop over `extracted` already works correctly (it iterates the array, runs consolidation, stores each)
- The key change: for task type memories, ALWAYS run enrichment regardless of autoAnalyzeMemories setting
- After the storage loop, check if any stored memory has `type === 'task'` -- if so, run `_enrichAsync()` unconditionally (don't check `autoAnalyze` setting for task type)
- Keep the existing autoAnalyze check for non-task memories (backward compat for old format memories that might still exist)

**Updated enrichment logic in add():**
Replace the current enrichment block:
```javascript
const autoAnalyze = await this._getAutoAnalyzeSetting();
if (autoAnalyze && this._extractor?.enrich) {
  for (const memory of stored) {
    this._enrichAsync(memory).catch(err => ...);
  }
}
```
With:
```javascript
if (this._extractor?.enrich) {
  for (const memory of stored) {
    // Task memories ALWAYS get enriched (per CONTEXT.md: no "Refine with AI" button)
    if (memory.type === 'task') {
      this._enrichAsync(memory).catch(err =>
        console.warn('[MemoryManager] Async enrichment failed:', err.message)
      );
    }
  }
  // Non-task memories still respect autoAnalyze setting
  const autoAnalyze = await this._getAutoAnalyzeSetting();
  if (autoAnalyze) {
    for (const memory of stored) {
      if (memory.type !== 'task') {
        this._enrichAsync(memory).catch(err =>
          console.warn('[MemoryManager] Async enrichment failed:', err.message)
        );
      }
    }
  }
}
```

No other changes to memory-manager.js. The consolidator wiring, search, update, delete, getAll, getStats all remain unchanged.
