**Guidance text (keyboard-first, slash commands primary):**

SLASH COMMANDS (primary block creation method):
```
Type "/" at the beginning of a new line to open the slash command menu:
  /text or /p       - Plain text paragraph
  /h1, /h2, /h3     - Headings
  /todo              - To-do checkbox
  /bullet            - Bulleted list
  /number            - Numbered list
  /toggle            - Toggle block
  /quote             - Quote block
  /divider           - Horizontal divider
  /code              - Code block
  /table             - Simple table
  /callout           - Callout box
  /page              - New sub-page
  /image             - Image embed
  /database          - Inline database (full page or inline)
  /link              - Link to page

After typing the command, press Enter to insert the block.
To search: type "/" then start typing any word to filter the menu.
```

Keyboard shortcuts:
- Ctrl+N: New page
- Ctrl+Shift+N: New page in sidebar
- Ctrl+K: Quick search / link to page
- Ctrl+P: Quick search (alternative)
- Ctrl+/: Toggle sidebar
- Ctrl+Shift+H: Toggle hide sidebar
- Enter: New block below current
- Tab: Indent block
- Shift+Tab: Outdent block
- Ctrl+Shift+0-9: Convert block type (0=text, 1=h1, 2=h2, etc.)
- Ctrl+D: Duplicate block
- Ctrl+Shift+D: Duplicate page
- /: Open slash command menu (at start of empty block)
- @: Mention person or page
- [[: Link to page

CONTENTEDITABLE INTERACTION:
- Notion uses contenteditable blocks. Always click the target block first to place cursor.
- Type text after clicking -- keystrokes go to the focused block.
- Press Enter to create a new block below.
- Blocks are individually selectable -- click to focus, then type.
- To edit an existing block's text, click directly on the text you want to edit.

**Workflows (6-8):**
1. `createPage`: Ctrl+N or click "New page" in sidebar > type title > Enter to start body > type content
2. `addBlocks`: Click at end of last block > type "/" > type block type name > Enter > type content
3. `createTodo`: Type "/todo" > Enter > type task text > Enter for next todo > repeat
4. `createTable`: Type "/table" > Enter > configure columns > Tab between cells > Enter for new row
5. `navigateDatabase`: Click database view tabs (Table/Board/Calendar) > use filter/sort buttons
6. `searchContent`: Ctrl+K or Ctrl+P > type query > navigate results with arrow keys > Enter to open
7. `moveBlock`: Click block drag handle OR use Ctrl+Shift+Up/Down to move block up/down
8. `linkToPage`: Type "[[" > start typing page name > select from dropdown > Enter

**Warnings (10-15):**
- Notion uses CSS Modules -- class names change on every deploy. NEVER rely on class selectors as primary strategy
- Slash commands only work at the start of an empty line or at cursor position -- not in the middle of existing text
- contenteditable blocks: click the block first to establish focus, then type. Without focus, keyboard shortcuts fire instead
- @ mentions and [[ links open popovers -- wait for them to render before selecting
- Database views (Table, Board, etc.) are separate views of the same data -- switching views does not lose data
- "New page" in sidebar creates at the top level. To create a sub-page, use /page inside an existing page
- Notion auto-saves continuously -- there is no save button
- Undo (Ctrl+Z) works within a session but has limits on cross-session undo
- The sidebar can be collapsed -- check for toggle button if sidebar is not visible
- Block drag handles only appear on mouse hover -- they are not visible by default
- Code blocks use a monospace editor with language selection dropdown
- Images and embeds use a separate upload/URL flow -- /image opens a picker

CREATE ISSUE -- FULL FORM WORKFLOW (THE critical workflow):
```
Press C (global shortcut) to open Create Issue modal.
Form fields (Tab order):
  1. Project:    Pre-selected, change via dropdown if needed
  2. Issue Type: Click dropdown > select Bug/Story/Task/Epic
  3. Summary:    Required. Type issue title.
```

Selector strategies: ALWAYS use aria/role first, then `[role="gridcell"]`, `[aria-colindex]`, `[data-rowindex]`. Class selectors as last resort only.

**Guidance text (keyboard-first, PER-FIELD-TYPE interaction):**

GRID NAVIGATION:
```
Arrow keys:  Navigate between cells
Enter:       Start editing selected cell
Escape:      Stop editing, return to navigation mode
Tab:         Move to next field (right)
Shift+Tab:   Move to previous field (left)
Shift+Enter: Add new record below current
Space:       Expand selected record to detail view
```

PER-FIELD-TYPE INTERACTION (THE critical reference):
```
TEXT / LONG TEXT:
  - Enter to edit > type content > Escape to save
  - Long text opens a multi-line editor

SINGLE SELECT:
  - Enter to open dropdown > type to filter options > Enter to select
  - Options are searchable -- type first few letters

MULTI SELECT:
  - Enter to open dropdown > type to filter > Enter to add > repeat for multiple
  - Click selected tags to remove them

DATE:
  - Enter to open date picker > type date (MM/DD/YYYY) or click calendar
  - Tab to time field if time is enabled

CHECKBOX:
  - Space to toggle checkbox (no Enter needed)

NUMBER / CURRENCY / PERCENT:
  - Enter to edit > type number > Escape to save

LINKED RECORD:
  - Enter to open linked record picker > type to search > Enter to select
  - Can link multiple records (each Enter adds one)

ATTACHMENT:
  - Enter to open attachment dialog > upload or paste URL
  - Cannot be automated via keyboard -- requires file picker

EMAIL / URL / PHONE:
  - Enter to edit > type value > Escape to save

FORMULA / ROLLUP / LOOKUP / COUNT:
  - READ-ONLY -- these are computed fields. Cannot be edited directly.
  - Guide should note: "This field is a formula/computed field and cannot be edited."
```

VIEW SWITCHING:
- Click view tabs at the top (Grid, Kanban, Calendar, Gallery, Form)
- Each view shows the same data in different layouts
- Grid is the default and most keyboard-friendly view

**Workflows (6-8):**
1. `navigateGrid`: Arrow keys to move between cells, Enter to edit, Escape to return to navigation
2. `editCellByType`: Select cell > Enter > type/select based on field type (reference per-field-type guide above)
3. `createRecord`: Shift+Enter or click "+" row button > fill fields from left to right using Tab
4. `switchViews`: Click view tabs > select Grid/Kanban/Calendar/Gallery
5. `sortFilter`: Click Sort button > select field > select direction > Apply. Same pattern for Filter.
6. `expandRecord`: Select row > Space to expand > view/edit all fields in detail modal > Escape to close
7. `searchRecords`: Ctrl+F or click search > type query > results highlight/filter in grid
8. `addField`: Click "+" at end of header row > select field type > name the field > customize options

**Warnings (10-15):**
- Airtable uses React with hashed CSS -- class selectors break on deploys. Use aria/role first.
- Grid cells use `[role="gridcell"]` with `aria-colindex` and `data-rowindex` -- these are stable selectors
- Formula, rollup, lookup, and count fields are READ-ONLY -- do not attempt to edit
- Checkbox fields use Space (not Enter) to toggle
- Single select and multi select use different interaction patterns (single replaces, multi appends)
- Date fields may have time enabled -- check for time field after date picker
- Linked records open a search dialog -- must type to search, then select
- Attachment fields require a file picker -- cannot fully automate via keyboard
- Airtable may use virtualized rows -- rows below the viewport may not be in DOM until scrolled to
- Expanding a record (Space) opens a modal with ALL fields including hidden ones
- Sort and filter controls are sticky buttons in the toolbar row
- Long text fields open an expanded editor that overlays the grid
- View tabs scroll horizontally if there are many views

**toolPreferences:** All three guides: `['navigate', 'click', 'type', 'keyPress', 'waitForTabLoad', 'getText', 'waitForElement', 'getAttribute', 'waitForDOMStable']`
