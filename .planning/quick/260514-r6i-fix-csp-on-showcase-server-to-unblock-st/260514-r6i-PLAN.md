# Plan: Fix showcase CSP to unblock /stats GitHub charts

## Context

The /stats Easter-egg page (`showcase/angular/src/app/pages/stats/stats-page.component.ts`) renders GitHub-derived charts (cumulative stars, weekly stars, issues open-vs-closed, forks growth, PRs opened-vs-merged, commits over time, maintenance) by calling browser-side `fetch()` against 7 endpoints on `https://api.github.com/repos/LakshmanTurlapati/FSB/...` (paginated, MAX_PAGES=2, 5-minute poll).

The showcase Express server (`showcase/server/server.js`) sets a strict CSP whose `connect-src` directive is currently `'self'` (line 59), so every one of those fetches is blocked at the browser layer. User-visible symptom: the GitHub-stats card on https://full-selfbrowsing.com/stats shows no data.

**Orchestrator pre-verified diagnostics (do not re-verify):**
- Production CSP header confirmed via `curl -D - https://full-selfbrowsing.com/stats` — `connect-src 'self'` is the active blocker.
- GitHub allows public-read CORS from any origin (`access-control-allow-origin: *`), so no server-side proxy is needed.
- Current page totals (66 stars / 4 forks / 50 issues / 47 pulls / 17 releases / commits-12wk window) all fit within MAX_PAGES=2.
- "Comments" in the user report = "commits" (no comments chart exists in the view-id enum).

**Out of scope for this quick task (orchestrator-locked):**
- Do NOT raise MAX_PAGES.
- Do NOT build a server-side GitHub proxy (that would be a phase, not a quick task).
- Do NOT add a comments chart.
- Do NOT touch any /stats Easter-egg crawler invariant (sitemap.xml, llms.txt, llms-full.txt, prerender-routes.txt, hreflang).

Branch is already checked out as `fix/csp-allow-github-api-for-stats`. One commit at the end.

## Task 1: Allow api.github.com in showcase CSP connect-src

**Files:** `showcase/server/server.js`

**Action:**

In `showcase/server/server.js`, locate the `SHOWCASE_CSP` array (currently spans lines 52–68; the offending line is line 59: `"connect-src 'self'"`).

Make two edits, both inside that array's textual region (do NOT modify the `app.use((req, res, next) => { ... })` middleware that emits the header — only the array values and the comment block above it):

1. Append a new paragraph to the existing comment block above `SHOWCASE_CSP` (the block that today spans lines 38–51 and ends with the script-src-attr explanation). The new paragraph documents the connect-src allowance so a future security tightening sees the rationale BEFORE editing. Use this exact prose (tailored to match the existing comment voice):

   ```

   // GitHubStatsService (showcase/angular/src/app/core/stats/github-stats.service.ts)
   // can fetch repo metrics directly from the browser. GitHub's public-read
   // endpoints return `access-control-allow-origin: *`, so no server-side
   ```
