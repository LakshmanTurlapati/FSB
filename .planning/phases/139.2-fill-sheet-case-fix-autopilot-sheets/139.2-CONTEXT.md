# Phase 139.2: Fill Sheet Case Fix & Autopilot Sheets Integration - Context

**Gathered:** 2026-04-01
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix three bugs preventing fill_sheet from working correctly, and fix the autopilot's Google Sheets interaction pattern.

</domain>

<decisions>
## Implementation Decisions

### Bug 1: Case sensitivity -- keyboard emulator drops uppercase
- **D-01:** Root cause: `utils/keyboard-emulator.js` line 314 sets `params.text = keyData.key` which is always lowercase, even when shift modifier is active for uppercase letters
- **D-02:** Fix: when shift is active and the key is a letter, set `params.text` to the uppercase version of the character
- **D-03:** Same fix needed for shift+special chars (e.g., '!' should have text='!')

### Bug 2: Parameter name mismatch -- csvData vs data
- **D-04:** `ai/tool-definitions.js` line 502 defines parameter as `csvData`
- **D-05:** `content/actions.js` line 4582 expects `data`
- **D-06:** Fix: add PARAM_TRANSFORMS entry for fill_sheet in `mcp-server/src/tools/schema-bridge.ts` mapping csvData->data
- **D-07:** Also fix in tool-executor.js for the autopilot path (same param rename before dispatch)

### Bug 3: MCP timeout too short for fill_sheet
- **D-08:** `mcp-server/src/tools/manual.ts` uses 30s timeout for all tools
- **D-09:** Content script allows 120s for fillsheet (content/messaging.js)
- **D-10:** Fix: increase MCP timeout for fill_sheet specifically to 120s

### Bug 4: Autopilot interacts with Sheets sidebar instead of calling fill_sheet
- **D-11:** System prompt in agent-loop.js already mentions fill_sheet but AI gets distracted by Tables sidebar DOM
- **D-12:** Fix: add Sheets-specific guidance to system prompt -- "When on Google Sheets, use fill_sheet tool directly. Do NOT interact with the Tables sidebar or search inputs."

### Claude's Discretion
- Exact wording of Sheets guidance in system prompt
- Whether to add a PARAM_TRANSFORMS entry or rename the parameter in tool-definitions.js directly

</decisions>

<code_context>
## Existing Code Insights

### The case bug in detail
keyboard-emulator.js sendKeyEvent line 314:
```javascript
if ((type === 'char' || type === 'keyDown') && isPrintableKey(keyData.key) && !hasShortcutModifier) {
  params.text = keyData.key;  // BUG: always lowercase 'p' even when shift=true for 'P'
}
```
Should be: `params.text = modifiers.shift ? keyData.key.toUpperCase() : keyData.key;` for letter keys

### The param mismatch
MCP sends: `{ startCell: 'A1', csvData: '...' }`
Content expects: `{ startCell: 'A1', data: '...' }`
The csvData never reaches the content script's `data` destructure

</code_context>
