  <name>Task 1: Fix keyboard-emulator case sensitivity (Bug 1)</name>
  <files>utils/keyboard-emulator.js</files>

  <action>
Read utils/keyboard-emulator.js lines 285-336 (sendKeyEvent method).

**The bug (per D-01):** Line 314 sets `params.text = keyData.key` which is always the lowercase base key from KEY_MAPPINGS (e.g., 'a', '1', ';'). When shift is active, Chrome CDP needs `params.text` to be the shifted character ('A', '!', ':') for correct text insertion.

**The fix (per D-02, D-03):** Replace line 314:
```javascript
params.text = keyData.key;
```
with:
```javascript
// When shift is active, produce the shifted character for params.text.
// For letters: 'a' + shift -> 'A'
// For digits/symbols: '1' + shift -> '!', ';' + shift -> ':'
if (modifiers.shift) {
  // Reverse lookup: find what shifted character maps to this base key
  const SHIFT_MAP = {
    '1': '!', '2': '@', '3': '#', '4': '$', '5': '%',
    '6': '^', '7': '&', '8': '*', '9': '(', '0': ')',
    '-': '_', '=': '+', '[': '{', ']': '}', '\\': '|',
    ';': ':', "'": '"', ',': '<', '.': '>', '/': '?',
    '`': '~'
  };
  if (/^[a-z]$/.test(keyData.key)) {
    params.text = keyData.key.toUpperCase();
  } else if (SHIFT_MAP[keyData.key]) {
    params.text = SHIFT_MAP[keyData.key];
  } else {
    params.text = keyData.key;
  }
} else {
  params.text = keyData.key;
}
```

This keeps the existing guard (`isPrintableKey && !hasShortcutModifier`) unchanged -- only the text assignment inside the `if` block changes.

**Do NOT change:**
- typeText method (lines 437-490) -- it correctly sets shift=true for uppercase/special chars
- pressKey method -- it correctly delegates to sendKeyEvent
- The KEY_MAPPINGS object -- it stores base keys by design
