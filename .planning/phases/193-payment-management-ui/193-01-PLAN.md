## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| options.js -> background.js | Message passing crosses extension context boundary; payment data flows through chrome.runtime.sendMessage |
| PIN input -> unlock handler | User-entered PIN crosses into credential verification |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-193-01 | Information Disclosure | renderPaymentCard | mitigate | Only display metadata from getAllPaymentMethods (last4, brand, maskedNumber) -- never request full card data for list display |
| T-193-02 | Spoofing | submitPaymentUnlock | accept | PIN verification happens in background.js SecureConfig; UI just forwards -- existing gate is sufficient |
| T-193-03 | Tampering | payment card HTML | mitigate | Use escapeHtml on all card data before inserting into innerHTML to prevent XSS from stored data |
</threat_model>
