Create a new file `content/dom-stream.js` following the established IIFE + window.FSB namespace pattern (see content/dom-state.js for reference).

The module must implement these capabilities:

**1. DOM Serializer function `serializeDOM()`:**
- Walk document.body using TreeWalker (NodeFilter.SHOW_ELEMENT | NodeFilter.SHOW_TEXT)
- Assign each element a stable numeric ID via a data attribute `data-fsb-nid` (node ID) -- a simple incrementing counter reset on each full snapshot. Store counter on the module: `let nextNodeId = 1;`
- For each element node:
  - Skip `<script>` and `<noscript>` tags entirely (return NodeFilter.FILTER_REJECT)
  - Skip elements that have `data-fsb-overlay` attribute or are inside a shadow root created by FSB (check `el.closest('[data-fsb-overlay]')` or if element's getRootNode() is a ShadowRoot with host having class containing 'fsb')
  - Absolutify `src`, `href`, `action`, `poster`, `data`, `srcset` attributes using `new URL(attrValue, document.baseURI).href`
  - Keep `<link rel="stylesheet">` tags with absolutified href
  - Keep `<img>`, `<video>`, `<picture>`, `<source>` tags with absolutified src/srcset
  - Replace `<iframe>` elements with `<div data-fsb-iframe-placeholder style="width:100%;height:200px;background:#e5e7eb;border:1px dashed #9ca3af;display:flex;align-items:center;justify-content:center;color:#6b7280;font-size:14px;">iframe</div>`
  - Capture computed styles for elements that have non-default styling (use a heuristic: if element has `style` attribute OR has classes, serialize key visual properties as inline style). Specifically capture: `display`, `position`, `width`, `height`, `margin`, `padding`, `background`, `color`, `font-size`, `font-family`, `border`, `border-radius`, `opacity`, `visibility`, `overflow`, `flex-direction`, `align-items`, `justify-content`, `gap`, `grid-template-columns`, `grid-template-rows`, `transform`, `box-shadow`, `text-align`, `line-height`, `font-weight`, `z-index`, `max-width`, `min-height`
  - Set `data-fsb-nid` attribute on the element (integer value)
- For text nodes: include as-is (cloneNode natural behavior handles this)
- Return an object: `{ html: serializedHTML, stylesheets: [absoluteURLs], scrollX: window.scrollX, scrollY: window.scrollY, viewportWidth: window.innerWidth, viewportHeight: window.innerHeight, pageWidth: document.documentElement.scrollWidth, pageHeight: document.documentElement.scrollHeight, url: location.href, title: document.title }`
- The serializedHTML should be the outerHTML of a cloned body with all transformations applied. Implementation: clone `document.body` via `document.body.cloneNode(true)`, then walk the clone applying transformations (strip scripts, absolutify URLs, add nid attributes, replace iframes), then return `clone.innerHTML`
- Collect stylesheet URLs from all `<link rel="stylesheet">` in document.head

**2. MutationObserver streaming via `startMutationStream()` / `stopMutationStream()`:**
- Create a MutationObserver watching document.body with config: `{ childList: true, attributes: true, characterData: true, subtree: true, attributeOldValue: true }`
- Do NOT use attributeFilter (we need all attribute changes for visual fidelity, unlike dom-state.js which filters for automation)
- Batch mutations on a 150ms debounce timer (use setTimeout, clear on each new mutation, fire after 150ms of quiet)
- When batch fires, encode mutations as an array of diff objects:
  ```javascript
  // For childList (added nodes):
  { op: 'add', parentNid: parentElement.dataset.fsbNid, html: addedNode.outerHTML, beforeNid: nextSibling?.dataset?.fsbNid || null }
  // Assign data-fsb-nid to new nodes before serializing (increment nextNodeId)
  // Absolutify URLs in added node HTML same as in serializeDOM

  // For childList (removed nodes):
  { op: 'rm', nid: removedNode.dataset.fsbNid }

  // For attributes:
  { op: 'attr', nid: target.dataset.fsbNid, attr: attributeName, val: target.getAttribute(attributeName) }

  // For characterData:
  { op: 'text', nid: target.parentElement?.dataset?.fsbNid, text: target.textContent }
  ```
- Skip mutations on elements that don't have `data-fsb-nid` (FSB-injected elements, elements added after snapshot that haven't been assigned IDs yet -- assign them on the fly when processing 'add' mutations)
- Skip mutations where target is inside a `[data-fsb-overlay]` element
- Send batched mutations via `chrome.runtime.sendMessage({ action: 'domStreamMutations', mutations: [...] })`

**3. Scroll tracker via `startScrollTracker()` / `stopScrollTracker()`:**
- Listen to `scroll` event on `window` with passive: true
- Throttle to max 1 event per 200ms (use a simple timestamp check, not setTimeout)
- Send scroll position via `chrome.runtime.sendMessage({ action: 'domStreamScroll', scrollX: window.scrollX, scrollY: window.scrollY })`

**4. Overlay event broadcaster via `broadcastOverlayState()`:**
- Expose a function that reads FSB's HighlightManager active highlight position and ProgressOverlay state
- For highlight: get the highlighted element's getBoundingClientRect() -> `{ x, y, w: width, h: height, state: 'active' }`
- For progress: read from FSB's ProgressOverlay if available -> `{ percent, phase, eta }`
- Send via `chrome.runtime.sendMessage({ action: 'domStreamOverlay', glow: {...}, progress: {...} })`
- This function will be called by the background script when it sends session status updates (we wire that in Task 2)

**5. Message listener for control commands from background:**
- Listen via `chrome.runtime.onMessage.addListener`
- Handle these actions:
  - `domStreamStart`: Call serializeDOM(), send result via `chrome.runtime.sendMessage({ action: 'domStreamSnapshot', snapshot: result })`, then call startMutationStream() and startScrollTracker(). Set internal flag `streaming = true`
  - `domStreamStop`: Call stopMutationStream() and stopScrollTracker(). Set `streaming = false`
  - `domStreamPause`: Call stopMutationStream() and stopScrollTracker() but keep `streaming = true` (paused state)
  - `domStreamResume`: Call serializeDOM() (fresh snapshot), send it, restart mutation stream and scroll tracker

**6. Module registration:**
- Register on FSB namespace: `FSB.domStream = { serializeDOM, startMutationStream, stopMutationStream, startScrollTracker, stopScrollTracker, broadcastOverlayState, isStreaming: () => streaming }`
- Register module: `window.FSB._modules['dom-stream'] = { loaded: true, timestamp: Date.now() }`

**5. Add `_forwardToContentScript` method to FSBWebSocket class in ws/ws-client.js:**

```javascript
/**
 * Forward a message to the content script on the active tab.
 * Used for dashboard-to-content-script communication (DOM stream control).
 * @param {string} action - Content script action name
 * @param {Object} payload - Additional payload data
 */
async _forwardToContentScript(action, payload) {
  try {
    var tabs = await chrome.tabs.query({ active: true, currentWindow: true });
    var tab = tabs[0];
    if (tab && tab.id) {
      chrome.tabs.sendMessage(tab.id, { action: action, ...payload }, { frameId: 0 })
        .catch(function() {}); // Ignore if content script not ready
    }
  } catch (e) {
    console.warn('[FSB WS] Failed to forward to content script:', action, e.message);
  }
}
```
