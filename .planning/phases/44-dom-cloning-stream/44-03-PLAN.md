**Hardening content/dom-stream.js:**

1. **SVG handling in serializeDOM:** SVG elements use `xlink:href` not `href`. In the URL absolutification step, also check for `xlink:href` attribute and absolutify it. Use `el.getAttributeNS('http://www.w3.org/1999/xlink', 'href')` to read and `el.setAttributeNS(...)` to write.

2. **Canvas snapshot:** For `<canvas>` elements, try to capture them as images:
```javascript
if (cloneEl.tagName === 'CANVAS') {
  try {
    var dataUrl = originalEl.toDataURL('image/png');
    var img = document.createElement('img');
    img.src = dataUrl;
    img.style.cssText = window.getComputedStyle(originalEl).cssText;
    img.setAttribute('data-fsb-nid', cloneEl.getAttribute('data-fsb-nid'));
    cloneEl.parentNode.replaceChild(img, cloneEl);
  } catch (e) {
    // Tainted canvas or security error -- leave as empty canvas
  }
}
```
Note: this requires walking original body alongside the clone. Implementation: use TreeWalker on both document.body and clone simultaneously, or use a pre-pass approach where canvas elements are identified on the original and their indices recorded, then replaced in the clone.

Alternative simpler approach: Before cloning, iterate all `canvas` elements in document.body, record their data URLs in a Map keyed by a temporary class, apply that class, then after cloning, find the canvas elements in the clone by that class and replace them with `<img>` tags.

3. **Empty/malformed attribute guards:** When absolutifying URLs, wrap each `new URL()` call in try/catch:
```javascript
try {
  var absUrl = new URL(attrValue, document.baseURI).href;
  cloneEl.setAttribute(attrName, absUrl);
} catch (e) {
  // Leave as-is if URL parsing fails (data: URIs, javascript: URIs, empty strings)
}
```

4. **Large page safety:** Add a size check to serializeDOM. If the resulting HTML string exceeds 2MB (2 * 1024 * 1024 characters), truncate by removing elements below the fold:
```javascript
if (html.length > 2 * 1024 * 1024) {
  // Re-serialize with only above-the-fold content
  // Walk clone, remove elements whose getBoundingClientRect().top > window.innerHeight * 3
  // This keeps first 3 viewports worth of content
}
```
Add a `truncated: true` field to the snapshot payload if truncation occurred.

5. **MutationObserver disconnect safety:** In `stopMutationStream`, check that observer exists before calling `.disconnect()`. Set observer to null after disconnect.

6. **Content script re-injection guard:** At the top of the message listener, check `streaming` flag for `domStreamStart` -- if already streaming, first stop existing stream, then restart. This prevents double-observers.

**Hardening showcase/js/dashboard.js:**

7. **Snapshot iframe error handling:** Add onerror handler to the iframe:
```javascript
previewIframe.onerror = function() {
  setPreviewState('error');
};
```

8. **Mutation application error boundary:** The existing `handleDOMMutations` already has try/catch per mutation. Ensure the outer function also has a try/catch:
```javascript
function handleDOMMutations(payload) {
  if (previewState !== 'streaming' || !previewIframe) return;
  try {
    var doc = previewIframe.contentDocument;
    if (!doc || !doc.body) return;
    // ... mutation loop
  } catch (e) {
    console.warn('[FSB-DASH] Mutation apply error:', e.message);
    // Don't change state -- keep showing last good content
  }
}
```

9. **Stale nid guard in mutations:** Before applying a mutation, verify the nid exists. For `op: 'add'`, if parentNid is not found, skip. For `op: 'rm'`, if nid not found, skip silently. For `op: 'attr'` and `op: 'text'`, if nid not found, skip. (Already handled by null checks in Plan 02, but verify and add console.debug logs for debugging.)

10. **Preview scale on container resize (ResizeObserver):** Add a ResizeObserver on the preview container for more accurate scaling than window resize alone:
```javascript
if (typeof ResizeObserver !== 'undefined' && previewContainer) {
  new ResizeObserver(function() {
    if (previewState === 'streaming') {
      updatePreviewScale();
    }
  }).observe(previewContainer);
}
```

11. **Glow overlay cleanup:** When a new snapshot arrives (handleDOMSnapshot), reset the glow overlay to hidden:
```javascript
if (previewGlow) previewGlow.style.display = 'none';
if (previewProgress) previewProgress.style.display = 'none';
```
Add these lines at the start of `handleDOMSnapshot`, before the iframe update.

12. **Scroll position memory on mutations:** After applying mutations, if the last known scroll position was received via ext:dom-scroll, re-apply it. Store the last scroll position in a variable `var lastPreviewScroll = { x: 0, y: 0 }` and update it in `handleDOMScroll`. After mutation application completes in `handleDOMMutations`, do:
```javascript
// Maintain scroll position after DOM changes
try {
  previewIframe.contentWindow.scrollTo(lastPreviewScroll.x, lastPreviewScroll.y);
} catch (e) { /* ignore */ }
```
