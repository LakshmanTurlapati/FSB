| Margin bottom | 24px | Matches `.dash-task-area` margin-bottom |
| Transition | opacity 200ms ease (fade in on appear, fade out on disappear) | Matches `dashTaskFadeIn` animation pattern |

### CSS implementation

```css
.dash-preview {
  position: relative;
  width: 100%;
  aspect-ratio: 16 / 9;
  background: var(--bg-card);
  border: 1px solid var(--border-color);
  border-radius: var(--radius-lg);
  overflow: hidden;
  margin-bottom: 24px;
  animation: dashTaskFadeIn 200ms ease;
}

.dash-preview-iframe {
  width: 100%;
  height: 100%;
  border: none;
  pointer-events: none;
  background: white;
}
```

### Iframe sandbox attributes

```
sandbox="allow-same-origin"
```

No `allow-scripts`, no `allow-forms`, no `allow-popups`. The iframe is purely display-only. `allow-same-origin` is required so the dashboard JS can access iframe DOM to apply mutations.

### Scaling approach

The iframe renders at the original page's dimensions. Use CSS `transform: scale()` to fit within the 16:9 container. The scale factor is `containerWidth / iframeContentWidth`. The iframe element itself should be sized to the original page viewport width and height, then scaled down.

```css
.dash-preview-iframe {
  transform-origin: top left;
  /* transform: scale(X) set dynamically by JS */
}
```

## State Machine

The preview container has four visual states:

### 1. Hidden (default)

- Preview container not in DOM or `display: none`
- Condition: task state is `idle`

### 2. Loading

- Preview container visible with centered spinner
- Spinner: reuse existing `.dash-spinner` (16x16, `var(--primary)` border-top)
- Text below spinner: "Connecting to browser..."
- Condition: `dash:dom-stream-start` sent, waiting for first `ext:dom-snapshot`

### 3. Streaming

- Iframe visible with cloned DOM content
- Glow overlay and progress overlay visible when data present
- Small green status dot (8px, #22c55e) in top-right corner of container, 8px inset
- Condition: receiving `ext:dom-mutations` or `ext:dom-scroll` messages

### 4. Disconnected

- Iframe shows last known content (frozen)
- Semi-transparent overlay: `rgba(0, 0, 0, 0.5)` covering entire preview
- Centered text: "Stream disconnected" in 14px `var(--text-primary)`
- Small yellow status dot (8px, #eab308) replaces green dot
- Condition: WS disconnected while preview was streaming, or `ext:dom-stream-stop` received but content still shown briefly

### 5. Error

- Preview container visible with error message
- Icon: `fa-solid fa-exclamation-triangle` in 24px `var(--text-muted)` centered
- Text: "Could not load page preview" in 14px `var(--text-secondary)`
- Subtext: "The extension may have navigated to a restricted page" in 12px `var(--text-muted)`
- Condition: snapshot serialization failed or content security policy blocked

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| > 1024px | Preview at full container width (up to 1200px), 16:9 aspect ratio |
| 768px - 1024px | Preview at full container width, 16:9 maintained |

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Click on preview | Nothing -- `pointer-events: none` on iframe |
| Task starts | Preview container fades in, shows loading state |
| First snapshot received | Loading state replaced by iframe with cloned DOM |
| Mutations received | Iframe DOM updated incrementally (no visible transition) |
| Scroll update received | Iframe scrolls to match extension viewport (smooth scroll, 100ms) |
| Overlay update received | Glow rect repositions, progress indicator updates |
| Task completes | Preview remains visible showing final state for 5 seconds, then fades out |
| Tab hidden (Page Visibility) | Stream pauses silently, preview freezes |
| Tab visible (Page Visibility) | Fresh snapshot requested, preview updates |
| WS disconnected | Disconnected overlay appears over frozen content |
| WS reconnected | Fresh snapshot requested, streaming resumes |
| Dashboard resize | iframe scale factor recalculated, glow rect repositioned |
