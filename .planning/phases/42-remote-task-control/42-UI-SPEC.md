| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, status dot margins |
| sm | 8px | Compact element spacing, metadata line gaps, input-to-button gap, progress metadata spacing |
| lg | 16px | Section inner padding, card padding |
| xl | 24px | Task area vertical padding, section margins, card padding |
| 2xl | 32px | Major section breaks (stats bar margin-bottom) |
| 3xl | 48px | Login card padding |

Exceptions: none. All new dash-task-* components use exclusively grid-aligned values from this scale. Existing dash-agent-* selectors outside this phase's scope are not governed by this contract.

## Interaction Contracts

### State Machine

```
idle --> running (on submit)
running --> success (on ext:task-complete with success: true)
running --> failed (on ext:task-complete with success: false)
success --> idle (on new task submit from success view input)
failed --> idle (on retry click or new task submit from failed view input)
```

### Transitions

| From | To | Animation |
|------|-----|-----------|
| idle | running | Input field fades out (opacity 0 over 200ms), progress view fades in (opacity 1 over 200ms, 100ms delay). Total: 300ms. Container height transitions with `max-height` + `overflow: hidden` for smooth resize. |
| running | success | Progress bar fill color transitions from #ff6b35 to #22c55e over 300ms. Metadata fades to success content. |
| running | failed | Progress bar fill color transitions from #ff6b35 to #ef4444 over 300ms. Metadata fades to failure content. |
| success | idle | Triggered by user interaction (typing in re-appeared input). No automatic transition. |
| failed | idle | Triggered by user interaction (retry click or typing in re-appeared input). No automatic transition. |

### Progress Bar Updates

- Width updates via inline style `width: {progress}%`
- CSS transition on width: `transition: width 300ms ease` -- creates smooth fill animation between WS updates
- Minimum visible width: `min-width: 2%` when progress > 0 to show activity has started
- Progress never decreases (enforced by extension's calculateProgress logic)

### Keyboard

| Key | Context | Action |
|-----|---------|--------|
| Enter | Input focused with text | Submit task |
| Enter | Input focused, empty | No action (button remains disabled) |

### Submit Validation

- Submit button disabled when input is empty (trim whitespace)
- Submit button disabled when `extensionOnline === false`
- Submit button disabled when `taskState === 'running'`
- No client-side length limit on task text

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Arrow icon button (no text -- icon-only submit, `aria-label="Submit task"`) |
| Input placeholder | "What should FSB do?" |
| Empty state heading | Not applicable -- task input is always visible when paired, no separate empty state |
| Running phase labels | "Navigating", "Reading page", "Filling form", "Clicking", "Extracting", "Writing" (from extension detectTaskPhase, rendered as-is) |
| Running action prefix | ">" (accent-colored, before action summary text) |
| Running fallback action | "Working..." (when no action description available yet) |
| Success status | "Complete" (preceded by checkmark character) |
| Success elapsed | "Completed in {duration}" |
| Failure status | "Failed" (preceded by x character) |
| Failure fallback error | "Task could not be completed" |
| Retry button label | "Retry Task" |
| Extension offline placeholder | "Extension offline..." |
| Extension busy error | "Another task is already running" |
| No active tab error | "No active browser tab" |

### Status Indicators (Unicode)

- Success checkmark: Unicode character rendered in markup, not Font Awesome icon, to keep it lightweight
- Failure x: Unicode character rendered in markup
- Action prefix ">": Plain text character, styled with accent color

## Layout Placement

The task area inserts into the existing dashboard HTML structure:

```
dash-content-section
  container
    dash-header          (existing -- "Your Agents" + badges + disconnect)
    dash-task-area       (NEW -- task input / progress / result)
    dash-stats-bar       (existing -- 6 stat cards)
    dash-agent-grid      (existing -- agent cards)
    dash-runs-panel      (existing -- run history)
    dash-empty           (existing -- no agents message)
```

The task area sits between the header and stats bar. It is visible only when `paired === true` and `extensionOnline === true` (or in offline-disabled state when paired but extension offline).

## Responsive Behavior

| Breakpoint | Change |
|------------|--------|
| > 768px | Input and button side by side in flex row. Full metadata row visible. |
| <= 768px | Input and button stack vertically (flex-direction: column). Button becomes full-width. Metadata wraps to two lines if needed. |
| <= 480px | Task area padding reduces to 16px. Font sizes remain the same (no reduction below 12px for accessibility). |
