  <action>
Add targeted relay diagnostics so server-side message direction and dropped deliveries are inspectable without replaying the entire session.

1. In `server/src/ws/handler.js`, add a bounded `roomDiagnostics` map keyed by `hashKey`. Each room keeps `{ events, receivedByType, deliveredByType, droppedByType, lastClose }` and stores at most 100 events.
2. In `setupWSHandler()`, record `connected` and `closed` events with concrete fields `{ role, hashKey, closeCode, closeReason }`. Keep `lastClose` updated for the room.
3. In the `message` handler, parse the message type when possible. Record concrete relay-direction events with `{ direction: 'dashboard->extension' | 'extension->dashboard', type, targetCount, deliveredCount, droppedCount }`. On malformed JSON, record a `malformed-json` diagnostic event.
4. Update `relayToRoom()` and `broadcast()` so they count open targets versus dropped targets. If `targetCount === 0` or any client is not `WebSocket.OPEN`, append a dropped-delivery event for that message type instead of only skipping the send.
5. Export `getRoomDiagnostics(hashKey)` from the module alongside the existing exports so later phases or manual server inspection can read the aggregated counters directly.
6. Do not store raw DOM payload bodies or task text in diagnostics. Store message type, direction, counts, and close metadata only.
  </action>
