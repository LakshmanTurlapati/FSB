- **Scroll-compare-stop loop not executed:** The core SCROLL-10 test (scrolling news feed checking article dates until reaching yesterday's articles) could not be performed without live browser MCP tool execution. All testing was HTTP-based.
- **time[datetime] ISO 8601 attributes NOT PRESENT:** The site guide's preferred timestamp strategy (time elements with datetime attributes) does not apply to BBC News. ## Tool Gaps Identified
- **WebSocket bridge availability (persistent gap, Phases 55-76):** The MCP server process runs on port 7225 with an established TCP connection, but the Chrome extension side returns HTTP 426 "Upgrade Required" for browser action dispatch. This has blocked every live MCP test since Phase 55. Without the bridge, no MCP tool can execute against the live browser DOM. The scroll-compare-stop loop -- the core test for SCROLL-10 -- requires live browser execution.
- **No time[datetime] parsing available in BBC DOM:** The site guide and SCROLL-10 workflow assume time[datetime] ISO 8601 attributes for reliable date comparison. BBC does not use these. The MCP tools would need to either: (a) parse relative text ("21 hrs ago" -> calculate actual date from current time), or (b) extract __NEXT_DATA__ JSON and parse numeric timestamps. Neither approach is currently documented in the tool chain.
- **Relative timestamp parsing not supported:** The visible timestamps ("3 hrs ago", "1 day ago") require knowing the current time and performing arithmetic to determine the actual publication date. This is unreliable across timezone boundaries and time-of-day variations. A "21 hrs ago" article could be from today or yesterday depending on when the page is loaded.
- **__NEXT_DATA__ JSON extraction not in current tool chain:** The most reliable date source (lastUpdated numeric timestamps) is embedded in a script tag, not in visible DOM elements. Current MCP tools (read_page, get_dom_snapshot) focus on visible DOM content. Extracting and parsing __NEXT_DATA__ JSON would provide precise date comparison but requires a dedicated extraction strategy.
- **BBC does not use date section headers:** The site guide mentions date section headers ("Today", "Yesterday") as quick stop signals. BBC News homepage uses topic-based sections ("Iran war", "More to explore", "Sport"), not date-based groupings. Date section headers are not a reliable stop signal for BBC.
- **Infinite scroll detection uncertain:** Without live browser testing, it is unclear whether BBC News homepage supports infinite scroll at all. The server HTML appears to be a complete finite page. The scroll-compare-stop loop may not be applicable if the page is static. Different BBC pages (e.g., /news/world, topic pages) may have different scroll behavior.
- **Screenshot/visual comparison not available:** For verifying that the article feed layout matches expectations and timestamps are visually positioned correctly, a screenshot tool would provide definitive evidence. Current MCP tools rely on DOM text extraction only.

## Autopilot Recommendations

1. **Dismiss cookie/notification banners before scrolling.** BBC cookie consent banner appears via client-side JavaScript, primarily for EU/UK visitors. It may not trigger for US-based requests. Autopilot should check for and dismiss any consent overlay before attempting to scroll or read articles. Use click on the accept/reject button if a consent banner is detected.

2. **Use __NEXT_DATA__ JSON timestamps instead of DOM datetime attributes for BBC.** BBC News does NOT use time[datetime] ISO 8601 attributes. The most reliable date source is the __NEXT_DATA__ JSON embedded in a script tag, which contains lastUpdated numeric Unix timestamps (milliseconds) for every article. Autopilot should extract this JSON, parse the lastUpdated fields, and convert to dates for comparison. Fallback: parse relative text ("21 hrs ago") from data-testid="card-metadata-lastupdated" span elements.

3. **Calculate yesterday date at start and compare throughout scroll loop.** Get today's date at the start of the task. Yesterday = today - 1 day, formatted as YYYY-MM-DD. Compare each article's date (from __NEXT_DATA__ or relative text) against yesterday. Stop when articles with dates OLDER than yesterday appear. On BBC, the initial page load already contains articles spanning today + yesterday + older, so the stop condition may be met without scrolling.

4. **Use 600-800px scroll increments for article-height cards.** BBC article cards vary in height: dundee-cards are large hero layouts, cambridge-cards are medium with images, manchester-cards are compact text-only. Scroll 600-800px per cycle to ensure at least 2-3 new article cards enter the viewport per scroll. Wait 1500-2000ms after scroll for any infinite scroll content to load.

5. **Stop condition: article date OLDER than yesterday (not just equal to yesterday).** When an article's lastUpdated date is before yesterday (e.g., 2026-03-19 when yesterday is 2026-03-20), stop scrolling. This means all yesterday articles have been passed. Do NOT stop when the first yesterday article appears -- continue scrolling until the oldest yesterday article is passed.

6. **Article deduplication by href to avoid counting articles visible across scroll boundaries.** BBC article links use /news/articles/{hash} format with unique hash IDs (e.g., c5yljdgwppzo). Some articles appear multiple times in different page sections (hero + "More to explore"). Use the href as a deduplication key. After each scroll, compare new article hrefs against the already-seen set.

7. **Handle both absolute timestamps and relative time text.** BBC uses relative text ("3 hrs ago", "1 day ago") in visible DOM. Other news sites (Reuters, CNN) may use absolute dates or ISO 8601 datetime attributes. Autopilot should implement multi-format parsing: (a) numeric Unix timestamps from JSON data, (b) ISO 8601 datetime attributes, (c) absolute date text ("March 20, 2026"), (d) relative text with current-time arithmetic.

8. **BBC homepage may be finite, not infinite scroll.** No load-more button or scroll sentinel was found in BBC News server HTML. The homepage appears to be a curated finite page with 47 articles across topic sections. Autopilot should detect the "end of feed" condition (2 consecutive scrolls with 0 new articles) and stop rather than scrolling indefinitely. For true infinite scroll behavior, try BBC topic pages or Reuters.

9. **Use data-testid selectors for BBC instead of generic article/time tags.** BBC article cards are identified by data-testid="dundee-card", "cambridge-card", "manchester-card", "london-card" -- not by article tags or class patterns. Timestamps are in data-testid="card-metadata-lastupdated" span elements. Headlines are in data-testid="card-headline" h2/h3 elements. These data-testid selectors are stable and specific.

10. **Timezone consideration: compare dates at day granularity (YYYY-MM-DD) to avoid timezone edge cases.** BBC serves timestamps in UTC (based on __NEXT_DATA__ lastUpdated values). A "21 hrs ago" article at 22:13 UTC means the article is from approximately 01:13 UTC today, which is still "today" in UTC. Compare dates at YYYY-MM-DD granularity only, ignoring hours/minutes. An article from "2026-03-20" is yesterday regardless of what hour it was published.
