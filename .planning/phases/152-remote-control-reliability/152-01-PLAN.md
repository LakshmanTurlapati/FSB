Output: Remote clicks and scrolls are clamped to the live preview viewport, remote typing only engages when the preview is actively focused for control, and printable text uses a non-duplicating dispatch path.
</objective>

Rework the website dashboard's remote-control capture path so it only forwards intentional preview interactions.

1. Make the remote overlay focusable and move keyboard capture from the whole document to the active preview-control surface.

3. Clamp remote click and scroll coordinates to the current preview viewport dimensions derived from the accepted snapshot so blank space, maximize/fullscreen layouts, and stale scale values do not generate out-of-bounds coordinates.
4. Keep remote-control interaction independent from dashboard form inputs and task controls so typing in the dashboard never leaks to the browser tab.
5. Preserve modifier keys and non-printable keys, but stop relying on document-level capture that steals all keyboard input while remote control is enabled.
  </action>
