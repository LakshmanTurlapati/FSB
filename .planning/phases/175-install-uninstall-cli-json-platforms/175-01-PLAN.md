   `const entry = getServerEntry();`

This is a backward-compatible change -- existing callers that pass 3 args get `entry = null`, which falls through to `getServerEntry()`.

  <name>Task 2: Create install.js orchestration module</name>

  <action>
Create `mcp-server/build/install.js` as a new ESM module exporting `runInstall(flags)` and `runUninstall(flags)`. This is the orchestration layer that connects Phase 174's engine to user-facing CLI commands.

The module must implement:

**Imports:**
```javascript
import { PLATFORMS, resolvePlatformConfig, getServerEntry } from './platforms.js';
import { installToConfig, removeFromConfig } from './config-writer.js';
import { FSB_MCP_VERSION, FSB_SERVER_NAME } from './version.js';
import { execSync } from 'node:child_process';
```

**Helper: `getMatchedPlatforms(flags)`** (internal, not exported)
Iterate `Object.entries(PLATFORMS)`. For each entry where `flags[key] === true`:
- If `platform.format === 'json' || platform.format === 'jsonc'`, add to matched array.
- Skip `cli`, `toml`, `yaml` formats (those are Phase 176 scope).
Return array of `{ key, platform }` objects.

**Helper: `printResult(result)`** (internal, not exported)
Map the result object from installToConfig/removeFromConfig to one-line console output per D-03:
- `status === 'created'` or `'updated'` or `'removed'`: `console.log('\u2713 ' + result.message)` (checkmark)
- `status === 'skipped'` or `'not-found'`: `console.log('\u25CB ' + result.message)` (circle)
- `status === 'error'`: `console.error('\u2717 ' + result.message)` (x-mark)

**Helper: `printInstallUsage()`** (internal, not exported)
Per D-04, when `install` has no platform flags:
```
FSB MCP Server {FSB_MCP_VERSION}

Usage: fsb-mcp-server install --<platform>

Platforms:
  --claude-desktop   Claude Desktop
  --claude-code      Claude Code
  --cursor           Cursor
  --vscode           VS Code
  --windsurf         Windsurf
  --cline            Cline
  --zed              Zed
  --codex            Codex CLI
  --gemini           Gemini CLI
  --continue         Continue
```
List ALL 10 platforms by iterating `Object.entries(PLATFORMS)` and using `platform.flag.padEnd(18)` and `platform.displayName`. This shows users what's available even though 3 are Phase 176.

**Helper: `printUninstallUsage()`** (internal, not exported)
Per D-05, same as above but with "uninstall" in the usage line:
```
FSB MCP Server {FSB_MCP_VERSION}

Usage: fsb-mcp-server uninstall --<platform>

Platforms:
  [same list as install]
```

**Export: `async function runInstall(flags)`**
Per D-01, supports multiple flags in one invocation:
1. Call `getMatchedPlatforms(flags)` to collect JSON/JSONC platforms with matching flags.
2. If `matched.length === 0` AND no non-JSON platform flags are set, call `printInstallUsage()` and return.
   - Check for non-JSON flags: if `flags['claude-code']` or `flags['codex']` or `flags['continue']` is true, print `'\u25CB --{flag} is not yet supported (coming in a future update)'` for each, then if no JSON matches either, return.
3. For each matched platform:
   a. Call `resolvePlatformConfig(key)` to get the config path.
   b. If `configPath` is null, print `'\u25CB {platform.displayName} is not supported on this OS'` and continue.
   c. Build the entry: `let entry = getServerEntry()`. If `key === 'vscode'`, augment: `entry = { type: 'stdio', ...entry }` (per PLAT-03).
   d. Call `await installToConfig(configPath, platform, FSB_SERVER_NAME, entry)`.
   e. Call `printResult(result)`.

**Export: `async function runUninstall(flags)`**
Per D-02, separate subcommand with same platform flags:
1. Special case for UNINST-05: if `flags['claude-code'] === true`, handle Claude Code uninstall:
   - Try `execSync('claude mcp remove fsb', { stdio: 'pipe', encoding: 'utf-8' })`.
   - On success, print `'\u2713 Removed FSB from Claude Code'`.
   - On error (e.g., `claude` CLI not found), print `'\u25CB Claude Code: run manually: claude mcp remove fsb'`.
2. Call `getMatchedPlatforms(flags)` for JSON/JSONC platforms.
3. If `matched.length === 0` AND no `flags['claude-code']` AND no non-JSON flags are set, call `printUninstallUsage()` and return.
   - Same non-JSON flag check as runInstall for `codex`/`continue`.
4. For each matched platform:
   a. Call `resolvePlatformConfig(key)` for the config path.
   b. If null, print `'\u25CB {platform.displayName} is not supported on this OS'` and continue.
   c. Call `await removeFromConfig(configPath, platform, FSB_SERVER_NAME)`.
   d. Call `printResult(result)`.

**Requirements covered:**
- PLAT-01 through PLAT-07: Each JSON/JSONC platform is handled by the PLATFORMS iteration (per D-01)
- UNINST-01: `runUninstall` calls `removeFromConfig` for matched platforms
- UNINST-02: `removeFromConfig` preserves other entries (built in Phase 174)
- UNINST-03: `removeFromConfig` creates `.bak` backup (built in Phase 174)
- UNINST-04: `removeFromConfig` returns `not-found` status when FSB not configured (built in Phase 174)
- UNINST-05: Claude Code uninstall via `execSync('claude mcp remove fsb')` with fallback

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI flags -> platform dispatch | User-supplied flags select which config files to modify |
| config-writer -> filesystem | Reads/writes user config files for third-party applications |
| execSync -> claude CLI | Shells out to external CLI for Claude Code uninstall |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-175-01 | T (Tampering) | installToConfig entry override | mitigate | Entry is constructed internally from getServerEntry() + hardcoded type:stdio augmentation; no user input flows into entry object |
| T-175-02 | D (Denial of Service) | execSync claude mcp remove | mitigate | Wrap in try/catch; execSync failure prints fallback message instead of crashing |
| T-175-03 | I (Information Disclosure) | printResult error messages | accept | Error messages may contain file paths, which is expected for a local CLI tool |
| T-175-04 | T (Tampering) | Flag injection via parseArgs | accept | parseArgs already sanitizes in index.js; install.js only checks boolean flag values |
</threat_model>
