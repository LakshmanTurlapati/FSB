**Target Site Selection:**
Use virtualpiano.net as the primary target. This is a free, no-auth online piano that maps keyboard letters to piano notes. Fallback sites: pianu.com, recursivearts.com/virtual-piano. Research the keyboard mapping:
- virtualpiano.net maps the middle row of the keyboard to piano notes. The standard mapping for the C major octave is: A=C, S=D, D=E, F=F, G=G, H=A, J=B (white keys). So to play E-D-C-D the keyboard presses would be: D, S, A, S.
- Some online pianos use different mappings. The site guide must document the virtualpiano.net mapping specifically and note that other pianos may differ.
- If the piano uses DOM-rendered keys (divs with click handlers), DOM click or click_at is an alternative.

**Create directory and file:**
Create the directory `site-guides/music/` if it does not exist, then create `site-guides/music/virtual-piano.js`.

**Site Guide Structure:**
Create `site-guides/music/virtual-piano.js` following the registerSiteGuide pattern. The site guide MUST include:

1. **Header comment** explaining this covers online virtual piano keyboards. Contrast keyboard-mapped pianos (press_key works) vs canvas-rendered pianos (click_at needed). Note that virtualpiano.net is the primary target.

2. **patterns** array: URL patterns matching virtual piano sites:
   - /virtualpiano\.net/i
   - /pianu\.com/i
   - /recursivearts\.com\/virtual-piano/i
   - /onlinepianist\.com/i
   - /autopiano\.com/i

3. **guidance** string covering these sections:
   - KEYBOARD MAPPING: virtualpiano.net maps keyboard letters to piano notes. White keys use the home row: A=C4, S=D4, D=E4, F=F4, G=G4, H=A4, J=B4. Sharp/flat keys (black keys) use the top row: W=C#4, E=D#4, T=F#4, Y=G#4, U=A#4. To play "Mary Had a Little Lamb" (E-D-C-D): press D, S, A, S in sequence using press_key.
   - DOM PIANO KEYS: Some pianos render keys as div or button elements with data attributes like data-note="E4" or class="white-key". These can be clicked with DOM click using selectors like [data-note="E4"] or .piano-key.white[data-key="d"].
   - CANVAS PIANO KEYS: If the piano renders keys on a canvas element, use click_at at the pixel coordinates of each key. White keys are evenly spaced across the piano width. The standard layout is 7 white keys per octave, with black keys in groups of 2 and 3 between them.
   - NOTE SEQUENCE: Mary Had a Little Lamb first four notes = E4, D4, C4, D4. Full first phrase = E D C D E E E (half rest) D D D (half rest) E G G.
   - TIMING: Add 300-500ms delay between notes for natural playback. Too fast may not register all key events.
   - STATE VERIFICATION: Look for visual feedback on played keys (CSS class change like .active, .pressed, .highlight; inline style changes; or canvas animation). Some pianos show the note name or highlight the key briefly after pressing.
   - COMMON OBSTACLES: Cookie consent banners, premium subscription modals, fullscreen tutorial overlays, audio autoplay policy (user must interact with page first before audio plays -- click anywhere on the page first).

4. **selectors** object:
   - `pianoContainer`: Selectors for the piano keyboard container element: `.piano`, `#piano`, `.keyboard`, `#keyboard`, `.piano-container`
   - `whiteKeys`: Selectors for individual white keys: `.white-key`, `.piano-key.white`, `[data-note]`, `.key.white`
   - `blackKeys`: Selectors for individual black keys: `.black-key`, `.piano-key.black`, `.key.black`
   - `noteLabels`: Selectors for note label overlays: `.note-label`, `.key-label`, `.key-text`
   - `consentDismiss`: Cookie/consent dismiss: `.cookie-accept`, `#accept-cookies`, `.consent-close`
   - `modalClose`: Modal/overlay close: `.modal-close`, `.close-btn`, `.overlay-close`, `[aria-label="Close"]`
   - Provide 2-3 fallback selectors per element

5. **workflows** object with these workflows:
   - `loadPiano`: Steps to navigate to the piano URL, dismiss cookie/consent popups, dismiss any subscription or tutorial modals, click once on the page body to satisfy browser audio autoplay policy, wait for piano keys to be visible
   - `playNoteKeyboard`: Steps to play a single note via press_key using the virtualpiano.net keyboard mapping. Input: note name (e.g., "E4"). Output: press_key call with the mapped keyboard letter. Include the mapping table in the workflow.
   - `playMaryHadALittleLamb`: Steps to play E4-D4-C4-D4 in sequence. For virtualpiano.net: press_key("d"), wait 400ms, press_key("s"), wait 400ms, press_key("a"), wait 400ms, press_key("s"). For DOM pianos: click on [data-note="E4"], wait 400ms, click on [data-note="D4"], etc.
   - `playNoteClickAt`: Fallback workflow for canvas pianos. Steps to get piano element bounds, calculate key position from note index and piano width, execute click_at at key center coordinates.

6. **warnings** array (at least 5):
   - "Browser audio autoplay policy requires at least one user interaction (click) before piano sounds will play -- always click the page body first"
   - "Keyboard mapping varies by piano site -- virtualpiano.net uses A=C, S=D, D=E, F=F, G=G, H=A, J=B but other sites may differ"
   - "press_key sends keyboard events to the active page -- ensure the piano has focus (click on it first) before sending key presses"
   - "Some pianos require the piano container to have focus, not just the page body -- click directly on the piano element"
   - "Canvas-rendered pianos have no DOM elements for individual keys -- use click_at at calculated pixel coordinates"
   - "Black keys are narrower and offset from white keys -- coordinate calculation must account for the 2+3 black key grouping pattern per octave"

7. **toolPreferences** array:
   - "PREFER press_key for pianos with keyboard mapping (virtualpiano.net, pianu.com) -- faster and more reliable than coordinate clicking"
   - "Use DOM click on [data-note] or .piano-key selectors if keyboard mapping is unknown or not working"
   - "Use click_at ONLY for canvas-rendered pianos where no DOM elements exist for individual keys"
   - "Always click the page body or piano container first to ensure focus and satisfy audio autoplay policy"
   - "Add 300-500ms wait between notes for reliable playback -- too fast may cause dropped notes"
   - "Use get_dom_snapshot after playing to check for visual feedback (.active, .pressed classes) on played keys"
