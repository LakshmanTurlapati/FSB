**Step 1: Navigate to the online piano**
Use MCP `navigate` to open https://virtualpiano.net/. Wait for page load with `wait_for_stable`.

**Step 2: Dismiss any overlays**
Use `get_dom_snapshot` to check for cookie consent dialogs, subscription modals, or tutorial overlays. Click dismiss/accept/close buttons using DOM `click` if found. Common selectors: `.cookie-accept`, `.modal-close`, `[aria-label="Close"]`, `.close-btn`.

**Step 3: Satisfy browser audio autoplay policy**
Click on the page body or the piano container using `click_at` at a neutral area (center of viewport) to ensure the browser allows audio playback. This is required because browsers block audio until user interaction. Alternatively, use DOM `click` on the piano container element.

**Step 4: Verify piano is loaded and identify interaction method**
Use `get_dom_snapshot` to inspect the piano:
- Look for DOM elements representing piano keys (div, button with data-note, class like .white-key, .piano-key)
- Check if piano keys are canvas-rendered (single canvas element, no individual key DOM nodes)
- Record the piano container position and dimensions
- Determine interaction method: press_key (if keyboard mapping documented) vs DOM click vs click_at

**Step 5: Play note E4 (first note of Mary Had a Little Lamb)**
Using the keyboard mapping from the site guide (virtualpiano.net: D key = E4 note):
- Execute `press_key("d")` to play E4
- Wait 400ms for the note to sound and visual feedback to appear
- Use `get_dom_snapshot` to check for visual feedback (key highlight, .active class, CSS change)
- Record whether the note registered

If press_key does not work (no visual/audio feedback), fall back to:
- Find the E4 key element via DOM selectors ([data-note="E4"] or equivalent)
- Use DOM `click` on that element
- Or use `click_at` at the E4 key pixel coordinates if canvas-rendered

**Step 6: Play note D4 (second note)**
Execute `press_key("s")` to play D4 (virtualpiano.net: S key = D4 note). Wait 400ms. Check for visual feedback.

**Step 7: Play note C4 (third note)**
Execute `press_key("a")` to play C4 (virtualpiano.net: A key = C4 note). Wait 400ms. Check for visual feedback.

**Step 8: Play note D4 again (fourth note)**
Execute `press_key("s")` to play D4 again. Wait 400ms. Check for visual feedback.

**Step 9: Verify all four notes were played**
After playing all four notes:
- Use `get_dom_snapshot` or `read_page` to check for any play history, visual note trail, or feedback
- The primary verification is that press_key (or click) executed without error for each note
- Visual feedback (key highlighting) confirms the piano responded to each input
- If no DOM-level verification possible, the successful execution of 4 press_key calls constitutes best-available verification
