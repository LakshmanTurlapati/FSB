  <action>
1. Install ws package: `cd mcp-server && npm install ws && npm install -D @types/ws`

2. Rewrite `mcp-server/src/bridge.ts` with a new `WebSocketBridge` class that has the SAME public interface as NativeMessagingBridge:
   - `connect(): Promise<void>` -- creates a `new WebSocketServer({ port: 7225 })` and listens for connections. The bridge is "connected" when a WebSocket client connects. Resolve immediately after server starts listening (don't wait for a client).
   - `disconnect(): void` -- closes all connected clients and the WebSocket server, rejects pending requests.
   - `sendAndWait(msg: Omit<MCPMessage, 'id'>, options?: { timeout?: number; onProgress?: (p: MCPResponse) => void }): Promise<Record<string, unknown>>` -- generates ID via `generateId()`, sends `MCPMessage` as JSON string to the connected client, sets up pending request with timeout. When response with matching ID arrives, resolves with `resp.payload`.
   - `get isConnected(): boolean` -- returns true when a WebSocket client is connected and ready.
   - `generateId(): string` -- same pattern: `mcp_${++counter}_${Date.now()}`

   Implementation details:
   - Import: `import { WebSocketServer, type WebSocket as WsWebSocket } from 'ws';`
   - Store the server as `private wss: WebSocketServer | null = null`
   - Store the connected client as `private client: WsWebSocket | null = null`
   - Only allow ONE connected client at a time (MCP server talks to one extension). If a second client connects, close the old one.
   - On client `message` event: parse JSON, check if it's MCPResponse. If `resp.type === 'mcp:progress'`, route to progressListeners map. Otherwise resolve pending request by ID.
   - On client `close` event: set `this.client = null`, `this.connected = false`, reject all pending requests with "Extension WebSocket disconnected".
   - On server `listening` event: log `[FSB MCP] WebSocket server listening on port 7225`
   - On server `error` event: if EADDRINUSE, log helpful message: `[FSB MCP] Port 7225 already in use. Is another MCP server running?`
   - Port: hardcoded `7225` (no configuration needed)
   - Pending requests, progress listeners, and ID counter: same pattern as existing code.

3. Update `mcp-server/src/types.ts`:
   - Remove the `BridgeMessage` interface entirely (it was IPC-specific, no longer needed)
   - Keep MCPMessage, MCPResponse, MCPProgress, MCPMessageType, ToolResult unchanged

4. Update `mcp-server/src/errors.ts`:
   - Change `native_messaging_error` key to `extension_not_connected`
   - Update its value from "Native messaging communication failed..." to: "Extension WebSocket not connected. The FSB Chrome extension must be running with a WebSocket connection to ws://localhost:7225. Verify the extension is installed and running."
   - In `bridge.ts`, use `FSB_ERROR_MESSAGES['extension_not_connected']` when sendAndWait is called with no connected client.
  </action>
