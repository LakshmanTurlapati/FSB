## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Status dot margins, icon gaps |
| sm | 8px | Tab gaps, compact padding, mobile grid gaps |
| md | 16px | Default element spacing, container padding (mobile), section margins |
| lg | 24px | Card padding (mobile), section padding, container padding (desktop) |
| xl | 32px | Feature card padding, section margins |
| 2xl | 48px | Login card padding (desktop), section header margin, footer padding |
| 3xl | 64px | Empty state padding, page-level spacing |

Exceptions: QR viewfinder container uses 250x250px fixed dimensions (html5-qrcode requirement). Countdown timer ring uses 44px diameter for touch-target compliance.

## Interaction States

### Login Card Tab Switching

| State | Visual | Trigger |
|-------|--------|---------|
| Scan QR active (default) | Tab underlined with `--primary`, viewfinder visible, paste tab hidden | Page load or click "Scan QR" tab |
| Paste Key active | Tab underlined with `--primary`, input+button visible, viewfinder hidden, camera stopped | Click "Paste Key" tab |
| Camera denied fallback | Auto-switch to Paste Key tab, brief error message above input: "Camera unavailable" in `--text-muted` | Camera permission denied or NotFoundError |
| Scan failure fallback | Auto-switch to Paste Key tab, brief error message: "Scan failed -- paste your key instead" | html5-qrcode fatal error |

### QR Code in Extension Control Panel

| State | Visual | Trigger |
|-------|--------|---------|
| Hidden (default) | "Pair Dashboard" button visible, QR container hidden (`display: none`) | Initial state |
| Generating | Button shows spinner, QR container still hidden | Click "Pair Dashboard" |
| Displayed | QR SVG visible below hash key area, countdown timer running (60s), button text changes to "Cancel Pairing" | Token received from server |
| Expiring (<10s) | Countdown turns `--primary` color (orange), pulse animation on timer | Timer reaches 10 seconds |
| Expired | QR fades out, message "Token expired", button reverts to "Pair Dashboard" | 60 seconds elapsed |
| Scanned successfully | QR collapses, brief success message "Dashboard paired!" in green | Server confirms token consumed (polled or WS notification) |

### Dashboard Pairing States

| State | Visual | Trigger |
|-------|--------|---------|
| Unpaired | Login card centered on page, dashboard content hidden | No valid session in localStorage |
| Scanning | Camera viewfinder active, scanning animation | User on "Scan QR" tab with camera active |
| Pairing in progress | Spinner in viewfinder area, "Connecting..." text | QR decoded, token exchange in progress |
| Paired | Login card fades out (opacity 0 over 400ms, then `display: none`), dashboard content fades in. Header shows "Paired" badge | Successful token exchange or valid session on load |
| Session expired | Dashboard content fades to 50% opacity, login card fades back in with message "Session expired. Scan QR code to reconnect." | Session token past 24h TTL |
| Extension offline | "Paired" badge stays, connection indicator shows "Disconnected" (gray badge) | WS connection lost while session valid |

### Disconnect / Unpair

| State | Visual | Trigger |
|-------|--------|---------|
| Hover | Disconnect button text turns `#ef4444` (destructive red) | Mouse hover on disconnect button |
| Clicked | Immediate transition: dashboard content fades out, login card fades in. No confirmation dialog needed (low-stakes action, easily reversible via re-scan). | Click "Disconnect" |

## Responsive Breakpoints

| Breakpoint | Behavior |
|------------|----------|
| > 1024px | Full desktop layout. Login card 480px max-width centered. |
| 769px -- 1024px | Feature grid 2-col. Dashboard stats 3-col. Login card unchanged. |
| <= 768px | Mobile layout. Nav hamburger active. Login card padding reduces to 32px 24px. Feature grid 1-col. Hero stacks vertically. QR viewfinder scales to fill card width (max 280px). Login form stacks vertically. |

## Transition Animations

| Transition | Duration | Easing | Description |
|------------|----------|--------|-------------|
| Login to paired | 400ms | ease | Login card opacity 1 to 0, then display none. Dashboard content opacity 0 to 1. |
| Paired to expired | 400ms | ease | Dashboard content opacity fades to 0.5, login card fades in from opacity 0 to 1. |
| Disconnect | 300ms | ease | Dashboard content opacity 1 to 0, login card opacity 0 to 1. |
| QR expand | 250ms | ease-out | Container height 0 to auto (use max-height trick: 0 to 400px). |
| QR collapse | 200ms | ease-in | Container height to 0, opacity to 0. |
| Tab switch | 150ms | ease | Active tab content fades in, inactive fades out. No layout shift. |
| Countdown pulse (<10s) | 1000ms | ease-in-out | Scale 1 to 1.05 and back, repeating. |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Tab keyboard navigation | Login tabs reachable via Tab key, activatable via Enter/Space |
| Camera viewfinder alt text | `aria-label="QR code camera viewfinder"` on scanner container |
| Countdown announcement | `aria-live="polite"` on countdown container, announce at 30s, 10s, and 0s |
| Color contrast | All text meets WCAG AA. Green (#22c55e) on dark bg passes. Orange (#ff6b35) on dark bg passes. |
| Focus indicator | Default browser outline on all interactive elements (buttons, tabs, inputs) |
| Screen reader paired status | `aria-live="polite"` on paired status badge area |
