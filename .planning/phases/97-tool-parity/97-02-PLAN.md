In ai/cli-parser.js, modify the existing dragdrop COMMAND_REGISTRY entry (currently at line 255) to include optional parameters that match the MCP manual mode capabilities.

Change FROM:
```javascript
  dragdrop:           { tool: 'dragdrop',            args: [{ name: 'sourceRef', type: 'string' }, { name: 'targetRef', type: 'string' }] },
```

Change TO:
```javascript
  dragdrop:           { tool: 'dragdrop',            args: [{ name: 'sourceRef', type: 'string' }, { name: 'targetRef', type: 'string' }, { name: 'steps', type: 'number', optional: true }, { name: 'holdMs', type: 'number', optional: true }, { name: 'stepDelayMs', type: 'number', optional: true }] },
```

The three new optional parameters:
- steps (number, optional): Number of intermediate move steps during the drag (default handled by tool implementation)
- holdMs (number, optional): Milliseconds to hold before releasing (default handled by tool implementation)
- stepDelayMs (number, optional): Delay between each intermediate move step (default handled by tool implementation)

These match the MCP manual mode drag_drop tool's parameter interface. The AI can now emit:
  `dragdrop e5 e10 15 500 30`
to drag element e5 to e10 with 15 steps, 500ms hold, and 30ms step delay.

The existing two positional args (sourceRef, targetRef) remain unchanged and mandatory.
