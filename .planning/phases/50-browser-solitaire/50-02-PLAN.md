**Step 1: Navigate to Google Solitaire**
Use MCP `navigate` to open `https://www.google.com/search?q=solitaire`. Wait for page load with `wait_for_stable`.

**Step 2: Dismiss any overlays**
Use `get_dom_snapshot` to check for cookie consent dialogs, promo banners, or other overlays. Click dismiss/accept buttons if found.

**Step 3: Launch the game**
Using selectors discovered in Plan 01 (from the site guide):
- Click the "Play" button on the solitaire game card in search results
- Wait for the game to initialize with `wait_for_stable`
- If difficulty selection appears, choose "Easy" (1-card draw) for simpler card moves

**Step 4: Inspect game state**
Use `get_dom_snapshot` to capture the current game state:
- Identify all visible cards and their positions
- Identify which cards are face-up vs face-down
- Identify the tableau columns, stock pile, waste pile, and foundation piles
- Record the viewport coordinates of key card elements

**Step 5: Attempt card move 1 -- draw from stock**
Click on the stock pile to draw a card to the waste pile. This is the simplest interaction to confirm game responsiveness.
- If DOM-based: `click` on the stock pile selector
- If canvas/iframe: `click_at` at the stock pile coordinates
- Use `get_dom_snapshot` or `wait_for_stable` to verify a card was drawn

**Step 6: Attempt card move 2 -- move card to valid target**
Identify a valid card move based on the current game state:
- Look for an Ace in the tableau or waste pile to move to an empty foundation
- Or look for a card that can be placed on another tableau column (alternating color, descending rank)
- Execute the move using the interaction pattern from the site guide:
  - Try double-click first (auto-sends to valid foundation if applicable)
  - If double-click does not work, try drag from source card to target pile
  - If drag does not work, try click-to-select then click-target

**Step 7: Attempt card move 3 (if moves 1-2 succeeded)**
Attempt one more move to demonstrate consistent card interaction capability. Pick any valid move from the current game state.

**Step 8: Record all results**
For each step, record: MCP tool used, target element/coordinates, result (OK/FAIL), and any notes about what happened.
