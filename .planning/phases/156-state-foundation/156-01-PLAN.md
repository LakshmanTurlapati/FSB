2. **`createSession(overrides = {})`** -- factory function that returns a new session object. Builds defaults from SESSION_FIELDS, then applies overrides via `Object.assign`. Per D-01: no ad-hoc property addition allowed -- all fields exist from creation. Example:
```javascript
function createSession(overrides = {}) {
  var session = {};
  for (var key in SESSION_FIELDS) {
    var field = SESSION_FIELDS[key];
    var defaultVal = field.default;
    // Deep-clone arrays and objects to avoid shared references
    if (Array.isArray(defaultVal)) {
      session[key] = [];
    } else if (defaultVal !== null && typeof defaultVal === 'object') {
      session[key] = JSON.parse(JSON.stringify(defaultVal));
    } else {
      session[key] = defaultVal;
    }
  }
  return Object.assign(session, overrides);
}
```

4. **`getHotFieldNames()`** -- returns an array of field names where tier is 'hot'. Used to identify what cannot be serialized.

Create `ai/state-emitter.js` with a `SessionStateEmitter` class. Per D-06: lives in the background service worker, translates internal state events to chrome.runtime messages. Per D-07: delta events only.

1. **SessionStateEmitter class**:
```javascript
class SessionStateEmitter {
  constructor() {
    this._handlers = {};  // Map<eventType, Set<Function>>
  }

  on(eventType, handler) { ... }
  off(eventType, handler) { ... }
  emit(eventType, data) { ... }
  removeAllListeners(eventType) { ... }
}
```

2. **Event types** -- define as a constant:
```javascript
var STATE_EVENTS = {
  STATUS_CHANGED: 'status_changed',        // Session status transition (idle -> running, etc.)
  ITERATION_COMPLETE: 'iteration_complete', // After each agent iteration
  COST_UPDATED: 'cost_updated',            // Token usage / cost change
  TOOL_EXECUTED: 'tool_executed',           // After a tool call completes
  SESSION_STARTED: 'session_started',       // New session created
  SESSION_ENDED: 'session_ended',           // Session completed/failed/stopped
  ERROR_OCCURRED: 'error_occurred'          // Error during session
};
```

3. **`emit(eventType, data)`** method:
   - Calls all registered handlers for that eventType with the data object
   - Also broadcasts via `chrome.runtime.sendMessage` for sidepanel/popup listeners (wrapped in try/catch, fire-and-forget)
   - The chrome.runtime.sendMessage payload format: `{ action: 'sessionStateEvent', eventType: eventType, ...data }`
   - Per D-07: data should be the delta only (what changed), not the full session

4. **`on(eventType, handler)`** method:
   - Registers a handler function for an event type
   - Returns an unsubscribe function: `return () => this.off(eventType, handler);`

5. **`off(eventType, handler)`** method:
   - Removes a specific handler

6. **`removeAllListeners(eventType)`** method:
   - If eventType provided, remove all handlers for that type
   - If no eventType, remove ALL handlers (cleanup for session end)

7. **Helper function `emitStatusChange(emitter, sessionId, oldStatus, newStatus, extras = {})`**:
   - Convenience wrapper that emits STATUS_CHANGED with `{ sessionId, oldStatus, newStatus, timestamp: Date.now(), ...extras }`
   - This is what replaces the scattered sendStatus calls

8. **Helper function `emitIterationComplete(emitter, sessionId, iteration, cost, inputTokens, outputTokens)`**:
   - Convenience wrapper that emits ITERATION_COMPLETE with `{ sessionId, iteration, cost, inputTokens, outputTokens, timestamp: Date.now() }`

Use `var` for module-level variables. Standard cross-environment export:
```javascript
if (typeof module !== 'undefined' && module.exports) {
  module.exports = { SessionStateEmitter, STATE_EVENTS, emitStatusChange, emitIterationComplete };
}
```

Guard `chrome.runtime.sendMessage` calls with `typeof chrome !== 'undefined' && chrome.runtime` check for Node.js testing compatibility.
