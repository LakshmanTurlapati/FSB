Research-based content to include:

SITE CONFIG:
- site: 'Miro'
- category: 'Design & Whiteboard'
- patterns: [/miro\.com/i]

GUIDANCE string covering:
1. KEYBOARD SHORTCUTS: N = sticky note tool, V = select/pointer, S = sticky note (alternate), Ctrl+A = select all, Delete = delete selected, Escape = deselect, Space+drag = pan canvas, Ctrl+D = duplicate
2. CANVAS ELEMENT: Miro renders to HTML5 canvas. Selector: canvas, [class*="canvas"]. All interactions via cdpClickAt/cdpDrag (content script events untrusted on canvas).
3. TOOLBAR STRUCTURE: Left-side toolbar or bottom toolbar with shape tools. Sticky note tool icon (square with folded corner). Look for: [data-testid*="sticky"], [aria-label*="Sticky"], [class*="sticky-note"], toolbar button with sticky note icon.
4. STICKY NOTE CREATION: Method 1 (preferred) -- double-click on empty canvas area via cdpClickAt to open quick-add, then type text. Method 2 -- press N to activate sticky note tool, then click on canvas to place note. Method 3 -- click sticky note tool in toolbar, then click on canvas.
5. TYPING IN STICKY NOTES: After creating a sticky note, it enters edit mode automatically. Use type_text to enter content. Press Escape to exit edit mode.
6. MOVING STICKY NOTES (for clustering): Use cdpDrag(startX, startY, endX, endY, steps=20, stepDelayMs=15) where startX/Y is the center of the sticky note and endX/Y is the target cluster location. Must be in selection mode (V) before dragging.
7. ONBOARDING/AUTH: Miro may require sign-in -- document as skip-auth if login wall blocks board creation. Onboarding tutorial overlays can be dismissed via Escape or clicking close/skip buttons. Look for: [class*="onboarding"], [class*="modal"], [class*="overlay"], [data-testid*="close"], [aria-label="Close"].
8. BOARD CREATION: Miro may offer templates on new board -- skip/dismiss and use blank board. Look for: [data-testid*="blank"], [class*="blank-board"], "Blank board" text.

SELECTORS object:
- canvas: 'canvas, [class*="canvas"], [class*="viewport"]'
- toolbar: '[class*="toolbar"], [role="toolbar"], [data-testid*="toolbar"]'
- stickyNoteTool: '[data-testid*="sticky"], [aria-label*="Sticky note"], [class*="sticky-note-tool"]'
- selectTool: '[data-testid*="select"], [aria-label*="Select"], [class*="select-tool"]'
- onboardingOverlay: '[class*="onboarding"], [class*="modal"], [class*="overlay"], [class*="tutorial"]'
- onboardingClose: '[class*="onboarding"] button[class*="close"], [aria-label="Close"], [data-testid*="close"], [class*="skip"]'
- boardTemplates: '[class*="template"], [data-testid*="template"]'
- blankBoard: '[data-testid*="blank"], [class*="blank"]'
- contextMenu: '[class*="context-menu"], [role="menu"]'
- noteEditor: '[class*="note-editor"], [contenteditable="true"], textarea'

WORKFLOWS object:
- createStickyNote: 7 steps -- dismiss overlays, activate sticky note tool (N key or toolbar click), click on canvas to place note, type_text for content, press Escape to exit edit, repeat for additional notes
- dragToCluster: 5 steps -- press V for selection tool, click on sticky note center to select it, cdpDrag from note center to target cluster position, verify note moved, repeat for each note
- fullClusteringWorkflow: 12 steps -- navigate to miro.com, handle auth/onboarding, create blank board, create sticky note 1 at position (400, 300), type "Idea A", create sticky note 2 at position (700, 500), type "Idea B", create sticky note 3 at position (300, 600), type "Idea C", drag note 2 to (450, 350), drag note 3 to (350, 400), verify all notes are in proximity (within ~200px of each other)

WARNINGS array (8 items):
1. Miro canvas requires CDP trusted events -- content script events are untrusted and ignored
2. Miro may require sign-in for board creation -- if login wall appears, document as skip-auth outcome
3. Dismiss onboarding/tutorial overlays before interacting with the board
4. After creating a sticky note, it auto-enters edit mode -- type_text works, then press Escape to exit
5. Must be in selection mode (V) before dragging notes -- N mode will create new notes on click
6. Minimum drag distance of 30px needed for Miro to register as a move, not a click
7. Use 20+ intermediate steps in cdpDrag with 15ms delay for smooth drag on Miro canvas
8. DOM snapshots of Miro can be very large due to React/framework DOM -- use targeted selectors

TOOL PREFERENCES: ['click', 'press_key', 'cdpClickAt', 'cdpDrag', 'type_text', 'waitForDOMStable', 'navigate', 'get_dom_snapshot', 'read_page']
