## Field Bundle

Every action tool in the canonical list above accepts and requires the
following field bundle on every MCP input call:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `visual_reason` | string | required | Short human-readable reason shown in the overlay (for example, "Logging in to GitHub"). |
| `client` | string | required | Allowlisted client label. Validated against the shared server/extension allowlist (see "Badge Allowlist (citation)" below). |
| `is_final` | boolean | optional | When `true`, the visual session clears immediately after the tool's `change_report` resolves. Default: `false`. |

Naming conventions:
- Field keys use `snake_case`, matching existing MCP tool argument conventions
  (`tab_id`, `session_token`, `change_report`, etc.).
- The `client` field reuses the same field name already in use by the v0.9.36
  explicit `start_visual_session` tool, minimising migration friction.
- `is_final` is the explicit task-completion signal documented under Phase 257.

Read-only MCP tools do NOT carry this bundle. See "Read-Only Tools (Schema
Lock List)" below.
