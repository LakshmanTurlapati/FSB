2. **`tokenizeLine(line)` function** -- a character-by-character state machine tokenizer:
   - States: NORMAL, DOUBLE_QUOTED, SINGLE_QUOTED
   - Handles backslash escaping in NORMAL and DOUBLE_QUOTED states (not in SINGLE_QUOTED per shell convention)
   - Splits on whitespace in NORMAL state, preserves everything inside quotes
   - Throws Error on unclosed quotes (strict parsing per user decision)
   - Extracts verb (first token), positional tokens, and flags (--name value, --bool-flag)
   - Flag value consumption: when `--name` is encountered, next non-flag token is its value; if next token is also a flag or end-of-tokens, treat as boolean flag (true)
   - Returns `{verb: string, tokens: string[], flags: Object}`
   - Use the exact tokenizer pattern from 15-RESEARCH.md Code Examples section as the starting point, but ensure flag parsing correctly handles boolean vs value flags

3. **`COMMAND_REGISTRY` constant** -- the complete lookup table mapping every CLI verb and alias to canonical tool names and argument schemas. Use the exhaustive table from 15-RESEARCH.md (section "COMMAND_REGISTRY Structure") as the definitive source. Key points:
   - All entries keyed by lowercase verb (case-insensitive lookup happens at call site)
   - Each entry: `{ tool: string, args: Array<{name, type, optional?}>, signal?: string, defaults?: Object }`
   - Arg types: 'ref' (element ref or CSS selector), 'string', 'number', 'json'
   - Include ALL tools from the research mapping table (70+ entries including aliases)
   - Fix the typo in research: `getstoredobs` should be `getstoredjobs` (both entries)
   - Signal commands: `done` with `signal: 'done'`, `fail` with `signal: 'fail'`
   - Scroll shorthand verbs (scrollDown, scrollUp, etc.) include `defaults: { direction: 'down' }` etc.
   - keyPress flag mapping: support `--ctrl`, `--shift`, `--alt`, `--meta` flags (mapped to ctrlKey, shiftKey, altKey, metaKey params in the command mapper)

4. **`classifyTarget(token)` helper function**:
   - If token matches `/^e\d+$/i` return `{ ref: token.toLowerCase() }`
   - Otherwise return `{ selector: token }`
   - Used by the command mapper for args with `type: 'ref'`

5. **`coerceValue(value, type)` helper function**:
   - 'number': parse with Number(), throw if NaN
   - 'ref': delegate to classifyTarget()
   - 'json': parse with JSON.parse()
   - 'string': return as-is
   - Returns the coerced value

<task type="auto">
  <name>Task 2: Build command mapper with param assignment and flag merging</name>
  <files>ai/cli-parser.js</files>
  <action>
Add the `mapCommand(tokenized)` function to `ai/cli-parser.js` (after the COMMAND_REGISTRY and helper functions):

1. **`mapCommand(tokenized)` function** -- takes tokenizer output, produces {tool, params} or error:
   - Look up `tokenized.verb.toLowerCase()` in COMMAND_REGISTRY
   - If not found, return `{ error: 'Unknown command: ${tokenized.verb}' }`
   - Create params object starting with `def.defaults || {}` (for scroll shorthand verbs)
   - Iterate over `def.args` schema and assign positional tokens:
     - For `type: 'ref'`: call coerceValue which calls classifyTarget. If result has `.ref`, set `params.ref = result.ref`. If result has `.selector`, set `params.selector = result.selector`. Only classify the token as a ref if the arg schema expects `type: 'ref'` AND it's a positional arg (not from flags).
     - For `type: 'string'`: set `params[argDef.name] = token`
     - For `type: 'number'`: call coerceValue to parse, throw on NaN
     - For `type: 'json'`: join ALL remaining positional tokens with spaces (JSON payloads may have been split by tokenizer if not quoted), then JSON.parse the joined string. If parsing fails, try joining the remaining token as-is.
     - For optional args: if no token available, skip (don't error)
   - Merge flags into params. Special handling for keyPress modifier flags:
     - `--ctrl` -> `params.ctrlKey = true`
     - `--shift` -> `params.shiftKey = true`
     - `--alt` -> `params.altKey = true`
     - `--meta` -> `params.metaKey = true`
     - All other flags: `params[flagName] = flagValue` (string or boolean)
   - For selectOption: handle `--by-value` flag (sets param name to 'value' instead of 'text') and `--by-index` flag (sets param name to 'index' and coerces to number)
   - For toggleCheckbox: handle `--checked` flag as `params.checked = true`, `--unchecked` as `params.checked = false`
   - If signal command (def.signal exists): return `{ signal: def.signal, message: params.message || null }`
   - Otherwise return `{ tool: def.tool, params }`

2. **pressKeySequence special handling**: When the tool is 'pressKeySequence', the text param contains comma-separated keys like "Tab,Enter,Escape". The mapper should split on comma and set `params.keys = text.split(',').map(k => k.trim())`.

3. **Edge case handling**:
   - Extra positional args beyond what the schema expects: ignore silently (don't error)
   - Missing required (non-optional) args: return `{ error: 'Missing required argument: ${argDef.name} for command: ${verb}' }`
   - Empty verb after lookup: should not happen (tokenizer always produces a verb from non-empty lines)

Coding conventions: same as Task 1.
