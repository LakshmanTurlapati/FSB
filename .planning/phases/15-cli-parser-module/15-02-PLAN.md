  <action>
Add two functions to `ai/cli-parser.js` after the existing mapCommand function:

1. **`preprocessResponse(text)` function** -- strips provider wrapping before line parsing:
   - Guard: if text is falsy or not a string, return empty string
   - Trim the text
   - Strip markdown code fences: remove lines that are exactly `` ``` ``, `` ```bash ``, `` ```shell ``, `` ```text ``, `` ```cli `` (use regex: `/^```(?:bash|shell|text|cli)?\s*$/gm` replaced with empty string)
   - Strip conversational preamble (Grok pattern): find the first line that is either empty, starts with `#`, or whose first whitespace-delimited word (lowercased) exists in COMMAND_REGISTRY. Discard all lines before that first recognized line.
   - Strip trailing conversational text: find the LAST line whose first word (lowercased) exists in COMMAND_REGISTRY or starts with `#`. Discard all lines after that.
   - Return the cleaned text, trimmed
   - Use the preprocessResponse pattern from 15-RESEARCH.md as starting point, but enhance with trailing text stripping

2. **`parseCliResponse(text)` function** -- the public entry point:
   - Call preprocessResponse(text) first
   - Split cleaned text on newlines
   - Initialize result object:
     ```
     {
       actions: [],
       reasoning: [],
       errors: [],
       taskComplete: false,
       taskFailed: false,
       result: null,
       confidence: 'medium',
       situationAnalysis: '',
       goalAssessment: '',
       assumptions: [],
       fallbackPlan: ''
     }
     ```
     (The last four fields are compatibility stubs for normalizeResponse shape in ai-integration.js)
   - Iterate over each line:
     - Trim the line
     - If empty or whitespace-only: skip silently (no error, no log)
     - If starts with `#`: push `trimmed.substring(1).trim()` to `result.reasoning`; continue
     - Otherwise: wrap in try/catch:
       - Call `tokenizeLine(trimmed)` -> tokenized
       - Call `mapCommand(tokenized)` -> mapped
       - If `mapped.error`: push `{line: trimmed, lineNumber: i+1, error: mapped.error}` to `result.errors`; continue
       - If `mapped.signal === 'done'`: set `result.taskComplete = true`, `result.result = mapped.message || 'Task completed'`; continue
       - If `mapped.signal === 'fail'`: set `result.taskComplete = true`, `result.taskFailed = true`, `result.result = mapped.message || 'Task failed'`; continue
       - Otherwise: push `{tool: mapped.tool, params: mapped.params}` to `result.actions`
     - catch(err): push `{line: trimmed, lineNumber: i+1, error: err.message}` to `result.errors`
   - After loop: set `result.reasoning` joined as `result.situationAnalysis = result.reasoning.join(' ')` (for compatibility)
   - Return result

Key behaviors (per requirements):
- CLI-03: Multiple command lines produce multiple entries in actions[]
- CLI-04: Lines starting with # go to reasoning[], not actions[]
- CLI-06: Per-line try/catch ensures parse failures are isolated
- CLI-02: Quoted strings handled correctly by tokenizeLine (from Plan 01)

Coding conventions: 2-space indent, single quotes, semicolons, camelCase, JSDoc on both functions.
