    **mcp-server/src/resources/index.ts:**

    Export function `registerResources(server: McpServer, bridge: NativeMessagingBridge): void`

    IMPORTANT: Check the actual McpServer API in the SDK type definitions before coding. The method may be `server.resource()` or `server.registerResource()`. Read the declaration file and use the correct method name.

    Register 5 resources:

    **1. Current Page DOM**
    - Name: `current-page-dom`
    - URI: `browser://dom/snapshot`
    - Metadata: `{ title: "Current Page DOM", description: "Structured DOM snapshot of the active tab with elements, selectors, forms, and page structure. Read this to understand what is on the current page before using action tools.", mimeType: "application/json" }`
    - Handler: Sends `mcp:get-dom` with `{ maxElements: 2000, prioritizeViewport: true }` to bridge. Returns `{ contents: [{ uri: uri.href, text: JSON.stringify(result.payload, null, 2), mimeType: "application/json" }] }`. If bridge not connected, returns error text explaining extension must be running.

    **2. Open Tabs**
    - Name: `open-tabs`
    - URI: `browser://tabs`
    - Metadata: `{ title: "Open Tabs", description: "All open browser tabs with title, URL, and active status. Read this to see available tabs before switching.", mimeType: "application/json" }`
    - Handler: Sends `mcp:get-tabs` to bridge. Returns tab array as JSON.

    **3. Site Guides**
    - Name: `site-guides`
    - URI: `fsb://site-guides`
    - Metadata: `{ title: "Site Guides", description: "All FSB site guide definitions with selectors and workflows for supported websites. Read this to use optimized selectors for known sites.", mimeType: "application/json" }`
    - Handler: Sends `mcp:get-site-guides` to bridge. Returns guide list as JSON.

    **4. FSB Memory**
    - Name: `fsb-memory`
    - URI: `fsb://memory`
    - Metadata: `{ title: "FSB Memory", description: "Episodic, semantic, and procedural memory from previous automation sessions. Read this to benefit from past experience on similar tasks.", mimeType: "application/json" }`
    - Handler: Sends `mcp:get-memory` to bridge. Returns memory data as JSON.

    **5. Extension Config**
    - Name: `extension-config`
    - URI: `fsb://config`
    - Metadata: `{ title: "Extension Config", description: "Current FSB extension configuration including provider, model, and connection status. API keys are redacted. Read this to check extension health.", mimeType: "application/json" }`
    - Handler: Sends `mcp:get-config` to bridge. Returns config data as JSON. The extension-side handler (background.js from Plan 01) already excludes API keys.

    Each handler follows this pattern:
    ```typescript
    async (uri) => {
      if (!bridge.isConnected()) {
        return {
          contents: [{
            uri: uri.href,
            text: "Extension not connected. The FSB Chrome extension must be running and connected via Native Messaging.",
            mimeType: "text/plain",
          }],
        };
      }
      const result = await bridge.sendAndWait(
        { type: 'mcp:get-xxx', payload: {} },
        { timeout: 10_000 }
      );
      return {
        contents: [{
          uri: uri.href,
          text: JSON.stringify(result.payload || result, null, 2),
          mimeType: "application/json",
        }],
      };
    }
    ```

    **mcp-server/scripts/install-host.js:**
    This is a standalone Node.js script (NOT TypeScript -- runs without build step).

    Shebang: `#!/usr/bin/env node`

    Functionality:
    1. Parse command-line args: `--extension-id=<id>` (required), `--uninstall` (optional flag)
    2. Determine OS via `process.platform` (darwin, linux, win32)
    3. Resolve absolute path to the native host shim: `path.resolve(__dirname, '..', 'build', 'native-host-shim.js')`
    4. Determine manifest output path per OS:
       - macOS: `path.join(os.homedir(), 'Library', 'Application Support', 'Google', 'Chrome', 'NativeMessagingHosts', 'com.fsb.mcp.json')`
       - Linux: `path.join(os.homedir(), '.config', 'google-chrome', 'NativeMessagingHosts', 'com.fsb.mcp.json')`
       - Windows: `path.join(os.homedir(), 'AppData', 'Local', 'Google', 'Chrome', 'User Data', 'NativeMessagingHosts', 'com.fsb.mcp.json')` (alternative: registry-based, but file-based works for user-level install)
    5. Create a launcher script next to the build output:
       - macOS/Linux: `mcp-server/build/fsb-native-host.sh` with content `#!/bin/bash\nexec node "SHIM_PATH"\n`, then `chmod +x`
       - Windows: `mcp-server/build/fsb-native-host.bat` with content `@echo off\nnode "SHIM_PATH"\n`
    6. Generate manifest JSON:
       ```javascript
       const manifest = {
         name: 'com.fsb.mcp',
         description: 'FSB Browser Automation MCP Bridge',
         path: launcherPath, // Absolute path to the .sh or .bat launcher
         type: 'stdio',
         allowed_origins: [`chrome-extension://${extensionId}/`]
       };
       ```
    7. Create parent directories if they don't exist: `fs.mkdirSync(dir, { recursive: true })`
    8. Write manifest to the OS-specific path
    9. On Windows, also create registry key:
       ```javascript
       const { execSync } = require('child_process');
       execSync(`reg add "HKCU\\Software\\Google\\Chrome\\NativeMessagingHosts\\com.fsb.mcp" /ve /t REG_SZ /d "${manifestPath}" /f`);
       ```
    10. If `--uninstall` flag: delete manifest file, delete launcher script, remove registry key on Windows
    11. Print success message with instructions:
        ```
        Native messaging host installed successfully!

        Manifest: <path>
        Launcher: <path>
        Extension ID: <id>

        Next steps:
        1. Reload the FSB extension in Chrome (chrome://extensions)
        2. Add to Claude Code: claude mcp add --transport stdio fsb-browser -- node <mcp-server-build-path>/index.js
        3. Or copy .mcp.json to your project root
        ```

    Use only Node.js builtins: `fs`, `path`, `os`, `child_process`. No external deps.
