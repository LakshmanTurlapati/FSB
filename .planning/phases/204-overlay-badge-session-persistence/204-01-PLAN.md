<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Canonical overlay state -> live DOM | Only trusted canonical `clientLabel` values should become visible badge text on the live page. |
| DOM-stream payload -> remote preview | Mirrored preview surfaces must show the same owner identity and lifecycle state, not a lossy or spoofable approximation. |
| Frozen preview state -> operator interpretation | Frozen/disconnected states must preserve the last trusted owner so operators know which MCP client produced the state they are looking at. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-204-01-01 | Spoofing | Live badge text | mitigate | Render only canonical `clientLabel` values already normalized by Phase 203; never accept arbitrary badge markup or freeform branding. |
| T-204-01-02 | Tampering | Mirrored overlay payload | mitigate | Forward identity via structured fields (`clientLabel`, `sessionToken`, `version`, `lifecycle`) instead of parsing display text. |
| T-204-01-03 | Information confusion | Frozen preview | mitigate | Preserve trusted badge identity through frozen states until the overlay actually clears. |
| T-204-01-04 | Regression | Existing preview overlays | mitigate | Keep glow/progress behavior intact and add focused preview source/runtime assertions. |
</threat_model>
