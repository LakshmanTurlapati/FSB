Execute the SCROLL-02 edge case test using MCP manual tools following the scrapeAllSearchResults workflow from the amazon.js site guide. Test steps:

1. Use MCP navigate tool to go to amazon.com. If redirected to a country-specific domain (amazon.com -> amazon.in, amazon.co.uk, etc.), note the actual domain.

2. Use read_page to verify the page loaded and the search box is accessible. Check for:
   - Cookie/consent banners: dismiss via click if present
   - Location popups: dismiss if present
   - CAPTCHA: if Amazon shows a CAPTCHA, document as a blocker
   Record the page title, URL, and whether the homepage loaded.

3. Execute a search for a broad product category. Use the following search queries in priority order (try next if current returns fewer than 500 total results):
   a. "wireless mouse" (typically 10,000+ results)
   b. "phone case" (typically 50,000+ results)
   c. "usb cable" (typically 10,000+ results)
   d. "water bottle" (typically 10,000+ results)
   Method: click the search box (selectors: #twotabsearchtextbox or [aria-label="Search Amazon"]), type the query, press Enter or click #nav-search-submit-button.

4. Use read_page to verify search results loaded. Check for:
   - Result count text (e.g., "1-48 of over 10,000 results")
   - Presence of [data-component-type="s-search-result"] elements
   - Note how many results are on the first page
   Record the result count text and number of visible product cards.

5. Use get_dom_snapshot to map product result elements on first page. Identify:
   - Result containers: [data-component-type="s-search-result"] elements
   - Each result's data-asin attribute value
   - Product names: h2 a span.a-text-normal (or h2 a .a-size-medium) within each result
   - Sponsored indicators: [aria-label="Leave feedback on Sponsored ad"]
   - Pagination: .s-pagination-next element
   Count how many products with valid (non-empty) data-asin exist on page 1.

6. Initialize tracking state:
   - seenASINs: Set of data-asin values already captured
   - productList: Array of {asin, productName, pageNumber} for each unique product
   - pageCount: 1
   - For each product on page 1, extract ASIN and name, add to productList and seenASINs

7. PAGINATION LOOP (repeat until productList.length >= 500 or no next page or max 35 page iterations):
   a. Click .s-pagination-next to navigate to next page
   b. Wait 1000ms-2000ms for page load (use waitForElement on [data-component-type="s-search-result"] or waitForDOMStable)
   c. Increment pageCount
   d. Execute get_dom_snapshot to capture product elements on new page
   e. For each [data-component-type="s-search-result"] with non-empty data-asin:
      - If ASIN NOT in seenASINs: extract name, add to seenASINs and productList
      - If ASIN IS in seenASINs: skip (already counted)
   f. Log: "Page {pageCount}: {newProductsThisPage} new products, {productList.length} total"
   g. If .s-pagination-next is disabled or absent: pagination ended, break loop
   h. If no new products found for 3 consecutive pages: results may be repeating, break loop

8. AFTER PAGINATION LOOP:
   - If productList.length >= 500: PASS -- report count and sample names
   - If productList.length > 0 but < 500 and pagination ended: PARTIAL -- report actual count
   - If productList.length == 0: FAIL -- couldn't extract any products
   - If CAPTCHA blocked access: document as blocker

9. REPORT PRODUCT SAMPLE:
   - First 5 product names (with ASINs) from productList
   - Last 5 product names (with ASINs) from productList
   - Total unique products collected
   - Total pages navigated
   - Average products per page

10. VERIFY: Confirm extraction quality:
    - All names are non-empty strings
    - All ASINs are non-empty and unique
    - No ASIN appears twice in productList
    - Names look like real product names (not HTML fragments or selector text)

Record every step outcome (tool used, target, result, notes) for the diagnostic report.
