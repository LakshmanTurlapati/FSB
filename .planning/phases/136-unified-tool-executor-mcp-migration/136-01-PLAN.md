Create ai/tool-executor.js as a CommonJS module (matching tool-definitions.js pattern) with the following structure:

1. Import TOOL_REGISTRY and getToolByName from ./tool-definitions.js using require().

2. Define a STRUCTURED RESULT format that all tool executions return:
```javascript
// Every tool result MUST have this shape
{
  success: Boolean,        // whether the tool achieved its intended effect
  hadEffect: Boolean,      // whether the page/state changed (for stuck detection in Phase 137)
  error: String|null,      // error message if success=false, null otherwise
  navigationTriggered: Boolean,  // whether a page navigation occurred
  result: Object|null      // tool-specific payload (text content, tab info, etc.)
}
```

3. Create `async function executeTool(name, params, tabId, options = {})`:
   - Lookup tool in TOOL_REGISTRY via getToolByName(name)
   - If tool not found, return structured error result immediately
   - Route based on tool._route:

   **Route 'content' (28 tools):**
   - Use tool._contentVerb to get the FSB.tools key
   - Special case: if _contentVerb is null (get_dom_snapshot), send message type 'mcp:get-dom' instead of 'executeAction'
   - For all others: `chrome.tabs.sendMessage(tabId, { action: 'executeAction', tool: tool._contentVerb, params })`
   - Wrap in try/catch, detect BF cache errors (message includes "Receiving end does not exist")
   - On BF cache: check if URL changed (navigation success), return navigationTriggered=true

   **Route 'cdp' (7 tools):**
   - Use tool._cdpVerb to get the CDP dispatch case
   - Reuse the executeCDPToolDirect pattern from background.js: attach debugger, dispatch Input.dispatchMouseEvent commands, detach
   - CDP tools: cdpClickAt, cdpClickAndHold, cdpDrag, cdpDragVariableSpeed, cdpScrollAt, cdpInsertText, cdpDoubleClickAt
   - DO NOT duplicate the full CDP logic -- instead, call the existing executeCDPToolDirect function via an options.cdpHandler callback. The executor is a routing layer, not a reimplementation.
   - Signature: `options.cdpHandler(toolVerb, params, tabId)` -- background.js will pass its executeCDPToolDirect when calling executeTool.

   **Route 'background' (7 tools):**
   - Sub-route by tool name:
     - navigate: chrome.tabs.update(tabId, { url: params.url }), wait 500ms, return with navigatingTo/fromUrl
     - go_back: chrome.tabs.goBack(tabId)
     - go_forward: chrome.tabs.goForward(tabId)
     - refresh: chrome.tabs.reload(tabId)
     - open_tab: chrome.tabs.create({ url: params.url, active: params.active !== false })
     - switch_tab: chrome.tabs.update(params.tabId, { active: true })
     - list_tabs: chrome.tabs.query({}) and map to {id, title, url, active, windowId}
   - Also handle data tools (fill_sheet, read_sheet) via options.dataHandler callback
   - Return structured result

4. Create `function isReadOnly(name)`:
   - Lookup tool, return tool._readOnly === true
   - Used by callers to decide whether to bypass mutation queue

5. Export via CommonJS: `module.exports = { executeTool, isReadOnly }`

Key design decisions:
- The executor is a ROUTING LAYER -- it dispatches to existing handlers (content scripts, CDP, chrome.tabs APIs), not a reimplementation
- CDP and data tool handlers are injected via options callbacks (options.cdpHandler, options.dataHandler) to avoid duplicating complex logic that lives in background.js
- Content script tools use chrome.tabs.sendMessage directly (the convergence point both paths already share)
- Background nav tools (navigate, go_back, go_forward, refresh, open_tab, switch_tab, list_tabs) use chrome.tabs APIs directly in the executor since they are simple one-liners
- All results normalized to the structured format with success/hadEffect/error/navigationTriggered/result

// 1. Verify all 42 tools are routable
let errors = [];
for (const tool of TOOL_REGISTRY) {
  if (!['content', 'cdp', 'background'].includes(tool._route)) {
    errors.push(`${tool.name}: invalid _route '${tool._route}'`);
  }
  if (tool._route === 'content' && !tool._contentVerb && tool.name !== 'get_dom_snapshot') {
    errors.push(`${tool.name}: content route but no _contentVerb`);
  }
  if (tool._route === 'cdp' && !tool._cdpVerb) {
    errors.push(`${tool.name}: cdp route but no _cdpVerb`);
  }
}

// 2. Verify read-only detection matches registry
const readOnlyNames = TOOL_REGISTRY.filter(t => t._readOnly).map(t => t.name);
for (const name of readOnlyNames) {
  if (!isReadOnly(name)) {
    errors.push(`${name}: _readOnly=true in registry but isReadOnly() returns false`);
  }
}

// 3. Verify non-read-only tools are correctly identified
const mutatingNames = TOOL_REGISTRY.filter(t => !t._readOnly).map(t => t.name);
for (const name of mutatingNames) {
  if (isReadOnly(name)) {
    errors.push(`${name}: _readOnly=false in registry but isReadOnly() returns true`);
  }
}

// 4. Verify unknown tool returns error
// (executeTool needs chrome APIs so we just verify the function signature)

  errors.forEach(e => console.error(' -', e));
  process.exit(1);
} else {
  console.log(`PASS: All ${TOOL_REGISTRY.length} tools routable`);
  console.log(`PASS: ${readOnlyNames.length} read-only tools detected correctly`);
