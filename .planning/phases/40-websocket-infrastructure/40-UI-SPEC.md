### Dashboard Header State Text

When the extension is online (WS connected and extension presence detected):
- Show "connected" badge next to agent count (green)

When the extension is offline (WS connected to server but no extension in room):
- Show "extension offline" in the status area using the disconnected style

When reconnecting:
- Show "reconnecting..." badge (amber) -- auto-updates on success

## Interaction States

### Dashboard Connection Lifecycle

```
Page load (has stored hashKey)
  -> Show login section briefly
  -> Auto-validate hashKey via REST
  -> On success: show dashboard, open WebSocket
  -> Badge: "reconnecting..." (amber)
  -> WS opens: badge -> "connected" (green)
  -> WS closes unexpectedly: badge -> "disconnected" (gray) -> immediate retry -> "reconnecting..." (amber) -> backoff
  -> WS reconnects: badge -> "connected" (green), extension sends state snapshot, dashboard auto-refreshes data
```

### Extension Badge Lifecycle

```
serverSyncEnabled = false -> no badge (clear)
serverSyncEnabled = true, connecting -> badge "!" red
WS connected -> badge " " green
WS disconnected -> badge "!" red -> auto-reconnect
Chrome closed -> WS dies naturally -> reconnect on next launch
```

### No Loading Spinners Needed

The existing `.dash-spinner` class is available but not needed for connection status. The "reconnecting..." text with amber color is sufficient feedback. Data fetching (agents, stats) continues to use existing loading patterns via REST API.
