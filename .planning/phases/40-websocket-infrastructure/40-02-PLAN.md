<!-- Server WS endpoint contract (created in Plan 01) -->

WebSocket endpoint: wss://host/ws?key={hashKey}&role={extension|dashboard}
- Server validates hashKey against DB on upgrade
- Server sends { type: 'pong', ts: N } in response to { type: 'ping', ts: N }
- Server relays all other messages to opposite side of room (extension <-> dashboards)
- Server broadcasts { type: 'ext:status', payload: { online: true|false }, ts: N } to dashboards on extension connect/disconnect

Message envelope format:
```javascript
{ type: string, payload: object, ts: number }
```

Types used by extension:
- 'ping' -- keepalive (server responds with 'pong', not relayed)
- 'ext:snapshot' -- state snapshot sent on connect/reconnect (relayed to dashboards)
- 'ext:status' -- generated by SERVER, not sent by extension

Create `ws/ws-client.js` at the extension root (same level as `agents/`). This is a self-contained WebSocket client for the Chrome MV3 service worker.

**Class: `FSBWebSocket`**

Properties:
- `this.ws = null` -- current WebSocket instance
- `this.keepaliveTimer = null` -- setInterval ID for 20s pings
- `this.reconnectTimer = null` -- setTimeout ID for reconnect
- `this.reconnectDelay = 0` -- current backoff delay (ms)
- `this.maxReconnectDelay = 30000` -- cap at 30s
- `this.connected = false` -- connection state flag
- `this.intentionalClose = false` -- true when disconnect() called explicitly

**Methods:**

`async connect()`:
1. Read config: `const { serverUrl, serverHashKey, serverSyncEnabled } = await chrome.storage.local.get(['serverUrl', 'serverHashKey', 'serverSyncEnabled']);`
2. If `!serverSyncEnabled || !serverHashKey`, call `_clearBadge()` and return
3. Build WS URL: `serverUrl.replace(/^http/, 'ws') + '/ws?key=' + encodeURIComponent(serverHashKey) + '&role=extension'`
4. `this.intentionalClose = false`
5. Create `this.ws = new WebSocket(wsUrl)`
6. Set `ws.onopen`: reset `reconnectDelay` to 0, set `connected = true`, call `_startKeepalive()`, `_sendStateSnapshot()`, `_updateBadge(true)`, log `[FSB WS] Connected`
7. Set `ws.onmessage`: parse JSON, call `_handleMessage(msg)`
8. Set `ws.onclose`: set `connected = false`, call `_stopKeepalive()`, `_updateBadge(false)`, if `!intentionalClose` call `_scheduleReconnect()`, log `[FSB WS] Disconnected`
9. Set `ws.onerror`: no-op (onclose fires after)

`disconnect()`:
1. Set `this.intentionalClose = true`
2. Call `_stopKeepalive()`
3. Clear `reconnectTimer` if exists
4. If `this.ws`, call `this.ws.close()`
5. Set `this.ws = null`, `this.connected = false`
6. Call `_clearBadge()`

`send(type, payload)`:
1. If `!this.ws || this.ws.readyState !== WebSocket.OPEN` return false
2. `this.ws.send(JSON.stringify({ type, payload, ts: Date.now() }))`
3. Return true

`_startKeepalive()`:
1. Call `_stopKeepalive()` first
2. `this.keepaliveTimer = setInterval(() => { ... }, 20000)` -- 20 seconds
3. Inside interval: if ws open, send `{ type: 'ping', ts: Date.now() }` via raw `this.ws.send()`

`_stopKeepalive()`:
1. If `this.keepaliveTimer`, `clearInterval(this.keepaliveTimer); this.keepaliveTimer = null;`

`_scheduleReconnect()`:
1. If `this.reconnectDelay === 0`: set delay to 1000, call `this.connect()` immediately (first retry is instant)
2. Else: `this.reconnectTimer = setTimeout(() => this.connect(), this.reconnectDelay)`
3. Increase delay: `this.reconnectDelay = Math.min(this.reconnectDelay * 2, this.maxReconnectDelay)`
4. Log: `[FSB WS] Reconnecting in ${this.reconnectDelay}ms`

`_sendStateSnapshot()`:
1. Gather current state: read from chrome.storage.local any relevant extension state (agents list, running tasks, etc.)
2. For now, send a minimal snapshot: `this.send('ext:snapshot', { version: chrome.runtime.getManifest().version, timestamp: Date.now() })`
3. This will be expanded in later phases to include running task state

`_handleMessage(msg)`:
1. Switch on `msg.type`:
   - `'pong'`: no-op (server responded to our ping, connection is healthy)
   - Default: log `[FSB WS] Received: ${msg.type}` (future phases will handle task commands, etc.)

`_updateBadge(connected)`:
1. If connected: `chrome.action.setBadgeText({ text: ' ' }); chrome.action.setBadgeBackgroundColor({ color: '#22c55e' });`
2. If disconnected: `chrome.action.setBadgeText({ text: '!' }); chrome.action.setBadgeBackgroundColor({ color: '#ef4444' });`

`_clearBadge()`:
1. `chrome.action.setBadgeText({ text: '' });`

**Global instance:**
At bottom of file: `const fsbWebSocket = new FSBWebSocket();`

This follows the same pattern as `agents/server-sync.js` which creates `const serverSync = new ServerSync();` at module level.

a. Connects on extension startup:
```javascript
// Initialize WebSocket connection if server sync is enabled
chrome.storage.local.get(['serverSyncEnabled'], (result) => {

    fsbWebSocket.connect();
  }
});
```

b. Listen for storage changes to connect/disconnect when user toggles serverSyncEnabled:
```javascript
chrome.storage.onChanged.addListener((changes, areaName) => {
  if (areaName === 'local' && changes.serverSyncEnabled) {
    if (changes.serverSyncEnabled.newValue) {
      fsbWebSocket.connect();
    } else {
      fsbWebSocket.disconnect();
    }
  }
});
```

Place the storage change listener near other `chrome.storage.onChanged` listeners if any exist, or near the importScripts block at the top of the file.
