<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Structured tool errors -> user-facing text | Layer classification errors can send operators toward the wrong recovery action. |
| Restricted recovery tool list -> operator action | If the wrong tools are suggested, users can be pushed into dead-end flows or unsafe automation assumptions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-200-02-01 | Tampering | Error-to-layer mapping | mitigate | Use explicit `errorCode` mappings and fixture tests for every supported layer. |
| T-200-02-02 | Information Disclosure | Raw error appendix | mitigate | Keep only the raw error string; do not append session bodies, DOM snapshots, logs, memory text, credentials, or payment data. |
| T-200-02-03 | Elevation of Privilege | Restricted recovery guidance | mitigate | Keep `run_task` excluded from restricted-page recovery guidance and assert that exclusion in tests. |
</threat_model>
