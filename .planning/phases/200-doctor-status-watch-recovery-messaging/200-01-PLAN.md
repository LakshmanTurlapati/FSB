<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP server diagnostics -> extension bridge | Diagnostic probes cross the local bridge into extension-owned state. |
| Background diagnostics helper -> service-worker state maps | The helper reads content-script and active-tab state that changes during navigation. |
| CLI output -> operator decisions | Diagnosis text can trigger restarts, refreshes, or config changes, so wrong layer selection is user-impacting. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-200-01-01 | Information Disclosure | Internal diagnostics probe | mitigate | Return bounded tab and content-script metadata only; do not expose logs, DOM, memory bodies, credentials, or payments. |
| T-200-01-02 | Tampering | Layer classifier | mitigate | Use deterministic ordered rules and regression fixtures for every supported layer. |
| T-200-01-03 | Denial of Service | `status --watch` | mitigate | Reuse the existing snapshot collector with a bounded poll interval and keep the watch loop read-only. |
| T-200-01-04 | Spoofing | Tool-routing diagnosis | mitigate | Only classify `tool_routing` from explicit `mcp_route_unavailable` signals, not from broad string matching. |
</threat_model>
