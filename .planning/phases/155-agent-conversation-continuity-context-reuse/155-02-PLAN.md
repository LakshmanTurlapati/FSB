<objective>
Move follow-up continuity into the native agent loop so resumed runs carry the right automation context without replaying stale full history every turn.

Purpose: `handleStartAutomation()` still leans on the legacy `ai.injectFollowUpContext()` hook, while `runAgentLoop()` resets `session.messages` to a fresh system prompt and sends the whole durable thread on every iteration. That defeats true follow-up continuity and bloats prompt cost.
Output: A dedicated agent resume-state contract in `background.js`, agent-loop hydration logic that preserves prior context across follow-ups, bounded per-turn prompt windows, and tool-result messages that keep tool names intact during compaction.
</objective>
