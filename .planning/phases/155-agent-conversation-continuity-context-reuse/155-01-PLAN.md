<objective>
Establish a durable conversation-thread contract across the popup, sidepanel, background session store, and session history so follow-up commands reuse the correct automation thread.

Purpose: The current continuity path only keeps a surface-local `conversationId` and a loose `conversationSessions` pointer. That is not enough to continue the thread a user selected from sidepanel history or to recover the correct thread cleanly after a service-worker restart.
Output: A richer background conversation/session mapping, persisted history metadata keyed by conversation, and UI request payloads that identify the current surface and selected history thread explicitly.
</objective>
