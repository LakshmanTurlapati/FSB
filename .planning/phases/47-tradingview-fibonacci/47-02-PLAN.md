**Step 1: Navigate to TradingView chart**
Use the MCP `navigate` tool to open `https://www.tradingview.com/chart/`. Wait for the chart to load.

**Step 2: Inspect DOM for modals and toolbar**
Use `get_dom_snapshot` to:
- Check for sign-up/cookie modals (look for `.tv-dialog`, overlay elements, cookie consent). Dismiss by clicking close buttons.
- Locate the left-side drawing toolbar. Look for elements containing drawing tool categories. The Fibonacci tool is likely under a "Gann and Fibonacci tools" group button.
- Record all discovered CSS selectors for: drawing toolbar container, Fib tool group button, Fib Retracement option, chart canvas element.

**Step 3: Select Fibonacci Retracement tool**
- Click the drawing tool group button on the left toolbar (standard DOM click works here since toolbar is HTML, not canvas)
- If Fibonacci is in a submenu/dropdown, click the group first, then select "Fib Retracement" from the expanded options
- Verify the tool is selected (toolbar button may change state/class)

**Step 4: Draw Fibonacci on chart canvas**
- Use `get_dom_snapshot` to find the chart canvas container element and get its bounding rect via `getAttribute` or by reading position data from the DOM
- Calculate two viewport-relative points: one in the lower-left area of the chart (representing a local low) and one in the upper-right area (representing a local high). Use approximate positions like chart-center-minus-100px and chart-center-plus-100px.
- Research note: TradingView may use click-click (two separate clicks) OR click-drag. Try TWO SEPARATE `click_at` calls first (simpler). If that does not produce a drawing, fall back to `drag` tool.
- For click-click approach: `click_at(lowerX, lowerY)` then `click_at(upperX, upperY)`
- For drag approach: `drag(startX=lowerX, startY=lowerY, endX=upperX, endY=upperY, steps=15, stepDelayMs=25)`
- Ensure drag distance exceeds 50px total (TradingView's 5px manhattan threshold)

**Step 5: Verify result**
- Use `get_dom_snapshot` after drawing attempt
- Check for new elements that indicate a Fibonacci drawing was created (look for SVG elements, new canvas layers, or drawing-related DOM nodes)
- Record the outcome: PASS (lines visible), PARTIAL (tool selected but drawing failed), FAIL (couldn't interact with canvas), or SKIP-AUTH (blocked by auth)

Update `site-guides/finance/tradingview.js` with all discoveries. Add to the `selectors` object:
