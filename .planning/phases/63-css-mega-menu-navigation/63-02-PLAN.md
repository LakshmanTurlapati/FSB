Execute the MICRO-07 edge case test using MCP manual tools following the navigateMegaMenu workflow from the mega-menu site guide. Test steps:

1. Use MCP navigate tool to go to https://www.bestbuy.com/ (free, no auth, robust mega-menus under Products/Brands/Deals nav items). If Best Buy is unavailable, try https://www.homedepot.com/ or https://www.lowes.com/ as alternatives.

2. Use read_page to verify the page loaded. Dismiss any cookie/consent banners via click if present. Note the page title and URL to confirm successful navigation.

3. Use get_dom_snapshot to map the page elements. Identify:
   - The top-level navigation bar (header nav, [role="menubar"], site-specific nav selectors)
   - Individual nav items with text labels (e.g., "Products", "Deals", "Brands")
   - Any aria-haspopup="true" or aria-expanded attributes indicating dropdown triggers
   - Record the CSS selector and bounding rect of the target nav item (e.g., "Products")

4. STRATEGY A -- DOM HOVER + CLICK:
   a. Use hover tool on the top-level nav item selector (e.g., hover('[data-lid="hdr_menu_Products"]') for Best Buy).
   b. Wait for mega-menu to animate open (natural MCP latency covers this).
   c. Use get_dom_snapshot to find the now-visible mega-menu panel and its sub-links.
   d. Use click tool on a specific nested sub-link inside the mega-menu (e.g., a link to "Computer Accessories" or "Appliances" or any sub-category).

5. VERIFY NAVIGATION: Use read_page to check if the URL changed to the target sub-category page. The page title should contain the sub-category name.

6. If Strategy A FAILED (hover did not open the menu):
   a. Use get_dom_snapshot to find nav item coordinates (center X, center Y).
   b. Use drag(navX, navY, navX, navY+1, steps=2, stepDelayMs=50) to CDP-hover the nav item.
   c. Use get_dom_snapshot to find the mega-menu panel and sub-link coordinates.
   d. Use L-shaped CDP drag path: first drag(navX, navY, navX, panelTopY+10, steps=5, stepDelayMs=30) to move into the panel, then click_at(linkX, linkY) on the sub-link.

7. If both strategies were attempted, document which worked and which failed.

Record every step outcome (tool used, target, result, notes) for the diagnostic report.
