Step 1 -- Read all 4 findings files produced by Plans 01 and 02.

Step 2 -- Identify CROSS-SUBSYSTEM regressions. These are cases where:
  - A change in agent-loop.js broke its contract with tool-executor.js (e.g., different arguments, different result format expected)
  - A change in ai-integration.js broke its contract with tool-use-adapter.js (e.g., different request payload format)
  - A change in background.js changed how it calls content/dom-analysis.js (e.g., different message format, missing parameters)
  - A change in tool-executor.js changed how it sends executeAction to content/actions.js (e.g., different action format)
  - A change in response parsing broke how agent-loop.js interprets AI responses (e.g., cleanResponse mangling tool_use blocks)

For each cross-subsystem regression, document which two subsystems are involved and which side needs to change.

Step 3 -- Assign each finding to its repair phase:
  - AL-* findings -> Phase 181 (Agent Loop Repair)
  - TE-* findings -> Phase 182 (Tool Execution Repair)
  - AC-* findings -> Phase 183 (AI Communication Repair)
  - DA-* findings -> Phase 184 (DOM Analysis Repair)
  - Cross-subsystem findings -> Primary repair phase with note about coordination
