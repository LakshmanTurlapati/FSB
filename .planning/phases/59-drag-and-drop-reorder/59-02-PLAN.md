Execute the MICRO-03 edge case test using MCP manual tools following the dragAndDropReorder workflow from the Trello site guide. Test steps:

1. Use MCP navigate tool to go to a Kanban board. Try in order:
   a. trello.com (if user is logged in and has boards) -- check if a board is accessible
   b. If Trello requires auth and blocks access, try a public Trello board URL
   c. Fallback: any free Kanban demo site with drag-and-drop cards (e.g., a public demo board)
   d. If all require auth, use skip-auth outcome

2. Use read_page to check for cookie/consent/onboarding popups and dismiss them via click

3. Use get_dom_snapshot to identify the board structure:
   - Find list containers: [data-testid="list"] or .list-wrapper
   - Find cards within lists: [data-testid="trello-card"] or .list-card
   - Identify the bottom card in list A (source) and the top position in list B (target)

4. Record source card text and target list name for verification later

5. Attempt METHOD 1 -- drag_drop(sourceSelector, targetSelector, steps=15, holdMs=200, stepDelayMs=25):
   - sourceSelector = element reference of bottom card in source list
   - targetSelector = element reference of first card in destination list (or list header if empty)
   - holdMs=200 gives react-beautiful-dnd time to recognize drag intent
   - Note whether drag_drop returns success and which method it used (html5_drag, pointer_events, or mouse_events)

6. After drag_drop, use get_dom_snapshot to verify:
   - Source card now appears at the top of the destination list
   - Source card is no longer in the source list

7. If drag_drop failed, attempt METHOD 2 -- CDP drag:
   - Get bounding rects of source and target from DOM snapshot
   - Calculate centers: sourceX = left + width/2, sourceY = top + height/2
   - drag(sourceX, sourceY, targetX, targetY, steps=20, stepDelayMs=30)
   - Verify card moved via get_dom_snapshot

8. If CDP drag failed, attempt METHOD 3 -- Move button:
   - Click the source card or press Enter to open card detail
   - Wait for [data-testid="card-back"] modal
   - Click Move button [data-testid="card-back-move-card-button"]
   - Select destination list from dropdown
   - Select position "1" (top)
   - Click Move to confirm
   - Verify via get_dom_snapshot

9. Final verification: confirm card appears at top of destination list and not in source list

Record every step outcome (tool used, target, result, notes) for the diagnostic report.
