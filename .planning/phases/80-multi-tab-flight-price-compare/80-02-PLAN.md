Execute the CONTEXT-04 edge case test using MCP manual tools following the compareFlightsMultiTab workflow from the google-travel.js site guide. Test steps:

1. IDENTIFY FLIGHT SEARCH TARGET: Primary target is Google Flights (google.com/flights). Search for a common domestic route.
   (a) Navigate to https://www.google.com/travel/flights -- check page loads and search form is present
   (b) If Google Flights loads: search for SFO to JFK (or any common route) with a future date
   (c) If Google Flights blocked or unavailable: try Kayak (kayak.com/flights) or direct airline URLs
   (d) Alternative simplest approach: open 5 separate airline/OTA URLs pre-constructed with the same route search query (google.com/flights, kayak.com, expedia.com, united.com, southwest.com)
   Record: target URL, page loaded confirmation, search form or results found

2. SEARCH FOR FLIGHTS (if starting from search page):
   - Use the searchFlights workflow: fill origin, destination, dates
   - Wait for results to load
   - Identify at least 5 result cards with prices
   Record: number of results found, price elements visible

3. OPEN TAB 1:
   - If results link to external sites: click first result to get the booking URL
   - Use open_tab({url: booking_url_1}) to open the first flight option in a new tab
   - Record the returned tabId
   - Wait for page load via wait_for_stable
   - Use read_page with targeted selectors to extract price text
   - Store {tabId, price, airline, url}
   Record: tabId returned, price extracted, airline name

4. OPEN TAB 2:
   - If still on results page (Google Flights keeps results in original tab): switch_tab back to results tab, click second result to get URL
   - Use open_tab for second booking URL
   - Extract price, store {tabId, price, airline, url}
   Record: same as step 3

5. OPEN TAB 3:
   - Repeat for third result
   Record: same as step 3

6. OPEN TAB 4:
   - Repeat for fourth result
   Record: same as step 3

7. OPEN TAB 5:
   - Repeat for fifth result
   Record: same as step 3

8. LIST ALL TABS:
   - Use list_tabs() to verify all 5 booking tabs are still open
   - Cross-reference returned tab IDs with stored tabId values
   Record: number of tabs open, all stored tabIds present (yes/no)

9. COMPARE PRICES:
   - Sort the stored prices array by price (numeric, ascending)
   - Identify the entry with the lowest price
   - Log all 5 prices with associated airlines and tabIds
   Record: cheapest price, cheapest airline, cheapest tabId, full sorted list

10. SWITCH TO CHEAPEST TAB:
    - Use switch_tab({tabId: cheapest_tabId}) to return to the cheapest flight tab
    Record: switch_tab success, active tab matches cheapest tabId

11. VERIFY CHEAPEST:
    - Use read_page on the now-active tab to re-read the price
    - Confirm it matches the stored cheapest price
    Record: re-read price, matches stored value (yes/no)

12. OUTCOME CLASSIFICATION:
    - PASS = 5 tabs opened, all prices extracted, comparison done, returned to cheapest tab, verification passed
    - PARTIAL = some tabs opened or some prices extracted, or comparison done but verification incomplete, or WebSocket bridge disconnect prevented interaction
    - FAIL = could not access flight search or no tabs could be opened
    - SKIP-AUTH = target requires authentication (unlikely for Google Flights)
