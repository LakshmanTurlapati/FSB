  <name>Task 1: Gut UniversalProvider to return raw text and delete JSON fixup pipeline</name>
  <files>ai/universal-provider.js</files>
  <action>
Modify UniversalProvider.parseResponse to return raw text instead of JSON-parsed content:

1. **Rewrite parseResponse method:**
   - Keep the provider-specific text extraction (Gemini candidates[0].content.parts[0].text, Anthropic content[0].text, OpenAI-compatible choices[0].message.content) -- these extract the text string from the HTTP response envelope
   - Remove the `this.cleanResponse(content)` call
   - Remove the `JSON.parse` call and its try/catch
   - Return `{ content: rawTextString, usage, model }` where content is the raw text string, NOT a parsed object
   - Keep all usage/token extraction logic unchanged

2. **Delete the following methods entirely** (they are JSON-specific and no longer needed -- the CLI parser's preprocessResponse handles all cleaning):
   - `cleanResponse()` -- strips markdown, fixes JSON malformations
   - `parseJSONSafely()` -- multiple JSON.parse strategies
   - `fixTruncatedJSON()` -- completes truncated JSON strings
   - `fixCommonMalformations()` -- fixes single quotes, trailing commas
   - `fixJSONStructure()` -- adds missing brackets
   - `extractJSONFallback()` -- regex-based JSON extraction

   These 6 methods comprise approximately 250 lines of JSON fixup code. Delete all of them.

3. **Do NOT delete or modify:**
   - The HTTP request methods (callAPI, makeRequestWithRetry, etc.)
   - Provider configuration (constructor, getHeaders, getBody, etc.)
   - Token tracking and usage calculation
   - Error handling in makeRequestWithRetry
   - The `removeUnsupportedParams` method
  </action>

  </verify>
  <done>UniversalProvider.parseResponse returns raw text string. All 6 JSON fixup methods deleted (~250 lines removed). No JSON parsing occurs in the provider layer.</done>

<task type="auto">
  <name>Task 2: Wire CLI parser into ai-integration.js as sole response path</name>
  <files>ai/ai-integration.js, background.js</files>

This task has 6 sub-changes in ai-integration.js and 1 in background.js:

**A. Add importScripts in background.js:**
- Add `importScripts('ai/cli-parser.js');` BEFORE the `importScripts('ai/ai-integration.js');` line in background.js so parseCliResponse is available when ai-integration.js loads.

**B. Create standalone sanitizeActions function in ai-integration.js:**
- Extract the security sanitization logic from the existing normalizeResponse function (the code that blocks `navigate` actions with `data:` or `javascript:` URIs, and `type` actions containing `<script`, `javascript:`, or `onerror=`).
- Create a new module-level function `sanitizeActions(actions)` that filters the actions array, removing any dangerous actions and logging warnings. Return the sanitized array.
- Place this function near the top of ai-integration.js (before the AIIntegration class) alongside other module-level functions.

**C. Rewrite processQueue parsing path:**
- In processQueue, after `const response = await this.callAPI(...)`:
  - The response from callAPI is now a raw text string (not a parsed object) because UniversalProvider.parseResponse returns raw text.
  - Modify callAPI's return to extract the content string: `const rawText = typeof response === 'string' ? response : (response?.content || String(response));`
  - Call `let parsed = parseCliResponse(rawText);`
  - Apply security sanitization: `parsed.actions = sanitizeActions(parsed.actions);`
  - If no valid output (parsed.actions.length === 0 && !parsed.taskComplete && !parsed.taskFailed && !parsed.helpRequested), implement CLI reformat retry:
    - Build a retry message asking the AI to reformat its response as CLI commands
    - Use the existing retry mechanism but with a CLI-format correction prompt: "Your response was not in CLI command format. Reformat as CLI commands (one per line, # for reasoning). Your response was: [first 500 chars of rawText]"
    - Parse the retry response through parseCliResponse again
    - Apply sanitizeActions to retry result
  - Attach raw text to parsed result: `parsed._rawCliText = rawText;`
  - Set batchActions compatibility shim: `if (parsed.actions.length > 1) parsed.batchActions = parsed.actions;`
  - Resolve with parsed result

**D. Delete the JSON parsing pipeline:**
- Delete `parseResponse()` method (the 4-strategy JSON pipeline: parseCleanJSON, parseWithMarkdownBlocks, parseWithJSONExtraction, parseWithAdvancedCleaning)
- Delete `parseCleanJSON()` method
- Delete `parseWithMarkdownBlocks()` method
- Delete `parseWithJSONExtraction()` method
- Delete `parseWithAdvancedCleaning()` method
- Delete `normalizeResponse()` method (security logic already extracted to sanitizeActions)
- Delete `getModelSpecificInstructions()` method (model-agnostic now -- no per-model JSON formatting instructions)
- Delete `isValidResponse()` if it only validates JSON structure; if it validates tool names, keep but update for parseCliResponse output shape (check actions array has valid tool names)

**E. Rewrite enhancePromptForRetry:**
- The current enhancePromptForRetry (around lines 4477-4494) adds JSON-specific retry instructions. Rewrite to add CLI format reinforcement instead:
  - "Respond with CLI commands only, one per line. Use # for reasoning. Example: click e5"
  - Remove all JSON-specific retry text ("respond with raw JSON only", "valid JSON object", etc.)

**F. Update createFallbackResponse:**
- The current createFallbackResponse returns `{ actions: [], taskComplete: false, reasoning: 'Fallback...', situationAnalysis: '...' }`.
- Update to include ALL fields that parseCliResponse returns: actions, reasoning (as array), errors (as array), taskComplete, taskFailed, result, helpRequested, helpVerb, confidence, situationAnalysis, goalAssessment, assumptions, fallbackPlan, _rawCliText.
- This ensures downstream consumers (background.js) never encounter undefined fields.
