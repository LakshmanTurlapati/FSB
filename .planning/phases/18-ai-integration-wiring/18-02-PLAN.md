Two changes -- one in ai-integration.js (compaction), one in cli-parser.js (YAML parsing):

**A. Update triggerCompaction's compaction prompt (ai-integration.js):**

  ```
  CRITICAL: Include 1-2 VERBATIM CLI command examples from the conversation to maintain format consistency. For example:

  ```
- The compaction output should be a text summary that naturally includes CLI command snippets, so the AI sees CLI format when the compacted history is injected into future turns.
- Keep the existing compaction structure (summarize actions taken, pages visited, errors, progress) but add the CLI preservation instruction.

**B. Add YAML block parsing for storeJobData (ai/cli-parser.js):**
- Create a `parseYAMLBlock(lines, startIndex)` function that:
  1. Takes the full lines array and the index of the storejobdata command line
  2. Reads subsequent lines that are indented by 2+ spaces (part of the YAML block)
  3. Parses the indented block into a JavaScript object using a simple state machine:
     - Top-level keys: `key: value` at 2-space indent
     - Array items: `- key: value` entries under a parent key
     - Nested object fields: `key: value` at deeper indent under array items
     - URLs are treated as opaque strings (don't split on colons after the scheme)
     - Colons in values: only the FIRST colon on a line is the key-value separator
  4. Returns `{ data: parsedObject, linesConsumed: N }` where N is how many indented lines were consumed
  5. The parsed data is placed in `params.data` of the storejobdata command
- No external YAML library (project rule: no new dependencies). The parser only needs to handle the storeJobData schema: company (string), jobs (array of objects with title, location, datePosted, applyLink fields).

- Update `mapCommand` in cli-parser.js:
  - For the `storejobdata` command, check if params are empty (no inline args) AND the command has type 'json' in COMMAND_REGISTRY
  - If the current line is `storejobdata` with no args, signal that subsequent indented lines should be consumed as a YAML block
  - In parseCliResponse's line processing loop, after calling mapCommand for storejobdata, check if YAML block consumption is needed and call parseYAMLBlock

- Update COMMAND_REGISTRY entry for storejobdata if needed to support the YAML block path.

**C. Update CLI_COMMAND_TABLE for fillSheetData documentation (ai/ai-integration.js):**
- In the CLI_COMMAND_TABLE constant, find the fillsheetdata entry.
- Update its example/documentation to show cell-reference YAML format:

  3. Mixed response: CLI commands + storejobdata YAML block + more CLI commands
