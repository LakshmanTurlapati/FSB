## Spacing Scale

Declared values (all multiples of 4, matching existing codebase):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, status dot margins, inline padding |
| sm | 8px | Compact element spacing, toggle internal padding, pill gaps |
| md | 16px | Card padding, section margins, input padding |
| lg | 24px | Card body padding, section gaps, header margins |
| xl | 32px | Stats bar bottom margin, major section breaks |
| 2xl | 48px | Login card padding, page-level section padding |
| 3xl | 64px | Empty state vertical padding |

Exceptions:
- 12px: Badge padding and form field internal gaps. Required because badge text at 12px font size needs proportional padding (mid-point between sm/8px and md/16px for visual balance at small element scale).
- Toggle switch: 20px height, 36px width (not on 4px grid; standard toggle sizing)
- Touch targets: 44px minimum for buttons (already established in dash-task-submit)

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (1rem) | 400 | 1.6 | Task descriptions, result text, modal body text |
| Label | 12px (0.75rem) | 600 | 1.3 | Stat labels, schedule summaries, meta text, badge text. Uppercase + letter-spacing: 0.04em for stat labels |
| Heading | 20px (1.25rem) | 600 | 1.2 | Card names, detail panel section headers, modal titles |
| Stat value | 24px (1.5rem) | 600 | 1.2 | Stats bar numeric values, cost saved highlight |

Small text exception: Badges and schedule pills use a fixed 12px (0.75rem) at weight 600, covered by the Label row above. No additional sizes or weights introduced.

Note: Existing codebase uses rem-based sizing. Phase 43 follows the same convention.

## Color

| Role | Value (Dark) | Value (Light) | Usage |
|------|-------------|---------------|-------|
| Dominant (60%) | #000000 / #0a0a0a | #ffffff / #f8f9fa | Page background, card backgrounds |
| Secondary (30%) | #141414 | #f1f5f9 | Elevated surfaces: modal overlays, input backgrounds, toggle tracks, pill backgrounds |
| Accent (10%) | #ff6b35 | #ff6b35 | See reserved-for list below |
| Destructive | #ef4444 | #ef4444 | Delete button, failed status, error text |

Primary focal point: The accent-colored "+ New Agent" button in the grid header and the selected agent card border glow (shadow-glow) serve as the primary visual anchors of this phase.

Accent reserved for:
- "+ New Agent" button (btn-primary gradient)
- "Save Agent" CTA in post-task inline save
- "Run Now" button in detail panel
- "Save Agent" button in create/edit modal
- Selected agent card border glow (shadow-glow)
- Active toggle switch track color
- Progress bar fill during agent run
- Input focus border highlight

Semantic colors (all pre-established in codebase):

| Semantic | Value | Usage in Phase 43 |
|----------|-------|-------------------|
| Success | #22c55e | Success rate >80% text, enabled status dot, "Replay" mode badge, success run status |
| Warning | #eab308 | Success rate 50-80% text, "AI Fallback" mode badge |
| Error | #ef4444 | Success rate <50% text, failed run status, delete button, delete confirmation CTA |
| AI blue | #818cf8 (indigo) | "AI" mode badge (rgba(99, 102, 241, 0.15) bg) |
| Muted | #64748b | Disabled status dot, disabled card overlay, secondary meta text |

Success rate color thresholds (from CONTEXT.md):
- Green (#22c55e): rate > 80%
- Orange (#eab308): rate 50%-80%
- Red (#ef4444): rate < 50%

Disabled agent card treatment: `opacity: 0.5` on the entire card container.

## Component Inventory

### New Components (Phase 43)

#### 1. Agent Card (enhanced)
Extends existing `dash-agent-card`. Additions:
- **Toggle switch**: Right-aligned in card header row, 36x20px, rounded-full track
  - Enabled: accent (#ff6b35) track, white circle
  - Disabled: muted (#64748b) track, white circle
  - Click fires REST API immediately, no confirmation (CONTEXT.md decision)
- **Success rate fraction**: "12/14" text, colored per threshold rules above
- **Cost saved**: "$0.42" in mono font, muted color
- **Last run**: "45m ago" relative time, muted color
- **Schedule summary**: "Every 2h" in existing dash-agent-schedule pill

#### 2. Agent Detail Side Panel
New element: `dash-agent-detail`. Right-side panel.
- **Width**: 420px fixed on desktop, 100% on mobile (<768px)
- **Position**: Placed after the agent grid in DOM flow (not overlay). Grid and panel sit side by side using CSS grid: `grid-template-columns: 1fr 420px` on the parent container. On mobile, panel replaces grid (display toggle).
- **Header**: Agent name (heading size) + [Run Now] (btn-primary btn-sm) + [Edit] (btn-outline btn-sm)
- **Config section**: Name, task, URL, schedule displayed as label+value pairs
- **Cost savings section**: Card with 4 values in 2x2 grid: replay runs, AI fallback count, tokens saved, cost saved ($X.XX in stat-value size)
- **Run history table**: Rows match existing `dash-run-entry` grid pattern (time, status badge, mode badge, result, duration, cost). Paginated with existing dash-runs-pagination pattern, 10 runs per page.
- **Execution mode badges**: Match existing CSS classes exactly:
  - `dash-mode-replay`: green bg rgba(34, 197, 94, 0.15), text #22c55e, label "Replay"
  - `dash-mode-ai`: accent bg rgba(255, 107, 53, 0.1), text var(--primary), label "AI"
  - `dash-mode-fallback`: warning bg rgba(234, 179, 8, 0.15), text #eab308, label "AI Fallback"
- **Recorded script section**: Collapsible with chevron icon. Collapsed by default. Shows numbered action steps when expanded. Uses mono font, bg-elevated background, 12px font size.
- **Delete button**: Full-width at bottom, `btn-outline` style with `color: #ef4444; border-color: #ef4444`. Label: "Delete Agent"
- **Close**: X button top-right (matching dash-runs-close pattern)

#### 3. Agent Creation Modal
New element: `dash-agent-modal`. Centered overlay modal.
- **Overlay**: rgba(0, 0, 0, 0.6) backdrop
- **Modal body**: max-width 520px, bg-card, border-radius var(--radius-xl), padding 32px
- **Header**: "New Agent" or "Edit Agent" (heading size) + close X button
- **Fields** (top to bottom):
  - Name: text input, placeholder "e.g., Check stock prices"
  - Task description: textarea, 3 rows, placeholder "Describe what this agent should do..."
  - Target URL: text input, mono font, placeholder "https://example.com"
  - Schedule type: 3-option segmented control (Interval / Daily / Once)
  - Schedule config (conditional on type):
    - **Interval**: Numeric input with "minutes" suffix label. Min 5, snaps to 5 if <5 with brief inline message "Minimum 5 minutes" that fades after 2s
    - **Daily**: Time input (HH:MM) + day-of-week pill row (M T W T F S S). Pills: 32px square, rounded-full, bg-elevated default, accent bg when selected. All selected by default. Click toggles.
    - **Once**: datetime-local input
- **Footer**: [Discard Changes] (btn-outline) right-aligned + [Save Agent] (btn-primary) right-aligned, 8px gap
- **Animation**: fadeIn 200ms ease (match existing dashTaskFadeIn pattern)

#### 4. Post-Task Save-as-Agent Section
New element: `dash-task-save-agent`. Inline expansion below task success view.
- **Trigger**: "Save as Agent" text button with down-chevron icon, muted color, appears only after successful task completion
- **Expanded state**: Slides down with 200ms ease transition (max-height animation)
- **Content**: Pre-filled fields (same as modal fields but inline layout):
  - Agent name (pre-filled by AI from task text)
  - Schedule type + config (pre-filled by AI if schedule words detected)
  - URL (pre-filled from task execution context)
  - All fields editable
- **CTA**: [Save Agent] (btn-primary btn-sm) at bottom-right
- **Collapsed visual**: Subtle divider line (border-top: 1px solid var(--border-color)) separating it from task result above

#### 5. Delete Confirmation Dialog
Small centered modal (not full creation modal size).
- **Width**: max-width 400px
- **Content**: "Delete [agent name]?" heading + "This removes all run history." body text in text-secondary
- **Buttons**: [Keep Agent] (btn-outline btn-sm) + [Delete] (btn with bg #ef4444, color white, btn-sm), 8px gap, right-aligned
- **Overlay**: Same as creation modal

#### 6. Toggle Switch
Standalone micro-component reused on agent cards.
- **Dimensions**: 36px wide, 20px tall, border-radius 10px (fully rounded)
- **Track**: bg-elevated when off, accent (#ff6b35) when on. Transition: background 150ms ease
- **Thumb**: 16px circle, white, 2px inset from track edge. Transform: translateX(16px) when on. Transition: transform 150ms ease
- **Implementation**: `<button role="switch" aria-checked="true/false">` with CSS pseudo-element for thumb
- **Click handler**: Immediate REST API call, optimistic UI update, revert on failure

## Interaction States

### Agent Card States

| State | Visual Treatment |
|-------|-----------------|
| Default | bg-card, border-color, normal opacity |
| Hover | border-hover, shadow-sm |
| Selected (detail open) | border: primary, shadow-glow |
| Disabled agent | opacity: 0.5 on entire card. Toggle shows off position |
| Agent running | Spinner icon (dash-spinner) next to agent name |

### Detail Panel States

| State | Visual Treatment |
|-------|-----------------|
| Idle | Normal display, Run Now button enabled |
| Agent running (via Run Now) | Progress bar appears below header (same as dash-task-bar pattern), Run Now button disabled with spinner |
| Run complete | Progress bar fills to 100%, turns green (success) or red (failed), auto-adds entry to run history |

### Modal States

| State | Visual Treatment |
|-------|-----------------|
| Open | Overlay visible, modal centered, first input focused |
| Validation error | Input border turns #ef4444, inline error text below field in 0.8rem #ef4444 |
| Submitting | Save button disabled, shows dash-spinner inside button, text changes to "Saving..." |
| Success | Modal closes, agent card appears/updates in grid with brief highlight animation (border-color accent for 1s then fades) |

### Post-Task Save States

| State | Visual Treatment |
|-------|-----------------|
| Hidden | display: none (task not yet completed, or task failed) |
| Collapsed | "Save as Agent" text link visible below task result |
| Expanded | Fields slide down, AI-pre-filled values shown |
| Saving | Save button disabled with spinner |
| Saved | Section collapses, brief success toast "Agent created" (match existing showToast pattern from Phase 41) |

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| Toggle switch | `role="switch"` + `aria-checked="true/false"` + `aria-label="Enable [agent name]"` |
| Modal | `role="dialog"` + `aria-modal="true"` + `aria-labelledby` pointing to modal title. Focus trap within modal. Escape key closes. |
| Delete confirmation | Same as modal. Focus auto-set to Keep Agent button (safer default). |
| Agent card | `role="button"` + `aria-expanded="true/false"` (reflects detail panel state) |
| Day-of-week pills | `role="checkbox"` + `aria-checked` + `aria-label="Monday"` etc. |
| Detail panel close | `aria-label="Close detail panel"` |
| Run history table | Semantic `<table>` with `<thead>` and `<th scope="col">` headers |
| Status badges | Include sr-only text: "Status: success" / "Status: failed" |
| Color-only indicators | Success rate fraction text provides numeric info alongside color |
