However, the CLI path through ai-integration.js callAPI does NOT set max_tokens -- it relies on UniversalProvider.buildRequest defaults. The legacyCallAPI does set `max_tokens: 4000` but only for direct xAI API calls.

**Fix: Ensure max_tokens is set in the CLI path's provider call.**

In the `callAPI` method (around line 4288), after `this.provider.buildRequest(prompt, {})` constructs the requestBody, inject max_tokens into the request if not already present:

Find the line:
```javascript
const requestBody = await this.provider.buildRequest(prompt, {});
```

Change it to:
```javascript
const requestBody = await this.provider.buildRequest(prompt, {});
// AL-13/XS-06 fix: Ensure max_tokens is set to prevent xAI defaulting to
// a tight internal limit (~87 tokens/iter) that truncates tool responses.
// The tool_use path (agent-loop.js callProviderWithTools) sets max_tokens: 4096.
// The CLI path must match to prevent truncated responses.
if (!requestBody.max_tokens) {
  requestBody.max_tokens = 4096;
}
```
