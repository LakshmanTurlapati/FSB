**Change B: Add the 5 handler functions.**

Place them AFTER the handleCDPInsertText function (after line ~9893) and BEFORE the handleMonacoEditorInsert function (line ~9895). Follow the exact CDP debugger pattern from handleCDPInsertText: attach debugger -> send Input commands -> detach debugger -> sendResponse.

Each handler must handle the KeyboardEmulator conflict (check `keyboardEmulator.isAttachedTo(tabId)` and detach first), handle "Another debugger is already attached" errors with force-detach retry, and always detach in a finally block.

Implementation details for each handler:

**handleCDPMouseClick(request, sender, sendResponse):**
- Extract: `x, y, shiftKey, ctrlKey, altKey` from request
- tabId from sender.tab.id
- Build modifiers bitmask: `(altKey ? 1 : 0) | (ctrlKey ? 2 : 0) | (shiftKey ? 8 : 0)`
- Send `Input.dispatchMouseEvent` type:'mousePressed', button:'left', clickCount:1
- Send `Input.dispatchMouseEvent` type:'mouseReleased', button:'left', clickCount:1
- Return `{ success: true, x, y }`

**handleCDPMouseClickAndHold(request, sender, sendResponse):**
- Extract: `x, y, holdMs` from request (holdMs defaults to 5000)
- Send `Input.dispatchMouseEvent` type:'mousePressed', button:'left', clickCount:1
- `await new Promise(r => setTimeout(r, holdMs))`
- Send `Input.dispatchMouseEvent` type:'mouseReleased', button:'left', clickCount:1
- Return `{ success: true, x, y, holdMs }`

**handleCDPMouseDrag(request, sender, sendResponse):**
- Extract: `startX, startY, endX, endY, steps, stepDelayMs, shiftKey, ctrlKey, altKey` from request
- steps defaults to 10, stepDelayMs defaults to 20
- Send mousePressed at (startX, startY)
- Loop `steps` times, interpolating x/y linearly from start to end, sending `Input.dispatchMouseEvent` type:'mouseMoved' at each step with a `stepDelayMs` delay between
- Send mouseReleased at (endX, endY)
- Return `{ success: true, startX, startY, endX, endY, steps }`

**handleCDPMouseDragVariableSpeed(request, sender, sendResponse):**
- Extract: `startX, startY, endX, endY, steps, minDelayMs, maxDelayMs` from request
- steps defaults to 20, minDelayMs defaults to 5, maxDelayMs defaults to 40
- Same as drag but delay per step is `Math.floor(Math.random() * (maxDelayMs - minDelayMs + 1)) + minDelayMs`
- Return `{ success: true, startX, startY, endX, endY, steps }`

**handleCDPMouseWheel(request, sender, sendResponse):**
- Extract: `x, y, deltaX, deltaY` from request (deltaX defaults to 0, deltaY defaults to -120)
- Send `Input.dispatchMouseEvent` type:'mouseWheel' at (x, y) with deltaX, deltaY
- Return `{ success: true, x, y, deltaX, deltaY }`

All handlers must log via `automationLogger.logActionExecution(null, actionName, 'start'/'complete', details)`.

All handlers must have try/catch with `sendResponse({ success: false, error: error.message })` in catch.
