Execute the SCROLL-06 edge case test using MCP manual tools following the expandAllThreads workflow from the hackernews.js site guide. Test steps:

1. Use MCP navigate tool to go to news.ycombinator.com. Look for a post with 1000+ comments in the subtext. If no 1000+ post is visible on the front page, try news.ycombinator.com/best or navigate to a known popular Ask HN thread. Record the chosen post URL and its displayed comment count.

2. Use read_page to verify the post thread loaded. Check for:
   - Post title visible
   - Subtext with comment count (e.g., "1523 comments")
   - Comment-tree table present with tr.athing.comtr rows
   - a.morelink at the bottom of the page
   Record: page URL, post title, header comment count, initial visible comment count (tr.athing.comtr count), morelink presence.

3. Count initial comments on page 1 using read_page or get_dom_snapshot:
   - Count all tr.athing.comtr elements
   - Record: "Page 1: {count} comments loaded"

4. EXPANSION LOOP (click morelink to navigate through comment pages, max 15 pages):
   a. Look for a.morelink at the bottom of the current page
   b. If morelink found: click it. This triggers a FULL PAGE NAVIGATION (URL changes to ?p=N)
   c. Wait 1000-2000ms for the new page to load (use waitForDOMStable or wait_for_stable)
   d. Use read_page to verify new page loaded. Count tr.athing.comtr elements on this page.
   e. Add page count to cumulative total
   f. Log: "Page {N}: {count} comments on this page, cumulative total: {cumulative}"
   g. Check for another a.morelink at the bottom of this page
   h. If morelink found: repeat from step (b)
   i. If no morelink found: all comment pages have been visited
   j. If cumulative total reaches 1000+: continue until no more pages (for completeness)

5. AFTER ALL PAGES VISITED:
   - Report: total pages visited, cumulative comment count across all pages, header comment count from step 2
   - Calculate: cumulative / header * 100 = percent of comments loaded
   - If cumulative >= 1000: threshold met
   - If cumulative < 1000 but no more pages: report total as final count

6. VERIFICATION:
   - Compare cumulative count against header comment count (within 10% is acceptable due to dead/flagged comments)
   - Confirm all morelinks were clicked and no pages were skipped
   - Record: "Final: {cumulative} comments expanded across {pages} pages vs {header_count} in header"

7. OUTCOME CLASSIFICATION:
   - PASS = expanded all comment pages, cumulative count >= 1000, count within 10% of header
   - PARTIAL = expanded some pages but not all (rate limiting, navigation error, or WebSocket bridge disconnect), OR count reached 1000+ but could not verify all pages
   - PARTIAL = expanded pages but cumulative count < 1000 (thread had fewer comments than expected)
   - FAIL = could not access HN or click morelink at all
   - Note: HN is public, no auth required -- SKIP-AUTH should not apply

Record every step outcome (tool used, target, result, notes) for the diagnostic report.
