Execute the CONTEXT-06 edge case test using MCP manual tools following the chatbot15TurnSummary workflow from the support-chatbot.js site guide. Test steps:

1. IDENTIFY CHATBOT TARGET: Primary target is tidio.com (their own chatbot widget).
   (a) Navigate to https://www.tidio.com -- check page loads and look for chat launcher icon
   (b) If tidio.com has no automated bot: try https://www.crisp.chat
   (c) If that fails: try https://www.drift.com
   (d) If that fails: try https://www.hubspot.com
   (e) Last resort: try https://www.intercom.com
   Record: target URL, page loaded, chat launcher visible, chatbot platform identified

2. OPEN CHAT WIDGET:
   - Click the chatbot launcher (chatLauncher selector or click_at if in bottom-right corner)
   - Wait for chat panel to open and input field to become visible
   - Check if chat panel is inside an iframe (chatIframe selector)
   Record: launcher clicked, panel opened, iframe detected (yes/no)

3. IFRAME HANDLING (if applicable):
   - If chat is in an iframe: attempt DOM tools first (type_text on chatInput)
   - If DOM tools cannot reach inside iframe: switch to CDP tools (click_at on input coordinates, press_key for typing)
   - Document which interaction method works
   Record: interaction method (DOM/CDP), any tool failures

4. PRE-CHAT FORM (if applicable):
   - Some chatbots ask for name/email before starting
   - If prompted: enter "Test User" for name, "test@example.com" for email
   - Submit the pre-chat form
   Record: pre-chat form present (yes/no), fields filled

5. TURN 1: Type "Hello, I need help with your product" into chatInput.
   - Click sendButton (or press Enter)
   - Wait for bot response to appear (wait_for_stable or check for new botMessage element)
   Record: {turn: 1, sent: true, botResponded: true/false, botResponseSnippet: "first 50 chars"}

6. TURNS 2-5: Continue conversation with product-related questions.
   - Turn 2: Respond to bot's greeting naturally
   - Turn 3: "What pricing plans do you offer?"
   - Turn 4: Follow-up on bot's pricing response
   - Turn 5: "What features are included in the basic plan?"
   - After each: wait for bot response, record turn status
   Record: turns completed, any failures

7. TURNS 6-10: Continue with broader questions.
   - Turn 6: "Do you integrate with other tools?"
   - Turn 7: "What support options are available?"
   - Turn 8: "How do I get started with setup?"
   - Turn 9: "Can I manage team members on my account?"
   - Turn 10: "What payment methods do you accept?"
   - If bot asks a question or presents menu options: respond naturally or click relevant option
   Record: turns completed, menu interactions, any failures

8. TURNS 11-15: Complete remaining turns.
   - Turn 11: "How do you handle data security?"
   - Turn 12: "Can I customize the appearance?"
   - Turn 13: "Do you have enterprise plans?"
   - Turn 14: "Can I migrate from a different provider?"
   - Turn 15: "Thank you, that has been very helpful"
   - Wait for final bot response
   Record: all 15 turns completed (yes/no), reason if fewer

9. VERIFY TURN COUNT:
   - Count total user messages sent: should be 15
   - Count total bot responses received: should be 15
   - If fewer than 15: note why (bot ended conversation, widget closed, error, timeout)
   Record: user messages count, bot responses count

10. SCROLL TO TOP OF CHAT:
    - Scroll up within the chat messages container (NOT the page)
    - Target the messagesContainer element with scroll tool
    - Continue scrolling until reaching the first messages in the conversation
    Record: scroll method used, reached top of chat (yes/no)

11. READ FIRST MESSAGES:
    - Use read_page targeting the chat panel after scrolling to top
    - Look for the first 3-4 bot messages (botMessage selector)
    - Identify which bot message contains the first actionable instruction (not just "Hi, how can I help?")
    Record: number of bot messages found at top, first instruction identified (yes/no)

12. EXTRACT AND SUMMARIZE FIRST INSTRUCTION:
    - Extract the first actionable bot instruction verbatim
    - Summarize in 1-2 sentences what the bot instructed/guided the user to do
    Record: verbatim first instruction text, 1-2 sentence summary

13. OUTCOME CLASSIFICATION:
    - PASS = 15 messages sent, bot responded to each, scrolled to top, first instruction identified and summarized in 1-2 sentences
    - PARTIAL = 10+ messages sent OR first instruction extraction failed OR WebSocket bridge disconnect prevented interaction (with HTTP/DOM validation of chatbot presence)
    - FAIL = could not open chatbot widget, or could not send messages, or fewer than 10 turns completed
    - SKIP-AUTH = all target chatbots require login or only offer live agent chat

Record every step outcome for the diagnostic report.
