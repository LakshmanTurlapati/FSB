**Step 1: Navigate to the canvas game**
Use MCP `navigate` to open the game URL documented in the site guide (from Plan 01). Wait for page load with `wait_for_stable`.

**Step 2: Dismiss any overlays on host page**
Use `get_dom_snapshot` to check for cookie consent dialogs, age gates, or promo banners on the host page (e.g., itch.io page). Click dismiss/accept buttons using DOM `click` if found.

**Step 3: Launch the game (if iframe-hosted)**
If the game is hosted in an iframe (common on itch.io):
- Look for a "Run game" or "Play" DOM button on the host page
- Click it using DOM `click` on the selector from the site guide
- Wait for the game iframe to load with `wait_for_stable`
- This step uses a DOM click because the launcher button IS an HTML element on the host page

**Step 4: Locate the canvas element and its viewport bounds**
Use `get_dom_snapshot` to find:
- The canvas element (using selectors from the site guide: `canvas`, `iframe canvas`, etc.)
- Its bounding rectangle in the viewport (x, y, width, height)
- Record these values -- they are needed to calculate pixel coordinates for the canvas button

**Step 5: Calculate canvas button pixel coordinates**
Using the canvas bounds from Step 4 and the percentage-based button positions from the site guide:
- Calculate the absolute viewport X coordinate: canvasLeft + (canvasWidth * horizontalPercentage)
- Calculate the absolute viewport Y coordinate: canvasTop + (canvasHeight * verticalPercentage)
- For a typical "Play" or "Start" button on a title screen: center-horizontal (~50% width), lower-center (~60-70% height)
- If the game is in an iframe, ensure coordinates are relative to the MAIN viewport (add iframe offset)

**Step 6: Wait for title screen / loading to complete**
Games often have a loading screen before the title screen with clickable buttons appears. After Step 4:
- Wait 3-5 seconds (use wait_for_stable or equivalent delay)
- Optionally use get_dom_snapshot again to check if loading indicators have disappeared

**Step 7: Click the canvas-painted button using click_at**
Execute `click_at(calculatedX, calculatedY)` to click the canvas-painted button at its pixel coordinates.
- This is the core CANVAS-07 test: a CDP trusted click at pixel coordinates targeting a canvas-rendered button with no DOM element
- Record the exact coordinates used

**Step 8: Verify the button click registered**
After click_at, verify that the game state changed:
- Wait 1-2 seconds with `wait_for_stable`
- Use `get_dom_snapshot` to check for any DOM changes (some games update a score div, change URL hash, or modify iframe attributes)
- Use `read_page` to check if page text content changed (e.g., game instructions appearing)
- The key signal is that the canvas content changed from "title screen with Play button" to "gameplay or next menu"
- If no DOM-level verification is possible (pure canvas rendering), the click_at completing without error plus a reasonable wait constitutes the best available verification

**Step 9: Attempt a second canvas interaction (if Step 7 succeeded)**
If the first button click worked, attempt one more canvas interaction to confirm consistent pixel-coordinate clicking:
- Click another canvas button (e.g., menu option, difficulty selection, in-game control)
- Or click the same button type on a different screen
- Record coordinates and result
