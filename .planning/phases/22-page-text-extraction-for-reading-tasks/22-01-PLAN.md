Add the following functions AFTER the existing `buildYAMLSnapshot` function (after ~L2560) and BEFORE the self-test section:

**1. `buildMarkdownSnapshot(options = {})`** -- Main entry point.
Parameters: `{ guideSelectors = null, charBudget = 12000, maxElements = 80 }`
Steps:
  a. Reset `FSB.refMap` and set `refMap.generationUrl = window.location.href`
  b. Get interactive elements via `getFilteredElements({ maxElements, mode: 'interactive' })`
  c. Build `interactiveSet` (Set of DOM elements) and register each in refMap using same pattern as buildYAMLSnapshot (~L2445-2452): `computeAccessibleName`, `refMap.register(el, primarySelector, role, nameStr)`
  d. Build guide annotations via `buildGuideAnnotations(guideSelectors, interactiveSet)`
  e. Group elements by region using `getRegion(el)` -- build a Map<regionKey, element[]>
  f. Build metadata header as markdown: page title as `# {title}` on first line, then `> URL: {url} | Scroll: {scrollPercent}% | Viewport: {width}x{height}` on next line. Use same data sources as existing `buildMetadataHeader` (document.title, location.href, scrollY/scrollHeight, innerWidth/innerHeight). Include focused element ref if applicable.
  g. Walk the DOM tree using recursive visitor `walkDOMToMarkdown(document.body, interactiveSet, refMap, guideAnnotationsMap)` to produce content lines
  h. Organize output: metadata header + blank line + region-grouped content with `## {RegionName}` headings
  i. Apply character budget: count chars line by line, stop at last complete line fitting within `charBudget`. If truncated, append `\n\n[...content continues below -- scroll down and observe]`
  j. Post-process: collapse 3+ consecutive blank lines to 1 blank line, trim trailing whitespace per line
  k. Return `{ snapshot, refGeneration: refMap.generationId, elementCount }`

**2. `walkDOMToMarkdown(root, interactiveSet, refMap, guideAnnotations)`** -- Recursive DOM visitor.
Uses a recursive `visit(node)` pattern (NOT TreeWalker API, per research recommendation).
Maintains state: `lines` array, `currentLine` string, `listDepth` counter.

For each node:
- **Text nodes**: Normalize whitespace (`text.replace(/\s+/g, ' ').trim()`), append to currentLine if non-empty
- **Hidden elements**: Skip entire subtree. Check `getComputedStyle(node).display === 'none'` or `visibility === 'hidden'` or `node.getAttribute('aria-hidden') === 'true'` or `node.offsetWidth === 0 && node.offsetHeight === 0`. Reuse the visibility check pattern from Stage 2 filter in getFilteredElements.
- **Interactive elements** (in `interactiveSet`): Emit inline ref via `formatInlineRef(node, refMap, guideAnnotations)`, then SKIP children (don't recurse -- the ref already captures the accessible name, avoiding Pitfall 1 from research).
- **Block elements** (h1-h6, p, div, section, article, li, blockquote, pre, table, tr, hr, br, figure, figcaption, details, summary, dt, dd, address, form, fieldset, aside, nav, header, footer, main): Flush currentLine to lines array, then set appropriate prefix:
  - `h1`-`h6`: `'#'.repeat(level + 1) + ' '` (offset by 1 since H1 is page title in metadata)
  - `li`: `'  '.repeat(listDepth) + prefix` where prefix is `'- '` for ul, `'{index}. '` for ol
  - `blockquote`: `'> '`
  - `hr`: `'---'`
  - `br`: flush line (acts as line break)
  - `pre`/`code` blocks: wrap content in triple backticks
  - `table`/`tr`: see table handling below
- **Inline formatting** (`strong`/`b`): wrap with `**`. (`em`/`i`): wrap with `*`. (`code`): wrap with backtick.
- **Images** (`img`): Emit `[Image: {alt || 'no description'}]`, skip children.
- **Links** (`a`) that are NOT in interactiveSet: Emit `[{text}]({href})` for non-interactive links (links excluded from element filtering but still visible).
- **Lists** (`ul`/`ol`): Increment `listDepth` before recursing children, decrement after.
- **Tables**: Simple pipe-delimited markdown. For each `tr`, collect cell text, join with ` | `, prepend and append `|`. After first row (thead or first tr), emit separator `|---|---|...`. Cap at 8 columns, truncate wide tables. For complex tables (colspan/rowspan), fall back to plain text with indentation.
- **FSB-injected elements**: Skip elements with FSB-specific classes/IDs (check for `__fsb` prefix or `data-fsb` attributes).
- After visiting all children of a block element, flush currentLine.

`flushLine()`: If `currentLine.trim()` is non-empty, push `currentLine.trimEnd()` to `lines`, reset `currentLine = ''`.

Return `lines` array.

**3. `extractPageText(root, options = {})`** -- Text-only extraction for readpage.
Parameters: `root` (DOM element), `options: { viewportOnly = true, format = 'markdown-lite' }`
Steps:
  a. If `viewportOnly`, determine viewport bounds (scrollY, scrollY + innerHeight)
  b. Walk DOM tree similarly to walkDOMToMarkdown but with NO element refs -- just text content with markdown formatting (headings, lists, links as `[text](url)`, tables, images as alt text)
  c. If `viewportOnly`, skip nodes whose `getBoundingClientRect().top` is outside viewport bounds
  d. No character budget (readpage is full extraction), but cap at 50K chars with `[...text truncated at 50K chars]`
  e. Apply same whitespace normalization
  f. Return the markdown text string

**4. `formatInlineRef(node, refMap, guideAnnotations)`** -- Backtick ref formatter.
Format: `` `{ref}: {type} "{name}" {attributes}` ``
- `ref`: from refMap (e.g., `e5`)
- `type`: element role/tag (button, input, a, select, checkbox, etc.)
- `name`: accessible name from computeAccessibleName, truncated to 60 chars
- Attributes in brackets:
  - `[disabled]` if node.disabled
  - `[required]` if node.required
  - `[checked]` if node.checked (for checkboxes/radios)
  - `[hint:{hintName}]` from guideAnnotations map
  - `= "{value}"` for inputs/selects with current value (node.value, truncated to 40 chars)
  - `href="{url}"` for links (truncated to 60 chars)
Examples:
  - `` `e5: button "Submit"` ``
  - `` `e2: input "Search" [hint:searchBox]` ``
  - `` `e3: input "Username" = "john_doe"` ``
  - `` `e7: checkbox "Remember me" [checked]` ``
  - `` `e8: select "Size" = "Standard"` ``
  - `` `e10: a "Cart (3)" href="/cart"` ``

**5. Helper functions:**
- `isBlockElement(tag)`: Returns true for block-level HTML tags (p, div, h1-h6, ul, ol, li, table, tr, th, td, blockquote, pre, hr, br, figure, figcaption, section, article, aside, nav, header, footer, main, details, summary, dt, dd, address, form, fieldset)
- `isVisibleForSnapshot(node)`: Visibility check for the walker. Check computed display/visibility, aria-hidden, zero dimensions. Similar to Stage 2 filter logic but as a standalone function.

**2. Add `readPage` case** in the `handleAsyncMessage` switch (after the new getMarkdownSnapshot case):

```javascript
case 'readPage':
  const rpStart = Date.now();
  const rpSelector = request.params?.selector || null;
  const rpFull = request.params?.flags?.full || request.params?.full || false;
  const rpRoot = rpSelector ? document.querySelector(rpSelector) : document.body;
  if (!rpRoot) {
    sendResponse({ success: true, text: '[Selector not found: ' + rpSelector + ']', charCount: 0 });
    break;
  }
  const rpText = FSB.extractPageText(rpRoot, {
    viewportOnly: !rpFull,
    format: 'markdown-lite'
  });
  const rpTime = Date.now() - rpStart;
  logger.logTiming(FSB.sessionId, 'DOM', 'extractPageText', rpTime, {
    selector: rpSelector,
    full: rpFull,
    charCount: rpText.length
  });
  sendResponse({
    success: true,
    text: rpText || '[No readable text content on page]',
    charCount: rpText ? rpText.length : 0
  });
  break;
```
