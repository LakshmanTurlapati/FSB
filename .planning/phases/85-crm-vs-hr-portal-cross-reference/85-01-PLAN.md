**Demo site targets (all auth-free with employee/people data):**

Primary CRM target: DemoQA (demoqa.com/webtables) -- a public practice website with an employee table containing First Name, Last Name, Age, Email, Salary, Department columns. No auth required. Has Add, Edit, Delete, Search functionality. Pagination built in.

Primary HR Portal target: The-Internet Heroku (the-internet.herokuapp.com/tables) -- a public test site with sortable data tables containing Last Name, First Name, Email, Due, Web Site, Action columns. No auth required. Table 1 and Table 2 with different employee records.

Fallback CRM: JSONPlaceholder UI (jsonplaceholder.typicode.com/users) -- public REST API rendered as JSON with 10 users (name, email, phone, company). Can be read via read_page.

Fallback HR Portal: ReqRes (reqres.in/api/users?page=1) -- public API with paginated user data (first_name, last_name, email, avatar). Pages 1-2 with 6 users each.

Additional fallback: Dummy JSON (dummyjson.com/users) -- public API with 30 users containing firstName, lastName, email, company.department.

1. `site`: 'CRM HR Cross-Reference'

   - SKIP-AUTH EXPECTATION: if CRM/HR sites require login, document as skip-auth
   - DEDUPLICATION: track names already cross-referenced to avoid repeating lookups
5. `selectors`: Object with selectors for each target site:

   - herokuapp: { table1: '#table1', table2: '#table2', tableRow: '#table1 tbody tr', lastName: 'td:nth-child(1)', firstName: 'td:nth-child(2)', email: 'td:nth-child(3)', sortHeader: 'th span' }
   - reqres: { userList: 'body', userData: 'pre' (JSON response) }
   - dummyjson: { userList: 'body', userData: 'pre' (JSON response) }

   Step 1: SETUP -- Navigate to CRM site (demoqa.com/webtables). Verify employee table loads (.rt-table visible). Record initial tab as crmTabId via list_tabs.
   Step 2: EXTRACT BATCH FROM CRM -- Read first 10 employee names from the table using read_page on .rt-tr-group elements. Extract firstName and lastName from each row. Store as compact array: [{name: "FirstName LastName", source: "crm"}]. If more than 10 rows visible, record only first 10 for this batch.
   Step 3: OPEN HR PORTAL TAB -- Use open_tab to navigate to the-internet.herokuapp.com/tables. Record as hrTabId. Verify table loads (#table1 visible).
   Step 4: CROSS-REFERENCE BATCH -- For each name in the current batch: search/scan the HR portal table rows for a matching firstName+lastName. Record match result: {name, crmFound: true, hrFound: true/false, match: true/false}.
   Step 5: SWITCH BACK TO CRM -- Use switch_tab to return to crmTabId. Navigate to next page or scroll to next batch of 10 names.
   Step 6: EXTRACT NEXT BATCH -- Read next 10 names from CRM. Append to the names array (avoid duplicates by checking against already-processed names).
   Step 7: SWITCH TO HR PORTAL -- Use switch_tab to hrTabId. Cross-reference the new batch against HR portal data.
   Step 8: REPEAT -- Continue steps 5-7 until 50 names have been extracted and cross-referenced (or all available names exhausted).
   Step 9: COMPILE RESULTS -- After all batches processed, compile final cross-reference report: { totalCrmNames: N, totalHrNames: M, matched: X, crmOnly: Y, hrOnly: Z, mismatched: W }.
   Step 10: VERIFY -- Switch between tabs to spot-check 3 random matches and 1 random mismatch.
   Step 11: CONTEXT TRACKING -- Throughout the workflow, maintain only: { batchNumber: N, namesProcessed: N, matched: N, mismatched: N, currentTab: "crm"|"hr" }. Total tracking under 200 characters per batch summary.
   Step 12: REPORT -- State final cross-reference outcome with counts and any discrepancies found.
