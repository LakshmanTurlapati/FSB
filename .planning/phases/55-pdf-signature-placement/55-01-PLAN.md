**Target Site Selection:**
Use Smallpdf.com as the primary target (https://smallpdf.com/edit-pdf). This is a free, no-auth online PDF editor that allows signature placement. Fallback sites: Sejda.com (https://www.sejda.com/sign-pdf), DocHub (https://dochub.com), iLovePDF (https://www.ilovepdf.com/sign-pdf). Research the interaction model:
- Smallpdf renders PDF pages as image-based views in a scrollable container. Each page is rendered as a div with an image or canvas child.
- The toolbar at the top has buttons for tools: Text, Image, Shape, Draw, and Sign/Signature.
- The Sign/Signature button opens a panel where you can draw, type, or upload a signature image. After creating a signature, clicking on the PDF page places it at the click location.
- Page navigation: pages render vertically in a scrollable container. Scroll down to reach page 3. Some editors also have a page number indicator/input or left sidebar thumbnails.

**Create file:**
Create `site-guides/productivity/pdf-editor.js`.

**Site Guide Structure:**
Create the file following the registerSiteGuide pattern. The site guide MUST include:

1. **Header comment** explaining this covers online PDF editors for editing and signing PDFs. Note that most online PDF editors render pages as images or canvas in a scrollable view with a toolbar overlay. Smallpdf.com is the primary target. Created for Phase 55, CANVAS-09 diagnostic.

2. **patterns** array: URL patterns matching online PDF editor sites:
   - /smallpdf\.com\/(edit-pdf|sign-pdf)/i
   - /smallpdf\.com\/result/i
   - /sejda\.com\/(sign-pdf|edit-pdf)/i
   - /dochub\.com/i
   - /ilovepdf\.com\/(sign|edit)/i
   - /pdfbuddy\.com/i
   - /pdf\.online/i

3. **guidance** string covering these sections:
   - PDF PAGE RENDERING: Online PDF editors render each page as an image (img or canvas) inside a scrollable container. Pages are stacked vertically. The editor toolbar floats above the page area. Page dimensions vary by PDF but typically fill the viewport width with margins. The actual PDF content area is NOT interactive HTML -- it is a rendered image or canvas, so individual text or form fields within the PDF are not clickable DOM elements. All interaction (placing signatures, annotations) happens via the editor's overlay system.
   - PAGE NAVIGATION: To reach page 3, scroll down through the rendered pages. Most editors render all pages in a vertical scroll container. Use scroll_at to scroll down through the page container. Some editors have a page number indicator (e.g., "Page 1 of 5") that can be clicked to jump to a specific page. Some have a left sidebar with page thumbnails. Keyboard shortcuts: Page Down or arrow keys may scroll through pages. CRITICAL: verify you are on page 3 by checking the page indicator or counting visible pages before placing the signature.
   - SIGNATURE TOOL ACTIVATION: The signature/sign tool is typically in the editor toolbar. On Smallpdf: look for a button labeled "Sign" or with a signature icon in the top toolbar. On Sejda: look for "Sign" in the toolbar. Clicking the Sign button opens a signature creation panel. The panel offers options: Draw (freehand), Type (typed text as signature), Upload Image. For simplest automation: use the "Type" option -- type a name, confirm, then click on the PDF page to place. Draw option requires mouse drag strokes which are more complex.
   - SIGNATURE PLACEMENT: After creating/selecting a signature in the Sign panel, the cursor enters placement mode. Click on the PDF page at the desired location to place the signature. The signature appears as a movable overlay at the click point. For the CANVAS-09 test: place signature on page 3 at the approximate location of a dotted line. Dotted signature lines are typically in the lower third of the page (y-position around 70-80% of page height), horizontally centered or left-aligned (x-position around 25-50% of page width).
   - COORDINATE CALCULATION: To place the signature on the dotted line on page 3: (1) scroll to page 3, (2) identify the page element bounds using get_dom_snapshot, (3) calculate the signature line position as approximately x=page_left + page_width*0.35, y=page_top + page_height*0.75 (lower third, left-center), (4) use click_at at those coordinates. Adjust based on actual page content visible in DOM snapshot.
   - SAMPLE PDF: For testing, navigate to smallpdf.com/edit-pdf and use their built-in sample document, or upload a PDF URL. The test needs a multi-page PDF with at least 3 pages. If no suitable sample is available, create a simple flow: navigate to editor, upload a known multi-page PDF URL, or use whatever sample the editor provides.
   - COMMON OBSTACLES: Cookie consent banners (GDPR), upgrade/premium subscription modals, file upload dialogs (may need to select a sample or upload), tutorial/onboarding overlays, browser popup blockers if editor opens in new tab.
   - VERIFICATION: After placing the signature, look for: a new DOM element representing the placed signature (div with class containing "signature", "annotation", "stamp"), visual change in the page area, signature tool state change (from placement mode back to select mode).

   - `pageContainer`: Selectors for the PDF page scroll container: `.pdf-viewer`, `.page-container`, `.document-container`, `[role="document"]`, `.viewer-container`
   - `pageIndicator`: Selectors for page number display: `.page-indicator`, `.page-number`, `[data-page]`, `.page-count`

   - Provide 3-5 fallback selectors per element

5. **workflows** object with these workflows:
   - `loadEditor`: Steps to navigate to the PDF editor URL (https://smallpdf.com/edit-pdf), dismiss cookie/consent popups, dismiss upgrade modals, wait for the editor interface to load (toolbar visible), handle any file upload prompt by either using a provided sample or the editor's default document
   - `navigateToPage3`: Steps to reach page 3 of the loaded PDF. Approach 1: scroll down through the page container using scroll_at (approximately 2 full viewport-height scrolls to pass pages 1 and 2). Approach 2: click page number indicator and type "3". Approach 3: use Page Down key presses. After scrolling, verify page 3 is visible by checking page indicator or DOM state.
   - `activateSignTool`: Steps to activate the signature tool from the toolbar. Click the "Sign" button in the toolbar. Wait for signature panel to appear. Select "Type" tab/option. Type a name (e.g., "John Doe") in the signature input field. Click "Apply" or "Insert" to confirm the signature. The editor should enter placement mode.
   - `placeSignatureOnDottedLine`: Steps to place the signature on the dotted line area of page 3. Get the page 3 element bounds via get_dom_snapshot. Calculate target coordinates: x = page_left + page_width * 0.35, y = visible_page_area_center_y (approximately center of the visible page 3 area, biased toward lower third). Execute click_at at the calculated coordinates. Verify a signature element appeared in the DOM or the page area changed.
   - `fullSignatureWorkflow`: Combined end-to-end workflow: loadEditor -> navigateToPage3 -> activateSignTool -> placeSignatureOnDottedLine. This is the complete CANVAS-09 test sequence.

   - "The signature Sign button location varies by editor -- Smallpdf has it in the top toolbar, Sejda in a sidebar, DocHub in a floating menu"
   - "After activating the signature tool, the editor enters placement mode -- the NEXT click on the PDF page places the signature. Do not click randomly or you will place it in the wrong location."
   - "Page navigation by scrolling requires knowing approximate page height -- each page is typically 800-1200px tall depending on viewport and zoom level"
   - "Some editors require file upload before the editor UI appears -- may need to click a sample document link or upload button first"
