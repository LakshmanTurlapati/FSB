**Step 1: Navigate to the online PDF editor**
Use MCP `navigate` to open https://smallpdf.com/edit-pdf. Wait for page load with `wait_for_stable`. If Smallpdf is inaccessible or requires auth, try fallback: https://www.sejda.com/sign-pdf or https://www.ilovepdf.com/sign-pdf.

**Step 2: Dismiss any overlays**
Use `get_dom_snapshot` to check for cookie consent dialogs, subscription modals, upgrade prompts, or tutorial overlays. Click dismiss/accept/close buttons using DOM `click` if found. Common selectors: `.cookie-accept`, `button:has-text("Accept")`, `.modal-close`, `[aria-label="Close"]`, `.dismiss-btn`. May need multiple rounds of dismissal (GDPR banner, then upgrade modal, then onboarding).

**Step 3: Handle file upload/sample document**
Check if the editor requires uploading a PDF before showing the editor UI. If a file upload prompt appears:
- Look for a "Choose File" or "Upload" button, or a "Try Sample" / demo document link
- If a sample document option exists, click it (simplest path)
- If file upload is required and no sample is available, note this as a blocker in the diagnostic
Wait for the editor to fully load with a document visible.

**Step 4: Verify editor loaded and identify page layout**
Use `get_dom_snapshot` to inspect the editor:
- Look for the toolbar (buttons for Text, Sign, Shape, Draw)
- Look for the page container (scrollable div containing rendered PDF pages)
- Identify how pages are rendered (img tags, canvas elements, or div containers with backgrounds)
- Count visible pages and check if page 1 is currently in view
- Record the page container position and dimensions for scroll calculation

**Step 5: Navigate to page 3**
Using scroll_at to scroll through the page container:
- Get the page container element bounds from DOM snapshot
- Calculate scroll target: approximately 2 page-heights down from current position
- Execute scroll_at at the center of the page container with negative deltaY (scroll down)
- Use multiple scroll_at calls if needed (each scrolling approximately one page height, ~800-1000px)
- After scrolling, use get_dom_snapshot or read_page to verify page 3 is visible (check page indicator, page number element, or count of pages scrolled past)
- If the editor has a page number input/indicator, click it and type "3" as an alternative approach

**Step 6: Activate the signature tool**
Locate the "Sign" or "Signature" button in the toolbar:
- Use get_dom_snapshot to find toolbar buttons
- Click the Sign button using DOM `click` if it matches a known selector, or `click_at` at its coordinates
- Wait for the signature panel/dialog to appear (wait_for_stable or wait_for_element)
- In the signature panel, select "Type" option if available (click the Type tab/button)
- Type a signature name (e.g., "John Doe") in the text input using type_text
- Click "Apply", "Insert", "Done", or equivalent button to confirm the signature
- The editor should now be in placement mode (next click on the page places the signature)

**Step 7: Place the signature on the dotted line area of page 3**
With the editor in placement mode:
- Use get_dom_snapshot to get the current page 3 element bounds
- Calculate the dotted line target coordinates:
  - x = approximately 35% of the page width from the left edge (left-center area where signature lines typically are)
  - y = approximately 70-75% of the visible page height from the top of the page 3 area (lower third where signature lines typically are)
- Execute click_at at the calculated coordinates
- Wait for the signature to be placed (wait_for_stable)

**Step 8: Verify signature was placed**
Use `get_dom_snapshot` to check for:
- A new DOM element representing the placed signature (div with class containing "signature", "annotation", "stamp", "placed")
- Changes in the page area DOM (new child elements in the page container)
- The signature tool state (should return to normal/select mode after placement)
- Any visual indicators that placement succeeded
Use `read_page` to check if "John Doe" or signature text appears in the page content
