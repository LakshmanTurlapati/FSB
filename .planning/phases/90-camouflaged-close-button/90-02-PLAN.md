       Tier 1: Check for aria-label, data-dismiss, data-close, id/class containing "close"

   (c) Record: total candidates found per tier, decoy count, real close button identified (yes/no), detection tier that found it

   (a) For each close button found, classify the camouflage technique:
       - Color matching: close button color matches or nearly matches the background
       - Low opacity: opacity < 0.5 making the element nearly transparent
       - Tiny size: element dimensions < 15px
       - Delayed appearance: close button not in initial DOM, added via JavaScript
       - Position hiding: absolute/fixed position outside visible ad boundary
       - Iframe nesting: close button inside an ad iframe, separate from parent page DOM
   (b) Record: camouflage technique per target, severity assessment

7. FALLBACK STRATEGY VALIDATION:
   (a) For pages where direct close button click would fail, assess fallback viability:
       - Escape key: does the overlay JavaScript listen for keydown/keyup events?
       - Backdrop click: does clicking outside the modal close it (check for backdrop click handlers)?
       - DOM removal: can the overlay container be identified and removed from DOM?
   (b) Record: which fallbacks are viable per target, recommended approach

   - FAIL = could not find any page with overlay/popup and close button elements for analysis, or DOM-based detection failed to identify any close button candidate
