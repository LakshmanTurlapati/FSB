<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP bridge message -> automation/session handlers | Autopilot and observability requests cross from the local MCP bridge into extension internals. |
| Extension memory/session/log state -> MCP client | Read-only data leaves the extension and must remain bounded and filtered. |
| Restricted-page response -> MCP server error mapper | Recovery hints cross from extension structured payloads into user-facing MCP text. |
| Route dispatcher -> task progress notifications | Long-running automation progress flows back through MCP notifications. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-199-03-01 | Spoofing | Autopilot route invocation | mitigate | `mcp:start-automation` is allowlisted and invokes `handleStartAutomation` with `source: 'mcp'`; unknown routes return `mcp_route_unavailable`. |
| T-199-03-02 | Information Disclosure | Logs/sessions/memory routes | mitigate | Filter prompt/raw response logs, cap log count to 200, cap memory text to 500 chars, expose memory stats separately, and do not add vault/payment data routes. |
| T-199-03-03 | Elevation of Privilege | Route fallback bypass | mitigate | Replace Phase 199 `_dispatchToBackground` calls with `dispatchMcpMessageRoute`; missing handlers fail loudly with route family metadata. |
| T-199-03-04 | Denial of Service | Route queue/progress behavior | mitigate | Preserve existing `run_task` queue path and five-minute completion timeout; read-only observability routes remain bounded and do not serialize through mutation fallback. |
| T-199-03-05 | Tampering | Restricted-tab recovery accuracy | mitigate | `validRecoveryTools` is fixed to navigation/tab tools and tests assert `run_task` is absent for blank/new-tab and browser-internal pages. |
</threat_model>
