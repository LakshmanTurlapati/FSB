<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP bridge message -> extension dispatcher | Local MCP requests cross into the extension service-worker route surface. |
| Dispatcher -> Chrome tabs/windows APIs | Route handlers can navigate, create, switch, or inspect tabs. |
| Dispatcher response -> MCP server | Browser/tab metadata leaves the extension and must stay sanitized. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-199-02-01 | Spoofing | `dispatchMcpToolRoute` | mitigate | Strict `MCP_PHASE199_TOOL_ROUTES` allowlist; unknown route returns `mcp_route_unavailable` with `tool` and `routeFamily`. |
| T-199-02-02 | Information Disclosure | Browser/tab responses | mitigate | Return sanitized fields only: `tabId`, `url` when appropriate, `domain`, `title`, `windowId`, `activeTabId`, and counts; never raw Chrome tab objects. |
| T-199-02-03 | Elevation of Privilege | Route fallback | mitigate | Browser/tab tools must not fall back to content-script routing or `chrome.runtime.sendMessage`; unsupported routes fail loudly. |
| T-199-02-04 | Denial of Service | Navigation/history handlers | mitigate | History failures return structured `navigation_unavailable` errors instead of retry loops or queued fallbacks. |
</threat_model>
