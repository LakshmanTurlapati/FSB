**Step 1: Navigate to Google Maps with Central Park Reservoir**
Use MCP `navigate` to open `https://www.google.com/maps/search/Central+Park+Reservoir+NYC`. Wait for the map to load using `wait_for_stable`.

**Step 2: Dismiss any consent/cookie popups**
Use `get_dom_snapshot` to check for:
- Google consent dialog (look for buttons with text "Accept all", "Reject all", or similar)
- Location permission prompts
- Any overlay modals
Click dismiss/accept buttons using `click` if found.

**Step 3: Inspect DOM for map canvas and UI elements**
Use `get_dom_snapshot` to discover:
- The main map canvas element (Google Maps uses a `<canvas>` or a div with `role="presentation"`)
- Zoom controls (+ / - buttons, if visible)
- Search bar and other UI overlay elements
- The map viewport dimensions and center coordinates
- Record all discovered CSS selectors for the site guide

**Step 4: Zoom in to street level using scroll_at**
- Determine the map canvas center coordinates from the DOM snapshot (typically around viewport center, e.g., x=640, y=400 for a standard window)
- Call `scroll_at` with x=center, y=center, deltaY=-120 (zoom in one tick)
- Repeat scroll_at 8-12 times with a `wait_for_stable` between each pair of calls to allow the map tiles to load
- Goal: zoom to street level where the walking path around the reservoir is visible
- After zooming, use `get_dom_snapshot` to verify zoom level changed (check for zoom indicator element or map scale text if present in DOM)

**Step 5: Trace walking path around reservoir with drag segments**
The Central Park Reservoir has a roughly oval shape. Trace its perimeter with 4-6 CDP drag segments:
- Use `get_dom_snapshot` to identify the current map viewport and reservoir position
- Drag segment 1: North side -- drag from left-center to right along the top of the reservoir
  - Example: `drag(startX=400, startY=250, endX=700, endY=250, steps=15, stepDelayMs=30)`
- Drag segment 2: East side -- drag from top-right downward along the east shore
  - Example: `drag(startX=700, startY=250, endX=700, endY=450, steps=15, stepDelayMs=30)`
- Drag segment 3: South side -- drag from right to left along the bottom
  - Example: `drag(startX=700, startY=450, endX=400, endY=450, steps=15, stepDelayMs=30)`
- Drag segment 4: West side -- drag from bottom-left upward to close the loop
  - Example: `drag(startX=400, startY=450, endX=400, endY=250, steps=15, stepDelayMs=30)`

IMPORTANT: On Google Maps, drag pans the map (moves the view), not draws a line. This is expected behavior -- the path tracing demonstrates that drag interactions are functional on the canvas. Each drag will pan the map in the direction of the drag movement.

After each drag, call `wait_for_stable` to allow map tiles to reload.

If Google Maps does not respond to drag, try alternative approaches:
1. Click two points using click_at to place markers (if marker mode available)
2. Use keyboard shortcuts (+/- for zoom) via press_key
3. Document what worked and what did not

**Step 6: Verify interactions worked**
After the zoom and drag sequence, use `get_dom_snapshot` to capture:
- Current map state (URL may update with coordinates)
- Any visual indicators of the zoom level
- Whether the map has panned (URL coordinates changed from initial)
