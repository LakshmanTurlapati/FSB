  <action>
Add five modular readiness check functions AFTER the existing `isElementActionable()` function (around line 1591) and BEFORE `waitForActionable()`. Each function returns a consistent result object: `{ passed: boolean, reason: string|null, details: object }`.

**1. checkElementVisibility(element)**
- Check element is not null/undefined
- Check bounding box has non-zero width AND height via `getBoundingClientRect()`
- Use `element.checkVisibility({ opacityProperty: true, visibilityProperty: true })` if available
- Fallback to `getComputedStyle()` checks for display:none, visibility:hidden, opacity:0
- Return `{ passed: true/false, reason: string|null, details: { rect, display?, visibility?, opacity? } }`

**2. checkElementEnabled(element)**
- Use `element.matches(':disabled')` to catch native disabled AND fieldset-disabled
- Check `element.getAttribute('aria-disabled') === 'true'`
- Check ancestor aria-disabled via `element.closest('[aria-disabled="true"]')`
- Check inert via `element.matches('[inert], [inert] *')`
- Return `{ passed: true/false, reason: string|null, details: { nativeDisabled?, ariaDisabled?, ancestorAriaDisabled?, inert? } }`

**3. checkElementStable(element, maxWaitMs = 300)**
- Get initial position via `getBoundingClientRect()`
- Use `requestAnimationFrame` to check position after one frame
- Compare positions with 1px tolerance for top, left, width, height
- If stable, return immediately; if not, wait up to maxWaitMs
- Return `{ passed: true/false, reason: string|null, details: { position, delta?, checkTime } }`
- NOTE: This is async function

**4. checkElementReceivesEvents(element)**
- Get bounding rect and calculate 5 check points: center + 4 quadrant points (25%/75% positions)
- Filter points to those within viewport bounds
- For each point, use `document.elementFromPoint(x, y)` and check if result is element OR element contains result OR result contains element
- Require at least center point (or 1+ point if center is outside viewport) to pass
- Return `{ passed: true/false, reason: string|null, details: { checkedPoints, passedPoints }, obscuredBy?: string }`

**5. checkElementEditable(element)**
- Check `element.matches(':disabled')` - disabled elements are not editable
- Check `element.readOnly` property
- Check `element.getAttribute('aria-readonly') === 'true'`
- Check contenteditable="false" for contenteditable elements
- Return `{ passed: true/false, reason: string|null, details: { disabled?, readonly?, ariaReadonly?, contenteditable? } }`

Add two more functions after the five check functions from Task 1:

**6. scrollIntoViewIfNeeded(element)** (async)
- Get bounding rect and check if element is fully visible in viewport:
  - `rect.top >= 0 && rect.left >= 0 && rect.bottom <= window.innerHeight && rect.right <= window.innerWidth`
- Also check if center is visible (even if edges are clipped)
- If not fully visible OR center not visible:
  - Call `element.scrollIntoView({ behavior: 'smooth', block: 'center', inline: 'center' })`
  - Wait 300ms for scroll animation
  - Set `scrolled: true`
- Return `{ scrolled: boolean, details: { initialRect, finalRect?, wasFullyVisible, wasCenterVisible } }`

**7. ensureElementReady(element, actionType = 'click')** (async)
- Orchestrator function that calls checks in order:
  1. checkElementVisibility(element) - fail fast if not visible
  2. checkElementEnabled(element) - fail fast if disabled
  3. scrollIntoViewIfNeeded(element) - scroll if needed
  4. checkElementStable(element) - wait for animations
  5. checkElementReceivesEvents(element) - check not obscured
  6. checkElementEditable(element) - ONLY if actionType is in ['type', 'fill', 'clear', 'clearInput', 'selectText']
- Return unified result object:
```javascript
{
  ready: boolean,
  element: element,
  scrolled: boolean,
  checks: {
    visible: { passed, reason, details },
    enabled: { passed, reason, details },
    stable: { passed, reason, details },
    receivesEvents: { passed, reason, details },
    editable?: { passed, reason, details }  // only for input actions
  },
  failureReason: string|null,
  failureDetails: object|null
}
```
- Early return on any failed check with appropriate failureReason
