Execute the SCROLL-08 edge case test using MCP manual tools following the scrollFeedForCatVideo workflow from the tiktok.js site guide. Test steps:

1. Use MCP navigate tool to go to tiktok.com/search?q=cat (primary target -- public search results page, pre-filtered for cat content). Record the URL used and response status.

2. Use read_page to verify the search results page loaded. Check for:
   - Search result video cards visible (div[data-e2e="search-card-container"] or div[data-e2e="recommend-list-item-container"])
   - Video descriptions visible (span[data-e2e="search-card-desc"] or span[data-e2e="new-desc"])
   - Video links visible (a[data-e2e="search-card-link"] or a[href*="/video/"])
   - Cookie consent banner presence (div[id="onetrust-banner-sdk"])
   - Login modal presence (div[data-e2e="modal-close-inner-button"])
   Record: page URL, card count, cookie banner present, login modal present.

3. Dismiss cookie consent banner if present:
   - Click button[id="onetrust-accept-btn-handler"] or button containing "Accept" text

4. Dismiss login modal if present:
   - Click div[data-e2e="modal-close-inner-button"] or the close/X button on the modal

5. READ SEARCH RESULTS (first batch):
   - Use read_page or get_dom_snapshot to capture visible search result cards
   - For each card, extract the description text from searchCardDesc selector
   - Check each description for cat keywords (case-insensitive): cat, kitten, kitty, meow, feline, purr, #cat, #cats, #catsoftiktok
   - Record: number of cards visible, descriptions found, cat keyword matches
   - Since search query is "cat", most results should contain cat-related content

6. IF CAT VIDEO FOUND:
   - Extract its URL from the searchCardLink selector (a[href*="/video/"])
   - Extract its full description text
   - Extract the author name from searchCardUser selector
   - Record: video URL, description, author
   - This constitutes a PASS outcome

7. IF NO CAT VIDEO IN FIRST BATCH (unlikely with "cat" search):
   - Scroll down 800-1000px using scroll(direction="down", amount=800)
   - Wait for new cards to render via wait_for_stable or waitForDOMStable
   - Read new search result cards and check descriptions for cat keywords
   - Repeat for up to 5 scroll cycles

8. IF SEARCH PAGE NOT ACCESSIBLE (auth wall):
   - Try fallback: navigate to tiktok.com/tag/cats
   - Read tag page video cards and check descriptions
   - If tag page also requires auth: outcome is SKIP-AUTH

9. OUTCOME CLASSIFICATION:
   - PASS = navigated to TikTok search/tag page AND found a cat video (description contains cat keyword) AND extracted its URL and description
   - PARTIAL = accessed TikTok but could not find cat content (selectors mismatch, descriptions unreadable), OR WebSocket bridge disconnect blocked execution, OR accessed page but could only verify via HTTP not live MCP
   - FAIL = could not access TikTok at all (page load failed, complete block)
   - SKIP-AUTH = both search page and tag page require authentication

Record every step outcome (tool used, target, result, notes) for the diagnostic report.
