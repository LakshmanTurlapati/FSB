Add three safety mechanisms to ai/agent-loop.js:

**1. checkSafetyBreakers(session) function**
Called at the START of each runAgentIteration, BEFORE the API call. Returns an object: `{ shouldStop: boolean, reason: string|null }`.

Checks in order:
a. **Cost circuit breaker (SAFE-01, D-01):**
   ```javascript
   const costLimit = session.safetyConfig?.costLimit || 2.00;  // Default $2
   if (session.agentState.totalCost >= costLimit) {
     return { shouldStop: true, reason: `Session cost ($${session.agentState.totalCost.toFixed(2)}) exceeded limit ($${costLimit.toFixed(2)}). Stopping to prevent excess spending.` };
   }
   ```

b. **Time limit (SAFE-02, D-02):**
   ```javascript
   const timeLimit = session.safetyConfig?.timeLimit || 10 * 60 * 1000;  // Default 10 minutes
   const elapsed = Date.now() - session.agentState.startTime;
   if (elapsed >= timeLimit) {
     const minutes = Math.floor(elapsed / 60000);
     return { shouldStop: true, reason: `Session duration (${minutes} min) exceeded limit (${Math.floor(timeLimit/60000)} min). Stopping automation.` };
   }
   ```

c. Returns `{ shouldStop: false, reason: null }` if all checks pass.

**2. detectStuck(session, toolResults) function**
Called AFTER tool execution, BEFORE scheduling next iteration. Analyzes whether the AI is stuck.

Logic (per D-03, SAFE-03, P8):
a. Check if ANY tool result had `hadEffect: true` -- if yes, reset `session.agentState.consecutiveNoChangeCount` to 0 and return `{ isStuck: false, hint: null }`.
b. If NO tool had effect, increment `session.agentState.consecutiveNoChangeCount`.
c. If `consecutiveNoChangeCount >= 3`, return:
   ```javascript
   {
     isStuck: true,
     hint: `WARNING: The last ${session.agentState.consecutiveNoChangeCount} tool calls produced no visible change on the page. ` +
       `Previous actions: [${recentActions.join(', ')}]. ` +
       `Suggestions: (1) Call get_dom_snapshot to refresh your view of the page -- elements may have changed. ` +
       `(2) Try scrolling to reveal hidden elements. ` +
       `(3) Try a completely different approach to the task. ` +
       `(4) If the task appears complete, respond with a summary instead of calling more tools.`
   }
   ```
   Where `recentActions` is the last 3-5 entries from session.actionHistory formatted as "toolName(key_param)".
d. If `consecutiveNoChangeCount < 3`, return `{ isStuck: false, hint: null }`.

**3. Integrate into runAgentIteration flow**

Modify runAgentIteration (the main loop body) to add safety checks at two points:

**Point A -- Before API call (top of iteration):**
After the existing guard clause (`if (!session || session.status !== 'running') return`), add:
```javascript
// Safety breaker check (SAFE-01, SAFE-02)
const safety = checkSafetyBreakers(session);
if (safety.shouldStop) {
  session.status = 'stopped';
  await sendSessionStatus(session.tabId, {
    phase: 'ended', reason: 'safety',
    taskName: session.task,
    statusText: safety.reason
  });
  await endSessionOverlays(session, 'safety');
  await persistSession(sessionId, session);
  return;  // Do NOT schedule next iteration
}
```

**Point B -- After tool execution, before scheduling next iteration:**
After the tool results are formatted and pushed to session.messages, add:
```javascript
// Stuck detection (SAFE-03, D-03)
const stuckCheck = detectStuck(session, toolResults);
if (stuckCheck.isStuck && stuckCheck.hint) {
  // Inject recovery hint as an additional message to the AI
  // This goes AFTER the tool results, so the AI sees: tool_results + recovery hint
  session.messages.push({
    role: 'user',
    content: stuckCheck.hint
  });
}
```
Note: For Anthropic, the recovery hint as a user message is fine since tool_results already use role:'user'. For OpenAI/Gemini, a user message after tool results is also valid. The hint does NOT replace any tool_result -- it is an ADDITIONAL message.
