Create the core agent loop engine that replaces startAutomationLoop with a native tool_use protocol loop.

**1. buildSystemPrompt(task, pageUrl)**
Returns a string of ~1-2KB (per D-05, D-06, D-07):
```
You are a browser automation agent. You control a web browser by calling tools.

Current task: {task}
Current page: {pageUrl}

Instructions:
- Call tools to interact with the page. Each tool returns a structured result.
- Call get_dom_snapshot to see page elements before interacting.
- When the task is complete, respond with a text message summarizing what you accomplished. Do NOT call any more tools.
- If you cannot complete the task, explain why in a text message.
- Do not ask the user questions. Execute the task autonomously.
```
No site guides (D-06), no conversation history instructions (D-07).

**2. callProviderWithTools(provider, model, apiKey, messages, tools, providerKey)**
- Creates a UniversalProvider instance: `new UniversalProvider(providerKey, model, apiKey)`
- Builds request body manually (NOT via buildRequest -- that method is for text completions):
  - For OpenAI/xAI/OpenRouter/Custom: `{ model, messages, tools: formatToolsForProvider(tools, providerKey), temperature: 0 }`
  - For Anthropic: `{ model, system: messages[0].content, messages: messages.slice(1), tools: formatToolsForProvider(tools, providerKey), max_tokens: 4096 }`
  - For Gemini: `{ contents: messages.filter(m => m.role !== 'system').map(convertToGeminiFormat), tools: formatToolsForProvider(tools, providerKey), systemInstruction: { parts: [{ text: messages[0].content }] } }`
- Calls `provider.sendRequest(requestBody)`
- Returns the raw API response object (NOT extracted text)
- Includes a Gemini message format converter that translates {role, content} to Gemini's {role, parts} format, handling tool_result content blocks

**3. runAgentLoop(sessionId, { activeSessions, persistSession, sendSessionStatus, broadcastDashboardProgress, endSessionOverlays, startKeepAlive, executeCDPToolDirect, handleDataTool })**
Entry point called from handleStartAutomation. Receives sessionId and an options object with all background.js callbacks (avoids tight coupling).
- Retrieves session from activeSessions.get(sessionId)
- Gets current tab URL via chrome.tabs.get(session.tabId)
- Builds system prompt via buildSystemPrompt(session.task, tabUrl)
- Initializes session.messages = [{ role: 'system', content: systemPrompt }]
- Initializes session.agentState = { iterationCount: 0, totalInputTokens: 0, totalOutputTokens: 0, totalCost: 0, startTime: Date.now(), lastDOMHash: null, consecutiveNoChangeCount: 0 }
- Gets public tools via getPublicTools()
- Gets provider config from chrome.storage.sync (settings.provider, settings.model, settings.apiKey)
- Calls runAgentIteration(sessionId, options) to start the first iteration

**4. runAgentIteration(sessionId, options)**
One iteration of the agent loop. Structured as a single async function that completes and schedules the next iteration via setTimeout (per D-08, per P4 prevention).

Step-by-step:
a. Retrieve session from options.activeSessions.get(sessionId)
b. Guard: if (!session || session.status !== 'running') return -- session was stopped or cleaned up
c. Increment session.agentState.iterationCount
d. Call sendSessionStatus with phase:'analyzing', iteration count, task name
e. Call broadcastDashboardProgress(session)
f. Get provider settings from session (cached at loop start into session.providerConfig = { provider, model, apiKey, providerKey })
g. Call callProviderWithTools(provider, model, apiKey, session.messages, session.tools, providerKey)
h. Extract usage via extractUsage(response, providerKey) -- accumulate into session.agentState.totalInputTokens/totalOutputTokens, compute cost using the pricing lookup (copy the calculateCost logic from background.js lines 4181-4207 into a local helper in agent-loop.js)
i. Check isToolCallResponse(response, providerKey):
   - **If FALSE (end_turn):** The AI is done. Extract final text from the response. Call options.endSessionOverlays(session, 'complete'). Set session.status = 'completed'. Send final status. Persist. Log. Return (do NOT schedule next iteration). Per D-04, LOOP-03 -- no fixed iteration cap, AI decides.
   - **If TRUE (tool_use):** Continue to step j.
j. Push formatAssistantMessage(response, providerKey) to session.messages
k. Parse tool calls via parseToolCalls(response, providerKey) -- returns [{id, name, args}]
l. Execute each tool call SEQUENTIALLY (per P12 -- browser actions must be serial):
   ```javascript
   const toolResults = [];
   for (const call of toolCalls) {
     const result = await executeTool(call.name, call.args, session.tabId, {
       cdpHandler: options.executeCDPToolDirect,
       dataHandler: options.handleDataTool
     });
     toolResults.push({ callId: call.id, name: call.name, result });
     // Update action history for progress tracking
     session.actionHistory.push({
       tool: call.name, params: call.args,
       result: { success: result.success, hadEffect: result.hadEffect },
       timestamp: Date.now(), iteration: session.agentState.iterationCount
     });
     // Send per-tool progress update
     sendSessionStatus(session.tabId, {
       phase: 'executing', taskName: session.task,
       statusText: `${call.name}${call.args?.selector ? ' ' + call.args.selector : ''}`,
       iteration: session.agentState.iterationCount
     });
   }
   ```
m. Format all tool results into messages and push to session.messages:
   ```javascript
   for (const tr of toolResults) {
     const resultMsg = formatToolResult(tr.callId, JSON.stringify(tr.result), providerKey);
     session.messages.push(resultMsg);
   }
   ```
n. Persist session state to chrome.storage.session (per SAFE-04, D-09):
   ```javascript
   await persistSession(sessionId, session);
   ```
   The existing persistSession only saves essential fields. Extend the persisted data to include session.agentState and a compact version of session.messages (last 5 messages only -- full history stays in memory for the active loop, but if SW dies and resurrects, the last 5 messages provide enough context for the stop button to work).
o. Schedule next iteration (per D-08, P4):
   ```javascript
   setTimeout(() => runAgentIteration(sessionId, options), 100);
   ```
   Use 100ms delay -- fast enough for responsive automation, long enough to yield the event loop and reset Chrome's execution timer.

**Error handling in callProviderWithTools:**
Wrap the API call in try/catch. On error:
- If status 401/403: set session.status = 'error', send error status "API key invalid or expired", return without scheduling next iteration
- If status 429 (rate limit): the UniversalProvider already handles retries internally, so this should be rare. If it reaches here, wait 5s and retry once.
- If network error / timeout: retry once after 2s. If second attempt fails, set session.status = 'error' with message.
- On any terminal error, call endSessionOverlays(session, 'error') and persistSession.
