1. Use MCP navigate tool to go to a public online voice recorder. Try in order:
   a. onlinevoicerecorder.com (primary target)
   b. Fallback: vocaroo.com
   c. Fallback: rev.com/voice-recorder
2. Use read_page to check for cookie/consent popups and dismiss them via click
3. Use get_dom_snapshot to locate the record button -- look for:
   - button elements with aria-label containing "record"
   - Elements matching site guide recordButton selectors
   - Large circular elements in the center of the page
4. Read the record button bounding rect (position, width, height) from the DOM snapshot
5. Calculate button center: centerX = left + width/2, centerY = top + height/2
6. Attempt Method 1 -- click_and_hold(centerX, centerY, 5000):
   - This should mousePressed, hold 5 seconds, then mouseReleased
   - Note: microphone permission may block recording start
7. After release, use get_dom_snapshot to check for recording indicators:
   - Timer display showing elapsed time
   - Playback controls (play button, audio element)
   - Download/save button
   - Changed button state (record -> stop -> done)
8. If click_and_hold did not produce a recording, attempt Method 2 -- toggleToRecord:
   - click_at(centerX, centerY) to start recording
   - Wait (MCP caller pauses ~5 seconds between tool calls)
   - click_at(centerX, centerY) again to stop, or click a separate stop button
9. Final verification: check for audio playback controls or timer >= 4 seconds

Record every step outcome (tool used, target, result, notes) for the diagnostic report.
