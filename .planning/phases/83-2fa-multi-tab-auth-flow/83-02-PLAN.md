1. IDENTIFY 2FA TARGET: Look for a demo site with working 2FA login flow.
   (a) Navigate to https://practiceautomation.com -- check for 2FA demo page
   (b) If no 2FA demo: try https://the-internet.herokuapp.com/login (Heroku test app)
   (c) If no 2FA: try https://demoqa.com/login
   (d) If no 2FA on any target: fall back to simulateMultiTabCodeFetch pattern
   (e) Last resort: Navigate to any login page + open disposable email in new tab to demonstrate multi-tab orchestration pattern
   Record: target URL, page loaded, 2FA available (yes/no), login form present

2. CAPTURE AUTH TAB ID:
   - Use list_tabs to get the current tab ID
   - Store as authTabId
   Record: authTabId value, tab URL confirmed

3. ENTER CREDENTIALS (if login form available):
   - Type username/email into usernameInput field
   - Type password into passwordInput field
   - If demo site: use demo credentials (e.g., standard_user/secret_sauce for saucedemo, tomsmith/SuperSecretPassword! for herokuapp)
   Record: credentials entered (yes/no), which selectors matched

4. SUBMIT LOGIN:
   - Click loginButton to submit credentials
   - Wait for page response
   Record: login submitted, response observed (2FA prompt / direct login / error)

5. DETECT 2FA PROMPT:
   - Check if a 2FA code input appeared (codeInput or splitDigitInput or twoFactorPrompt selector)
   - If 2FA prompt present: proceed to step 6
   - If no 2FA prompt (direct login or error): note as "no 2FA on this target"
   Record: 2FA prompt detected (yes/no), selector matched, prompt type

6. OPEN EMAIL TAB:
   - If real 2FA triggered: open_tab to email provider (Gmail, Outlook, or disposable email)
   - If simulating: open_tab to https://www.guerrillamail.com or https://www.mailinator.com
   - Record the new tab ID as emailTabId
   Record: email tab opened, emailTabId value, URL loaded

7. SEARCH FOR CODE EMAIL (or read disposable inbox):
   - On the email tab: search for verification code email or read the inbox
   - For disposable email: check inbox for latest email
   - For Gmail/Outlook: search "verification code" or "login code"
   Record: email found (yes/no), email subject/sender if found

8. EXTRACT CODE:
   - Read the email body using read_page
   - Look for a 4-8 digit number (the verification code)
   - Store ONLY the code string
   - If simulating (no real 2FA email): note that no code was available
   Record: code extracted (yes/no), code value (or "N/A" if simulating)

9. SWITCH BACK TO AUTH TAB:
   - switch_tab(authTabId) to return to the original authentication tab
   - Verify the page is still the auth/2FA page
   Record: switch_tab result, correct tab confirmed (yes/no)

10. ENTER CODE AND VERIFY (if code was extracted):
    - Type code into codeInput field
    - Click verifyButton
    - Check for loginSuccess indicators
    Record: code entered (yes/no), login succeeded (yes/no)

11. OUTCOME CLASSIFICATION:
    - PASS = 2FA code triggered, fetched from email tab, entered on auth tab, login completed
    - PARTIAL = multi-tab pattern demonstrated (open_tab, switch_tab, read_page across tabs worked) but 2FA not fully completed (no demo site has real 2FA, or email code not deliverable, or WebSocket bridge disconnect)
    - FAIL = could not demonstrate multi-tab orchestration pattern at all
    - SKIP-AUTH = no demo 2FA site available and multi-tab simulation could not execute
