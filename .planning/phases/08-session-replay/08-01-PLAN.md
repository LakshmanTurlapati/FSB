**Part B: Add replay engine to background.js**

1. Add `getReplayDelay(tool)` function near the other utility functions (after `slimActionResult` around line 1625):
```javascript
function getReplayDelay(tool) {
  if (['navigate', 'searchGoogle', 'goBack', 'goForward'].includes(tool)) return 1500;
  if (['click', 'doubleClick', 'rightClick'].includes(tool)) return 500;
  if (['type', 'keyPress', 'pressEnter'].includes(tool)) return 300;
  return 200;
}
```

2. Add `loadReplayableSession(sessionId)` async function:
- Load session from chrome.storage.local fsbSessionLogs
- Return null if session not found or has no actionHistory
- Define replayable tools set: click, rightClick, doubleClick, type, clearInput, pressEnter, keyPress, selectOption, toggleCheckbox, navigate, searchGoogle, scroll, goBack, goForward, refresh, hover, focus, moveMouse, waitForElement
- Filter actionHistory to entries where result.success === true AND tool is in replayable set
- Return { session, replayableActions, originalTask: session.task, originalUrl: session.logs?.[0]?.data?.url || null }

3. Add `executeReplaySequence(replaySessionId)` async function:
- Get session from activeSessions, return if not found or status !== 'replaying'
- Loop through session.replaySteps from session.currentStep to end
- For each step:
  a. Check isSessionTerminating(replaySessionId), return if true
  b. Update session.currentStep = i
  c. If step.tool === 'type', prepend a clearInput action first targeting the same selector (step.params.selector). Execute via sendMessageWithRetry with {action: 'executeAction', tool: 'clearInput', params: {selector: step.params.selector}}. Brief 200ms delay after.
  d. Send statusUpdate with: action 'statusUpdate', sessionId, message from getActionStatus(step.tool, step.params), iteration i+1, maxIterations session.totalSteps, progressPercent Math.round(((i+1)/session.totalSteps)*100)
  e. Execute action via sendMessageWithRetry(session.tabId, {action: 'executeAction', tool: step.tool, params: step.params, visualContext: {taskName: session.task, stepNumber: i+1, totalSteps: session.totalSteps, iterationCount: 1, isReplay: true}})
  f. Record result in session.actionHistory: {timestamp: Date.now(), tool: step.tool, params: step.params, result: slimActionResult(actionResult), replayStep: i+1}
  g. If !actionResult?.success: check if tool is in criticalTools set ['navigate', 'searchGoogle']. If critical, set session.status = 'replay_failed', break. If non-critical, log warning and continue.
  h. Inter-action delay using getReplayDelay(step.tool) -- skip if last step
- After loop: if session.status is still 'replaying', set to 'replay_completed'
- Count successful and failed steps from session.actionHistory
- Send automationComplete: { action: 'automationComplete', sessionId: replaySessionId, result: `Replay complete: ${successCount}/${session.totalSteps} steps executed successfully.${failedCount > 0 ? ` ${failedCount} steps skipped.` : ''}` }
- Send session status ended to content script via sendSessionStatus
- Log session end and cleanup: automationLogger.logSessionEnd, cleanupSession(replaySessionId)
- If replay_failed: send automationError instead with appropriate message

4. Add `handleReplaySession(request, sender, sendResponse)` async function:
- Extract sessionId from request.sessionId
- Check if automation is already running: if activeSessions.size > 0 for any non-idle session, respond with error "Another automation is already running"
- Get current active tab: chrome.tabs.query({active: true, currentWindow: true})
- Call loadReplayableSession(sessionId)
- If null or no replayable actions, respond with error "No replayable actions found in this session"
- Create replay session ID: `replay_${Date.now()}_${Math.random().toString(36).slice(2,6)}`
- Create session data object in activeSessions Map with isReplay: true flag and all required fields (see research Pattern 1, but use a minimal set -- no AI-related fields):
  - task: `Replay: ${replayData.originalTask}`, tabId, status: 'replaying', startTime, actionHistory: [], isReplay: true, originalSessionId: sessionId, replaySteps: replayData.replayableActions, currentStep: 0, totalSteps: replayData.replayableActions.length
- Start keep-alive: startKeepAlive()
- Log session start
- Send response: { success: true, sessionId: replaySessionId, totalSteps: replayData.replayableActions.length }
- Execute executeReplaySequence(replaySessionId) -- do NOT await (run async, response already sent)

5. Add message handler case in the switch statement (around line 3189, after existing cases):
```javascript
case 'replaySession':
  handleReplaySession(request, sender, sendResponse);
  return true;
```
