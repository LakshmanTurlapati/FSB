   - The dropzone element (look for .dropzone, .dz-clickable, elements with "drag" or "drop" text)

   - Upload area boundaries and visual indicators

   - A file preview element (.dz-preview) appeared
   - The dropzone state changed from empty to has-file
   - Any success indicator (.dz-success) is present

6. If Strategy A FAILED (no visual change after drop_file):
   a. Try clicking the dropzone element first (click(".dropzone") or click(".dz-clickable")) to activate it, then retry drop_file.
   b. If retry also fails, look for hidden input[type="file"] near the dropzone and document as fallback.
