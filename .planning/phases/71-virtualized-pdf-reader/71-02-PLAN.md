Execute the SCROLL-05 edge case test using MCP manual tools following the readVirtualizedDocument workflow from the pdf-viewer.js site guide. Test steps:

1. Use MCP navigate tool to go to https://mozilla.github.io/pdf.js/web/viewer.html (official pdf.js demo viewer with built-in sample PDF). If this URL is not accessible, fall back to any other publicly hosted pdf.js viewer with a multi-page document.

2. Use read_page to verify the viewer loaded. Check for:
   - The viewer toolbar is visible (page number input, zoom controls)
   - The first page (div.page[data-page-number="1"]) is rendered
   - The textLayer on page 1 has span children with text content
   - Total page count from the numPages indicator
   Record: page title, URL, total pages, whether page 1 textLayer has content.

3. EXTRACT PAGE 1 TEXT:
   - Use get_dom_snapshot or read_page to find all span elements inside .page[data-page-number="1"] .textLayer
   - Concatenate the span text content to get page 1's text
   - Record: first 200 characters of page 1 text as "page1_sample"
   - Verify the text is non-empty and contains recognizable words

4. NAVIGATE TO PAGE 3:
   - Click on the page number input (#pageNumber or equivalent)
   - Clear the input and type "3"
   - Press Enter to navigate to page 3
   - Wait 500-1000ms for page 3 to render
   - Verify page 3 is now visible (check page indicator shows 3, or check .page[data-page-number="3"] is in viewport)

5. EXTRACT PAGE 3 TEXT:
   - Use get_dom_snapshot to find textLayer spans on page 3
   - Concatenate text content
   - Record: first 200 characters of page 3 text as "page3_sample"
   - Verify page 3 text differs from page 1 text (different content)

6. NAVIGATE TO PAGE 5 (or later):
   - Use page input to jump to page 5
   - Wait for render
   - Extract page 5 text and store as "page5_sample"

7. CHECK PAGE 1 VIRTUALIZATION:
   - Use get_dom_snapshot to inspect .page[data-page-number="1"] .textLayer
   - Check if textLayer spans are empty or absent (indicating page 1 was virtualized/unloaded)
   - Record: "Page 1 textLayer status after navigating to page 5: {loaded/unloaded/partially_loaded}"
   - This is the KEY test: does the viewer virtualize earlier pages?

8. SCROLL BACK TO PAGE 1:
   - Use page input to jump back to page 1 (type "1", press Enter)
   - Wait 500-1000ms for page 1 to re-render
   - Use get_dom_snapshot to verify textLayer has repopulated

9. RE-READ PAGE 1 TEXT:
   - Extract textLayer spans from page 1 again
   - Record: first 200 characters as "page1_reread"
   - Compare with original "page1_sample": do they match?
   - If match: virtualization round-trip successful
   - If different: document the discrepancy

10. OUTCOME CLASSIFICATION:
    - PASS = navigated to multiple pages + extracted text from textLayer + confirmed page virtualization (unloading) + re-read matched original
    - PARTIAL = navigated and read text but virtualization not confirmed (pages stayed loaded) OR re-read did not match
    - PARTIAL = text extraction worked but page navigation via input failed (used scroll fallback)
    - FAIL = could not load pdf.js viewer or extract text from any page
    - Note: pdf.js demo viewer is a public site with no auth requirement

11. VERIFY:
    - PDF viewer was loaded and pages were rendered
    - Text was extracted from at least 2 different pages with different content
    - Page navigation (via page input or scroll) worked
    - Virtualization behavior was observed or documented as absent
    - Re-reading a previously visited page returned consistent text

Record every step outcome (tool used, target, result, notes) for the diagnostic report.
