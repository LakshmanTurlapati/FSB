**Step 1: Navigate to product page**
Use MCP `navigate` to open the Nike product page URL discovered in Plan 01 (from the site guide or SUMMARY). If Nike was not usable in Plan 01, navigate to the Sketchfab fallback URL from the site guide. Wait for page load with `wait_for_stable`.

**Step 2: Dismiss cookie consent and popups**
Use `get_dom_snapshot` to check for:
- Cookie consent banner (Nike uses OneTrust-style banner)
- Country/region selection modal
- Any overlay blocking the product page content
Click dismiss/accept buttons as needed using `click` or `click_at`. Wait for stable after dismissal.

**Step 3: Activate 3D view**
Following the site guide's activate3DView workflow:
- Use `get_dom_snapshot` to find the 3D/360 view trigger button in the product image area
- Use `click_at` on the 3D button coordinates to activate the viewer
- Wait for the WebGL viewer to load (1-3 seconds) with `wait_for_stable`
- Use `get_dom_snapshot` again to confirm the canvas/model-viewer element appeared

If no 3D button exists on the page:
- Try a different Nike product page URL
- Or switch to Sketchfab fallback where the 3D viewer loads automatically

If on Sketchfab:
- The viewer loads automatically, skip the activation step
- Just confirm the canvas/viewer is present in DOM

**Step 4: Calculate drag coordinates for 180-degree rotation**
- Use `get_dom_snapshot` to find the canvas/model-viewer element and its bounding box
- Calculate coordinates for a half-width horizontal drag:
  - centerY = element.top + element.height / 2
  - startX = element.left + element.width * 0.25
  - endX = element.left + element.width * 0.75
- These coordinates produce a drag covering 50% of the canvas width, which should rotate approximately 180 degrees

**Step 5: Execute 180-degree rotation drag**
- Use `drag(startX, centerY, endX, centerY, steps=30, stepDelayMs=20)` for smooth rotation
- The drag should be purely horizontal (same Y for start and end) to avoid vertical tilt
- Wait for rendering to settle with `wait_for_stable`

**Step 6: Verify rotation occurred**
- Use `get_dom_snapshot` to check if the viewer is still active and responsive
- There is no DOM-based way to read the exact rotation angle (it is GPU-rendered)
- Note the before/after state for the human checkpoint to verify visually
- If `read_page` shows different text (e.g., tooltip changed), record it

**Step 7: Record all results**
For each step, record: MCP tool used, target element/coordinates, result (OK/FAIL), and notes about what happened.
