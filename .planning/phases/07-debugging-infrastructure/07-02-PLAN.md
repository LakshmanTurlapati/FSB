  <files>content.js</files>
  <action>
Add ElementInspector class in content.js AFTER the ActionRecorder class (or if Plan 01 hasn't run yet, after the tools object definition).

ElementInspector class structure:

constructor():
- this.isActive = false
- this.hoverOverlay = null (element showing bounds of hovered element)
- this.inspectionPanel = null (element showing detailed info)
- this.currentElement = null
- Bind event handlers: this.handleMouseMove = this.handleMouseMove.bind(this), this.handleClick = this.handleClick.bind(this)

enable():
- If already active, return
- Set isActive = true
- Call createHoverOverlay()
- Call createInspectionPanel() (hidden initially)
- Add document event listeners with capture: true
  - 'mousemove' -> handleMouseMove
  - 'click' -> handleClick
- Add visual indicator (e.g., border on body or floating badge)

disable():
- If not active, return
- Set isActive = false
- Remove event listeners (same handlers)
- Remove hoverOverlay from DOM
- Remove inspectionPanel from DOM
- Remove visual indicator
- Set currentElement = null

createHoverOverlay():
- Create div with Shadow DOM for style isolation (same pattern as HighlightManager)
- Position: fixed, pointer-events: none
- Style: 2px dashed orange border, light orange background (rgba(255,165,0,0.1))
- z-index: 2147483645 (below FSB highlights)
- Initially hidden (display: none)
- Append to document.body

createInspectionPanel():
- Create div with Shadow DOM for style isolation
- Position: fixed, bottom: 20px, right: 20px
- Width: 350px, max-height: 400px, overflow: auto
- Background: white, border-radius: 8px, box-shadow
- z-index: 2147483647 (top layer)
- Initially hidden
- Append to document.body

handleMouseMove(e):
- If not active, return
- Get element: document.elementFromPoint(e.clientX, e.clientY)
- Skip if element is our overlay or panel
- If element !== currentElement:
  - currentElement = element
  - Update hoverOverlay position and size to match element.getBoundingClientRect()
  - Show hoverOverlay

handleClick(e):
- If not active, return
- e.preventDefault()
- e.stopPropagation()
- Get element: document.elementFromPoint(e.clientX, e.clientY)
- Skip if element is our overlay or panel
- Call showInspectionPanel(element)

Create singleton: const elementInspector = new ElementInspector();
  </action>

<task type="auto">
  <name>Task 2: Add getElementInspection and showInspectionPanel methods</name>
  <files>content.js</files>
  <action>
Add these methods to the ElementInspector class:

getElementInspection(element):
- If !element, return null
- Call generateSelectors(element) to get selector array (existing function from Phase 1)
- Get rect = element.getBoundingClientRect()
- Get style = window.getComputedStyle(element)
- Return object:
  {
    tagName: element.tagName,
    id: element.id,
    className: element.className,
    selectors: selectors (array of selector strategies FSB would try),
    preferredSelector: selectors[0] (highest scored),
    attributes: {
      'data-testid': element.getAttribute('data-testid'),
      'aria-label': element.getAttribute('aria-label'),
      'role': element.getAttribute('role'),
      'type': element.type,
      'name': element.name,
      'href': element.href,
      'value': element.value?.substring(0, 50)
    },
    interactability: {
      isVisible: style.display !== 'none' && style.visibility !== 'hidden' && parseFloat(style.opacity) > 0,
      isEnabled: !element.disabled,
      isInViewport: rect.top >= 0 && rect.left >= 0 && rect.bottom <= window.innerHeight && rect.right <= window.innerWidth,
      receivesPointerEvents: style.pointerEvents !== 'none'
    },
    boundingRect: {
      x: Math.round(rect.x),
      y: Math.round(rect.y),
      width: Math.round(rect.width),
      height: Math.round(rect.height)
    },
    text: element.innerText?.substring(0, 100)
  }

showInspectionPanel(element):
- Get inspection = getElementInspection(element)
- If !inspection, hide panel and return
- Build HTML content for panel (use Shadow DOM's inner content):
  - Header: Tag name with ID/class badges
  - Section: "Selectors FSB Would Try" - list selectors with index numbers
  - Section: "Attributes" - key-value pairs (skip null values)
  - Section: "Interactability" - green check or red X for each boolean
  - Section: "Position" - x, y, width, height
  - Close button at top right
- Update inspectionPanel's shadow DOM content
- Show panel (display: block)
- Add click handler to close button to hide panel

Style the panel content inside Shadow DOM:
- Font: system-ui, 12px base
- Sections with headers in bold
- Monospace for selectors
- Color coding: green for passing checks, red for failing
  </action>

<task type="auto">
  <name>Task 3: Add message handler for inspection mode toggle</name>
  <files>content.js</files>
  <action>
Add a message handler in the existing chrome.runtime.onMessage listener (in handleMessage function or similar) to toggle inspection mode:

Handle action: 'toggleInspectionMode':
- If elementInspector.isActive, call elementInspector.disable()
- Else call elementInspector.enable()
- Return { success: true, active: elementInspector.isActive }

Handle action: 'getInspectionModeStatus':
- Return { active: elementInspector.isActive }

This allows the popup/sidepanel to toggle inspection mode via:
chrome.tabs.sendMessage(tabId, { action: 'toggleInspectionMode' })

Also add keyboard shortcut listener for Ctrl+Shift+I:
- In content.js initialization (at the bottom), add:
  document.addEventListener('keydown', (e) => {
    if (e.ctrlKey && e.shiftKey && e.key === 'I') {
      e.preventDefault();
      if (elementInspector.isActive) {
        elementInspector.disable();
      } else {
        elementInspector.enable();
      }
    }
  });

Note: Ctrl+Shift+I conflicts with Chrome DevTools, so use Ctrl+Shift+E instead (E for Element inspector).
  </action>
