## Accomplishments

- `background.js` now caches the last canonical overlay status, identity, and target tab so a reconnecting content script can be rehydrated immediately.
- The content-script `ready` path replays the cached overlay state for the active running session and requests a matching DOM-stream overlay refresh for the dashboard when streaming is active.

- `ai/agent-loop.js` now refreshes canonical overlay state before provider waits, during `report_progress`, and after iterations so overlays stay current during ordinary execution.
