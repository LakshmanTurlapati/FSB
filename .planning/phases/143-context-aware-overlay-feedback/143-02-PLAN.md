# Phase 143 Plan 02: Overlay Rendering & Stream Parity

## Objective

Carry the new target-aware overlay behavior through both the live page highlight and the dashboard DOM stream so the rendered geometry stays consistent across both surfaces.

## Tasks

1. Render text-style highlights and box-style highlights through the live action glow overlay.
2. Expose the computed overlay geometry to the DOM stream layer instead of mirroring raw element bounds.
3. Preserve cleanup and tracking behavior so highlights disappear correctly and continue to follow moving targets.

## Acceptance Criteria

- Live page overlays render the new text-first treatment for compact links and inline text.
- The dashboard DOM stream mirrors the computed overlay geometry instead of the original element rectangle.
- Overlay cleanup and movement tracking still work during execution and teardown.
