# Phase 143 Plan 01: Overlay Target Classification

## Objective

Replace the one-size-fits-all overlay targeting logic with target-aware rules so compact links and inline text use text-style emphasis while controls and larger surfaces continue to use fitted box highlights.

## Tasks

1. Add target classification logic for compact links, inline text, and larger controls.
2. Compute text-fragment geometry from client rects for text-first highlights.
3. Keep fitted box geometry for buttons, inputs, and larger interaction surfaces.

## Acceptance Criteria

- Compact links and inline text no longer receive oversized rectangular overlays.
- Inputs, buttons, and larger controls still receive clear visible targeting.
- The target-selection rules are explicit in code rather than implicit side effects of element bounds.
