Research-based content to include:

FILE HEADER COMMENT:
Document that HTML5 video players use custom slider controls (divs, spans, or input[type=range]) instead of native browser controls. Two interaction approaches: (1) click_at on the slider track at the calculated percentage position, (2) drag the slider thumb from current position to target position. Primary targets: Vimeo, Dailymotion, JW Player demo pages. These are free, no-auth, publicly accessible.

SITE CONFIG:
- site: 'HTML5 Video Player'
- category: 'Media'
- patterns: [/vimeo\.com/i, /dailymotion\.com/i, /jwplayer\.com\/developers\/tools\/player-demos/i, /plyr\.io/i, /videojs\.com/i]

GUIDANCE string covering:

1. VOLUME SLIDER ANATOMY: Custom HTML5 video players render volume as a horizontal or vertical slider track with a draggable thumb. The track is typically a div/span with a progress fill child showing current volume. The thumb (handle/knob) sits at the current volume position. Key attributes: aria-valuenow, aria-valuemin (0), aria-valuemax (100 or 1), data-value, style width percentage.

2. SLIDER SELECTORS BY PLATFORM:
   - Vimeo: .vp-volume-control, .vp-slider, [aria-label*="Volume"], input[type="range"][aria-label*="Volume"]. Vimeo uses an input[type="range"] for volume -- can set value directly via DOM if available, or click_at on the track.
   - Dailymotion: .dmp_VolumeSlider, [aria-label*="volume" i], .dmp_ControlBar [class*="volume"]. Dailymotion uses a custom div-based slider.
   - JW Player: .jw-slider-volume, .jw-slider-container, .jw-rail, .jw-knob, [aria-label*="Volume"]. JW Player uses a custom div slider with jw-slider-volume container, jw-rail track, and jw-knob thumb.
   - Plyr: .plyr__volume input[type="range"], [data-plyr="volume"], .plyr__slider. Plyr uses an input[type="range"] for volume.
   - Video.js: .vjs-volume-bar, .vjs-volume-level, .vjs-slider, [role="slider"][aria-label*="Volume"]. Video.js uses role="slider" with aria-valuenow.
   - Generic fallback: input[type="range"][aria-label*="volume" i], [role="slider"][aria-label*="volume" i], [class*="volume"][class*="slider"], [class*="volume-bar"], [class*="volume-control"]

3. PERCENTAGE CALCULATION FORMULA: For a horizontal slider track at 37%:
   - Get track bounding rect via get_dom_snapshot or read_page
   - targetX = track_left + (0.37 * track_width)
   - targetY = track_top + (track_height / 2)
   - For vertical sliders (some players): targetY = track_bottom - (0.37 * track_height), targetX = track_left + (track_width / 2)

4. METHOD 1 -- CLICK ON TRACK (preferred for precision):
   - Use click_at(targetX, targetY) directly on the slider track at the 37% position
   - Most custom players listen for click events on the track and set volume to the click position
   - This is a single action and most reliable for setting an exact value

5. METHOD 2 -- DRAG THUMB (fallback):
   - First locate the thumb element position (current volume position on track)
   - Use drag(thumbX, thumbY, targetX, targetY, steps=10, stepDelayMs=20) to slide the thumb to 37%
   - Some players only respond to drag events on the thumb, not click on track
   - Use 10 intermediate steps with 20ms delay for smooth slider movement

6. VERIFICATION: After setting volume, verify the value using:
   - get_attribute on the slider element for aria-valuenow (should be approximately 37 or 0.37)
   - get_attribute for aria-valuetext (may show "37%")
   - Read the slider fill width via style attribute (should be approximately "37%" or "width: 37%")
   - Some players show volume tooltip text -- read_page may capture it

7. VOLUME BUTTON AND MUTE: Most players have a speaker icon button next to the volume slider. Clicking it toggles mute (volume 0). Make sure to NOT click the mute button -- only interact with the slider track/thumb. Selectors: [aria-label*="Mute"], [aria-label*="Volume"], button[class*="mute"], button[class*="volume"].

8. PLAYER ACTIVATION: Some players hide volume controls until the user hovers over the player or control bar. Use hover on the player container or control bar first. YouTube requires clicking in the player area first. Vimeo shows controls on hover.

9. COOKIE/CONSENT POPUPS: Dismiss cookie banners before interacting. Common selectors: #onetrust-accept-btn-handler, [class*="cookie"] button[class*="accept"], [class*="consent"] button.

SELECTORS object:
- volumeSlider: '[role="slider"][aria-label*="Volume" i], input[type="range"][aria-label*="volume" i], .vp-slider, .jw-slider-volume, .vjs-volume-bar, [data-plyr="volume"], [class*="volume"][class*="slider"], [class*="volume-bar"]'
- volumeThumb: '.jw-knob, .vjs-volume-handle, [class*="volume"][class*="thumb"], [class*="volume"][class*="knob"], [class*="slider-thumb"]'
- volumeTrack: '.jw-rail, .vjs-volume-level, [class*="volume"][class*="track"], [class*="volume"][class*="rail"], [class*="slider-track"]'
- volumeFill: '.jw-progress, .vjs-volume-level, [class*="volume"][class*="fill"], [class*="volume"][class*="progress"], [class*="volume"][class*="level"]'
- muteButton: '[aria-label*="Mute" i], [aria-label*="Unmute" i], button[class*="mute"], button[class*="volume-button"], .jw-icon-volume, .vjs-mute-control'
- playerContainer: 'video, .video-player, .jw-wrapper, .vjs-tech, .plyr, [class*="player-container"]'
- controlBar: '.jw-controlbar, .vjs-control-bar, .plyr__controls, [class*="control-bar"], [class*="controls"]'
- cookieConsent: '#onetrust-accept-btn-handler, [class*="cookie"] button[class*="accept"], [class*="consent"] button, [id*="cookie"] button'

WORKFLOWS object:
- adjustVolumeByClick: 6 steps -- navigate to video page, hover over player to reveal controls, locate volume slider track via get_dom_snapshot, calculate targetX = track_left + (0.37 * track_width) and targetY = track_top + (track_height / 2), click_at(targetX, targetY) on the track, verify via get_attribute for aria-valuenow approximately 37
- adjustVolumeByDrag: 7 steps -- navigate to video page, hover over player to reveal controls, locate volume thumb current position via get_dom_snapshot, calculate target position for 37% on track, drag(thumbX, thumbY, targetX, targetY, 10, 20) to slide thumb, verify via get_attribute for aria-valuenow approximately 37, if verification fails try click_at method as fallback
- fullVolumeSliderTest: 10 steps -- navigate to target video page, dismiss any cookie/consent popup, hover over player container to reveal control bar, use get_dom_snapshot to identify volume slider element and its bounding rect, read current volume via get_attribute (aria-valuenow or value), calculate pixel position for 37% along slider track, attempt click_at on track at 37% position, verify volume changed via get_attribute, if click did not work attempt drag method, final verification of aria-valuenow approximately 37 (accept 35-39 range)

WARNINGS array (8 items):
1. Volume sliders can be horizontal (most common) or vertical -- check slider dimensions to determine orientation before calculating position
2. Some players use 0-1 range instead of 0-100 -- 37% is 0.37 on a 0-1 scale or 37 on a 0-100 scale
3. Do NOT click the mute/volume speaker icon button -- it toggles mute, not adjusts volume
4. Volume controls may be hidden until hover -- hover on player container or control bar first
5. Vimeo and some players use input[type="range"] which may respond to value setting via DOM rather than click events
6. CDP click_at is preferred for track clicks as content script click events may be ignored on custom player controls
7. After volume change, some players show a transient tooltip with the volume percentage -- capture via read_page if visible
8. Accept 35-39% range as success (slider precision limited by pixel granularity on narrow tracks)

TOOL PREFERENCES: ['click_at', 'drag', 'hover', 'get_attribute', 'click', 'navigate', 'get_dom_snapshot', 'read_page', 'waitForDOMStable', 'press_key']
