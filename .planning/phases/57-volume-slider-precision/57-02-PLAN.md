Execute the MICRO-01 edge case test using MCP manual tools following the fullVolumeSliderTest workflow from the video player site guide. Test steps:

1. Use MCP navigate tool to go to a public video page with a custom volume slider. Try in order:
   a. vimeo.com (search for any public video, e.g., vimeo.com/channels/staffpicks)
   b. Fallback: dailymotion.com (any popular video)
   c. Fallback: a JW Player demo page (jwplayer.com/developers/tools/player-demos)
2. Use read_page to check for cookie/consent popups and dismiss them via click
3. Use get_dom_snapshot to locate the video player and its control bar
4. Hover over the player container to reveal volume controls (some players hide controls)
5. Use get_dom_snapshot to identify the volume slider element -- look for:
   - input[type="range"] with aria-label containing "volume"
   - [role="slider"] with aria-label containing "volume"
   - Elements matching site guide volumeSlider selectors
6. Read the slider bounding rect (position, width, height) from the DOM snapshot
7. Calculate the 37% target position:
   - For horizontal slider: targetX = slider_left + (0.37 * slider_width), targetY = slider_top + (slider_height / 2)
   - For vertical slider: targetX = slider_left + (slider_width / 2), targetY = slider_bottom - (0.37 * slider_height)
8. Attempt Method 1 -- click_at(targetX, targetY) on the slider track at the 37% position
9. Verify result: use get_attribute on the slider for aria-valuenow (expect approximately 37 or 0.37)
10. If click did not change the volume, attempt Method 2 -- locate the thumb position and drag(thumbX, thumbY, targetX, targetY, 10, 20)
11. Final verification: get_attribute for aria-valuenow, accept 35-39 range as PASS

Record every step outcome (tool used, target, result, notes) for the diagnostic report.
