Execute the DARK-01 edge case test using MCP manual tools following the downloadRealFile workflow from the freeware-download.js site guide. Test steps:

1. VALIDATE PRIMARY TARGET DOWNLOAD PAGE (sourceforge.net):
   (a) Navigate to https://sourceforge.net/projects/vlc/files/latest/download -- VLC is a well-known freeware project, this URL goes directly to the download page
   (b) HTTP fetch as fallback: check server response, page size, presence of download links in HTML
   (c) Also try alternative URLs if primary fails: https://sourceforge.net/projects/7zip/ or https://sourceforge.net/projects/notepadplusplus/
   (d) Look for download link elements: a[href*="/download"], a.button, any element with text "Download"
   (e) Look for ad indicators in the HTML: iframe elements, elements with ad-related class names, data-ad-* attributes, links to ad network domains
   (f) Record: URL loaded (yes/no), HTTP status, page size, download link count, ad element count, page title

2. ANALYZE ALL DOWNLOAD-LIKE ELEMENTS:
   (a) From the page HTML, extract every element that contains "download" in its text, href, class, or aria-label
   (b) Categorize each as: REAL DOWNLOAD (href points to sourceforge.net or downloads.sourceforge.net) or FAKE AD (href points to ad domain, element is in iframe, parent has ad class) or UNKNOWN
   (c) Count: total download-like elements, confirmed real, confirmed fake ads, unknown
   (d) For each fake ad, document which heuristic identified it (ad domain, iframe, CSS class, data attribute, tracking URL, domain mismatch, aria label, parent container)

3. IDENTIFY AD IFRAMES:
   (a) Extract all iframe elements from the page HTML
   (b) Check each iframe src for ad network domains: googleadservices.com, doubleclick.net, adsrvr.org, taboola.com, outbrain.com, pagead2.googlesyndication.com
   (c) Record: total iframes found, iframes with ad domains, iframes without ad domains
   (d) Note: content inside iframes is not visible in the main page's get_dom_snapshot -- the iframe boundary is itself the detection signal

4. VALIDATE REAL DOWNLOAD LINK:
   (a) Among non-ad download-like links, identify the one most likely to be the real download
   (b) For SourceForge: look for href matching sourceforge.net/projects/*/files/*/download or downloads.sourceforge.net/*
   (c) Use get_attribute (or HTTP header analysis) to check the href of the identified real download link
   (d) Verify the real link is NOT inside an iframe and NOT in a parent container with ad-related classes
   (e) Record: real download link href, element selector, confidence level, verification method

5. ATTEMPT FULL DOWNLOAD WORKFLOW (if MCP bridge available):
   (a) Navigate to the download page
   (b) Use get_dom_snapshot to capture all download-like elements
   (c) Apply ad detection heuristics from the site guide (iframe check, href domain check, parent class check, data attribute check)
   (d) Identify the real download link
   (e) Use get_attribute to verify the real link href before clicking
   (f) Click the verified real download link
   (g) Check if the page transitions to a download confirmation/countdown page (not an ad landing page)
   (h) If MCP bridge unavailable: document workflow as designed but not executed

6. VALIDATE FALLBACK TARGETS (if primary fails):
   (a) Fetch https://filehippo.com/ -- check for download page structure and ad elements
   (b) Fetch https://fosshub.com/ -- check for download page structure
   (c) Fetch https://majorgeeks.com/ -- check for download page structure and mirror links
   (d) Record: status, download link selectors found, ad element count, page size for each

7. DARK PATTERN HEURISTIC EFFECTIVENESS:
   (a) For each of the 8 ad detection heuristics documented in the site guide, assess:
       - Heuristic 1 (Ad domains in href): How many fake buttons did this catch?
       - Heuristic 2 (Iframe wrapper): How many iframes with ad content were found?
       - Heuristic 3 (Ad CSS classes): How many elements with ad-related classes?
       - Heuristic 4 (data-ad-* attributes): How many elements with data-ad attributes?
       - Heuristic 5 (Tracking redirects): How many tracking redirect URLs?
       - Heuristic 6 (Domain mismatch): How many external domain links?
       - Heuristic 7 (aria/title labels): How many accessibility-labeled ads?
       - Heuristic 8 (Parent container markers): How many ad-marked parent containers?
   (b) Rank heuristics by effectiveness (most fake buttons caught)
   (c) Note any false positives (real links incorrectly flagged as ads) or false negatives (ads not caught)

8. OUTCOME CLASSIFICATION:
   - PASS = real download link identified and clicked, fake ad buttons correctly ignored, download confirmation page reached via MCP tools
   - PARTIAL = HTTP validation confirms download page structure accessible with ad elements identifiable in HTML, real download link href validated, but live click execution blocked by WebSocket bridge disconnect
   - FAIL = could not identify the real download link on any target site, or real link could not be distinguished from ads
   - SKIP-AUTH = no freeware download site accessible (unlikely -- these sites are public)
