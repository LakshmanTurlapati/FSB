<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP caller -> trusted client label | Untrusted callers must not be able to inject arbitrary brand text into the overlay surface. |
| MCP server -> extension bridge | Start/end lifecycle messages cross the local bridge and must remain explicit and bounded. |
| Client-owned visual session -> autopilot session | The visible surface must not be double-owned by manual MCP lifecycle and FSB autopilot at the same time. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-203-01-01 | Spoofing | Client label validation | mitigate | One shared allowlist helper plus extension-side revalidation; arbitrary labels rejected before overlay state is emitted. |
| T-203-01-02 | Tampering | Stale end call | mitigate | Token + version metadata ensures stale end requests cannot clear a newer same-tab session. |
| T-203-01-03 | Denial of Service | Surface ownership | mitigate | Reject same-tab autopilot conflicts and keep client-owned session state separate from `activeSessions`. |
| T-203-01-04 | Information Disclosure | Overlay metadata | mitigate | Start/end responses and overlay payloads include only canonical client label, token, version, and tab metadata; no prompt bodies, logs, secrets, or memory content. |
</threat_model>
