## Goal

Every prerendered `index.html` emitted by `ng build --localize` carries:

1. Exactly 7 `<link rel="alternate">` tags: one per `en`, `es`, `de`, `ja`, `zh-CN`, `zh-TW`, plus one `x-default` pointing at the `en` URL for that route.
2. Exactly 1 `<link rel="canonical">` pointing at the locale-specific URL.
3. `<html lang="...">` matching the served locale (verified, not set — Angular's i18n compiler sets it automatically).

`lint:i18n` and `extract-i18n-clean` stay green; sibling target XLIFFs untouched.

## Approach

**Shared helper** (CONTEXT D-01 + D-02 + D-06 + D-07): `showcase/angular/src/app/core/seo/locale-seo.ts` exports two functions:

- `buildLocaleUrl(localeId: string, routePath: string): string` — computes `${HOST}${LOCALE_SUBPATHS[L]}${routePath}` with the no-trailing-slash invariant for `routePath === ''`.
- `emitLocaleHead(renderer, doc, localeId, routePath)` — upserts canonical (re-using the existing per-page pattern) and replaces the hreflang block (idempotent: removes existing `<link rel="alternate">` tags first, then appends 7 fresh ones in declaration order).

Each marketing page component:
- Injects `@Inject(LOCALE_ID) private localeId: string`.
- Replaces its existing `this.upsertCanonical(url)` call with `emitLocaleHead(this.renderer, this.doc, this.localeId, ROUTE_PATH)` where `ROUTE_PATH` is `''` for home, `/about`, `/agents`, `/privacy`, `/support` for the rest.
- The page's `og:url` meta tag also gets the locale-aware URL (reuses `buildLocaleUrl`).

`<html lang>` requires no code change — Angular's i18n compiler sets it from `angular.json:locales.{locale}` at build time. Verification in step 5.

## Tasks

### Task 1 — Create shared helper
- Create `showcase/angular/src/app/core/seo/locale-seo.ts` exporting:
  - `HOST = 'https://full-selfbrowsing.com'` constant.
  - `buildLocaleUrl(localeId, routePath)` — returns `HOST` when both are empty/en+root; else concatenates with single `/` separators; never emits trailing slash. Falls back to `'en'` if `!isValidLocale(localeId)` (CONTEXT D-11).
  - `emitLocaleHead(renderer, doc, localeId, routePath)`:
    1. Compute canonical = `buildLocaleUrl(localeId, routePath)`.
    2. Upsert `<link rel="canonical" href={canonical}>` (existing pattern).
    3. Remove all `<link rel="alternate">` tags from `<head>` (idempotency).
    4. For each `L` in `LOCALES` (declaration order): append `<link rel="alternate" hreflang={L} href={buildLocaleUrl(L, routePath)}>`.
    5. Append `<link rel="alternate" hreflang="x-default" href={buildLocaleUrl('en', routePath)}>` (CONTEXT D-03).

### Task 2 — Wire pages
For each of `home`, `about`, `agents`, `privacy`, `support` page components:
- Add `import { LOCALE_ID, Inject } from '@angular/core'`.
- Add constructor/inject: `@Inject(LOCALE_ID) private localeId: string`.
- Define `ROUTE_PATH` constant in the component (`''`, `/about`, `/agents`, `/privacy`, `/support`).
- In `ngOnInit` (or `applyMeta`): replace `url` calc with `const url = buildLocaleUrl(this.localeId, ROUTE_PATH);` so canonical AND `og:url` both reflect locale.
- Replace `this.upsertCanonical(url)` with `emitLocaleHead(this.renderer, this.doc, this.localeId, ROUTE_PATH)`.
- Remove the now-unused `upsertCanonical` private method (or leave it as dead code only if it has callers elsewhere — verify with grep).

### Task 3 — Unit tests
For each of 5 page components, add (or extend) a `*.component.spec.ts` test:
- `TestBed.configureTestingModule({ providers: [{ provide: LOCALE_ID, useValue: 'en' }] })` — assert canonical = `HOST + ROUTE_PATH` (home: `HOST` exact); assert 7 `<link rel="alternate">` tags in document.head; assert one has `hreflang="x-default"`.
- Repeat with `useValue: 'es'` — assert canonical = `HOST/es${ROUTE_PATH}` (home: `HOST/es`); assert 7 alternate tags.
- Failure-mode test: `useValue: 'fr'` (unknown) — falls back to `en` URL form, `console.warn` called once.

### Task 4 — Post-build verification script
- Create `showcase/angular/scripts/verify-hreflang.mjs`:
  - Reads `showcase/angular/dist/showcase/browser/` recursively for `index.html` files.
  - For each file, parses with `jsdom` (already in devDeps from prior CI tooling) or regex.
  - Asserts: exactly 7 `<link rel="alternate">` tags; 6 with hreflang in `LOCALES`; 1 with `hreflang="x-default"`; exactly 1 `<link rel="canonical">` and its `href` matches the file's locale + route path; `<html lang>` matches the directory's locale (`en` root, `es/`, `de/`, `ja/`, `zh-CN/`, `zh-TW/`).
  - Exit 0 on all-pass; exit 1 with diff listing on first failure.
- Wire as `npm run verify-hreflang` in `package.json`.

### Task 5 — CI integration
- Extend `.github/workflows/ci.yml` website job: add `npm run verify-hreflang` step **after** the `build` step (Phase 262's step ordering: install → verify-locale-sync → lint:i18n → extract-i18n-clean → build → **verify-hreflang** → verify-bundle-budgets).
- Gate runs only on prerender output; if `dist/` is empty, exit 1 with clear error.

## Threat Model

- **T-264-01: Hreflang link tags accumulate on CSR route revisits.** Mitigation: Task 1 step 3 removes existing alternates before appending. Unit test verifies count is exactly 7 after a re-`ngOnInit`.
- **T-264-02: LOCALE_ID injection fails in SSR/prerender context.** Mitigation: `@Inject(LOCALE_ID)` is a built-in Angular token guaranteed by the i18n compiler; CONTEXT D-11 fallback handles tests / misconfig.
- **T-264-03: Sibling XLIFFs accidentally touched.** Mitigation: this plan does not edit anything under `showcase/angular/src/locale/`. CI's `extract-i18n-clean` gate will catch any accidental drift.
- **T-264-04: Prerender output path mismatch.** `angular.json:i18n.locales.{L}.subPath` already maps to URL prefix; `outputPath` is `dist/showcase/browser/{en|es|de|ja|zh-CN|zh-TW}/{route}/index.html`. Task 4 script must walk this exact structure.
- **T-264-05: `<html lang>` set incorrectly by manual code conflicting with Angular i18n compiler.** Mitigation: plan explicitly forbids manually setting `<html lang>` — CONTEXT D-05.
