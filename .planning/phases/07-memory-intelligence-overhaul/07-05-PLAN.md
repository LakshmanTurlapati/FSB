       ```javascript
       // Memory auto-refresh on storage changes
       let _memoryRefreshTimer = null;
       let _memoryRefreshInProgress = false;
       ```

       Inside the onChanged listener (or a new one), add:
       ```javascript
       // Auto-refresh Memory tab when fsb_memories changes
       if (changes.fsb_memories || changes[MEMORY_STORAGE_KEY]) {
         // Only refresh if memory section is currently visible
         if (dashboardState.currentSection !== 'memory') return;

         // Prevent refresh loops: skip if a refresh is already in progress
         if (_memoryRefreshInProgress) return;

         clearTimeout(_memoryRefreshTimer);
         _memoryRefreshTimer = setTimeout(async () => {
           _memoryRefreshInProgress = true;
           try {
             await _smartMemoryRefresh();
           } finally {
             _memoryRefreshInProgress = false;
           }
         }, 1000); // 1 second debounce
       }
       ```

    2. Add a `_smartMemoryRefresh()` function that preserves expanded state:
       ```javascript
       async function _smartMemoryRefresh() {
         // Save expanded panel state
         const expandedItem = document.querySelector('.memory-item.detail-expanded, .memory-item.graph-expanded');
         const expandedMemoryId = expandedItem?.dataset?.memoryId || null;
         const scrollContainer = document.getElementById('memoryListContainer');
         const scrollTop = scrollContainer?.scrollTop || 0;

         // Reload data
         await loadMemoryDashboard();

         // Restore expanded state
         if (expandedMemoryId) {
           const restoredItem = document.querySelector(`.memory-item[data-memory-id="${expandedMemoryId}"]`);
           if (restoredItem) {
             // Use a small delay to let the DOM settle after render
             setTimeout(() => {
               // Re-expand (toggleMemoryDetail handles both regular and site_map types)
               if (typeof toggleMemoryDetail === 'function') {
                 toggleMemoryDetail(restoredItem);
               }
             }, 100);
           }
         }

         // Restore scroll position
         if (scrollContainer) {
           scrollContainer.scrollTop = scrollTop;
         }
       }
       ```

    3. CRITICAL: Prevent infinite refresh loops. The `MemoryStorage.recordAccess()` method writes to chrome.storage.local which fires onChanged. To prevent this:
       - The `_memoryRefreshInProgress` flag prevents nested refreshes
       - The debounce timer (1 second) batches rapid writes
       - The `dashboardState.currentSection !== 'memory'` check prevents off-screen refreshes
       - Do NOT call `recordAccess()` during `loadMemoryDashboard()` or `renderMemoryList()`. Check the existing code: `memoryManager.search()` calls `recordAccess()` for each result. Make sure `loadMemoryDashboard()` uses `memoryManager.getAll()` (which does NOT call recordAccess) instead of search. This is already the case in the current code.

    4. NOTE on MEMORY_STORAGE_KEY: The constant `MEMORY_STORAGE_KEY` is defined in memory-schemas.js which is loaded in the background service worker, NOT in the options page. In options.js, use the literal string `'fsb_memories'` in the onChanged check, OR define a local constant at the top of the memory section: `const MEMORY_STORAGE_KEY_LOCAL = 'fsb_memories';`

    5. Also handle the case where the user switches TO the Memory section while it is stale. In the section switching logic (wherever `switchSection` is called), if switching to 'memory', call `loadMemoryDashboard()` to ensure fresh data.
