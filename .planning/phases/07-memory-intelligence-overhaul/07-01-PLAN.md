<task type="auto">
  <name>Task 1: Add AI enrichment pipeline to MemoryExtractor and update schemas</name>
  <files>lib/memory/memory-extractor.js, lib/memory/memory-schemas.js</files>
  <action>
    1. In memory-schemas.js `createBaseMemory()`, add two new fields to the base memory object:
       - `aiAnalysis: null` (will be populated by enrichment)
       - In metadata, add: `aiEnriched: false, enrichedAt: null`

    2. In memory-extractor.js, add an `async enrich(memory, context = {})` method to the MemoryExtractor class. This method:
       - Takes a single already-stored memory object and returns an AI analysis object (or null on failure)
       - Calls `this._getProvider(context)` to get the AI provider. If no provider, return null.
       - Builds a type-specific enrichment prompt using a new `_buildEnrichmentPrompt(memory)` method
       - Sends the prompt via `provider.buildRequest()` / `provider.sendRequest()` / `provider.parseResponse()`
       - Parses the response into a structured analysis object
       - On any error, logs warning and returns null (never throws)

    3. Add `_buildEnrichmentPrompt(memory)` method that generates SHORT prompts (~50 tokens system, ~100 tokens user) based on memory type:
       - For episodic: "Analyze this automation session. Return JSON with: { lessonsLearned: string[], patterns: string[], riskFactors: string[], optimizationTips: string[] }"
       - For semantic: "Analyze this learned fact. Return JSON with: { category: string, relatedPatterns: string[], reliability: string, usageContext: string }"
       - For procedural: "Analyze this action workflow. Return JSON with: { optimizationSuggestions: string[], reliabilityAssessment: string, alternativeApproaches: string[], complexityRating: string }"
       - User prompt includes: memory.text, memory.type, memory.metadata.domain, memory.typeData (JSON-stringified, truncated to 500 chars)

    4. Add `_parseEnrichmentResponse(rawContent)` method that:
       - Strips markdown code blocks
       - Tries JSON.parse
       - On failure, returns a minimal `{ rawAnalysis: rawContent.substring(0, 500) }` fallback
       - Never throws

    5. Ensure the enrich() method tracks AI usage via analytics by calling:
       `if (typeof analytics !== 'undefined') analytics.trackUsage(model, inputTokens, outputTokens, true, 'memory');`
       NOTE: This requires the analytics source field from Task 2 to be in place. If analytics.trackUsage does not accept a source param yet, the call still works (extra param is ignored until Task 2 lands).
  </action>
  <verify>
    - memory-schemas.js: `createBaseMemory()` returns object with `aiAnalysis: null` and `metadata.aiEnriched: false`
    - memory-extractor.js: `MemoryExtractor` class has `enrich()`, `_buildEnrichmentPrompt()`, and `_parseEnrichmentResponse()` methods
    - The enrich() method catches all errors and returns null on failure
    - Prompts are type-specific (check for 'episodic', 'semantic', 'procedural' in _buildEnrichmentPrompt)
  </verify>
  <done>MemoryExtractor has an enrich() method that accepts any memory type, builds a type-specific prompt, calls the AI provider, and returns structured analysis. Schema includes aiAnalysis and aiEnriched fields.</done>

<task type="auto">
  <name>Task 2: Wire enrichment into MemoryManager and add analytics source field</name>
  <files>lib/memory/memory-manager.js, utils/analytics.js</files>
  <action>
    1. In utils/analytics.js `trackUsage()` method:
       - Add a 5th parameter: `source = 'automation'`
       - Add `source: source` to the entry object that gets pushed to `this.usageData`
       - Add `getStatsBySource(timeRange, source)` method that filters `this.usageData` by source field before aggregating (same logic as `getStats()` but with the source filter). Treat entries with no `source` field as `'automation'`.
         Return shape: `{ totalCost: number, totalRequests: number, totalTokens: number }` -- these three fields are required because plan 07-06 accesses them directly in `loadDashboardCostBreakdown()` and `loadMemoryCostPanel()`.

    2. In lib/memory/memory-manager.js `add()` method, after the existing `this._storage.add(memory)` call succeeds:
       - Check `autoAnalyzeMemories` setting: `const autoAnalyze = await this._getAutoAnalyzeSetting();`
       - If enabled and `this._extractor` has an `enrich` method, fire off enrichment asynchronously:
         ```
         if (autoAnalyze && this._extractor?.enrich) {
           this._enrichAsync(memory).catch(err =>
             console.warn('[MemoryManager] Async enrichment failed:', err.message)
           );
         }
         ```
       - Add `async _enrichAsync(memory)` private method that:
         a. Calls `const analysis = await this._extractor.enrich(memory, {});`
         b. If analysis is not null, calls `await this._storage.update(memory.id, { aiAnalysis: analysis, metadata: { ...memory.metadata, aiEnriched: true, enrichedAt: Date.now() } });`
       - Add `async _getAutoAnalyzeSetting()` that reads `autoAnalyzeMemories` from `chrome.storage.local`. Default to `true` if not set.

    IMPORTANT: The store-first-enrich-second pattern means enrichment never blocks memory storage. If enrichment times out or fails, the memory is already stored with `aiEnriched: false`.
  </action>
  <verify>
    - analytics.js: `trackUsage()` accepts a 5th `source` parameter and includes it in entry objects
    - analytics.js: `getStatsBySource()` method exists and filters by source
    - memory-manager.js: `add()` fires off `_enrichAsync()` after successful storage
    - memory-manager.js: `_enrichAsync()` updates memory via `_storage.update()` with aiAnalysis data
    - memory-manager.js: `_getAutoAnalyzeSetting()` reads from chrome.storage.local, defaults to true
    - The entire enrichment path is non-blocking (uses `.catch()` on the promise)
  </verify>
  <done>MemoryManager.add() stores memories first, then fires async AI enrichment. Analytics entries have a source field ('automation' | 'memory' | 'sitemap'). Enrichment failure never prevents memory storage.</done>
