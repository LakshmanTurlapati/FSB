  <action>
    Create a new file `lib/memory/cross-site-patterns.js` with the following:

    1. `async function analyzeCrossSitePatterns(allMemories)` -- the main entry point:
       - Filters memories to only sitemap type: `memory.typeData?.category === 'site_map'`
       - Groups sitemaps by `memory.metadata?.domain` into a `byDomain` map
       - If fewer than 2 domains have sitemaps, return null (not enough data for cross-site analysis)
       - For each domain, extract structural patterns from sitePattern data:
         - `formTypes`: types of forms found (login, search, contact, etc.) from sitePattern.forms
         - `navPatterns`: navigation structures (header nav, sidebar, footer links) from sitePattern.navigation
         - `commonSelectors`: frequently used selector patterns (data-testid, aria-label, role attributes) from sitePattern.keySelectors
         - `pageTypes`: types of pages found (landing, list, detail, form, dashboard) from sitePattern.pages
       - Find patterns that appear in 2+ domains (intersection analysis):
         - `commonFormTypes`: form types appearing across multiple sites
         - `sharedSelectorPatterns`: selector patterns (not literal selectors, but patterns like "uses data-testid", "uses aria-label") that appear across sites
         - `commonNavPatterns`: navigation patterns shared across sites
         - `commonWorkflows`: page type sequences (e.g., search -> results -> detail) that appear across sites
       - Create a semantic memory using `createSemanticMemory()`:
         - text: `Cross-site patterns: ${domainCount} domains analyzed, ${totalPatterns} common patterns found`
         - metadata: `{ domain: null, tags: ['cross-site', 'patterns', 'auto-generated'], confidence: 0.7 }`
         - typeData: `{ category: 'cross_site_pattern', sitePattern: { domains: Object.keys(byDomain), commonFormTypes, sharedSelectorPatterns, commonNavPatterns, commonWorkflows, analyzedAt: Date.now() } }`
       - Return the created memory object

    2. `function groupSitemapsByDomain(sitemapMemories)` -- helper:
       - Groups an array of sitemap memories by their `metadata.domain` field
       - Returns an object: `{ "example.com": [memory1, memory2], "other.com": [memory3] }`
       - Used by the analysis function above and potentially by future UI rendering

    3. `function extractDomainPatterns(sitemapMemory)` -- helper:
       - Takes a single sitemap memory and returns a structured object of patterns found:
         `{ formTypes: string[], navPatterns: string[], selectorPatterns: string[], pageTypes: string[] }`
       - Extracts from sitePattern.forms (map form fields to type: login, search, registration, etc.)
       - Extracts from sitePattern.navigation (count nav items, categorize as header/sidebar/footer)
       - Extracts from sitePattern.keySelectors (identify selector strategy: id-based, data-attr, aria, class)
       - Extracts from sitePattern.pages (categorize by content: list pages, detail pages, form pages)
       - Uses simple heuristics (no AI call needed):
         - Form with password field -> "login"
         - Form with search/query field -> "search"
         - Navigation with > 5 items in header -> "mega-nav"
         - Pages with /search or /results in path -> "search-results"

    4. Export pattern: Use `if (typeof self !== 'undefined') { self.analyzeCrossSitePatterns = ...; self.groupSitemapsByDomain = ...; }` to match project convention.

    NOTE: This module does NOT call AI. It uses local heuristic analysis only. AI enrichment of the resulting memory happens via the pipeline from plan 07-01 if autoAnalyze is enabled.

  <action>
    1. In memory-manager.js, update the `consolidate()` method:
       - After the existing consolidation logic (whether using this._consolidator or _basicCleanup), add cross-site pattern analysis:
         ```javascript
         // Cross-site pattern analysis (runs only during consolidation)
         if (typeof analyzeCrossSitePatterns === 'function') {
           try {
             const allMemories = await this._storage.getAll();
             const patternMemory = await analyzeCrossSitePatterns(allMemories);
             if (patternMemory) {
               // Check if a cross-site pattern memory already exists and update it
               const existing = allMemories.find(m => m.typeData?.category === 'cross_site_pattern');
               if (existing) {
                 await this._storage.update(existing.id, {
                   text: patternMemory.text,
                   typeData: patternMemory.typeData,
                   metadata: { ...existing.metadata, ...patternMemory.metadata },
                   updatedAt: Date.now()
                 });
               } else {
                 await this._storage.add(patternMemory);
               }
             }
           } catch (error) {
             console.warn('[MemoryManager] Cross-site pattern analysis failed:', error.message);
           }
         }
         ```
       - The cross-site analysis is wrapped in try/catch and uses typeof guard so it degrades gracefully if cross-site-patterns.js is not loaded.

    NOTE: Loading cross-site-patterns.js into the background service worker is handled by plan 07-07 Task 1 (integration verification), which adds it to manifest.json and options.html script tags.

    NOTE: The consolidate() method is called manually by the user via the "Consolidate" button in the Memory tab. It is NOT called on every memory add. This keeps cross-site analysis costs low.
