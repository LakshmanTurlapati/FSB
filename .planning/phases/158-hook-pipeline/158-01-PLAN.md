**LIFECYCLE_EVENTS constant (7 events per HOOK-01):**
```javascript
var LIFECYCLE_EVENTS = {
  BEFORE_ITERATION: 'beforeIteration',
  AFTER_API_RESPONSE: 'afterApiResponse',
  BEFORE_TOOL_EXECUTION: 'beforeToolExecution',
  AFTER_TOOL_EXECUTION: 'afterToolExecution',
  AFTER_ITERATION: 'afterIteration',
  ON_COMPLETION: 'onCompletion',
  ON_ERROR: 'onError'
};
```

**HookPipeline constructor (per D-01):**
- Use function/prototype pattern (not class syntax) matching state-emitter.js convention for importScripts compatibility.
- Internal storage: `this._hooks = {}` where keys are event names from LIFECYCLE_EVENTS values and values are arrays (not Sets -- need order preservation per D-01 "registration order execution").
- Use `var` for module-level declarations.

**HookPipeline.prototype.register(event, handler):**
- Validate `event` is a value in LIFECYCLE_EVENTS. If invalid, console.warn and return a no-op unsubscribe function.
- Push `handler` to `this._hooks[event]` array (create array if first handler).
- Return an unsubscribe function that calls `this.unregister(event, handler)`.

**HookPipeline.prototype.unregister(event, handler):**
- Find `handler` in `this._hooks[event]` array by reference equality.
- Splice it out. If array becomes empty, delete the key.

**HookPipeline.prototype.emit(event, context):**
- Make this an async function (per Claude's Discretion -- async allows hooks that do chrome.storage reads).
- Get the handlers array for `event`. If none, return `{ stopped: false, results: [] }`.
- Iterate handlers **in registration order** (per D-01).
- Each handler call wrapped in try/catch (per D-05):
  - On success: collect result in results array.
  - On error: `console.warn('[hook-pipeline] handler error on ' + event + ':', err)` and continue.
- After each handler, check if result is an object with `shouldStop === true`. If so, set `stopped = true` and break the loop (per D-05: "Only explicit { shouldStop: true } can halt").
- Return `{ stopped: boolean, results: Array, stoppedBy: string|null }` where stoppedBy is the reason from the stopping handler.

**HookPipeline.prototype.removeAll(event):**
- If event provided, delete `this._hooks[event]`.
- If no event, reset `this._hooks = {}`.

**HookPipeline.prototype.getHandlerCount(event):**
- Return the number of registered handlers for an event (or total if no event specified). Useful for testing.

**Module boilerplate:**
- `'use strict';` at top.
- JSDoc header comment referencing Claude Code hooks subsystem adaptation.
- `typeof module !== 'undefined' && module.exports` guard exporting `{ HookPipeline: HookPipeline, LIFECYCLE_EVENTS: LIFECYCLE_EVENTS }`.
- 2-space indentation, single quotes, semicolons always.
