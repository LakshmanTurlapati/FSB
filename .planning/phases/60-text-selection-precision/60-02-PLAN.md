  <action>
Execute the MICRO-04 edge case test using MCP manual tools following the highlightSentence workflow from the Wikipedia site guide. Test steps:

1. Use MCP navigate tool to go to https://en.wikipedia.org/wiki/Albert_Einstein (well-known article with many paragraphs, no auth required)

2. Use read_page to verify the article loaded and get the text content. Check for any consent banners and dismiss via click if present.

3. Use get_dom_snapshot to identify paragraph element references inside #mw-content-text .mw-parser-output. Locate all `<p>` elements and identify which are non-empty (textContent.trim().length > 10).

4. Identify the THIRD non-empty paragraph. Get its CSS selector or element reference. On Wikipedia, some early `<p>` elements may be empty or contain only short class/style info.

5. Use getText on the third paragraph element to get its full text content. Record the full text for offset calculation.

6. Calculate sentence boundaries in the paragraph text:
   - Scan for ". " followed by uppercase letter [A-Z] as sentence boundaries
   - Skip abbreviations: "Dr.", "Mr.", "Mrs.", "U.S.", "St.", "Jr.", "Sr.", "Inc.", "Ltd.", "Corp.", "e.g.", "i.e.", "vs.", "al.", "Prof."
   - Account for citation brackets [1], [2] etc. which are part of textContent
   - First sentence boundary = first qualifying ". [A-Z]" position
   - Second sentence starts at: position after the space following the period
   - Second sentence ends at: next qualifying ". [A-Z]" position + 1 (include the period), or end of paragraph if no further boundary

7. Use select_text_range(selector, startOffset, endOffset) with the calculated offsets to highlight the second sentence.

8. Verify the result:
   - Check the returned selectedText from select_text_range
   - It should start with an uppercase letter and end with a period
   - It should be a complete, coherent English sentence
   - Compare against the expected sentence from step 5

9. If select_text_range returned incorrect text, attempt corrections:
   - Recalculate offsets (check if citation text shifted boundaries)
   - Try CDP drag as fallback: calculate getBoundingClientRect coordinates for start and end characters using Range API, then drag(startX, startY, endX, endY, steps=30, stepDelayMs=15)
   - Document which method worked or why both failed

Record every step outcome (tool used, target, result, notes) for the diagnostic report.
