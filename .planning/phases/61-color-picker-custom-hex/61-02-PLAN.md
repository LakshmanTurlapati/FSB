Execute the MICRO-05 edge case test using MCP manual tools following the selectCustomHex workflow from the color picker site guide. Target hex: #2196F3 (Material Design Blue 500). Test steps:

1. Use MCP navigate tool to go to https://colorpicker.me/ (free, no auth, clean DOM)

2. Use read_page to verify the page loaded. Dismiss any cookie/consent banners via click if present.

3. Use get_dom_snapshot to map the page elements. Identify:
   - The shade/saturation area (large colored square -- typically a canvas or div with gradient background)
   - The hue strip (narrow vertical rainbow bar on the right side of the shade area)
   - The hex value input field (text input showing current hex color like #FF0000)
   Record element references and note their approximate bounding rectangles.

4. IDENTIFY HUE STRIP BOUNDS: Determine the hue strip element's position and dimensions. On colorpicker.me, the hue strip is a vertical bar. Record top, left, width, height values from the DOM snapshot or element position data.

5. SET HUE: Calculate coordinates for blue hue (~207 degrees):
   - X = hueStrip.left + hueStrip.width / 2 (center of strip)
   - Y = hueStrip.top + (207 / 360) * hueStrip.height (57.5% down from top)
   Use click_at(x, y) to set the hue. If click_at does not update the color, try drag from current hue reticle position to target position.

6. VERIFY HUE: Use read_page or get_dom_snapshot to verify the shade area gradient changed to show blue tones.

7. IDENTIFY SHADE AREA BOUNDS: Determine the shade area element's position and dimensions.

8. SET SHADE: Calculate coordinates for target saturation/brightness (S~82%, V~95%):
   - X = shadeArea.left + 0.82 * shadeArea.width (82% from left = high saturation)
   - Y = shadeArea.top + 0.05 * shadeArea.height (5% from top = high brightness/value)
   Use click_at(x, y) to set the shade. If click_at does not update, try drag.

9. READ HEX: Read the hex value from the input field using get_attribute on the hex input element (read "value" attribute), or use getText on the hex display.

10. VERIFY: Compare the read hex value with target #2196F3. Check each RGB channel is within +/-15 tolerance. If outside tolerance, attempt fine-tuning with adjusted coordinates.

11. If both click_at interactions succeeded but hex is wrong, try the direct approach: click the hex input field, clear it, type "#2196F3" using type_text.

Record every step outcome (tool used, target, result, notes) for the diagnostic report.
