<objective>
Establish the core partial-completion lifecycle contract in the runtime so blocked-but-useful sessions are no longer forced into the error path.

Purpose: The current architecture only allows `complete_task` or `fail_task`, so useful runs that hit an external blocker are misclassified as errors. This plan introduces the generic runtime mechanism for a first-class partial/blocked terminal outcome and updates the transport contract that turns terminal messages into dashboard/MCP execution results.
Output: A dedicated partial/blocked lifecycle tool, session/result schema support, agent-loop terminal handling, and background completion resolution that preserves partial completion as non-crash state.
</objective>
