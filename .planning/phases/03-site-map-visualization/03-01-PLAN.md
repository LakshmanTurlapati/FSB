<task type="auto">
  <name>Task 2: Create site-graph.js -- data transformer, SVG renderer, and interaction handlers</name>
  <files>lib/visualization/site-graph.js</files>
  <action>
    Create lib/visualization/site-graph.js exposing a global `SiteGraph` namespace with these functions:

    **1. SiteGraph.transformData(sitePattern)**
    Converts a sitePattern object (from sitemap-converter.js / sitemap-refiner.js) into {nodes, links} for d3-force.

    Data mapping:
    - pages[path] -> node with type:'page', id:'page:'+path, label: title or path, size: Math.max(8, Math.min(25, 8 + elementCount/10 + formCount*3))
    - forms[i] -> node with type:'form', id:'form:'+page+':'+fields.join(','), label: 'Form: '+fields.slice(0,3).join(', '), size: 10
    - navigation[i] -> link from root page node (path='/') or first page to target page, type:'navigation'
    - workflows[i] -> parse the '->' delimited string into sequential edges, type:'workflow'. If parsing fails, skip (do NOT error).
    - pagePurposes[path] -> stored as `purpose` property on page nodes (used for color coding and tooltips)

    CRITICAL: Deep-clone all data before passing to d3-force. d3-force MUTATES node objects (adds x, y, vx, vy, index). Never pass the original sitePattern data.

    Give each node random initial x/y positions within [0, width] x [0, height] to prevent force explosion. Use a default canvas of 600x400.

    **2. SiteGraph.render(container, graphData, options)**
    - container: DOM element to append SVG into
    - graphData: {nodes, links} from transformData
    - options: {width: 600, height: 400} (optional, defaults provided)

    Implementation:
    a) Create SVG element using createElementNS('http://www.w3.org/2000/svg', 'svg') -- NEVER use createElement.
       Set width='100%', height=options.height, viewBox='0 0 width height'.
       Add CSS classes: 'site-graph-svg'.

    b) Create arrow marker defs for directed edges.

    c) Create a <g> group for links first (so they render behind nodes).
       For each link:
       - navigation: solid line, stroke color var(--info-color, #0891b2), marker-end arrow
       - form: solid line, stroke color var(--warning-color, #d97706), no arrow
       - workflow: dashed line (stroke-dasharray: 5,3), stroke color var(--success-color, #059669), marker-end arrow

    d) Create a <g> group for nodes on top.
       For each node:
       - page: <circle> with r=node.size, fill based on purpose category:
         - default: var(--primary-color, #ff6b35)
         - If purpose contains 'login/auth': #8b5cf6 (purple)
         - If purpose contains 'search': #06b6d4 (cyan)
         - If purpose contains 'dashboard/admin': #f59e0b (amber)
         - If purpose contains 'profile/account': #ec4899 (pink)
         - Otherwise: var(--primary-color, #ff6b35)
       - form: <rect> with width/height=node.size*1.6, rotated 45deg (diamond shape via transform), fill var(--warning-color, #d97706)
       Add a <text> label below each node with the node label (truncated to 15 chars).

    e) Set up d3.forceSimulation:
       - forceLink with id accessor, distance 80 for nav, 50 for form
       - forceManyBody strength -250 for pages, -100 for forms
       - forceCenter at width/2, height/2
       - forceCollide radius = node.size + 5
       - alphaDecay 0.02 for smooth animation
       On each tick, update SVG element positions (setAttribute cx/cy for circles, transform for rects, x1/y1/x2/y2 for lines).

    f) Store the simulation reference on the container element as container._siteGraphSim for cleanup.

    **3. Interaction handlers (inside render):**

    a) Node drag: mousedown on node -> set node.fx/fy, alphaTarget(0.3).restart(). mousemove -> update fx/fy converting screen coords to SVG coords via viewBox ratio. mouseup -> clear fx/fy, alphaTarget(0). Use e.stopPropagation() to prevent pan during drag.

    b) Zoom: wheel event on SVG -> adjust viewBox width/height by scale factor (1.1 for zoom out, 0.9 for zoom in). Zoom toward cursor position. Clamp zoom between 0.3x and 3x of original dimensions. e.preventDefault() to prevent page scroll.

    c) Pan: mousedown on SVG background (not on a node) -> track mouse movement and adjust viewBox x/y. mouseup -> stop panning.

    d) Tooltip: Create a single div.site-graph-tooltip, position absolute relative to container. On node mouseenter, populate with:
       - Page nodes: title, path, elementCount, formCount, linkCount, purpose (if AI-refined)
       - Form nodes: fields list, action, parent page
       Position tooltip near the cursor. On mouseleave, hide tooltip.

    **4. SiteGraph.destroy(container)**
    - Stop the simulation (container._siteGraphSim.stop())
    - Remove the SVG element
    - Remove the tooltip div
    - Delete container._siteGraphSim reference

    **5. SiteGraph.cache**
    - Map of memoryId -> {nodes, links, settled: boolean}
    - On first render for a memoryId, cache the data
    - On subsequent renders for same memoryId, restore cached positions (skip animation)
    - SiteGraph.clearCache(memoryId) to remove a specific entry
    - SiteGraph.clearAllCache() to clear everything

    Export: Attach SiteGraph to `self` (window) for use by options.js:
    ```javascript
    if (typeof self !== 'undefined') {
      self.SiteGraph = SiteGraph;
    }
    ```

    PITFALL AVOIDANCE:
    - ALWAYS use document.createElementNS for ALL SVG elements
    - ALWAYS use addEventListener, never inline handlers (CSP)
    - Deep-clone nodes/links before d3-force (it mutates them)
    - Random initial positions + forceCenter to prevent explosion
    - Stop simulation on destroy to prevent memory leaks
  </action>
