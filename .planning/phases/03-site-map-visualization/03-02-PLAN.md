    - Add `data-has-graph="true"` attribute on site_map memory items
    - The chevron goes after the refine button and before the delete button

    **2. Add click handler on memory items for graph expand/collapse:**

    After the existing delete and refine handlers in renderMemoryList, add a click handler on each `.memory-item[data-has-graph="true"]`:

    ```javascript
    container.querySelectorAll('.memory-item[data-has-graph="true"]').forEach(item => {
      item.addEventListener('click', (e) => {
        // Don't trigger on button clicks
        if (e.target.closest('.control-btn')) return;
        toggleMemoryGraph(item);
      });
      item.style.cursor = 'pointer';
    });
    ```

    **3. Create toggleMemoryGraph(memoryItem) function:**

    ```javascript
    function toggleMemoryGraph(memoryItem) {
      const memoryId = memoryItem.dataset.memoryId;

      // If already expanded, collapse
      if (memoryItem.classList.contains('graph-expanded')) {
        collapseMemoryGraph(memoryItem);
        return;
      }

      // Collapse any other expanded graph first
      const existingExpanded = document.querySelector('.memory-item.graph-expanded');
      if (existingExpanded) {
        collapseMemoryGraph(existingExpanded);
      }

      // Find the memory data
      // Use the memories array from the last loadMemoryDashboard call
      expandMemoryGraph(memoryItem, memoryId);
    }
    ```

    **4. Create expandMemoryGraph(memoryItem, memoryId) async function:**

    a) Get memory data: Load from memoryManager.getAll(), find by memoryId. If not found, show toast and return.

    b) Get sitePattern from memory.typeData.sitePattern. If missing, show toast and return.

    c) Call SiteGraph.transformData(sitePattern) to get {nodes, links}.

    d) If nodes.length === 0, show toast "No pages found in site map" and return.

    e) Mark item: memoryItem.classList.add('graph-expanded')

    f) Create wrapper div with class 'site-graph-container', set position: relative (for tooltip positioning).

    g) Create a legend bar inside the wrapper showing color coding:
       - Orange circle = Page
       - Amber diamond = Form
       - Cyan line = Navigation link
       - Green dashed line = Workflow
       Only show legend items for types present in the data.

    h) Insert wrapper AFTER memoryItem in the DOM (memoryItem.after(wrapper)).

    i) Use requestAnimationFrame to delay graph render until container is visible (Pitfall 6 avoidance):
       ```javascript
       requestAnimationFrame(() => {
         const rect = wrapper.getBoundingClientRect();
         const width = Math.max(rect.width - 16, 300); // account for padding
         SiteGraph.render(wrapper, graphData, { width, height: 380, memoryId });
       });
       ```

    j) Scroll the item into view: memoryItem.scrollIntoView({ behavior: 'smooth', block: 'nearest' })

    **5. Create collapseMemoryGraph(memoryItem) function:**

    a) Remove 'graph-expanded' class from memoryItem.

    b) Find the .site-graph-container that follows memoryItem (memoryItem.nextElementSibling with class check).

    c) Call SiteGraph.destroy(graphContainer) to stop simulation and clean up.

    d) Remove the container from DOM.

    **6. Handle memory list re-renders:**

    When the memory list is re-rendered (search, filter, delete), any expanded graphs should be cleaned up. At the top of renderMemoryList, before clearing innerHTML, find and destroy any active graphs:
    ```javascript
    // Clean up any active graph before re-render
    const activeGraphs = container.querySelectorAll('.site-graph-container');
    activeGraphs.forEach(gc => {
      if (typeof SiteGraph !== 'undefined') SiteGraph.destroy(gc);
      gc.remove();
    });
    ```

    IMPORTANT: Guard all SiteGraph calls with `typeof SiteGraph !== 'undefined'` to handle cases where the script failed to load.
