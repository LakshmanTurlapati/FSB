```javascript
function getGuideSelectorsForUrl(url) {
  const guide = getGuideForUrl(url);
  if (!guide || !guide.selectors) return null;
  return guide.selectors;
}
```

This returns the `selectors` object from the matched site guide (e.g., `{searchBox: '#search-input', submitBtn: '.btn-submit'}`) or null. The background script will call this and pass the result to the content script via the `getYAMLSnapshot` message.

**In `content/messaging.js`:**

Add a new case in the `handleAsyncMessage` function's switch statement (after the `getCompactDOM` case):

```javascript
case 'getYAMLSnapshot':
  const yamlMode = request.options?.mode || 'interactive';
  const yamlGuideSelectors = request.options?.guideSelectors || null;
  const yamlStart = Date.now();
  const yamlResult = FSB.buildYAMLSnapshot({
    mode: yamlMode,
    guideSelectors: yamlGuideSelectors
  });
  const yamlTime = Date.now() - yamlStart;
  logger.logTiming(FSB.sessionId, 'DOM', 'buildYAMLSnapshot', yamlTime, {
    mode: yamlMode,
    elements: yamlResult.elementCount
  });
  sendResponse({
    success: true,
    yamlSnapshot: yamlResult.snapshot,
    refGeneration: yamlResult.refGeneration,
    elementCount: yamlResult.elementCount
  });
  break;
```

This handler:
1. Reads mode ('interactive' or 'full') and guideSelectors from request.options
2. Calls `FSB.buildYAMLSnapshot()` with those options
3. Logs timing via the existing logger
4. Returns the snapshot string, ref generation, and element count

  <name>Task 2: Add inline self-test validating snapshot format and token reduction</name>
  <files>content/dom-analysis.js</files>
  <action>
Add a `_runYAMLSnapshotSelfTest()` function at the end of the YAML DOM SNAPSHOT section in `content/dom-analysis.js`. This is a validation function that can be called from the browser console to verify the snapshot engine works correctly.

The self-test should:

1. **Build a snapshot** -- Call `buildYAMLSnapshot({ mode: 'interactive' })` on the current page
2. **Validate metadata header format** -- Check that the snapshot string contains:
   - A line starting with `url:`
   - A line starting with `title:`
   - A line starting with `scroll:` with the pattern `N/N (N%)`
   - A line starting with `viewport:` with the pattern `NxN`
   - A line starting with `state:`
   - A line starting with `captcha:`
   - A line starting with `forms:`
   - A line starting with `headings:`
3. **Validate element line format** -- Check that at least one line matches the pattern `  e\d+: \w+` (ref + tag)
4. **Validate filter footer** -- Check that the snapshot ends with a line matching `--- \d+/\d+ shown`
5. **Validate region grouping** -- Check that at least one region header exists (`@nav` or `@main` or `@header` or `@footer`)
6. **Token comparison** -- Build the old format using `generateCompactSnapshot()` on the same page. Compare character counts (chars / 3.5 heuristic per existing ai-integration.js convention). Report:
   - Old format char count (compact snapshot text only -- not the full JSON)
   - New YAML format char count
   - Reduction percentage
   - PASS if reduction >= 0% (the YAML format replaces BOTH the compact snapshot AND the HTML context + page structure blocks, so even if the YAML is slightly longer than the compact snapshot alone, the total token savings from eliminating the other blocks still hits 40%+. The true comparison is done at the prompt level in Phase 18.)
7. **Output results** as a structured object:
   ```javascript
   {
     passed: N,
     failed: N,
     results: [ { name: '...', passed: true/false, detail: '...' } ],
     tokenComparison: { oldChars, newChars, reductionPct }
   }
   ```

Expose the self-test on the FSB namespace:
```javascript
FSB._runYAMLSnapshotSelfTest = _runYAMLSnapshotSelfTest;
```

Also expose it on the global self/window for console access:
```javascript
if (typeof self !== 'undefined') self._runYAMLSnapshotSelfTest = _runYAMLSnapshotSelfTest;
```

Log the test results to console with `console.log('[FSB YAML Self-Test]', ...)`.
