  <action>
Add a new section after the COMPACT SNAPSHOT GENERATION section (after `generateCompactSnapshot`) in `content/dom-analysis.js` titled "YAML DOM SNAPSHOT". Build three functions:

**1. `buildMetadataHeader(refMap)`** -- Returns a multi-line string with YAML-style key:value metadata:
- `url:` -- `window.location.href`
- `title:` -- `document.title`
- `scroll:` -- format as `{scrollTop}/{pageHeight} ({pct}%)` using `window.scrollY`, `document.documentElement.scrollHeight`, `window.innerHeight`
- `viewport:` -- `{width}x{height}`
- `state:` -- `document.readyState`
- `captcha:` -- boolean from `document.querySelector('[class*="captcha"], [id*="captcha"], iframe[src*="recaptcha"], iframe[src*="hcaptcha"]')`
- `focus:` -- find the active element's ref from `refMap` (iterate refMap.map entries checking if `entry.element.deref() === document.activeElement`). Omit if activeElement is body/document.
- `forms:` -- count of `document.forms`, with brief description of up to 5 forms using form id/name/aria-label/legend text + field count. Format: `2 -- contact (5 fields), search (2 fields)`
- `headings:` -- up to 10 headings (h1-h3) as `h1 "text" > h2 "text" > ...`. If none: `none`

**2. `getRegion(el)`** -- Returns a region string using `element.closest()`:
- Priority order (most specific first): `@dialog` (dialog, [role="dialog"], [role="alertdialog"], [aria-modal="true"]), `@nav` (nav, [role="navigation"]), `@header` (header, [role="banner"]), `@footer` (footer, [role="contentinfo"]), `@aside` (aside, [role="complementary"]), `@main` (main, [role="main"]), default `@main`
- Note: forms are NOT a region -- they are grouped WITHIN regions per CONTEXT.md

**3. `buildElementLine(el, ref, guideAnnotations)`** -- Returns a single formatted line like `  e5: button "Submit Form" .btn-primary [disabled]`:
- Prefix with 2-space indent (or 4-space if inside a form group)
- Format: `{ref}: {tagName} "{accessibleName}" #{id} .{primaryClass} type="{type}" val="{value}" href="{href}" selected="{optionText}" placeholder="{placeholder}" [editable] [{states}] [hint:key:action]`
- Use full HTML tag names: `button`, `input`, `select`, `textarea`, `a`, `div`, `span`, etc.
- Accessible name: use existing `computeAccessibleName(el)` -- truncate to 60 chars
- ID: only if present and NOT auto-generated. Use existing `isAutoGeneratedId` pattern (check for common patterns: starts with `:`, contains random hex strings > 8 chars, starts with `__`, etc.). Simple heuristic: skip if id contains more than 8 consecutive hex chars or starts with `:` or `__`.
- Primary class: first class that is NOT a utility/framework class. Simple heuristic: skip classes starting with `css-`, `sc-`, `chakra-`, `MuiGrid`, or shorter than 3 chars. Take the first remaining class.
- Input type: only for INPUT elements with type != "text"
- Value: for INPUT/TEXTAREA, truncate to 40 chars, format as `val="{value}"`
- href: for A elements, show path-only for same-origin, full URL (truncated to 60 chars) for cross-origin
- Selected option: for SELECT elements, show `selected="{optionText}"` truncated to 30 chars
- Placeholder: only if no accessible name exists, truncated to 40 chars
- Contenteditable: append `[editable]`
- State flags: `[disabled]`, `[checked]`, `[focused]`, `[readonly]` -- joined with commas in a single bracket pair
- `guideAnnotations` is a Map(element -> {key, action}) -- if element is in the map, append `[hint:key:action]`

Keep the existing `generateCompactSnapshot` function intact (it will be deprecated later in Phase 18, but must not break now).

<task type="auto">
  <name>Task 2: Build buildYAMLSnapshot with region grouping, duplicate collapse, form grouping, and filter footer</name>
  <files>content/dom-analysis.js</files>
  <action>
Build the main orchestrator function `buildYAMLSnapshot(options)` in the same YAML DOM SNAPSHOT section, plus helper functions:

**1. `getElementFingerprint(el)`** -- Returns a string fingerprint for duplicate detection:
- Concatenate: `tag|text(50chars)|type|role|href|name|id`
- Include `href` for links, `name`/`id` for inputs -- prevents collapsing distinct elements that look similar

**2. `buildFilterFooter(shownCount, totalCandidates, offscreenByType)`** -- Returns footer line:
- Format: `--- {shown}/{total} shown | {N} links, {M} buttons, {K} inputs offscreen`
- If capped: `--- {shown}/{total}+ shown (capped) | ...`
- `offscreenByType` is an object like `{link: 5, button: 3, input: 10}`

**3. `buildSelectOptions(selectEl)`** -- For SELECT elements, return indented option lines:
- Each option as `      "{optionText}"` (6-space indent, under the select element line)
- Limit to 10 options, add `... and {N} more` if truncated

**4. `buildYAMLSnapshot(options = {})`** -- The main function:

Parameters:
- `options.mode`: `'interactive'` (default) or `'full'`
- `options.guideSelectors`: object of CSS selectors from the matching site guide (or null)

Steps:
a. Reset `FSB.refMap` and set `generationUrl`
b. Get elements using `getFilteredElements()`:
   - Interactive mode: `maxElements: 80`, `prioritizeViewport: true`
   - Full mode: `maxElements: 200`, `prioritizeViewport: false`
c. For interactive mode, filter to viewport-only elements using `getBoundingClientRect()` check (element bottom >= 0 AND top <= window.innerHeight AND right >= 0 AND left <= window.innerWidth). Track offscreen elements by type for the footer.
d. Build guide annotations Map if `guideSelectors` provided:
   - For each key/selector pair in guideSelectors, skip if selector starts with `/` (XPath)
   - Try `document.querySelector(selector)` -- if it matches one of the snapshot elements, add to Map
   - Infer action verb: INPUT/TEXTAREA -> "type", SELECT -> "select", BUTTON/[role=button] -> "click", A -> "click", checkbox/radio -> "check", contenteditable -> "type", default -> "click"
e. Build fingerprints Map for duplicate detection
f. For each element:
   - Compute fingerprint. If seen 1-2 times before, skip. If seen 0 times, register ref and build line.
   - Determine region via `getRegion(el)`
   - Determine form grouping: check `el.form` property (authoritative) OR `el.closest('form')` as fallback
   - Register element in `FSB.refMap` with selector from `FSB.generateSelectors(el)`
   - Build element line via `buildElementLine(el, ref, guideAnnotationsMap)`
   - For SELECT elements, also build option lines via `buildSelectOptions(el)`
g. After all elements processed, update lines for collapsed duplicates (fingerprint count >= 3: append ` (x{count})` to the line)
h. Group output by region:
   - Within each region, forms are sub-grouped: `  form "{formName}" #{formId}` header line, then indented form elements (4-space indent), then non-form elements (2-space indent)
   - Region order: @dialog first (if present), then @nav, @header, @main, @aside, @footer
i. Assemble: metadata header + blank line + region sections + blank line + filter footer
j. Return: `{ snapshot: string, refGeneration: number, elementCount: number }`

Expose `buildYAMLSnapshot` on the FSB namespace at the bottom of the function alongside existing exports:
```javascript
FSB.buildYAMLSnapshot = buildYAMLSnapshot;
```
