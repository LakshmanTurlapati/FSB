Execute the CONTEXT-03 edge case test using MCP manual tools following the readPdfAndFillForm workflow from the pdf-viewer.js site guide. Test steps:

1. IDENTIFY PDF TARGET: The pdf.js demo viewer default sample has only 14 pages. For a 50-page test, try one of these approaches in order:
   (a) Navigate to https://mozilla.github.io/pdf.js/web/viewer.html -- check the default sample page count. If less than 50, document this limitation.
   (b) Try loading a known long public PDF via URL parameter: https://mozilla.github.io/pdf.js/web/viewer.html?file=URL (URL-encode a public PDF URL)
   (c) If no 50-page PDF can be loaded in pdf.js viewer, use the 14-page sample and test with pages 4, 7, 14 instead (closest available pages to the 4/17/42 targets)
   (d) Alternative: try an arxiv.org PDF URL (many are 50+ pages) or any online pdf.js deployment with a long document
   Record: PDF target URL, page count, which pages will be tested

2. Use MCP navigate tool to go to the pdf.js viewer URL. Record: response status, page loaded confirmation.

3. Wait for PDF to load via wait_for_stable. Use read_page to verify:
   - Viewer container visible (#viewerContainer or .pdfViewer)
   - Page elements in DOM (div.page with data-page-number)
   - Toolbar with page number input (#pageNumber) and total pages indicator (#numPages)
   - textLayer with content on page 1
   Record: page count, viewer structure, textLayer populated (yes/no)

4. EXTRACT PAGE 4 (or adjusted target):
   - Click on page number input (#pageNumber) to focus
   - Use type_text to clear and type "4"
   - Use press_key with Enter to navigate
   - Wait 500-1000ms for page 4 to render
   - Extract text from .page[data-page-number="4"] .textLayer spans
   - Store first 300 characters as page4Text
   Record: page 4 navigation success, textLayer populated, text extracted (first 100 chars shown)

5. EXTRACT PAGE 17 (or adjusted target):
   - Repeat page input navigation for page 17
   - Wait for render, extract textLayer text
   - Store first 300 characters as page17Text
   Record: same as step 4

6. EXTRACT PAGE 42 (or adjusted target):
   - Repeat page input navigation for page 42
   - Wait for render, extract textLayer text
   - Store first 300 characters as page42Text
   Record: same as step 4

7. VERIFY ALL EXTRACTIONS:
   - Confirm all 3 text excerpts are non-empty
   - Verify they contain different content (not identical text on all pages)
   Record: all 3 non-empty (yes/no), texts differ (yes/no)

8. NAVIGATE TO FORM:
   - Use navigate tool to go to a web form. Primary target: https://httpbin.org/forms/post (simple HTML form with customer name, size, topping, comments fields). Fallback: any public form page.
   - Wait for form page to load
   Record: form URL, page loaded, form fields found

9. IDENTIFY FORM FIELDS:
   - Use get_dom_snapshot or read_page to find text input fields
   - Map available fields to extracted page data
   Record: number of fields found, field types (text, textarea, select), field identifiers

10. FILL FORM FIELDS:
    - For each available text input: click to focus, Ctrl+A to select existing text, type_text with stored page text
    - Field 1 = page4Text, Field 2 = page17Text, Field 3 = page42Text
    - If form has fewer than 3 text inputs: fill as many as possible, document limitation
    Record: fields filled (count), type_text success per field

11. VERIFY FORM FILLED:
    - Use read_page or get_dom_snapshot to confirm fields contain typed text
    - Cross-reference: field text should match the stored page excerpts
    Record: verification result per field

12. OUTCOME CLASSIFICATION:
    - PASS = text extracted from all 3 PDF pages AND web form filled with extracted data AND cross-site context retained
    - PARTIAL = some pages read or some form fields filled, or PDF accessed but form filling incomplete, or WebSocket bridge disconnect prevented interaction
    - FAIL = could not access PDF viewer or no text extracted from any page
    - SKIP-AUTH = target requires authentication (unlikely for pdf.js demo and httpbin)

Record every step outcome for the diagnostic report.
