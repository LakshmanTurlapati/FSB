**URL patterns:** `keep.google.com`

**fsbElements (15-20 elements, 5 strategies each):**
Target these elements with 5-strategy selectors (aria/role strategies are good for Keep since it uses standard ARIA):
- `new-note-input`: "Take a note" input area
- `new-note-title`: Note title field when editing
- `new-note-body`: Note body field when editing
- `search-bar`: Search/filter notes input
- `pin-button`: Pin note button
- `archive-button`: Archive note button
- `delete-button`: Delete note button
- `label-button`: Add/change labels button
- `color-button`: Change note color button
- `checklist-button`: Show checkboxes / convert to checklist
- `reminder-button`: Add reminder button
- `collaborator-button`: Add collaborator button
- `more-menu`: Three-dot more options menu
- `note-list`: Main note list/grid container
- `sidebar-labels`: Labels section in sidebar
- `settings-gear`: Settings gear icon

For selector values: use aria-label based selectors as primary strategies since Keep uses descriptive ARIA labels. Use class selectors cautiously (Keep uses relatively stable class names but they may be hashed). Use `[role="button"]`, `[role="listbox"]`, `[aria-label="..."]` patterns. Context selectors should use parent-child relationships.

**Guidance text (keyboard-first, detailed):**
Document these interaction patterns:
- CREATE NOTE: Click "Take a note" area (or press C when not editing), type title, Enter for body, Escape to close and save
- KEYBOARD SHORTCUTS: J/K to navigate notes, / for search, E to archive, # to delete, L for labels, C for new note
- CHECKLIST: Click checklist icon or Ctrl+Shift+9 to convert note to checklist, Enter for new items, Ctrl+Enter to check/uncheck
- NOTE COLORS: Right-click or use palette icon, select color
- LABELS: L key on selected note, or click label icon, type to filter labels
- AUTO-SAVE: Keep auto-saves on every keystroke, no explicit save needed

**Workflows (6-8, each with VERIFY and STUCK steps):**
1. `createNote`: Open Keep > click "Take a note" > type title > Enter > type body > Escape to close
2. `createChecklist`: Click "New list" icon or create note + checklist button > type items separated by Enter > Escape
3. `pinNote`: Navigate to note (J/K) > click pin icon or open note > pin
4. `archiveNote`: Navigate to note > press E (archive shortcut) or click archive icon
5. `deleteNote`: Navigate to note > click more menu > delete, or # shortcut
6. `addLabel`: Navigate to note > press L or click label icon > type label name > select
7. `searchNotes`: Press / to focus search > type query > Enter > results filter
8. `editNote`: Click note to open > edit title/body > Escape to close and save

Each workflow must include:
- Exact keystroke sequences (e.g., "press J three times to reach the third note")
- VERIFY step: "Verify by checking [what to look for in DOM]"
- STUCK recovery: "If [problem], try [fallback approach]"

**Warnings (8-12):**
- Keep auto-saves -- there is no save button, changes persist immediately
- J/K navigation only works when no note is open for editing
- The "Take a note" bar requires a click -- no keyboard shortcut opens it from empty state (use C shortcut instead)
- Checklist items use Enter to create new items, not Tab
- Labels are global -- creating a label on one note makes it available to all notes
- Color picker is a popover -- wait for it to render before clicking a color
- Archive is NOT delete -- archived notes are still searchable
- Deleted notes go to Trash for 7 days before permanent deletion
- Pinned notes always appear at the top of the list regardless of sort order
- Search (/) focuses the search bar -- pressing / while editing types a literal slash

**toolPreferences:** `['navigate', 'click', 'type', 'keyPress', 'waitForTabLoad', 'getText', 'waitForElement', 'getAttribute', 'waitForDOMStable']`

**Guidance text (keyboard-first, CRITICAL warnings about shortcuts):**

MOST IMPORTANT: Document Todoist Quick Add natural language syntax as THE primary creation method:
```
Quick Add syntax: "Task name #ProjectName @label1 @label2 p1 tomorrow at 3pm"
  #ProjectName  - assigns to project (case-insensitive, partial match)
  @label        - adds label
  p1/p2/p3/p4   - sets priority (p1 = urgent red, p4 = no priority)
  Date phrases  - natural language: "tomorrow", "next Monday", "every weekday", "Jan 15", "in 3 days"
  Time phrases  - "at 3pm", "at 14:00"
  Duration      - "for 30min", "for 2h"
```

Document keyboard shortcuts:
- Q: Open Quick Add (CRITICAL -- this is THE primary way to create tasks)
- A: Add task below current selection
- /: Open search
- G then I: Go to Inbox
- G then T: Go to Today
- G then U: Go to Upcoming
- J/K: Navigate tasks up/down (with Vim-style keys)
- E: Open task editor
- X: Select task (for bulk actions)
- C: Add comment
- Shift+D: Set due date
- Enter: Open selected task
- # key: Complete selected task
- Number 1-4: Set priority when task editor open

CRITICAL WARNING about single-key shortcuts:
```
TODOIST SHORTCUT HAZARD: Todoist uses 20+ unmodified single-key shortcuts
(Q, A, E, C, X, etc.). If the AI types text without focus on an input field,
letters trigger shortcuts instead of text entry. EVERY workflow that involves
typing must include a focus verification step. Before typing content:
1. Verify cursor is in an input/textarea/contenteditable element
2. If not, click the target input field first
3. Then type the content
```

**Workflows (6-8, each with VERIFY and STUCK steps):**
1. `quickAddTask`: Press Q > type task with natural language syntax > Enter to create (document full syntax)
2. `completeTask`: Navigate to task (J/K) > press # or click checkbox > verify task disappears from active view
3. `editTask`: Navigate to task > press E or click task > modify fields > Escape to save
4. `organizeByProject`: Press Q > include #ProjectName in task text, OR drag task, OR use move shortcut
5. `setPriority`: Open task editor (E) > press 1-4 for priority, OR use p1-p4 in Quick Add
6. `setDueDate`: Open task editor > Shift+D for date picker, OR include date in Quick Add text
7. `searchTasks`: Press / > type query > Enter > results displayed
8. `navigateViews`: G+I for Inbox, G+T for Today, G+U for Upcoming

Each workflow must include:
- Exact keystroke sequences with Quick Add natural language examples
- VERIFY step: "Verify task appears in [view] with [attributes]"
- STUCK recovery: "If Quick Add doesn't open, click the + button directly"

**Warnings (10-15):**
- CRITICAL: Todoist uses 20+ unmodified single-key shortcuts -- typing without input focus triggers commands
- Quick Add (Q) is the fastest and most reliable way to create tasks with all attributes at once
- Quick Add natural language parsing is LEFT TO RIGHT -- put task name first, then modifiers
- p1 in Quick Add means URGENT (red), p4 means NO PRIORITY -- this is the reverse of some systems
- Date parsing accepts natural language ("tomorrow", "next week") but NOT relative hour math ("in 2 hours")
- Project names with spaces in Quick Add use # followed by the name: "#My Project"
- Labels use @ prefix: "@urgent @work" -- labels must already exist
- Completing a task (# or checkbox) moves it to Completed -- undo with Ctrl+Z immediately
- Todoist does NOT auto-save inline edits -- press Enter or Escape to save
- The sidebar collapses on narrow viewports -- look for hamburger menu
- Filters view uses Todoist filter syntax which is different from search syntax
- Sub-tasks are created by pressing Ctrl+Right Arrow on a task to indent it
