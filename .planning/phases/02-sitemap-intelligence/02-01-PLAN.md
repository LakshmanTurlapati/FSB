## Goal

Create the foundation: pre-bundled site map directory, local converter that transforms recon data into memory format, memory schema extension, and "Save to Memory" button on research results.

## Tasks

### Task 1: Extend Memory Schema with `site_map` Category

**File:** `lib/memory/memory-schemas.js`

Add `site_map` to the semantic memory category options. The `typeData.sitePattern` field will hold the structured site map.

**Changes:**
- In `createSemanticMemory()` (~line 72), document that `category` now supports: `selector`, `site_pattern`, `user_preference`, `general`, `site_map`
- No code change needed to the function itself -- `category` is already a free-form string field
- Add a new factory function `createSiteMapMemory(domain, sitePattern, metadata)` that wraps `createSemanticMemory()` with the correct structure:

```javascript
function createSiteMapMemory(domain, sitePattern, metadata = {}) {
  const text = `Site map for ${domain}: ${sitePattern.pageCount || 0} pages, ${sitePattern.formCount || 0} forms`;
  return createSemanticMemory(text, {
    domain,
    tags: ['site_map'],
    confidence: sitePattern.refined ? 0.95 : 0.8,
    ...metadata
  }, {
    category: 'site_map',
    sitePattern
  });
}
```

**Verify:** `createSiteMapMemory('example.com', { pageCount: 5, ... })` returns a valid semantic memory with `typeData.category === 'site_map'`.

### Task 2: Build Local Site Map Converter (Tier 1)

**File:** `lib/memory/sitemap-converter.js` (NEW)

Pure JS module that transforms raw recon JSON (from Site Explorer `saveResearch()` format) into a `sitePattern` object.

**Input:** Research data object (same shape as stored in `fsbResearchData[researchId]`)

**Output:** `sitePattern` object:

```javascript
{
  domain: 'example.com',
  pages: {
    '/': { title: 'Home', elementCount: 45, formCount: 0, linkCount: 12 },
    '/careers': { title: 'Careers', elementCount: 85, formCount: 2, linkCount: 8 },
    ...
  },
  navigation: [
    { label: 'Home', path: '/' },
    { label: 'Careers', path: '/careers' },
    ...
  ],
  forms: [
    { page: '/careers', fields: ['search', 'location'], action: 'filter' },
    ...
  ],
  keySelectors: {
    '/careers': ['#job-search-input', '.filter-dropdown'],
    ...
  },
  workflows: [],  // Empty for Tier 1, populated by Tier 2
  tips: [],       // Empty for Tier 1, populated by Tier 2
  source: 'recon',
  refined: false,
  refinedAt: null,
  reconId: research.id,
  pageCount: research.pages.length,
  formCount: totalForms,
  crawledAt: research.startTime
}
```

**Logic:**
1. Iterate `research.pages[]` -- build `pages` object keyed by pathname
2. Deduplicate navigation items across all pages -- build `navigation` array
3. Extract form metadata from each page's `forms` array
4. Collect key selectors per page from `keySelectors` array
5. Compute summary counts

**Verify:** Pass a mock research object with 3 pages, get back a valid sitePattern with correct page count and navigation.

In the event delegation handler (~line 2731), add the `saveMemory` action:

else if (action === 'saveMemory') saveResearchToMemory(researchId);

2. Call `convertToSiteMap(research)` from sitemap-converter.js
3. Call `createSiteMapMemory(domain, sitePattern)` from memory-schemas.js
4. Save via `memoryStorage.store(memory)` (or the appropriate storage function)
5. Show toast: "Site map saved to memory for {domain}"
6. Update the research item to show a "Saved" indicator

**Verify:** Click "Save to Memory" on a research result -> memory appears in Memory tab with type filter showing as semantic/site_map.
