Create a new file site-guides/utilities/premium-highlighting.js. This site guide targets airline and travel booking sites that use misleading visual highlighting to steer users toward premium fare options instead of the cheapest available option. This is the DARK-06 dark pattern avoidance edge case.

**Distinction from other DARK patterns:** DARK-06 is NOT about hidden elements (DARK-04/05) or deceptive buttons (DARK-01/03). It is about visual misdirection: the cheapest option IS visible and clickable, but the site's UI design makes expensive options visually dominant while the cheapest option is visually subdued. The counter-strategy is numeric price comparison from DOM text, ignoring all visual styling.

**Misleading premium highlighting techniques documented in the guide:**

1. **Color and contrast emphasis:** Premium/business fares shown in bold colors (blue, green, gold) with high contrast backgrounds; basic/economy fares shown in muted gray or low-contrast colors
2. **Size scaling:** Premium fare cards are physically larger (wider, taller) with larger font sizes for price and details; economy cards are compact with smaller text
3. **Badge and label manipulation:** Premium options marked with "Recommended", "Best Value", "Most Popular", "Best Deal" badges; economy options have no badge or marked "Basic" or "Light"
4. **Position manipulation:** Premium options placed at the top of the results list or in the center column (eye-tracking sweet spot); cheapest options placed at the bottom or far left/right
5. **Default pre-selection:** Some airline sites pre-select the premium fare tier with a radio button or highlighted border, requiring the user to actively deselect and choose the cheaper option
6. **Hidden total pricing:** Displayed "from" prices may exclude taxes, baggage fees, or seat selection fees that make the "cheap" option more expensive; but the DOM text usually contains the actual total somewhere
7. **Fare class obfuscation:** Cheapest fares labeled with unappealing names ("Basic Economy", "Light", "Saver") while expensive fares have aspirational names ("Premium", "Comfort+", "Main Cabin Extra")

The AI must IGNORE all visual styling (colors, sizes, badges, positions, borders, highlights) and select PURELY by numeric price comparison.    - PRICE-BASED SELECTION STRATEGY section: The core selection approach is numeric price comparison from DOM text:
     Step A: EXTRACT ALL PRICES -- Use get_dom_snapshot to find all flight result elements. For each result element, extract the price text.      Step B: PARSE TO NUMERIC VALUES -- Strip currency symbols ($, EUR, GBP, etc.), commas, spaces, and "From" prefix. Convert to float. Handle edge cases: "1,234" = 1234, "$1,234.56" = 1234.56, "From $199" = 199.
     Step C: COMPARE AND IDENTIFY MINIMUM -- Sort all parsed prices ascending. The minimum value is the cheapest flight. Record which DOM element (result card) contains this minimum price.

   - FARE CLASS DARK PATTERNS section: Airlines present 2-5 fare tiers per flight (e.g., Basic Economy, Main Cabin, Comfort+, First) with different prices. The dark pattern is making the mid-tier or premium tier visually dominant while the cheapest tier is visually suppressed.    - AGGREGATOR VS AIRLINE SITE DIFFERENCES section: Aggregator sites (Google Flights, Kayak, Expedia) show one price per result row (usually the cheapest for that flight).
