Execute the MICRO-06 edge case test using MCP manual tools following the scrollCarouselHorizontally workflow from the carousel site guide. Test steps:

1. Use MCP navigate tool to go to https://www.amazon.com/ (free, no auth, multiple horizontal carousels on homepage). If Amazon is unavailable or blocks automation, try https://www.bestbuy.com/ or https://www.target.com/ as alternatives.

2. Use read_page to verify the page loaded. Dismiss any cookie/consent banners via click if present. Note the page title and URL to confirm successful navigation.

3. Use get_dom_snapshot to map the page elements. Identify:
   - Carousel containers (look for .a-carousel, [class*="carousel"], or similar)
   - Carousel arrow buttons (look for .a-carousel-goto-nextpage or [aria-label*="Next"])
   - Carousel items (product cards, deal cards, recommendation items)
   Record element references and note 2-3 visible item texts for before/after comparison.

4. RECORD VERTICAL SCROLL POSITION: Check the current vertical scroll position from the DOM snapshot or page state (scrollPosition field, window.scrollY). This is the baseline. Expected: scrollY = 0 (top of page) or near top.

5. CHOOSE AND EXECUTE INTERACTION METHOD:
   - METHOD A (preferred): If arrow buttons found (e.g., .a-carousel-goto-nextpage), click the next button.
   - METHOD B: If no arrow buttons, use scroll_at at the carousel center coordinates with deltaY=0, deltaX=300. The x,y MUST be inside the carousel container bounds.
   - METHOD C (fallback): Use drag from (carouselCenterX+150, carouselCenterY) to (carouselCenterX-150, carouselCenterY) with steps=15, stepDelayMs=20.

6. VERIFY HORIZONTAL SCROLL SUCCEEDED:
   - Use get_dom_snapshot or read_page to check if new/different items are visible in the carousel
   - Check that vertical scroll position has NOT changed from the baseline in step 4
   - PASS = new items visible AND vertical scroll unchanged
   - PARTIAL = carousel moved but vertical scroll also changed, OR unable to verify
   - FAIL = could not interact with carousel at all

7. If the first method did not work or triggered vertical scroll, try the next method in priority order.

8. If all three methods are attempted, document which ones worked and which failed.

Record every step outcome (tool used, target, result, notes) for the diagnostic report.
