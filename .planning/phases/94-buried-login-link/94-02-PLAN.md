   (b) slack.com -- secondary target. "Try for Free" and "Get Started" CTAs dominant, "Sign in" in top navigation.

2. VALIDATE DROPBOX.COM HOMEPAGE (primary target):
   (a) Fetch https://www.dropbox.com/ to analyze homepage structure
   (b) Find ALL a[href] and button elements on the page
   (c) For each clickable element found, extract: text content, href, class, id, aria-label
   (d) Classify each as LOGIN-INTENT (text contains "log in", "sign in", "login", "signin"; href contains /login, /signin) or SIGNUP-INTENT (text contains "sign up", "get started", "create account", "try free"; href contains /signup, /register, /plans)
   (e) Count: how many signup-intent vs login-intent elements? What is the ratio?
   (f) Analyze login link placement: Is it in header? Footer? Small text? Behind a menu?
   (g) Check the login link href: does it point to a recognizable login URL?
   (h) Record: URL tested, HTTP status, page size, total clickable elements, login-intent count, signup-intent count, login link location, login href

       - Did text keyword matching correctly identify the login link?

       - Were there false positives (signup elements misclassified as login)?
       - Were there false negatives (login links missed by keyword matching)?
   (b) Analyze login link hiding techniques per site:
       - Small font / muted color: Does the login link have less visual emphasis in the HTML (fewer classes, no button styling)?
       - Footer burial: Is the login link in a footer element?
       - Hamburger menu: Is the login link inside a nav element with mobile toggle?
       - "Already have an account?" pattern: Is there de-emphasized text near the signup form?
       - CTA vs text link asymmetry: Is the signup a button element while login is an anchor?
   (c) Record: classification results per site, hiding techniques identified
