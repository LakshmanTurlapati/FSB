Create utils/cli-validator.js with the following components. This file will be loaded via script tag in options.html (browser context, NOT service worker). It will access parseCliResponse from window scope (cli-parser.js loaded before it via script tag).

**1. FAILURE_TYPES constant:**
```javascript
const FAILURE_TYPES = {
  JSON_REVERT: 'Provider returned JSON instead of CLI commands',
  MARKDOWN_WRAP: 'Response wrapped in markdown code blocks (handled by preprocessor)',
  CONVERSATIONAL: 'Response includes conversational preamble/postscript (handled by preprocessor)',
  UNKNOWN_VERB: 'Response contains verbs not in COMMAND_REGISTRY',
  MALFORMED_ARGS: 'Command verb recognized but arguments are malformed',
  WRONG_ACTION: 'Commands parsed but target wrong elements or produce wrong params',
  EMPTY_RESPONSE: 'No commands found in response at all',
  PARTIAL_CLI: 'Mix of CLI commands and non-CLI text that partially parsed',
  SPECIAL_CHAR_MANGLED: 'Special characters in arguments were escaped or corrupted',
  URL_TRUNCATED: 'URL arguments were split or truncated at special characters',
};
```

**2. MockElement class:**
- Constructor: (ref, type, text, attributes = {})
- Properties: ref, type, textContent, text, attributes, value (for inputs), clicked (false), focused (false), typed (null), selectedOption (null), keyPressed (null)
- Method: `applyAction(tool, params)` that simulates action effects:
  - click: sets clicked = true
  - type: sets value = params.text, typed = params.text
  - selectOption: sets selectedOption = params.value
  - keyPress/pressEnter/key: sets keyPressed = params.key || tool
  - getText: returns textContent
  - navigate: (no element needed, tracked at MockDOM level)
  - focus: sets focused = true
  - hover: (no-op, tracked)

**3. MockDOM class:**
- Constructor: accepts array of {ref, type, text, attrs} objects
- `elements`: Map<string, MockElement> keyed by ref
- `resolveRef(ref)`: returns MockElement or null
- `navigations`: array tracking navigate commands
- `applyAction(action)`: resolves element by ref from action.params, calls element.applyAction. For navigate/scroll/searchGoogle (no element needed), track on MockDOM directly. Returns {success, element, error}.

**4. Test case structure:**
```javascript
// Each test case:
{
  name: string,
  taskType: string,  // navigation, form-fill, data-extraction, search-click, google-sheets, career-search
  domSnapshotFile: string,  // path to YAML snapshot
  mockDOMElements: Array<{ref, type, text, attrs}>,  // extracted from snapshot
  expected: {
    minActions: number,
    maxActions: number,
    requiredTools: string[],  // at least one of each must appear
    forbiddenTools: string[], // none of these should appear
    noJSON: boolean,          // response must NOT contain JSON objects
    taskComplete: boolean,    // expect done signal
    customValidation: function|null  // optional custom validator
  }
}
```

**5. CLIValidator class:**
- `constructor()`: initializes testSuites (empty array), results (empty array), mode ('golden'), onProgress callback (null)
- `registerSuite(suite)`: adds test suite to testSuites array
- `async loadGoldenResponse(provider, testName)`: fetches from chrome.runtime.getURL(`test-data/golden-responses/${provider}/${testName}.txt`)
- `async loadDOMSnapshot(snapshotFile)`: fetches YAML snapshot from test-data directory
- `buildMockDOM(elements)`: creates MockDOM from element array
- `validateParsing(parsed, expected)`: checks minActions, maxActions, requiredTools, forbiddenTools, noJSON, taskComplete. Returns {passed, failures[]} where each failure has {type: FAILURE_TYPE, detail, expected, actual}.
- `validateExecution(actions, mockDOM)`: iterates actions, calls mockDOM.applyAction for each. Checks elements were targeted correctly (ref exists), tracks which tools executed. Returns {passed, executedCount, failures[], elementStates: Map}.
- `categorizeFailure(response, parsed)`: analyzes a failed response and returns the FAILURE_TYPE -- checks for JSON patterns (/{.*"tool".*}/), markdown wrapping, empty actions, unknown verbs, etc.
- `async runSuite(suiteName, provider, options)`: main entry point. For each test case in suite: load golden response (or use options.liveResponse if provided), run parseCliResponse, run validateParsing, run validateExecution with mock DOM. Collect results. Call onProgress after each test.
- `async runAll(providers)`: runs all registered suites for each provider. Returns full results array.
- `generateReport(results)`: produces structured report object with: perProvider (pass/fail counts per provider), perTaskType (pass/fail counts per task type), perTest (individual test results with diffs), failures (categorized failures with expected vs actual), summary (total pass/fail, compliance percentage per provider).
- `diffActions(expected, actual)`: produces human-readable diff between expected and actual action lists.

**6. Test suite definitions:**
Define the DEFAULT_SUITES constant with 6 test suites (one per task type). Each suite contains the test case definition with mockDOMElements inline. Use realistic element counts (20-50 elements per mock DOM, not 3-5).

For google-sheets suite: mock elements must include Name Box input (e5), formula bar, waffle cells, toolbar buttons.

For career-search suite: mock elements must include job listing cards, apply links, navigation elements, and the expected customValidation function that checks storeJobData YAML data integrity (company name, jobs array length, URL query string preservation).


Each .txt file starts with a metadata comment header:
```
# Golden response file: {provider}/{task-type}.txt

# Captured: 2026-03-01

```

**Provider-Specific Quirks to Include:**
Per Phase 15 research and CONTEXT.md, each provider has known output patterns:

- **xai (grok-4-1-fast):** Clean CLI output. Occasionally includes conversational preamble like "Sure, I'll help with that." before the CLI commands. Include this preamble in 2-3 of the 6 golden files so preprocessResponse stripping is exercised.

- **openai (gpt-4o):** Clean CLI output. Sometimes adds extra flags or verbose comments. Include slightly more detailed # reasoning comments than other providers.

- **anthropic (claude-sonnet-4-5):** Cleanest CLI output. Well-structured with minimal extra text. Include clean, well-formatted CLI commands.

- **gemini (gemini-2.5-flash):** Most wrapping. Occasionally wraps output in ```\n...\n``` markdown code blocks. Include markdown wrapping in 2-3 of the 6 golden files so preprocessResponse stripping is exercised. May also prefix with "Here are the commands:" text.

**Task-Specific Golden Response Content:**

1. **navigation.txt** (per provider): Navigate to a specific URL and confirm arrival.
   - Commands: `navigate "https://example.com/products"` then optional scroll/getText to verify
   - Element refs from: search-page.yaml DOM snapshot
   - Must end with `done "navigated to products page"`

2. **form-fill.txt** (per provider): Fill out a contact form with name, email, message.
   - Commands: click on name input, type name, click email input, type email, click message textarea, type message, click submit button
   - Element refs from: form-page.yaml DOM snapshot
   - Must include at least 4 type commands with realistic text
   - Must end with `done "submitted contact form"`

3. **data-extraction.txt** (per provider): Extract text data from a table/list page.
   - Commands: multiple getText commands to extract product names, prices, descriptions
   - Element refs from: data-table.yaml DOM snapshot
   - Must include at least 3 gettext commands
   - Must end with `done "extracted 5 product details"`

4. **search-click.txt** (per provider): Search for something and click a result.
   - Commands: type in search box, click search button or pressEnter, wait, click first result link
   - Element refs from: search-results.yaml DOM snapshot
   - Must include type + click/pressEnter + click sequence

5. **google-sheets.txt** (per provider): Name Box navigation + cell data entry.
   - Commands: key "Escape", click Name Box (e5), type cell reference, pressEnter, type cell data, key "Tab", repeat for more cells

   - Must end with `done "filled row 1 with job data"`
