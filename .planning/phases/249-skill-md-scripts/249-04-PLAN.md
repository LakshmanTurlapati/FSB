<objective>
Author `skills/FSB Skill/scripts/install-host.mjs`: a Node ESM script that detects other MCP hosts on the user's machine via the existing `npx -y fsb-mcp-server install --list` CLI, prints them, prompts `y/n` per host using `readline`, and runs `npx -y fsb-mcp-server install --<host>` ONLY on explicit yes.

Purpose: After a user installs the FSB skill into OpenClaw, the same FSB MCP server can also be installed into Claude Desktop, Cursor, VS Code, etc. This script offers that convenience -- but with explicit per-host consent. Eager auto-install is forever out of scope (per CONTEXT.md anti-scope and ROADMAP hard constraints). OpenClaw's own MCP config is NEVER auto-written; the user pastes that block from `print-stdio.mjs`.

Output: A new file at `skills/FSB Skill/scripts/install-host.mjs`.
</objective>

Detection:
  npx -y fsb-mcp-server install --list

  Stdout shape (per Phase 248 + v0.9.35 convention): one host per line, possibly with a leading detection marker.
  The host names FSB recognizes (matches `mcp/src/install.ts` `getSetupSections()` titles, lower-kebab-cased):
    claude-code
    claude-desktop
    cursor
    vscode
    windsurf
    codex

    (others may be added; do NOT hard-code a closed enum)

  The script MUST extract host names by:
    1. Reading the full --list stdout.
    2. Line-by-line scan for tokens that match `^[a-z][a-z0-9-]*$` AND appear AFTER any of these markers in the line (case-insensitive substring): 'detected', 'found', 'available', 'configured'.
    3. If --list output is empty or has no marker matches, print `[WARN] no other MCP hosts detected. Re-run after installing one (Claude Desktop, Cursor, VS Code, etc.).` and exit 0.

  The script MUST NOT hard-code the host enum -- it accepts whatever --list emits. This keeps it forward-compatible as fsb-mcp-server adds new hosts.

Per-host install:
  npx -y fsb-mcp-server install --<host>

  e.g. `npx -y fsb-mcp-server install --claude-desktop`
       `npx -y fsb-mcp-server install --cursor`

  The --<host> flag is the kebab-case host identifier with `--` prefix.

FORBIDDEN flags:
  --all              # never; consent is per-host
  --openclaw         # never; OpenClaw config is print-only via print-stdio.mjs

Prompt UX:
  For each detected host, print:
    Detected host: <host>
    Install fsb-mcp-server into <host>? [y/N]:
  Read one line via readline. Treat ONLY 'y' or 'Y' (after .trim()) as yes. Any other input (including 'yes', 'YES', '', 'n', 'no', whitespace) is NO.
  This is intentionally strict: 'y' or 'Y' only. The CONTEXT.md decision is "explicit yes" -- do not accept 'yes' to avoid accidental confirmations from the model agent.

Final summary:
  After all hosts processed, print:
    --- FSB skill install summary ---
    [OK] installed: <comma-separated list, or 'none'>
    [WARN] skipped:  <comma-separated list, or 'none'>
  Always exit 0 (the script's job is to RUN to completion; install failures are reported per-host but do not poison the overall exit code).

      #!/usr/bin/env node
      // install-host.mjs -- consent-gated multi-host installer (SKILL-05 / SKILL-06)
      // Detects other MCP hosts via `npx -y fsb-mcp-server install --list`,
      // prompts y/n per host, runs `npx -y fsb-mcp-server install --<host>` only on explicit yes.
      // NEVER passes --all. NEVER auto-writes OpenClaw's MCP config (use print-stdio.mjs).

    Imports (Node stdlib only):
      import { spawn } from 'node:child_process';
      import { createInterface } from 'node:readline';
      import process from 'node:process';

    Implementation contract (you implement the logic; this is what must be true):

    1. Detect step: spawn `npx -y fsb-mcp-server install --list` with `stdio: ['ignore', 'pipe', 'pipe']`, `shell: false`. Capture stdout into a string `listOut`. Stream raw output through to `process.stdout` so the user sees what FSB reported. On ENOENT (no `npx`), print:
         [FAIL] cannot launch `npx -y fsb-mcp-server install --list` -- is Node 18+ on PATH?
         next-step: Install Node 18+ (https://nodejs.org/en/download) and reopen the session.
       Then `process.exit(2);`. (Same exit-code contract as doctor.mjs so the two scripts feel consistent.)

    2. Parse step: derive `hosts` array from `listOut`:
       - Split on \n.
       - For each line, lowercase it; if it contains any of 'detected', 'found', 'available', 'configured', extract every substring matching the regex `/(?:^|\\s|:)([a-z][a-z0-9-]+)(?=\\s|$|[,.;:])/g` and keep matches that are NOT one of: `mcp`, `host`, `hosts`, `server`, `client`, `clients`, `config`, `version`, `the`, `with`, `from`, `into`, `at`, `on`. The intent is to extract the kebab-case host identifier without false-positives from English filler.
       - Dedupe. Preserve first-seen order.
       - EXPLICITLY filter out 'openclaw' if it appears (OpenClaw config is print-only; this script never installs into OpenClaw).
       - If `hosts` ends up empty, print `[WARN] no other MCP hosts detected. Re-run after installing one (Claude Desktop, Cursor, VS Code, etc.).` and `process.exit(0);`.

    3. Confirm + install loop: open a single readline interface (`createInterface({ input: process.stdin, output: process.stdout })`). For each host in `hosts`:
       a. Print:
            Detected host: <host>
            Install fsb-mcp-server into <host>? [y/N]:
       b. Await `rl.question(...)` (await/Promise pattern -- readline's question is callback-based; wrap it in a Promise).
       c. `const yes = answer.trim() === 'y' || answer.trim() === 'Y';`  // strict: only single-letter y/Y
       d. If yes: spawn `npx -y fsb-mcp-server install --<host>` with `shell: false`, stream stdio through to the parent. Await the child's `close` event. If the child exits 0, push host to `installed[]`; otherwise push to `failed[]` with the exit code.
       e. If no: push host to `skipped[]`.

       Forbidden in this loop:
       - Building the flag any way other than `'--' + host` (no string templating that could inject characters).
       - Allowing `host` to contain anything other than `[a-z0-9-]` -- if it does (defensive), skip with [WARN].
       - Passing `--all`.
       - Calling `npx -y fsb-mcp-server install --openclaw` (already filtered, but add a runtime guard: if host === 'openclaw', skip with `[WARN] OpenClaw config is print-only -- run print-stdio.mjs instead`).

    4. Close the readline interface when all hosts are processed: `rl.close();`.

    5. Final summary print:
         --- FSB skill install summary ---
         [OK] installed: <comma-separated installed[] or 'none'>
         [WARN] skipped:  <comma-separated skipped[] or 'none'>
       If `failed[]` is non-empty, additionally print:
         [FAIL] errored:   <comma-separated 'host(exit=N)' tuples>

       Then `process.exit(0);` (the script always exits 0; per-host failures are reported, not propagated, because mid-run partial success is the common case and the user is the one who should decide whether to retry).

    Forbidden in this file:
    - emojis / non-ASCII bytes
    - `require(...)`
    - `--all` flag literal anywhere in the file
    - `--openclaw` flag literal anywhere in the file (defense-in-depth: if reviewer greps for it, they should find zero hits)
    - any code that writes to a config file directly (no `node:fs` writes, no `writeFile`, no `writeFileSync`)
    - `exec` / `execSync` (use spawn)
    - shell wrapping (`shell: true` is forbidden -- prevents argument injection if a host name ever contained metacharacters)
