      #!/usr/bin/env node
      // doctor.mjs -- FSB skill diagnostic dispatcher (SKILL-03 / SKILL-06)
      // Wraps `npx -y fsb-mcp-server doctor`, parses the failing layer,
      // and prints a focused [OK]/[FAIL]/[WARN] branch with a one-line next step.
      // Cross-platform: Node ESM only; no shell-specific assumptions.

    Imports (from Node stdlib, no third-party deps):
      import { spawn } from 'node:child_process';
      import process from 'node:process';

    Implementation skeleton (you implement the actual logic; this is the contract):

    1. Define a constant `LAYERS` array IN THIS EXACT ORDER (matches v0.9.35 doctor output ordering, most-fundamental first):
         ['package', 'bridge', 'extension', 'active-tab', 'content-script', 'config']

    2. Define `NEXT_STEPS` map keyed by layer:
         package:        "Run `npx -y fsb-mcp-server --version` to confirm npm can reach the package."
         bridge:         "Start the FSB extension and run `npx -y fsb-mcp-server status --watch` to confirm ws://localhost:7225 is reachable."
         extension:      "Install the FSB Chrome extension from https://chromewebstore.google.com/detail/badgafnfchcihdfnjneklogedcdkmjfk -- then reopen this session."
         active-tab:     "Open a normal http(s) tab (not chrome://, edge://, or the Web Store) and run `npx -y fsb-mcp-server status --watch` to re-attach."
         content-script: "Reload the active tab. The FSB content script attaches on page load; reloads after extension install are required."
         config:         "Re-run `npx -y fsb-mcp-server install --<host>` for your MCP host (replace <host> with claude-desktop, cursor, etc.)."
         ok:             "All layers green. Run a small read_page or list_tabs call to confirm end-to-end."

    3. Spawn the doctor child:

         const child = spawn('npx', ['-y', 'fsb-mcp-server', 'doctor'], {
           stdio: ['ignore', 'pipe', 'pipe'],
           shell: false,                          // cross-platform: NO shell wrapping; npx itself is cross-platform
         });

       On Windows, `npx` resolves through the standard PATH lookup; do NOT special-case `cmd /c` here -- Node's child_process locates `npx.cmd` via PATHEXT automatically when shell:false. (Tested convention -- matches mcp/scripts pattern.)

       If spawn errors with ENOENT (no `npx` on PATH), catch via `child.on('error', ...)` and print:
         [FAIL] cannot launch `npx -y fsb-mcp-server doctor` -- is Node 18+ installed and on PATH?
         next-step: Install Node 18+ (https://nodejs.org/en/download) and reopen the session.
       Then exit with code 2.

    4. Capture both stdout and stderr into a single buffered string `combined` (the doctor CLI emits diagnostics on both streams in v0.9.35). Stream each chunk through to process.stdout/process.stderr unchanged so the user sees the raw output AS WELL as the wrapper's branch summary.

    5. On child `close` event, parse `combined` to identify the FIRST failing layer:
       - Lowercase the buffer once.
       - For each layer in `LAYERS` (in declared order), check if the buffer contains BOTH the layer keyword AND a failure marker within the same line (use a line-by-line scan; a "failure marker" is any of: 'fail', 'error', 'missing', 'timeout', 'not reachable', 'not found', 'not detected', 'unreachable').
       - First match wins -> `failingLayer = layer; break;`.
       - If no layer matched as failing AND the buffer contains 'all green' OR ('[OK]' near each layer), set `failingLayer = 'ok'`.
       - If parsing is ambiguous (no green marker, no failure marker) set `failingLayer = null` and emit a [WARN] branch (see step 6).

    6. Emit the branch summary AFTER the raw doctor output, separated by a blank line and a fence line `--- FSB skill doctor summary ---`. Then:

         if failingLayer === 'ok':
           print "[OK] all six layers green"
           print "next-step: " + NEXT_STEPS.ok
           exit 0
         else if failingLayer is one of the six:
           print "[FAIL] failing layer: " + failingLayer
           print "next-step: " + NEXT_STEPS[failingLayer]
           exit 1
         else (null / ambiguous):
           print "[WARN] doctor output did not match a known layer pattern; raw output above"
           print "next-step: re-run `npx -y fsb-mcp-server doctor` and report this output if it persists."
           exit 3

    7. Exit codes are MEANINGFUL: 0 = ok, 1 = a known layer failed, 2 = could not launch the doctor at all, 3 = doctor ran but its output was unparseable. Document these as a top-of-file comment block.

    Forbidden in this file:
    - emojis (per ./CLAUDE.md)
    - non-ASCII bytes
    - `require(...)` (ESM only)
    - shelling out via `exec` or `execSync` for the main doctor call (use spawn for streaming)
    - hard-coded `cmd /c` (let Node resolve npx cross-platform)
    - any path manipulation that assumes `/` (use `node:path` if a path is ever joined; this script does not need paths)
    - any reference to a host other than the bridge URL `ws://localhost:7225` (the bridge port is locked; do not reinvent discovery)
