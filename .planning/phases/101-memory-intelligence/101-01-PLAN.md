```javascript
// Phase 101 (MEM-03): Auto-consolidation triggers
try {
  // Increment persistent session counter
  const counterResult = await chrome.storage.local.get('fsb_session_count');
  const sessionCount = (counterResult.fsb_session_count || 0) + 1;
  await chrome.storage.local.set({ fsb_session_count: sessionCount });

  let shouldConsolidate = false;
  let reason = '';

  // Trigger 1: Every 10 sessions
  if (sessionCount % 10 === 0) {
    shouldConsolidate = true;
    reason = `session count ${sessionCount} (every 10)`;
  }

  // Trigger 2: Any memory type at 80% capacity (80 out of 100 per type)
  if (!shouldConsolidate) {
    const stats = await memoryStorage.getStats();
    const CAPACITY_THRESHOLD = 0.8;
    for (const [type, count] of Object.entries(stats.byType || {})) {
      if (count >= Math.floor(MAX_MEMORIES * CAPACITY_THRESHOLD / Object.keys(MEMORY_TYPES).length)) {
        shouldConsolidate = true;
        reason = `${type} memory at ${count} (80% capacity)`;
        break;
      }
    }
  }

  if (shouldConsolidate) {
    // Fire-and-forget -- consolidation must never block automation
    memoryManager.consolidate().then(result => {
      console.log(`[Memory] Auto-consolidation complete (${reason}):`, result);
      // Reset counter after successful consolidation
      chrome.storage.local.set({ fsb_session_count: 0 });
    }).catch(err => {
      console.warn('[Memory] Auto-consolidation failed:', err.message);
    });
  }
} catch (err) {
  console.warn('[Memory] Auto-consolidation trigger check failed:', err.message);
}
```

IMPORTANT NOTES:
- Use `memoryStorage.getStats()` directly (not memoryManager.getStats()) since memoryStorage is already available in background.js scope.
- The capacity check uses MAX_MEMORIES (500 total) divided by number of active memory types. With 3 types (semantic, procedural, task) that is ~167 per type, 80% = ~133. However, the CONTEXT.md says "80 out of 100 per type" -- use the simpler threshold of 80 per type as the trigger point since that matches the user's stated capacity model.
- Fire-and-forget pattern: use `.then().catch()` not `await`. This matches the established pattern from `generateActionSummary`.
- Reset the session counter to 0 after successful consolidation.
- The entire block is wrapped in try-catch so it never crashes extractAndStoreMemories.

For the capacity threshold, use a flat 80 per type (matching CONTEXT.md "80 out of 100 per type"):
```javascript
const PER_TYPE_CAPACITY = 100;
const CAPACITY_THRESHOLD = 80; // 80% of 100
for (const [type, count] of Object.entries(stats.byType || {})) {
  if (count >= CAPACITY_THRESHOLD) {
    shouldConsolidate = true;
    reason = `${type} memory at ${count}/${PER_TYPE_CAPACITY} (80% capacity)`;
    break;
  }
}
```
