## Context

The in-flight 21-platform installer expansion WIP (working tree on `Refinements`) introduced three regressions caught by the audit on 2026-04-29:

1. **Allowlist parity broken** -- the WIP added `'OpenClaw ΏθοΑ'` to the MCP server's visual-session client-label allowlist (`mcp-server/src/tools/visual-session.ts:10`) but did NOT update the matching extension allowlist (`utils/mcp-visual-session.js:4-16`). The comment at `visual-session.ts:7` ("must match the extension's allowlist") is now a lie. A client labeling itself `'OpenClaw ΏθοΑ'` would pass server-side `normalizeMcpVisualClientLabel` and then be rejected by the extension at `ws/mcp-tool-dispatcher.js:559-563` with `errorCode: 'invalid_client_label'`.
2. **`tests/mcp-setup-guidance.test.js` regression** -- the WIP removed the OpenClaw entry from `getSetupSections()` in `mcp-server/src/install.ts` (replacing it with the new ChatGPT / Claude.ai / Warp instructions sections). The existing test at `tests/mcp-setup-guidance.test.js:59-63` asserts that `setup` output contains `OpenClaw`, `manual`, `unsupported` -- this now fails 3/35 cases.
3. **Stale README version** -- the WIP rewrote root `README.md` content but left the version badge, document title, and intro `Note` saying `0.9.36` even though the active milestone is `v0.9.45rc1`.

None of these were caught by automated tests (the allowlist parity is unobservable to tests because the existing visual-session contract test only checks one synthetic invalid label) or by review (the WIP came in as a single uncommitted working tree).

## Tasks

### Task 1 -- FIX-01: Restore visual-session client-label allowlist parity

**File**: `utils/mcp-visual-session.js`
**Change**: add `'OpenClaw ΏθοΑ'` to the `MCP_VISUAL_CLIENT_LABELS` array (line 4-16). Match the order used in `mcp-server/src/tools/visual-session.ts:9-11` so future audits can diff the two arrays line-by-line.

**Why this works**: the `CLIENT_LABEL_MAP` at line 18-21 is built dynamically from `MCP_VISUAL_CLIENT_LABELS.forEach`, so adding the label auto-populates the lookup. The normalization key for `'OpenClaw ΏθοΑ'` is `'openclawΏθοΑ'` (lowercase + `[\s_-]+` stripping leaves the crab); the same normalization runs on both sides, so once both arrays contain the label the lookup succeeds.

**Acceptance**: a payload with `clientLabel: 'OpenClaw ΏθοΑ'` round-trips through `_mcp_normalizeVisualClientLabel` without returning null.

### Task 2 -- FIX-02: Restore OpenClaw section in `getSetupSections()`

**File**: `mcp-server/src/install.ts`
**Change**: re-insert the `{ title: 'OpenClaw', lines: [...manual / unsupported text...] }` block in `getSetupSections()` AFTER the new instructions-only platforms (ChatGPT, Claude.ai, Warp) and BEFORE the existing trailing entry. Keep the original wording ("manual / unsupported for now", "stdio command manually") so the test's `OpenClaw`, `manual`, `unsupported` needles all match.

**Rebuild**: run `npm --prefix mcp-server run build` to regenerate `mcp-server/build/install.js` from source.

**Acceptance**: `node tests/mcp-setup-guidance.test.js` reports 35 passed / 0 failed.
