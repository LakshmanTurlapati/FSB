<objective>
Detect and dismiss cookie consent overlays proactively before they block MCP tool interactions. The function uses a 3-tier detection approach (CMP framework identifiers, generic overlay patterns, text-based fallback), prefers reject/decline buttons over accept, and never touches non-cookie overlays like login prompts or paywalls.

Purpose: Cookie consent banners block clicks, obscure content, and cause read_page to return overlay text instead of page content. Clearing them proactively eliminates the "invisible blocker" problem where the AI cannot see or work around the overlay.

Output: A `dismissCookieConsent()` function in content/accessibility.js, wired into the readPage handler and smartEnsureReady pipeline.
</objective>

**Dismiss logic (per D-04):**

Once a cookie consent container is found, find the dismiss button in priority order:

1. **Reject/Decline buttons** (PREFERRED per D-04 -- minimize tracking):
   - For CMP-specific: use the `rejectButtons` selectors from the matching config
   - For generic/text-based: scan buttons/links inside the container for text matching (case-insensitive):
     `reject|decline|refuse|deny|necessary only|essential only|tout refuser|ablehnen|rifiuta|rechazar|refuser|nur notwendige|solo necessari|solo esenciales`
   - Match against `element.innerText.trim()` (not innerHTML) to avoid matching hidden text

2. **Accept as last resort** (per D-04 -- fallback only if no reject button found):
   - Only fall back to accept if NO reject/decline button was found
   - For CMP-specific: use the CMP's known accept button selectors
   - For generic: scan for text matching `accept|agree|ok|got it|allow|i understand|j'accepte|akzeptieren|accetta|aceptar`
   - Before clicking accept, log a warning: `FSB.logger.warn('dismissCookieConsent: no reject button found, falling back to accept', { cmp })`

3. **Close/X button** as alternative:
   - If neither reject nor accept button found, look for a close button: `button[aria-label*="close" i], button[aria-label*="dismiss" i], .close-button, [class*="close"]` within the container
   - Only click if the close button is inside the cookie container (not a page-level close button)
