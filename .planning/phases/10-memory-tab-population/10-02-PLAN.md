  <action>
Add duplicate extraction prevention for follow-up commands within the same session. The problem: each follow-up command triggers `extractAndStoreMemories`, and actionHistory grows, so overlapping actions produce near-identical memories.

In the `_localFallbackExtract` method (added in 10-01), filter actionHistory to only include actions since the last extraction:

1. In the `extract` method, before calling `_localFallbackExtract` or the AI extraction, record the current actionHistory length on the session:
   ```javascript
   const actionStartIndex = session._lastExtractionActionIndex || 0;
   const newActions = (session.actionHistory || []).slice(actionStartIndex);
   ```

2. After successful extraction (whether AI or local fallback), update the tracking index:
   ```javascript
   session._lastExtractionActionIndex = (session.actionHistory || []).length;
   ```

3. In `_localFallbackExtract`, use `newActions` for the procedural memory step generation instead of the full actionHistory. But still create the episodic memory based on the full session (since episodic summarizes the whole task, not just new actions).

4. In `_buildExtractionPrompt`, use `newActions` for the action summary if available, falling back to full actionHistory. Pass it as parameter or use session._lastExtractionActionIndex to slice.

This ensures that follow-up commands only extract memories from NEW actions, not re-process the entire history. The consolidator (Jaccard >= 0.85 NOOP) provides a second safety net.

Keep changes minimal -- only add the index tracking and action filtering. Do NOT restructure the class or change the method signatures.
