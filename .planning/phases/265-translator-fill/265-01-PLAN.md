## Goal

## Threat Model

- **T-265-01 (translation mistranslates a brand name):** Mitigated structurally -- brands live in `<x>` placeholders, not translatable surface. Translator literally cannot emit "Crom" for "Chrome" because Chrome is not a `<source>` token.
- **T-265-02 (placeholder drift):** Quality gate `<x id="">` count check per file. If counts don't match between source and target, file rejected and rewritten before commit.
- **T-265-03 (XML malformation):** Each Write is followed by `xmllint --noout` if available, else `node -e` parse check. CI will catch on push anyway.
- **T-265-04 (register drift across the 420 strings):** Single-session author ensures consistency; tone register table locked at top of CONTEXT.
- **T-265-05 (policy flip break in development):** Flip happens AFTER all 5 fills land, in a separate commit. If reverting needed, one-line revert restores the warning policy.
- **T-265-06 (translation memory loss across re-runs):** Out of scope -- this is a one-time fill. Future re-extractions of new strings will be handled in the next i18n milestone.
