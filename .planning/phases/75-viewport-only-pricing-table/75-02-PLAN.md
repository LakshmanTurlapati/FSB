Execute the SCROLL-09 edge case test using MCP manual tools following the extractPricingTableRows workflow from the pricing-table.js site guide. Test steps:

1. Use MCP navigate tool to go to https://www.notion.so/pricing (primary target -- SaaS pricing page with feature comparison table). Record the URL used and response status.

2. Use read_page to verify the pricing page loaded. Check for:
   - Pricing hero cards visible (plan names, prices)
   - Feature comparison table/section visible (look for "Compare plans" heading or table element)
   - Table header row with plan column names (Free, Plus, Business, Enterprise or similar)
   - Cookie consent or overlay banners
   Record: page URL, pricing cards found, table container found, plan column count.

3. Dismiss any cookie consent banners or promotional overlays if present.

4. Scroll down past the hero pricing cards to locate the feature comparison table section:
   - Use scroll(direction="down", amount=600) to move past pricing cards
   - Use read_page to check if comparison table is now visible
   - Repeat scroll if needed until the table header row is in viewport

5. EXTRACT TABLE HEADERS (plan names):
   - Use read_page or get_dom_snapshot to capture the table header row
   - Extract plan column names from planColumnHeader selectors (thead th, role=columnheader)
   - Record: plan names found, column count

6. FIRST BATCH -- READ VISIBLE FEATURE ROWS:
   - Use read_page or get_dom_snapshot to capture all visible feature rows
   - For each row, extract: feature name (from featureNameCell), value per plan column (check/cross/text from featureValueCell)
   - Identify category header rows vs feature value rows
   - Record: number of rows visible, feature names extracted, category headers found
   - Store all extracted feature names in a deduplication set

7. SCROLL-READ-DEDUPLICATE LOOP:
   - Scroll down 400-600px using scroll(direction="down", amount=500)
   - Wait for DOM to stabilize via wait_for_stable
   - Read newly visible feature rows
   - Compare feature names against already-captured set
   - Add only NEW rows (feature names not yet seen) to result set
   - Record: scroll cycle number, new rows found, cumulative total
   - Repeat until 2 consecutive scrolls yield 0 new feature rows

8. COMPLETENESS VERIFICATION:
   - Check that the last category section (typically "Admin", "Security", or "Support") was captured
   - Verify total row count is reasonable for a SaaS comparison table (expect 20-80+ rows)
   - Check if all plan columns have values for the extracted rows

9. IF NOTION PAGE INACCESSIBLE OR TABLE NOT FOUND:
   - Try fallback: navigate to https://www.airtable.com/pricing
   - Repeat the table detection and extraction workflow
   - If fallback also fails: document what was found and what is missing

10. OUTCOME CLASSIFICATION:
    - PASS = navigated to pricing page AND found comparison table AND extracted all feature rows with plan values across multiple scroll cycles AND deduplication worked correctly
    - PARTIAL = accessed pricing page but could not extract all rows (selectors mismatch, lazy loading issues, incomplete table), OR WebSocket bridge disconnect blocked live execution, OR accessed page but table only partially extractable via HTTP
    - FAIL = could not access pricing page at all or no table found
    - SKIP-AUTH = pricing page or comparison table requires authentication

Record every step outcome (tool used, target, result, notes) for the diagnostic report.
