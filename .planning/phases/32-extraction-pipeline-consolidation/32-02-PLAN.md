  <name>Task 1: Update consolidator for domain + task similarity matching</name>
  <files>lib/memory/memory-consolidator.js</files>
  <action>
Update MemoryConsolidator to handle task type memories with domain + task similarity instead of pure text similarity.

**Changes to resolve():**
- After finding similar memories via retriever, add special handling when `newMemory.type === 'task'`:
  - Extract domain from `newMemory.metadata.domain` and task from `newMemory.typeData.session.task`
  - Filter `similar` results to only those that are also type='task' AND same domain
  - For matching task memories, compare task descriptions using `_textSimilarity` on the task field (not the full text)
  - If task similarity >= 0.7 on same domain: treat as repeat run of same task -> UPDATE
  - If task similarity < 0.7 on same domain: ADD (different task, same site)
- For non-task memories, existing behavior unchanged (backward compat)

**Changes to _resolveHighSimilarity():**
- Add task type handling: when both are type='task', merge run data
  - Return UPDATE with reason "merging repeat task run on {domain}"
  - The actual merge (combining timelines, updating outcome, incrementing run count) happens in memory-manager when it receives the UPDATE action -- the consolidator just decides the action

**Changes to _resolveMediumSimilarity():**
- For task type: if same domain and task similarity >= 0.7 from the task description fields, UPDATE (merge)
- Otherwise ADD (different enough to be separate task)

**Changes to consolidateAll():**
- In Pass 2 (merge duplicates), group task memories by domain (not domain:type since all are type='task')
- Within each domain group, compare task descriptions (`typeData.session.task`) with `_textSimilarity`
- If task similarity >= 0.7: merge -- keep the one with more accesses, merge learned selectors/patterns from both into the kept memory
- When merging task memories, update the kept memory's typeData:
  - Append new timeline entries to existing timeline (run history)
  - Merge learned.selectors (deduplicate by selector name)
  - Merge learned.patterns (deduplicate by string equality)
  - Merge procedures (deduplicate by name)
  - Update outcome to most recent run's outcome
  - Increment a `runCount` field on session (add if not present, default 1)

**No changes to _textSimilarity()** -- reuse existing Jaccard index.
