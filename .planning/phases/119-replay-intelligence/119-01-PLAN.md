**2. Step-level retry (REPLAY-02):** Wrap the step execution in a retry loop (max 2 retries, so 3 total attempts). On failure, wait 500ms then retry the same step. Only report failure after all attempts exhausted. The retry loop wraps the `sendMessageWithRetry` call and its success check. Track which attempt succeeded for logging. Add `retriesUsed` to the step failure result for downstream tracking.

Structure:
```javascript
const MAX_STEP_RETRIES = 2;
for (const step of script.steps) {
  if (step.isReadOnly) continue;

  let stepSuccess = false;
  let lastError = null;

  for (let attempt = 0; attempt <= MAX_STEP_RETRIES; attempt++) {
    if (attempt > 0) {
      console.log('[FSB Executor] Retrying step', step.stepNumber, 'attempt', attempt + 1);
      await new Promise(r => setTimeout(r, 500));
    }

    try {
      const actionResult = await sendMessageWithRetry(tabId, { ... });
      if (actionResult && actionResult.success !== false) {
        stepSuccess = true;
        break;
      }
      lastError = actionResult?.error || 'action returned failure';
    } catch (error) {
      lastError = error.message;
    }
  }

  if (!stepSuccess) {
    return { success: false, failedAtStep: step.stepNumber, retriesUsed: MAX_STEP_RETRIES, error: ... };
  }

  // navigation handling stays the same...
  // smart delay replaces hardcoded delay...
}
```
