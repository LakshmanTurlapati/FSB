We need to track which steps are unreliable so the system can proactively re-record scripts rather than repeatedly failing and falling back.
Output: Updated agent-executor.js and agent-manager.js with step-level success tracking and re-record trigger.

```javascript
if (!agent.replayStats) {
  agent.replayStats = {
    totalReplays: 0, totalAISaves: 0, estimatedCostSaved: 0,
    stepSuccessRates: {},
    needsReRecord: false
  };
}
if (!agent.replayStats.stepSuccessRates) {

}
```

2. When `executionMode === 'replay'` and `runResult.stepResults` exists, update per-step stats:
```javascript
if (runResult.stepResults) {
  for (const sr of runResult.stepResults) {
    const key = String(sr.stepNumber);
    const existing = agent.replayStats.stepSuccessRates[key] || { successes: 0, total: 0 };
    existing.total++;
    if (sr.success) existing.successes++;
    agent.replayStats.stepSuccessRates[key] = existing;
  }
}
```

3. After updating step stats, check for unreliable steps (below 50% success rate with at least 4 data points to avoid premature triggers):
```javascript
const unreliableSteps = Object.entries(agent.replayStats.stepSuccessRates)
  .filter(([_, s]) => s.total >= 4 && (s.successes / s.total) < 0.5);

if (unreliableSteps.length > 0) {
  agent.replayStats.needsReRecord = true;
  console.log('[FSB Agent] Steps below 50% success rate:', unreliableSteps.map(([k]) => k).join(', '),
    '-- flagging for re-record');
}
```

4. When `executionMode === 'ai_fallback'` or `executionMode === 'ai_initial'` and a new script was saved (success === true), reset `stepSuccessRates` to `{}` and `needsReRecord` to `false` since the script is fresh.

5. Also in `createAgent`, add `stepSuccessRates: {}` and `needsReRecord: false` to the initial `replayStats` object.

Do NOT change `saveRecordedScript` -- it stays focused on script storage. The rate tracking belongs in `recordRun` which is already the stats aggregation point.
