**Step 1: Navigate to Photopea**
Use MCP `navigate` to open `https://www.photopea.com`. Wait for page load with `wait_for_stable`.

**Step 2: Dismiss splash dialog and ads**
Use `get_dom_snapshot` to check for:
- Welcome/splash dialog (New Project, Open, Recent Files)
- Ad overlays or cookie consent banners
- Any modal blocking the editor canvas
Click dismiss/close buttons as needed. If no close button visible, try pressing Escape key via `press_key`.

**Step 3: Load a sample image**
The goal is to have an image with a distinct background on the canvas. Options (try in order):
1. Check if Photopea opens with a sample/default image already loaded
2. If not, try File menu > Open > look for Sample Images option
3. If no sample images, navigate to `https://www.photopea.com` with a simple test image -- Photopea supports URL hash: try navigating to a URL that loads a simple image
4. As last resort: use File > New to create a new document, then use the Brush tool to paint a simple shape on a white background (white background can then be selected with magic wand)

After loading, use `get_dom_snapshot` to confirm an image/document is active on the canvas.

**Step 4: Activate Magic Wand tool**
Try keyboard shortcut first (per site guide and CONTEXT.md):
- Use `press_key` with key "w" to activate Magic Wand tool
- Use `get_dom_snapshot` to verify the tool options bar changed (tolerance control should appear)
- If W key did not work (e.g., Photopea focused on wrong element), try clicking the Magic Wand toolbar button using its selector from the site guide

**Step 5: Click on the background area with Magic Wand**
- Use `get_dom_snapshot` to identify the canvas element and its viewport position
- Calculate a coordinate that should be in the background area of the image (e.g., a corner of the canvas where background is likely visible)
- Use `click_at` at the calculated background coordinate
- Wait briefly for selection computation
- Use `get_dom_snapshot` to check if marching ants selection appeared (look for selection-related DOM state changes, or toolbar state indicating an active selection)

**Step 6: Delete the selected background**
- Use `press_key` with key "Delete" (or "Backspace" on Mac) to clear the selected area
- Wait briefly for the operation to complete
- Use `get_dom_snapshot` to check for any visual state change (checkerboard pattern indicates transparency, but this is canvas-rendered and not DOM-visible -- check for undo history change or layer state change instead)

**Step 7: Deselect (cleanup)**
- Use `press_key` with key "d" and ctrl=true (Ctrl+D) to deselect
- This confirms keyboard shortcuts work for menu operations

**Step 8: Record all results**
For each step, record: MCP tool used, target element/coordinates, result (OK/FAIL), and notes about what happened.
