Create `mcp-server/build/config-writer.js` as an ESM module. This is the format-aware config read-merge-write engine.

**Imports:**
```javascript
import { readFile, writeFile, copyFile } from 'node:fs/promises';
import { existsSync } from 'node:fs';
import { dirname } from 'node:path';
import stripJsonComments from 'strip-json-comments';
import { parse as parseToml, stringify as stringifyToml } from 'smol-toml';
import { parse as parseYaml, stringify as stringifyYaml } from 'yaml';
```

**Internal helper: `parseByFormat(raw, format)`**
Dispatches on format string to parse file content into a JS object:
- `'json'`: `JSON.parse(raw)`
- `'jsonc'`: `JSON.parse(stripJsonComments(raw, { trailingCommas: true }))` -- CRITICAL: must pass `{ trailingCommas: true }` because it defaults to false
- `'toml'`: `parseToml(raw)`
- `'yaml'`: `parseYaml(raw) || {}` -- yaml parse returns null for empty docs
- Throw `Error('Unsupported format: ...')` for any other value

**Internal helper: `serializeByFormat(data, format)`**
Dispatches on format string to serialize a JS object back to string:
- `'json'` or `'jsonc'`: `JSON.stringify(data, null, 2) + '\n'` -- JSONC is written back as plain JSON (comments are lost, documented trade-off; the .bak backup preserves the original)
- `'toml'`: `stringifyToml(data) + '\n'`
- `'yaml'`: `stringifyYaml(data)`
- Throw for unsupported

**Internal helper: `mergeServerEntry(data, serverMapKey, serverName, entry)`**
Handles the merge logic:
- For `serverMapKey === 'mcpServers'` where the platform is Continue (YAML array format): Check if `data.mcpServers` is an array. If so, find index where `item.name === serverName`. If found, replace at that index. If not found, push `{ name: serverName, ...entry }`. If `data.mcpServers` is not an array, create it as `[{ name: serverName, ...entry }]`.
- For all other platforms (object map format): Ensure `data[serverMapKey]` is an object (create `{}` if missing). Set `data[serverMapKey][serverName] = entry`.

**Detection of Continue's array format:** The Continue platform has `serverMapKey: 'mcpServers'` AND `format: 'yaml'`. Use this combination to detect array mode: if `format === 'yaml' && serverMapKey === 'mcpServers'`, use array merge logic. All other platforms use object merge logic.

**Internal helper: `checkIdempotent(data, serverMapKey, serverName, entry, format)`**
Checks if FSB is already configured identically. Returns `true` if skip is appropriate:
- For Continue (yaml + mcpServers array): Check if `Array.isArray(data.mcpServers)` and find item where `item.name === serverName`. If found, compare `JSON.stringify({ name: serverName, ...entry })` with `JSON.stringify(found)`.
- For all other platforms: Check if `data[serverMapKey]?.[serverName]` exists and `JSON.stringify(data[serverMapKey][serverName]) === JSON.stringify(entry)`.

**Exported function: `installToConfig(configPath, platform, serverName)`**
Parameters:
- `configPath` (string): Resolved absolute path to the config file
- `platform` (object): The platform entry from PLATFORMS (has `format`, `serverMapKey`, `displayName`)
- `serverName` (string): The server name key, always `'fsb'`

Returns an object: `{ status: string, path: string, message: string }`
Where status is one of: `'created'`, `'updated'`, `'skipped'`, `'error'`

Logic:
1. **Check directory exists (INST-08, D-02):** `const dir = dirname(configPath)`. If `!existsSync(dir)`, return `{ status: 'error', path: configPath, message: 'Directory does not exist: ' + dir + ' -- is ' + platform.displayName + ' installed?' }`. Do NOT create the directory.

2. **Build the server entry:** Import and call `getServerEntry()` from `./platforms.js`.

3. **Read existing file or start fresh:** If `existsSync(configPath)`, read with `readFile(configPath, 'utf-8')` and parse with `parseByFormat(raw, platform.format)`. If file does not exist, start with `{}`.

4. **Check idempotency (INST-04):** Call `checkIdempotent(data, platform.serverMapKey, serverName, entry, platform.format)`. If true, return `{ status: 'skipped', path: configPath, message: platform.displayName + ' already configured' }`.

5. **Determine if this is create or update:** `const isUpdate = (file existed && data[platform.serverMapKey]?.[serverName] !== undefined)`. For Continue array format, check array includes an item with matching name.

6. **Backup (INST-03):** If the config file exists, `await copyFile(configPath, configPath + '.bak')`.

7. **Merge (INST-02):** Call `mergeServerEntry(data, platform.serverMapKey, serverName, entry)`.

8. **Serialize and write (D-03):** `const output = serializeByFormat(data, platform.format)`. Then `await writeFile(configPath, output, 'utf-8')`.

9. **Return result:** `{ status: isUpdate ? 'updated' : 'created', path: configPath, message: (isUpdate ? 'Updated' : 'Installed to') + ' ' + platform.displayName + ': ' + configPath }`.

Wrap the entire function body in try/catch. On error, return `{ status: 'error', path: configPath, message: 'Failed to configure ' + platform.displayName + ': ' + err.message }`.

**Exported function: `removeFromConfig(configPath, platform, serverName)`**
Parameters same as installToConfig. Returns `{ status: string, path: string, message: string }` with status: `'removed'`, `'not-found'`, `'error'`.

Logic:
1. Check directory and file exist. If file doesn't exist, return `{ status: 'not-found', path: configPath, message: platform.displayName + ' config file not found: ' + configPath }`.
2. Read and parse the file.
3. For Continue (yaml array): Find and remove the array entry where `item.name === serverName`. If not found, return `{ status: 'not-found', ... }`.
4. For all others: Check if `data[platform.serverMapKey]?.[serverName]` exists. If not, return `{ status: 'not-found', path: configPath, message: 'FSB not configured in ' + platform.displayName }`. If yes, `delete data[platform.serverMapKey][serverName]`.
5. Backup the file (same as install).
6. Serialize and write.
7. Return `{ status: 'removed', path: configPath, message: 'Removed from ' + platform.displayName + ': ' + configPath }`.

Wrap in try/catch same as installToConfig.

Do NOT add a `//# sourceMappingURL` comment -- this is hand-authored JS.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File system read/write | Config files on disk could contain unexpected content, malformed data |
| JSONC comment stripping | Malicious JSONC could exploit parser edge cases |
| Config file merge | Existing config could have unexpected key types (array vs object) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-174-04 | T (Tampering) | Config file content | mitigate | Parse entire file, modify only target key, write back full content to prevent data loss (Pitfall 4 prevention) |
| T-174-05 | D (Denial of Service) | Malformed config file | mitigate | Wrap all parse calls in try/catch, return error status with message instead of crashing |
| T-174-06 | T (Tampering) | Backup overwrite | accept | .bak is overwritten on each install. Acceptable for a one-shot CLI tool -- user can recover from most recent backup |
| T-174-07 | I (Information Disclosure) | Config file contents | accept | Config files are user-owned, read/written with user's own permissions. No elevation. |
</threat_model>
