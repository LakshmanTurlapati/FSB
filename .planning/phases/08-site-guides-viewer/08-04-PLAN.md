  <action>
    Modify the `_buildTaskGuidance(taskType, siteGuide, currentUrl)` method in ai-integration.js (around line 3921).

    **Current behavior (line ~3930-3946):** When building selector guidance, it extracts domain from URL, then looks up `siteGuide.selectors[domain]` (nested format). This worked because old format was `selectors: { amazon: { searchBox: '...' }, ebay: { searchBox: '...' } }`.

    **New behavior:** Per-site guides have flat selectors: `selectors: { searchBox: '...', addToCart: '...' }`. The guide object also has a `.site` property.

    **Changes to make:**

    1. In the selectors section (~line 3930), add a check for the new per-site format:
    ```javascript
    // Add known CSS selectors for the current domain
    if (siteGuide.selectors && currentUrl) {
      let siteSelectors;

      if (siteGuide.site) {
        // New per-site format: selectors are flat on the guide object
        siteSelectors = siteGuide.selectors;
      } else {
        // Old category format: selectors nested under domain key
        const domain = (typeof extractDomain === 'function') ? extractDomain(currentUrl) : null;
        if (domain) {
          siteSelectors = siteGuide.selectors[domain];
          if (!siteSelectors) {
            const matchKey = Object.keys(siteGuide.selectors).find(key =>
              domain.includes(key) || key.includes(domain)
            );
            if (matchKey) siteSelectors = siteGuide.selectors[matchKey];
          }
        }
      }

      if (siteSelectors) {
        guidance += `\n\nKNOWN ELEMENT IDENTIFIERS FOR THIS SITE (use refs from snapshot to target these elements):\n${(typeof formatSelectors === 'function') ? formatSelectors(siteSelectors) : JSON.stringify(siteSelectors, null, 2)}`;
      }
    }
    ```

    2. At the beginning of _buildTaskGuidance, add category guidance integration:
    ```javascript
    // For per-site guides, include category-level guidance first
    let categoryGuidanceText = '';
    if (siteGuide.site && siteGuide.category && typeof getCategoryGuidance === 'function') {
      const catMeta = getCategoryGuidance(siteGuide.category);
      if (catMeta && catMeta.guidance) {
        categoryGuidanceText = `CATEGORY GUIDANCE (${siteGuide.category}):\n${catMeta.guidance}\n\n`;
      }
    }
    ```
    Then prepend categoryGuidanceText to the guidance string:
    ```javascript
    let guidance = categoryGuidanceText + `SITE-SPECIFIC GUIDANCE (${siteGuide.site || siteGuide.name}):\n...`;
    ```

    3. In the header line (~3927), use `siteGuide.site || siteGuide.name` instead of just `siteGuide.name` so it works with both formats.

    4. Update the `detectTaskType` method (~line 3984): where it references `siteGuide.name`, also check `siteGuide.category`:
    ```javascript
    const guideTaskType = guideToTaskType[siteGuide.category || siteGuide.name];
    ```

    5. In `getGuideForTask` (in index.js, line ~142), the fallback keyword matching uses `SITE_GUIDES_REGISTRY.find(g => g.name === categoryName)`. This should ALSO match per-site guides by category. However, since getGuideForTask tries URL first and falls back to keywords, and keyword matching returns the category-level match, this may need updating. BUT -- since old category files are now deleted, the keyword fallback in getGuideForTask won't find any match (it searches for `g.name === categoryName` but per-site guides don't have `.name`).

    **Fix in index.js `getGuideForTask()`:** Update the keyword fallback to also match by `g.category`:
    ```javascript
    const guide = SITE_GUIDES_REGISTRY.find(g => g.name === categoryName || g.category === categoryName);
    ```
    This returns the first per-site guide in that category, which is fine because the keyword fallback is a best-effort match and the category guidance will be attached by _buildTaskGuidance.
