    Read `site-guides/ecommerce.js` and `site-guides/social.js`. For each site that has dedicated selectors in the old file's `.selectors` object, create a per-site file.

    **Per-site file format (CRITICAL -- follow exactly):**
    ```javascript
    // site-guides/{category}/{site-name}.js
    registerSiteGuide({
      site: '{Display Name}',           // e.g., 'Amazon', 'LinkedIn'
      category: '{Category Name}',       // e.g., 'E-Commerce & Shopping' -- must match _shared.js
      patterns: [                        // URL patterns for THIS SITE ONLY (not the whole category)
        /amazon\.(com|co\.\w+|in|de|fr|jp|ca|com\.au|com\.br|com\.mx)/i
      ],
      guidance: `{SITE-SPECIFIC guidance text}`,  // Only the guidance specific to THIS site
      selectors: {                       // FLAT object -- NOT nested by domain name
        searchBox: '...',
        addToCart: '...',
        // ... all selectors for this site directly on the object
      },
      workflows: { ... },               // Site-relevant workflows
      warnings: [ ... ],                // Site-specific warnings only
      toolPreferences: ['click', 'type', 'scroll', ...] // Copy from parent category
    });
    ```

    **CRITICAL difference from old format:** The selectors are FLAT (directly on the guide object), NOT nested under a domain key. Old format: `selectors: { amazon: { searchBox: '...' } }`. New format: `selectors: { searchBox: '...' }`.
