    **Create ui/site-guides-viewer.js** with the following functionality:

    1. **initSiteGuidesViewer()** -- Called on DOMContentLoaded. Reads guide data from registry, renders the guide list, and sets up event handlers.
       - Call `getSiteGuidesByCategory()` to get grouped data
       - Call `getTotalSiteCount()` to update the header count
       - Render the guide list into `#siteGuidesList`

    2. **renderSiteGuidesList(groupedGuides)** -- Builds the HTML for all categories and sites.
       - For each category, create a collapsible category header with:
         - FontAwesome icon from `getCategoryGuidance(category).icon`
         - Category name
         - Site count badge (e.g., "(5 sites)")
         - Chevron toggle icon (fa-chevron-down)
       - Under each category header, render individual site items:
         - Site name
         - Category badge (small colored label)
         - Detail-toggle chevron icon (fa-chevron-right)

    3. **Category collapse/expand:**
       - Category headers are clickable. Clicking toggles a `.collapsed` class on the category container.
       - All categories start EXPANDED (per user decision: "Section always visible, not collapsed by default").
       - CSS hides `.guide-category.collapsed .guide-category-sites` via `display: none`.

    4. **Site accordion (one at a time):**
       - Clicking a site item calls `toggleGuideDetail(siteItem, guide)`.
       - If already expanded, collapse it (remove panel, remove `.guide-expanded` class).
       - If not expanded, collapse any other expanded site first (accordion behavior), then create and insert a detail panel after the site item.
       - Use the same pattern as `toggleMemoryDetail()` in options.js.

    5. **Guide detail panel rendering -- renderGuideDetailPanel(guide):**
       - Returns HTML string with collapsible sub-sections.
       - Sub-section order: Category Guidance (from _shared.js), Selectors, Workflows, Warnings.
       - All sub-sections COLLAPSED by default.
       - Each sub-section has a clickable header with chevron and title.
       - Clicking a sub-section header toggles its `.collapsed` class.

       Sub-section content formats:
       - **Category Guidance:** Show the shared guidance from `getCategoryGuidance(guide.category)`. Render as `<pre>` inside a scrollable container. Also show category-level warnings if any.
       - **Selectors:** Two-column table using `detail-table` CSS class. Left column: selector name (in `detail-code` class). Right column: CSS selector string (in `detail-code` class). Show count in header: "Selectors (N)".
       - **Workflows:** For each workflow key, show the workflow name as a label, followed by an ordered list of steps. Show count in header: "Workflows (N)".
       - **Warnings:** Bulleted list (`<ul>`). Show count in header: "Warnings (N)".
       - **Site Guidance:** If the site has its own guidance text (guide.guidance), show it as pre-formatted text. Title: "Site Guidance" (no count).

       IMPORTANT: Do NOT show `toolPreferences` in the viewer (per user decision).

    6. **Search integration -- filterSiteGuides(query):**
       - Called when the memory search input changes.
       - If query is empty, show all sites and categories (remove all `.guide-hidden` classes).
       - If query has text, match against site names and category names (case-insensitive).
       - Non-matching sites get `.guide-hidden` class (CSS: `display: none`).
       - Category headers with NO visible sites also get `.guide-hidden`.
       - Use CSS class toggle (NOT re-rendering) to preserve collapse/expand state.

    7. **Hook into existing search:**
       - Find the `memorySearchInput` element.
       - Add an input event listener that calls `filterSiteGuides(query)` alongside the existing memory search.
       - This should be done in `initSiteGuidesViewer()`, NOT by modifying options.js.
       - The existing `searchMemories()` in options.js is called by its own debounced listener. The viewer adds a SEPARATE listener on the same input for guide filtering (no debounce needed for CSS class toggle -- it's instant).

    8. **escapeHtml usage:**
       - The existing `escapeHtml()` function is defined in options.js. Since site-guides-viewer.js loads BEFORE options.js, define a local `_escapeGuideHtml()` function in site-guides-viewer.js or check for `escapeHtml` availability.
       - Simplest: define a small local escape function in site-guides-viewer.js.
