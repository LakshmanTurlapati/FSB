<objective>
Execute live MCP test for CONTEXT-08 (edit Google Doc replacing every "synergy" with "collaboration" without Find/Replace) and generate the autopilot diagnostic report.

Purpose: Validate whether the MCP navigate + read_page + press_key (Ctrl+F) + click_at (double-click to select word) + type_text chain can perform manual word-by-word replacement in a canvas-rendered Google Doc without using the Find/Replace dialog (Ctrl+H). Google Docs renders text on HTML5 canvas -- visible text is NOT in the DOM as text nodes, so select_text_range (Range API) will not work. The manual replacement strategy uses Ctrl+F (Find only) to locate each occurrence, double-click to select the word, then type the replacement. This is a "Context Bloat & Long-Running Workflows" edge case -- the challenge is efficiently replacing multiple occurrences without re-reading the full document after each replacement. Expected outcome is SKIP-AUTH (Google Docs editing requires a signed-in Google account) or PARTIAL (can navigate and read but not edit). Produces a structured diagnostic report for the autopilot enhancement milestone.

Output: 84-DIAGNOSTIC.md with CONTEXT-08 test results
</objective>
