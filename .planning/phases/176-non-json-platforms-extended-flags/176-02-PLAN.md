**2. Add dry-run preview helper function** (place after `printUninstallUsage()`):
```javascript
function printDryRunPreview(platform, configPath, entry) {
  console.log('[DRY RUN] ' + platform.displayName);
  if (configPath) {
    console.log('  Config: ' + configPath);
  }
  // Wrap the single entry in the platform's root key structure for realistic preview
  const wrapper = {};
  if (platform.format === 'yaml' && platform.serverMapKey === 'mcpServers') {
    // Continue uses array format
    wrapper[platform.serverMapKey] = [{ name: FSB_SERVER_NAME, ...entry }];
  } else {
    wrapper[platform.serverMapKey] = { [FSB_SERVER_NAME]: entry };
  }
  const preview = serializeByFormat(wrapper, platform.format === 'jsonc' ? 'json' : platform.format);
  console.log('  Entry:');
  for (const line of preview.split('\n')) {
    if (line.trim()) console.log('    ' + line);
  }
}
```
Note: Uses `FSB_SERVER_NAME` (already imported from version.js). For JSONC platforms, previews as JSON since JSONC serialization outputs JSON anyway.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI flags -> iteration scope | --all sets all platform keys; bounded by hardcoded PLATFORMS registry |
| dry-run -> file system | dry-run MUST prevent all writes; trust boundary between preview and mutation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-176-04 | Tampering | dry-run bypass | mitigate | dry-run check placed BEFORE every installToConfig/removeFromConfig/execSync call. No code path skips the check. |
| T-176-05 | Denial of Service | --all iterating all platforms | accept | 10 platforms is a fixed, small set. No amplification risk. Each platform either succeeds quickly or skips with a message. |
| T-176-06 | Information Disclosure | dry-run preview showing entry content | accept | Entry content is the same data that would be written to a user-owned config file. No secrets exposed. |
</threat_model>
