    - scheduleAgent with type='cron' creates alarm with correct when timestamp
    - isValidAlarmFire for cron type returns true if enough time elapsed
  </behavior>
  <action>
Add three new methods and extend two existing methods in AgentScheduler:

1. **parseCron(expression)** - Lightweight cron parser (no library needed). Parse a standard 5-field cron expression (minute hour dayOfMonth month dayOfWeek). Support:
   - Numeric values: `0`, `9`, `14`
   - Wildcards: `*`
   - Ranges: `1-5` (expands to [1,2,3,4,5])
   - Lists: `1,3,5` (expands to [1,3,5])
   - Validate: minute 0-59, hour 0-23, dayOfMonth 1-31, month 1-12, dayOfWeek 0-6
   - Return parsed object `{ minute, hour, dayOfMonth, month, dayOfWeek }` where each field is either `'*'` or a number/array. Return `null` on invalid input.

2. **_calculateNextCronRun(cronExpression)** - Given a cron expression string, parse it and find the next matching timestamp from now. Algorithm:
   - Start from current time, round up to next minute
   - Check each minute for match (iterate up to 366 days ahead, checking hour/minute/day matches)
   - Optimization: jump by hours/days when possible (if current hour doesn't match, skip to next matching hour)
   - Return timestamp or null if invalid expression

3. **getScheduleDescription(schedule)** - Return human-readable string for any schedule type:
   - `interval`: "Every {N} minutes" or "Every {N} hours" (if divisible by 60)
   - `daily` without daysOfWeek: "Daily at {HH:MM}"
   - `daily` with daysOfWeek: "{Mon, Wed, Fri} at {HH:MM}" using day name abbreviations
   - `cron`: Parse expression and generate description. For common patterns produce natural language (e.g., "Every weekday at 09:00", "1st of every month at 00:00"). For complex expressions, fall back to "Cron: {expression}"
   - `once`: "One-time execution"
   - Day name map: `['Sun', 'Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat']`

4. **Extend scheduleAgent()** - Add `case 'cron'` that:
   - Calls `_calculateNextCronRun(schedule.cronExpression)`
   - If null, log warning and return false
   - Otherwise, create alarm with `{ when: nextRun }`
   - Log: `[FSB Scheduler] Cron alarm created: {alarmName} next at {date}`

5. **Extend isValidAlarmFire()** - Add cron case:
   - For cron type, require at least 50 seconds since last run (similar to interval's 80% check but simpler since cron intervals vary)

6. **Add rescheduleCron(agent)** method - Similar to rescheduleDaily, calls scheduleAgent to set the next cron alarm after each run.
  </action>
