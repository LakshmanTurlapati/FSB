Execute the SCROLL-01 edge case test using MCP manual tools following the scrollAndCountPosts workflow from the twitter.js site guide. Test steps:

1. Use MCP navigate tool to go to a public X/Twitter profile page. Prioritized targets (try in order until one loads with a public feed):
   a. x.com/NASA (high-volume public account, science content, unlikely to be restricted)
   b. x.com/nytimes (news account, many posts, public)
   c. x.com/BBCWorld (international news, public)
   d. x.com/elonmusk (very high volume, public)
   e. Any public X/Twitter profile with 150+ posts visible without login
   The goal is to find a profile page showing a tweet timeline that can be scrolled without authentication. Avoid the home feed (x.com/home) which requires login.

2. Use read_page to verify the page loaded and a tweet timeline is visible. Check for:
   - Login wall: if page redirects to x.com/i/flow/login or shows [data-testid="loginButton"] prominently, classify as skip-auth
   - Cookie/consent banners: dismiss via click if present
   - Partial content: some profile pages show a few tweets then prompt login -- note this behavior
   Record the page title, URL, and whether tweets are visible.

3. Use get_dom_snapshot to map initial tweet elements. Identify:
   - Tweet containers: [data-testid="tweet"] elements
   - Tweet text: [data-testid="tweetText"] within each tweet
   - Tweet permalinks: a[href*="/status/"] within each tweet (for unique ID tracking)
   - Timeline container: [data-testid="primaryColumn"] section
   - Ad indicators: [data-testid="placementTracking"] elements to filter out
   - Count how many tweets are visible in the initial snapshot
   Record CSS selectors that matched and the initial tweet count.

4. Initialize tracking state:
   - seenPermalinks: Set of permalink hrefs already captured
   - postList: Array of {index, permalink, textSnippet} for each unique tweet
   - scrollCount: 0
   - For each tweet in initial snapshot, extract permalink and text, add to postList and seenPermalinks

5. SCROLL LOOP (repeat until postList.length >= 150 or max 30 scroll iterations):
   a. Execute scroll(direction="down", amount=800) to trigger infinite scroll loading
   b. Wait 500ms-1000ms for new content (use waitForDOMStable if available, else short delay)
   c. Execute get_dom_snapshot to capture current tweet elements
   d. For each [data-testid="tweet"] in snapshot:
      - Extract permalink href from a[href*="/status/"]
      - If permalink NOT in seenPermalinks: add to seenPermalinks and postList with text snippet
      - If permalink IS in seenPermalinks: skip (already counted)
   e. Increment scrollCount
   f. Log: "Scroll {scrollCount}: {newTweetsThisScroll} new tweets, {postList.length} total"
   g. If no new tweets found for 3 consecutive scrolls: feed may have ended or auth wall appeared -- break loop
   h. If auth/login prompt detected mid-scroll: note and break

6. AFTER SCROLL LOOP:
   - If postList.length >= 150: SUCCESS -- extract text of postList[149] (0-indexed)
   - If postList.length < 150 and feed ended: PARTIAL -- document how many posts were accessible
   - If auth wall blocked further scrolling: document at what count the wall appeared

7. EXTRACT 150th POST TEXT:
   - The text from postList[149].textSnippet (captured during scroll)
   - If the text was only a snippet, try to get full text via read_page or getText on the specific tweet element
   - Record the full tweet text, its permalink, and the user who posted it

8. VERIFY: Confirm the extracted text is a real tweet:
   - Non-empty text content
   - Has a valid permalink (contains "/status/{numeric_id}")
   - Is not an ad or suggestion module

Record every step outcome (tool used, target, result, notes) for the diagnostic report.
