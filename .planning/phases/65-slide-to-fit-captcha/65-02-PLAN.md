   b. https://www.geetest.com/en/ (GEETEST official site with slider demo)

   d. Search for "slider puzzle captcha codepen" or "slide to verify demo"
   The goal is to find a page with a visible slider CAPTCHA widget that can be interacted with.

3. Use get_dom_snapshot to map the page elements. Identify:
   - The CAPTCHA container (look for geetest, captcha, slider, verify classes)
   - The slider track element (horizontal bar)
   - The slider thumb/handle (the draggable element)
   - The puzzle background image (if present)
   - The gap/target position (if exposed in DOM)
   - Any iframe wrapping the CAPTCHA (common for third-party providers)
   Record CSS selectors for the thumb and track elements.

4. Determine slider thumb coordinates:
   - Get the thumb element's bounding rectangle or position from DOM snapshot
   - Calculate startX = thumb center X, startY = thumb center Y

5. Estimate target position:
   - If gap position visible in DOM: use that X coordinate
   - If not visible: estimate targetX = trackLeft + trackWidth * 0.55
   - endY = startY (horizontal slider, same Y coordinate)

6. STRATEGY A -- VARIABLE SPEED DRAG:
   Use drag_variable_speed tool on the slider:
   drag_variable_speed(startX=thumbCenterX, startY=thumbCenterY,
   endX=targetX, endY=thumbCenterY, steps=25, minDelayMs=5, maxDelayMs=45)
   The tool creates slow-fast-slow movement via CDP trusted events.

7. VERIFY: Use get_dom_snapshot to check for success/failure indicators:
   - Success: green slider, "verified"/"success" text, CAPTCHA removed
   - Failure: slider reset, "try again" text, red indicator
   - If failed: retry with adjusted position (+/-20px), maximum 3 attempts

8. If both strategies attempted, document which worked and which failed.
